STOP c000021a Unknown hard error (Solution to system crash caused by Norton mis-killing)

by xiaopang on 2007-05-19 17:25:33

**STOP c000021a Unknown Hard Error (Solution for Norton Misidentification Leading to System Crash)**

---

### **Issue Description:**

After updating Norton to the May 17th version, it mistakenly identified the following system files as Backdoor.Haxdoor malware:

- `C:\Windows\System32\netapi32.dll`

- `C:\Windows\System32\lsasrv.dll`

Norton then quarantined these files, causing the system to fail during reboot. Users were unable to enter the system or safe mode, resulting in a blue screen error:

**STOP c000021a Unknown Hard Error**

Preliminary investigations confirmed that `lsasrv.dll` and `netapi32.dll` are legitimate system files.

---

### **Resolution Methods:**

#### **Emergency Countermeasures for Users Not Yet Affected:**

1. Disconnect the network cable before turning on the computer.

2. After booting up, disable Norton's real-time protection before reconnecting the network cable and accessing the internet.

3. Wait for Symantec to resolve the issue before re-enabling Norton's real-time protection.

4. Disabling real-time protection may expose the computer to other viruses, trojans, or malicious programs. Consider temporarily using another antivirus software.

---

#### **Solution for Systems Already Unable to Boot:**

For users who have already experienced this issue and cannot boot their computers after restarting, follow these steps:

1. Use an XP system installation disk and set the BIOS to boot from the CD drive.

2. Select "Repair Console" during startup by pressing the "R" key.

3. Copy the `netapi32.dll` and `lsasrv.dll` files from the XP installation disk to the appropriate directories on the hard drive.

**Steps:**

- Insert the Windows installation CD and select booting from the CD-ROM.

- Choose "Repair Console" and press "R."

- Locate the necessary files in the `X:\I386` directory of the installation disk (replace "X:" with your actual CD drive letter).

- Copy the files using the following commands:

```

copy x:\I386\netapi32.dl_ c:\windows\system32\netapi32.dll

copy x:\I386\lsasrv.dl_ c:\windows\system32\lsasrv.dll

```

4. Restart the computer and modify the BIOS settings to boot from the hard drive. The system should now be accessible.

---

### **Detailed Analysis:**

After updating Norton's virus definitions on May 17th, systems running Windows XP with the KB924270 patch started experiencing frequent blue screens (**STOP c000021a Unknown Hard Error**). This occurred because Norton misidentified the updated system files `netapi32.dll` and `lsasrv.dll` as the Backdoor.Haxdoor malware.

Once these files were quarantined by Norton, the system would fail to boot properly, showing the blue screen error upon restart.

---

### **Temporary Solution Provided by Symantec:**

#### **Immediate Actions:**

1. From the System Center, right-click the server, go to "All Tasks," then "Symantec Antivirus," and open the "Virus Definition Manager."

2. In the dialog box, click "Configuration," then select "Virus Definition File" and revert to a previous definition file.

3. Ensure the server does not distribute today's virus definition file to clients.

4. For clients that have already updated the virus definitions, avoid restarting the computer.

5. If the `netapi32.dll` and `lsasrv.dll` files still exist and their modification dates are not today, they have not been fully quarantined. Restore these files from the quarantine folder or copy them from a working machine to `C:\Windows\System32`.

6. Delete the `20070517` folder located at `C:\Program Files\Common Files\Symantec Shared\VirusDefs`.

#### **If the Blue Screen Has Already Occurred:**

Try the following steps:

1. Use a Windows PE tool or an XP system installation disk to access the system.

2. Use a USB drive to copy the original `netapi32.dll` and `lsasrv.dll` files from a working environment to replace the corrupted ones in the `C:\Windows\System32` directory.

3. Delete the `20070517` folder from `C:\Program Files\Common Files\Symantec Shared\VirusDefs` on the affected machine.

4. Restart the computer.

5. If SAVCE (Symantec AntiVirus Corporate Edition) raises an alert again, the system should now function normally.

---

### **Latest Resolution from Symantec (As of 14:18):**

Symantec has released an update to address the false positive issue and restore the quarantined system files. Clients can download the fix from the following FTP site:

**Download Link:**

```

ftp://ftp.symantec.com/public/english_us_canada/antivirus_definitions/symantec_antivirus_corp/rapidrelease/sequence/

```

Navigate to the `68639` directory (or a newer one if available).

**Files to Download:**

- `symrapidreleasedefsx86.exe`: Executable for both server and client installations.

- `vd256247.xdb`: Copy this file to the enterprise antivirus installation directory on the management server, such as:

- `C:\Program Files\SAV` (if using Symantec Antivirus Corporation Edition)

- `C:\Program Files\Symantec` (if using Symantec Client Security)

Manually restart the antivirus service after applying the update.

---

### **Update (16:00):**

The issue has been resolved, and users can now directly update their virus definitions to the latest version without further problems.
