11. Write out the process and specific steps for domain name resolution.

Domain name resolution is the process of converting international, domestic, or Chinese domains into IP addresses after registration.

Example: A client requests from a local server to resolve www.baidu.com's IP address.

Steps:

1. The local server searches its database for corresponding entries (if not found, it queries the root domain server for the ".com" domain server address).

2. The root domain receives the request and sends the ".com" domain server IP to the local server, which then queries the "baidu.com" domain server IP.

3. The local server queries the "www" host IP from the "baidu.com" domain server and returns the final result to the client, enabling communication with www.baidu.com.

12. If customers report a 500 error on the server, how should it be handled? List at least three possible causes.

1. Confirm that the appropriate scripting language is defined in IIS (default ASP language set to VBScript or JScript if you prefer JavaScript).

2. Check your code's first line for any typos overriding the settings.

3. Sometimes incomplete DLL registration of the script engine can cause this issue; try downloading the latest script engine.

4. Ensure certain DLL files are registered during script engine installation, e.g., regsvr32 %windir%\system32\VBscript.dll, regsvr32 %windir%\system32\JScript.dll, regsvr32 %windir%\system32\Inetsrv\Asp.dll.

5. If unresolved, it may be due to permission issues.

Common causes include lack of permissions, incorrect script configurations, etc.

13. If we can normally access the customer's website but they cannot, with the server and firewall confirmed to have no issues, how should we explain and troubleshoot?

1. Check if it's a local ISP DNS resolution problem—notify the customer to wait and collaborate with their ISP.

2. Verify if there's an issue with the customer's HOST file (or system)—inform them about potential costs for on-site service.

3. Network issues at the customer’s location—advise them to contact their ISP or network provider.

14. What are the two modes of FTP upload and what are their differences?

FTP supports two modes: Standard (Active) and Passive (PASV).

Standard mode: The FTP client sends a PORT command to the FTP server.

Passive mode: The FTP client sends a PASV command to the FTP server.

Differences: Standard mode establishes a new connection for data transfer, while Passive mode uses a random high port for data exchange without requiring a new connection.

16. Write the specific steps for sending an email from test1@test1.com to test2@test2.com.

Typically, emails are sent either through an SMTP relay provided by the ISP or free email services, or directly via an SMTP server configured on the local machine.

Process:

1. Sender sends the email to the SMTP server.

2. The SMTP server queues and forwards the email to the recipient's MX server.

3. The MX server forwards it to the POP3 server.

4. The recipient retrieves the email using a client.

17. Create a virtual hosting site that supports secure FSO.

FSO stands for File System Object. To enable file uploads, ensure write permissions are enabled in IIS. For NTFS directories, configure appropriate directory permissions.

18. Design a periodic full backup and incremental backup plan for customer data, and set the server to automatically restart at 5 AM every Monday, Wednesday, and Friday.

Enable scheduled tasks—backup—select weekly schedule for Mondays, Wednesdays, and Fridays at 5 AM.

20. Use the built-in IP security policy to create a new IP strategy named NEW, disable PING, block ports 139 and 135, and block the 192.168.0.0 subnet.
