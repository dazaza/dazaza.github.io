### Linux Server Configuration

#### BIND DNS Server

-------------------

1. **System Installation**

- Navigate to System -> "Software Package" -> Search and install the corresponding "Package" (upload file & Install Package).

2. **DNS Client Configuration**

- In Webmin, go to Hardware -> Network Configuration -> Select "DNS Client" -> Change "Resolution Order" to "DNS, Hosts" -> Save and return.

- Go to Services -> "BIND 8 DNS Server". You will see sections such as "Global Server Options", "Existing DNS Zones", and others.

- By default, BIND can function as a caching server (with Root zone, reverse for 127.0.0, and forward for localhost) -> Installation order: bind-9.1.3-4 -> bindconf-1.6.1 -> caching-nameserver7.2-1.

3. **Create Master Server**

- In Webmin's Server section, select "BIND DNS Server" -> Under "Existing Zones", choose "Create New Master Zone" -> Set "Zone Type" to "Forward", input "test.com" as the domain name, enter "master.test.com" as the master server, and set the email address to "root@localhost" -> Click "Create".

- Create another master zone for reverse lookup by selecting "New Master Zone" -> Set "Zone Type" to "Reverse", input "192.168.0" as the network, enter "master.test.com" as the master server, and set the email address to "root@localhost" -> Click "Create".

- Add forward address records in the master server -> Select "test.com" -> Choose "A Record" -> Input "www" in the "Name" field and "192.168.0.100" in the address field -> Click "Add".

- Optionally, you can add "CNAME Records", "MX Records", and "NS Records".

- After completing all record additions, return to "BIND DNS Server" and click "Apply Changes".

4. **Create Slave Server**

- In "Existing Zones", choose "Create New Secondary Zone" -> Select "Forward" (Forward Lookup), input the domain "abc.com", and provide the "Master Server" address "192.168.0.101" -> Click "Create" and then "Save". Return to "BIND DNS Server" and click "Apply Changes" to activate the configuration.

- Use the same steps to configure the reverse records for the slave server.

#### Postfix Mail Server Configuration

1. **Postfix Installation**

- In the Webmin interface, navigate to System -> Software Package -> Install a New Package -> Upload file -> Install postfix-1.1.7-2.i386.rpm and imap-2001a-10.i386.rpm.

2. **Configure Postfix Startup**

- In System -> Bootup and Shutdown -> Select "postfix" -> Change "Start at boot time" to "Yes" -> Save and return.

- In Server -> Extended Internet Services -> Select both "imap" and "pop3" -> Change "Service Enable" to "Yes" -> Save and return -> Apply Changes (equivalent to `service xinetd restart`).

3. **Register Mail Server in DNS**

- Note: After adding, press "Apply Changes" to make the configuration effective. If the hostname is fully qualified, it must end with a ".".

4. **Configure Postfix**

- In Server -> Postfix Configuration -> General Options -> Set "What domain to use in outbound" to "Use domainname" (to use the current mail server's domain as the sender's domain).

- Set "What domains to receive mail for" to "Whole domain" (to receive mail for the entire domain).

- Set "Network interfaces for receiving mail" to "All" (to receive mail through all network cards).

- Set "Hostname of this mail system" to the current mail server's hostname (e.g., "mail.test.com").

- In "Local internet domain name", input the current host's domain name (e.g., "test.com").

- Keep other options as default -> Click "Save and Apply".

5. **Add User Accounts for the Mail Server**

- In Webmin -> System -> Users and Groups -> Local Groups -> Create a new group -> Input "mailuser" as the Group name.

- In Local User -> Create a new user -> Input "mailuser1" as the Username, "/home/mailuser1" as the Home directory, set Password to "Normal Password", and in Group Membership -> Primary group -> Existing group -> Select "mailuser" -> Click "Create".

6. **Mail Aliases and Virtual Domains**

- **Mail Aliases**:

- In Webmin -> Server -> Postfix Configuration -> Mail Aliases -> Create a new alias -> In Alias Options, set "Alias databases used by the local delivery agent" and "Alias databases built by newaliases command" to "hash:/etc/postfix/aliases".

- Create a new alias -> In Alias Properties Edition, input "mailuser3" as Name and "mailuser2" as Alias to... (so mail sent to mailuser3 will automatically forward to mailuser2).

- **Virtual Domains**:

- In Webmin -> Server -> Postfix Configuration -> Virtual Domain -> Set "Domain mapping lookup tables" to "hash:/etc/postfix/virtual" -> Save and Apply.

- Enter Virtual Domains -> New Mapping -> In Edit A Map, set "Name" to "@newtest.com" and "Maps to..." to "@test.com" (indicating that all emails sent to @newtest.com will be mapped to @test.com) -> Save mapping.

#### Wu-ftpd Server

1. **FTP Server Installation**

- In Webmin -> System -> Software Package -> From uploaded file -> Install "wu-ftpd-2.6.2-5.i386.rpm" and "anonftp-4.0-9.i386.rpm".

2. **Wu-ftpd Server Startup Settings**

- In Webmin -> Server -> Extended Internet Services -> Select "ftp" -> Modify "Service Enabled?" to "Yes" -> Save settings -> In Extended Internet Services page, click "Apply Changes" to make the settings effective.

3. **Set Anonymous Account Rules**

- In Webmin -> Server -> WU-FTP Server -> Select "Anonymous FTP" -> Set "Anonymous FTP root directories" to "/var/ftp" -> Save settings and test successfully using a client.

4. **Set Guest Account**

- Add account ftpuser1:

- In Webmin -> System -> Users and Groups -> Create a new user -> Set "Username" to "ftpuser1", "Shell" to "/sbin/nologin", "Home directory" to "/home/ftpuser/user1", "Password" to "Normal password", and "Copy files to home directory" to "NO" -> Click "Create".

- Set Guest account:

- In Webmin -> Server -> WU-FTP Server -> Select "Users and Classes" -> Set "Unix users and UIDs to treat as guests" to "*" (meaning all user accounts are treated as guest type by default) -> Save and test successfully using a client.

5. **Set Real Account**

- Add account ftpuser2:

- In Webmin -> System -> Users and Groups -> Create a new user -> Set "Username" to "ftpuser2", "Shell" to "/sbin/nologin", "Home directory" to "/home/ftpuser/user2", "Password" to "Normal password", and input the password -> Click "Create".

- Set Real account:

- In Webmin -> Server -> WU-FTP Server -> Select "Users and Classes" -> In "Unix groups and UIDs not to treat as guest", input "ftpuser2" -> Save and test successfully using a client.

6. **Access Control for Wu-ftpd Server**

- **IP Address Access Control**:

- In a possible login directory, edit a file like /var/ftp/ipdeny with content "Sorry, your ip address is blocked".

- In Webmin -> Servers -> WU-FTP Server -> Select "Limits and Access Control" -> In "Deny access from", add "Deny from address" as "192.168.0.2" and "Error message file" as "/var/ftp/ipdeny" -> Save and test successfully using a client.

- **Class-Based Access Control**:

- In a possible login directory, edit a file like /var/ftp/toomany with content "Sorry, too many users now. Please try again later."

- In Webmin -> Servers -> WU-FTP Server -> Define class name "unknownuser", "User types" as "anonymous", and "Matching address" as "!192.168.0.0/24" -> Save.

- In "Limits and Access Control", set concurrent user connection limits: In "Concurrent user limits", set "Apply to class" to "unknownuser", "Maximum users" to "15", "At times" to "Any time", and "Error message file" to "/var/ftp/toomany" -> Save and test successfully using a client.

- Additionally, set "File and data transfer limits": Set "Limit type" to "Files per session", "Maximum" to "5", and "Apply to class" to "unknownuser".

#### Apache Server Configuration

1. **Apache Installation and Startup**

- In Webmin -> System -> Software Package -> From uploaded file -> Select "Install" to install "apache-1.3.20-16.i386.rpm".

- To start: Select System -> Bootup and Shutdown -> Select "httpd" -> Change "Start at boot time?" to "Yes" -> Save (or select "Start Selected" to start the service).

2. **Apache Configuration Interface**

- In Webmin -> Servers -> Apache Webserver, you will see three sections: "Global Configuration", "Virtual Servers", and "Create a New Virtual Server".

3. **Create Virtual Host Based on Domain Name**

- In Webmin -> Servers -> Apache Webserver -> Create a New Virtual Server -> Set IP "Address" to "192.168.0.100", "Document Root" to "/home/ftpuser/user1", and "Server Name" to "ftpuser1.test.com" -> Select "Create" to complete creation -> In the Apache Webserver page, click "Apply Changes" to make the modified configuration effective -> Test successful access via the client (domain must be pre-configured).

4. **Create Virtual Host Based on IP Address**

- Bind multiple IP addresses:

- In Webmin -> Hardware -> Network Configuration -> Network Interfaces -> Select "Interfaces Active Now" -> Add a new interface -> Input "Name" as "eth0:0", "IP Address" as "192.168.0.101" -> Click "Create" to complete creation.

- In Webmin -> Servers -> Apache Webserver -> Create a New Virtual Server -> Set IP "Address" to "192.168.0.101", "Document Root" to "/home/ftpuser/user2", and "Server Name" to "ftpuser2.test.com" -> Select "Create" to complete creation -> In the Apache Webserver page, click "Apply Changes" to make the modified configuration effective -> Test successful access via the client (domain must be pre-configured).

5. **Access Restrictions in Apache**

- **Username/Password-Based Access Control**:

- Select "Virtual Server" "ftpuser1.test.com" -> In "Create Per-Directory, Files or Location Options", set "Type" to "Directory", "Path" to "/home/ftpuser/user1", and create -> Select the created "Directory /home/ftpuser/user1" -> Select "Access Control" -> Set "Authentication realm name" to "ftpuser1.test.com", "Authentication type" to "Basic", "Restrict access by login" to "All valid users", and "Text file authentication" with "User text file" as "/etc/httpd/conf/accesscontrolfile" -> Save -> Re-enter the "Access Control" page to set "Edit users" and add user "httpuser1" -> Apply Changes to make the setting effective -> Test successful access via the client.

- **Client Host IP Address-Based Access Control**:

- Similarly, select "ftpuser2.test.com" -> Set Directory "/home/ftpuser/user2" -> Select "Access Control" for the created "Directory /home/ftpuser/user2" -> Set "Resist Access" with "Access checking Order" as "Allow then deny" -> Add "Action" "Allow" "All requests" and "Deny" "Request from IP" "192.168.0.2" -> Verify successful access via the client.
