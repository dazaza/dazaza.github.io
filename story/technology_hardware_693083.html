Design Principle of Car Decoder Encryption System: One advantage of the PWM car decoder is that all signals from the processor to the controlled system are in digital form, eliminating the need for digital-to-analog conversion. Keeping the signal in digital form minimizes the impact of noise. Noise only affects the digital signal when it is strong enough to change a logic 1 to a logic 0 or vice versa.

Another advantage of PWM over analog control is its strong resistance to noise, which is also why PWM is sometimes used for communication. Transitioning from analog signals to PWM can significantly extend the communication distance. At the receiving end, appropriate RC or LC networks can filter out the modulated high-frequency square wave and restore the signal to its analog form.

Many microcontrollers have built-in PWM controllers. For example, in car decoders, car interference devices, car unlocking tools, and car remote decoding devices, the STC12C2052 contains two PWM controllers, each of which can select the on-time and period. The duty cycle is the ratio of on-time to period; the modulation frequency is the reciprocal of the period. Before performing PWM operations, this microprocessor requires the following tasks to be completed in software:

1. Set the period of the on-chip timer/counter that provides the modulating square wave www.kaisuo888.net

2. Set the on-time in the PWM control register;

3. Set the direction of the PWM output, which here is a general-purpose I/O pin;

4. Start the timer;

5. Enable the PWM controller (although specific PWM controllers may differ in programming details, their basic ideas are the same).

3 Hardware Design

The design of the CTCSS car decoder system revolves around a set of low-frequency audio signals (67.0~250.3 Hz) (32 or 38 depending on different standards). These sub-audio signals are pure sine waves with strict frequency differences. In most designs, the approach involves calculating fixed-point sine function values: to generate a sine wave, the method of fixed points is used, dividing one cycle of the sine wave into several points according to 360 degrees, calculating the sine function value at each point, and converting it to the corresponding D/A converter input value, thus obtaining a sine function table. This table is stored in the microcontroller's program memory via the program, using the microcontroller's timer to generate timing. Whenever the timing period expires, the table is looked up to get the output value corresponding to that point, and then the corresponding voltage value is obtained through D/A conversion. Repeatedly looking up and outputting the table in this way produces the desired sine wave. Since the number of points in one cycle of a sine wave is fixed, changing the timer's timing value changes the interval time between adjacent points, thereby changing the frequency of the sine wave.

Here, another simpler method is used to produce various waveforms. Using the PWM modulation signal from the microcontroller to generate multiple waveforms, but the drawback of this method is the limited frequency of the generated waveforms.
