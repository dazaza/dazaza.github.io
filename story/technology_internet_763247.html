Linux knowledge is as deep as the sea.

by anonymous on 2013-07-29 17:35:38

If you have a 4.2GB CSV file with over 12 million records, and each record has 50 columns. Now, your task is to sum up the values of one specific column.

What would you do?

This was the opening part of an article I recently came across. The article discusses how to use Unix commands to analyze large files. As a developer who mostly works on the Windows platform, my first thought upon seeing such a problem would be about memory overflow, CPU usage hitting 100%, or even letting it run overnight. However, for a Linux/Unix expert, this is a piece of cake. As shown in the article, the task can be completed with just one line of command.

(Assume the file is named `data.csv`, the rows are separated by pipes (`|`), and we need to sum up the fourth column.)

```bash

cat data.csv | awk -F "|" '{ sum += $4 } END { printf "%.2f\n", sum }'

```

It’s concise and clean, and the `{ sum += $4 }` part has a closure-like flavor. This made me admire Linux even more.

The purpose of writing this blog post isn’t to express my feelings about Linux, but rather to focus on what happened in the comments section of that article. The article attracted quite a few comments from Linux enthusiasts, and they were all experts. Just the first comment alone left me speechless.

The commenter, using the pseudonym "California Lotto," wrote:

"If you think you’re a Linux command-line expert, congratulations—you’ve just won today’s 'Most Useless Use of Cat' award. You should write the command like this:"

```bash

awk -F "|" '{ sum += $4 } END { printf "%.2f\n", sum }' < data.csv

```

Indeed, `cat` seems unnecessary here. I was impressed at first, but then I immediately felt that this person was annoying and showing off. Although I didn’t like how he disregarded the author's effort in writing such a great blog post, as an outsider, both he and the author are far beyond my level.

However, when I read the second comment, things took a dramatic turn. This second comment was clearly in response to the first one:

"If you think you’re a Linux command-line expert, congratulations—you’ve just won today’s 'Most Useless Use of Redirection' award. You should write the command like this:"

```bash

awk -F "|" '{ sum += $4 } END { printf "%.2f\n", sum }' data.csv

```

As the saying goes, "There’s always someone better than you." I suddenly realized that there’s no point in comparing who’s stronger here. In this vast internet world, there will always be people who are better than you in some way. What matters is discussion and participation. Through such discussions, not only do we enrich our knowledge, but we also gain a deeper understanding of the solution to the problem. As pointed out in the following comments, the placement of the redirection symbol doesn’t matter; this also works:

```bash

< data.csv awk -F "|" '{ sum += $4 } END { printf "%.2f\n", sum }'

```

Amazing, right? And someone else further pointed out that the author initially wrote the following during testing:

```bash

head -1 data.psv | awk -F'' '{print NF}'

```

After the experiment succeeded, it was natural to replace `head` with `cat`.

In any case, this is a great article. These people are all experts and my teachers, not only teaching me programming knowledge but also how to be a better person.
