Everything is JavaScript

by anonymous on 2013-11-16 20:09:44

JavaScript originated from Netscape's LiveScript language, which is an object-based and event-driven client-side scripting language. It was initially designed to validate HTML form inputs.

In the early years, JavaScript was unpopular among developers due to its complex Document Object Model (DOM), poor implementations, and lack of debugging tools, as well as inconsistent browser support. However, with technological advancements, JavaScript has become increasingly powerful and complete. For instance, Ajax technology allows for more engaging web applications, Node.js extends JavaScript's application scope to the server side, and various frameworks make JavaScript development more efficient. Especially in recent years, the emergence of HTML5 has elevated JavaScript to unprecedented heights.

Today, JavaScript has become an essential language for web development and is gradually expanding into mobile domains. Due to its cross-platform nature, JavaScript plays a significant role in the mobile internet era.

Anything that can be written in JavaScript will eventually be written in JavaScript — Atwood’s Law (proposed by Jeff Atwood in 2007).

What if you don’t know JavaScript?

Nowadays, JavaScript has become the "standard" language for web development. Whether you like it or not, JavaScript is inevitably involved in web development. JavaScript enables true "write once, run anywhere" capability and controls all common browsers, so it is also referred to as the "virtual machine of the Web" and the "assembly language of the Web."

Even if you haven't started learning JavaScript, it doesn't prevent you from building web applications. You can write applications in your preferred language (Java, .NET, C/C++, Python, Ruby, etc.), then compile them into JavaScript and run them in the browser.

If you're dissatisfied with JavaScript's syntax, you can use some enhanced versions of JavaScript. For example, if you prefer classic object-oriented programming and want some syntactic sugar, you can try CoffeeScript. If you prefer a stricter type system, you might find Dart or TypeScript more interesting. If you prefer functional programming, ClojureScript or Roy can help you.

Compiling your familiar language into JavaScript

Compiling Java code into JavaScript

Despite the name "JavaScript" containing "Java," they are actually unrelated. The name was chosen at the time because Java was very popular, and JavaScript borrowed its fame for promotion.

The emergence of powerful tools has established a connection between JavaScript and Java code: Java code can be compiled into JavaScript code to run in the browser. Moreover, other JVM-based languages can also be compiled similarly.

Tools that enable this include:

BicaVM: A JavaScript implementation of the JVM that can run 60% of Java bytecode.

Ceylon: A modular, statically-typed JVM language that can compile into JavaScript.

Doppio: A Coffeescript implementation of the JVM, enabling Java programs to run on any JavaScript engine.

GrooScript: A small framework that converts Groovy code into JavaScript.

j2js: Converts Java bytecode into JavaScript.

Java2Script: An Eclipse plugin that compiles Java into JavaScript and implements a JavaScript version of Eclipse SWT.

Kotlin: A statically-typed programming language that can compile into both JVM bytecode and JavaScript.

Processing: A visualization language based on Java that can compile into JavaScript.

STJS: Strongly-Typed JavaScript, a Maven plugin that converts Java source code into JavaScript.

Compiling C/C++ code into JavaScript

C/C++ can now be used to develop web applications and can also be compiled into JavaScript. Some tools include:

Emscripten: A Mozilla-developed LLVM backend that compiles LLVM IR (generated by Clang) into JavaScript.

mala: Compiles Vala code (which first compiles into C) into JavaScript.

Mandreel: Compiles C++ and Objective-C code into highly optimized JavaScript.

Clue: A C compiler that compiles C code into high-quality Lua, JavaScript, or Perl code.

Since Emscripten can compile LLVM binary code into JavaScript, and LLVM is a popular intermediate compilation format supported by almost all languages, Emscripten can serve as a universal tool for converting most languages into JavaScript.

Compiling Python code into JavaScript

Tools that convert Python code into JavaScript include:

Brython: Replaces JavaScript in web pages, allowing Python scripts to execute directly on the web.

PYXC-PJ: Converts Python into JavaScript and generates a line/column mapping file.

Pyjaco: Converts Python into JavaScript.

Pyjamas: A Python-to-JS converter.

Pyjs: Another Python-to-JS converter.

PyCow: Converts Python code into MooToolsJS code.

PyvaScript: A programming language with Python-like syntax and JavaScript features.

RapydScript: An enhanced version of PyvaScript.

Skulpt: Client-side Python interpreter.

Compiling Ruby code into JavaScript

A Ruby standard library implemented in JavaScript called RubyJS provides all Ruby methods, such as Array, Numbers, Time, etc. Ruby creator Matz said, "If I have to write JavaScript, I would use RubyJS."

Additionally, you can use the following tools to compile Ruby code into JavaScript:

8ball: Compiles Ruby (or a subset of Ruby) source code into JavaScript.

ColdRuby: A Ruby 1.9 MRI bytecode compiler and JS runtime with a C++ runtime for native execution via V8.

HotRuby: Runs YARV-compiled Ruby opcodes inside the browser or Flash platform.

Opal: A Ruby-to-JavaScript compiler for any JS environment.

rb2js: A Ruby-to-JavaScript compiler.

Red: Allows writing Ruby-style code and running it as JavaScript.

Compiling .NET code into JavaScript

You can use the following tools to compile C#, F#, and other .NET code into JavaScript:

Apps in Motion: Allows building web applications using C# that run on any device.

Blade: A Visual Studio extension that converts C# code into JavaScript.

FunScript: Uses TypeScript type providers to convert F# code into JavaScript or jQuery code.

jsc: Re-compiles .NET programs into JavaScript, ActionScript, PHP, or Java.

JSIL: Converts MSIL (.NET bytecode) into JavaScript.

NemerleWeb: Compiles Nemerle (a multi-paradigm Microsoft language) into JavaScript.

Pit: Compiles F# code into JavaScript.

Prefix (in development): Compiles C# code into JavaScript.

Saltarelle: Compiles C# code into JavaScript.

SharpKit (commercial tool): Compiles C# code into JavaScript.

Script#: Compiles C# code into JavaScript.

WebSharper: Compiles F# code into JavaScript.

Compiling Scala code into JavaScript

You can use the following tools to compile Scala code into JavaScript:

Scala.js: A Scala-to-JavaScript compiler supporting all Scala language features, enabling full-stack web development in Scala.

js-scala: Treats JavaScript as an embedded DSL language within Scala.

Scala+GWT: Compiles Scala code into JavaScript through the GWT toolchain.

Converting Flash into JavaScript

The biggest rival to HTML5 (HTML/JavaScript/CSS3 technology stack) is Flash. Although Flash still holds a 70% market share, it's undeniable that the future belongs to HTML5.

Even Flash developer Adobe has embraced HTML5 and released Toolkit for CreateJS, making it easier for web developers to convert Flash into HTML5 format.

Last year, Google released Swiffy, a web-based tool that converts SWF-format Flash files into HTML5 format online.

Moreover, Mozilla launched Shumway, a "JavaScript-based Flash virtual machine and runtime" aimed at creating a reliable and effective SWF (Flash file format) renderer in the browser without plugins.

This shows how these tools accelerate the replacement of Flash by HTML5.

Other compilation tools

Go2js: Compiles Go code line-by-line into JavaScript.

Perlito: Compiles Perl 5/6 code into JavaScript, Ruby, SBCL, and Go.

Issues after compiling into JavaScript

Debugging issues

Since it involves cross-compilation, the output code is difficult to trace and debug.

Some tools generate mapping files during compilation (mapping compiled JavaScript back to the source language), allowing debugging through these files.

Performance issues

Due to JavaScript's execution in the browser, the speed of locally compiled code may decrease after being converted into JavaScript.

Mozilla proposed a solution—asm.js (see "asm.js: JavaScript's 'Assembly Language'"). Asm.js is a strict subset of JavaScript that provides a C/C++-like virtual machine abstraction, including an efficiently loadable and storable large binary heap, integer and floating-point operations, higher-order function definitions, and function pointers. It can serve as a low-level, high-performance compiler target language.

Additionally, Mozilla's JavaScript engine OdinMonkey offers faster execution speeds. Developers can manually write their code as asm.js code after compiling it into JavaScript, and when the browser executes the program, the OdinMonkey engine will make the program run closer to native application speeds. Mozilla may develop tools in the future (similar to LLJS) to automatically generate asm.js code.

As technology advances, JavaScript may one day run as fast as native C.

Demonstrations

In 2012, Mozilla introduced a web-based first-person shooter game demo called BananaBread. In May of this year, Mozilla announced a collaboration with Epic Games to port the Unreal Engine 3 (C/C++ implementation) to the browser and released an HTML5 demo game based on Unreal Engine 3 called Epic Citadel. These games were built using WebGL, Emscripten, and the high-performance JavaScript subset asm.js, redefining the performance of 3D games running in the browser and showcasing how advanced games can easily be ported to JavaScript and WebGL platforms without sacrificing performance or visual effects.

Mozilla's multiplayer online HTML5 game—BrowserQuest.

With the improvement of new HTML5 features such as Full Screen API, Gamepad API, and Mouse Lock API, browsers could become the new battleground for future gaming.

Enhanced JavaScript

JavaScript's extensive extensions significantly compensate for certain shortcomings in its features. For example, you can use the following tools or extensions to make JavaScript more powerful:

Security enhancements

ADsafe: Provides client-side static verification functions and related APIs to enhance third-party script security.

Caja: Converts traditional HTML and JavaScript into secure-restricted JavaScript formats.

Dojo Secure: A framework for building secure mashup applications.

FBJS: Facebook's JavaScript toolkit for developing Facebook widgets.

Jacaranda: Supports object-based static verifiers.

Gatekeeper: A JavaScript password protection script.

Microsoft Web Sandbox: Ensures security through isolation.

Static typing features

JSX: A faster, safer, and more user-friendly JavaScript alternative.

Elm: A type-safe functional language that compiles into HTML, CSS, and JavaScript.

Lambdascript: A strongly-typed pure functional prototype language.

MileScript: A strongly-typed language similar to C# and Java that compiles into JS.

SafeJS: Adds syntactic sugar to JavaScript.

Mascara: Adds powerful features to JavaScript, such as classes, namespaces, and type checking.

Roy: A functional language close to JavaScript semantics with type inference, structural types, sum types, and pattern matching.

JavaScript derivatives

Although JavaScript's position in web development cannot be shaken today, new languages continue to emerge, hoping to replace JavaScript by providing more powerful, complete, and elegant features.

CoffeeScript: Written in pure Ruby, CoffeeScript uses only the "essence" of JavaScript, discarding problematic parts such as global variable declarations and "with." It also adds many syntactic sugars to make the code more elegant and readable.

Dart: A class-based optionally typed programming language introduced by Google, which can run natively in a virtual machine or be compiled into JavaScript.

TypeScript: A superset of JavaScript that adds optional static typing and class-based object-oriented programming.

ClojureScript: Created by Rich Hickey, it aims to "do what JavaScript does" and compiles into JavaScript.

IcedCoffeeScript: An extension of CoffeeScript that adds two keywords, "await" and "defer," providing strong asynchronous control capabilities for both browsers and servers.

LiveScript: A branch of Coco and an indirect subset of CoffeeScript, improving object-oriented and imperative programming aspects while compiling into JavaScript.

Kaffeine: An extension of JavaScript syntax that closely resembles JavaScript and compiles directly into JavaScript.

For more tools, see: http://altjs.org/

Finally

Almost all mainstream languages can be compiled into JavaScript and executed in browsers across all platforms, demonstrating JavaScript's power and importance in web development. Additionally, there are numerous tools available on the internet that allow these languages to interoperate with JavaScript, which we will detail in future articles.

Most of the above tools are open-source projects. If you're interested, you can explore the relevant principles and technologies behind language conversion through their source code.
