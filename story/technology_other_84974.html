1. Reasons for not being able to open the door:

a. The "server1name=ktest" in the client's ftp.ini has been changed.

Please restore this or change it to match the name on the server side (mir200/!setup.txt).

Note: This is actually the server name that appears when you open the game.

b. The IP address on the server side hasn't been set properly. The IPs in the 5 files are all 127.0.0.1; if you're doing single-machine debugging, leave it as is;

For network debugging, change it to your machine's IP address (in Win9x, run winipcfg to see the IP address).

2. Unable to register an account:

a. The reasons can be complex, possibly due to the server-side IP not being set properly or issues with DBC.

First, try registering again. If it still doesn’t work, you can delete the HeroDB field and recreate it.

If it still doesn’t work, delete all the server-side software and reinstall it. It's a bit cumbersome,

but I can only do it this way. [.] b. When creating an ID or character, illegal characters are prompted. All registration data must be in English; you can also try Korean.

Be sure not to use spaces and illegal characters, fill in all fields, and try a few times.

3. Black screen after entering:

a. Version issue with mir.exe and mir.dat. Try to use the client files provided in the private server files.

b. After entering and pressing the prompt window, black screen appears: Reinstall the client, you've probably installed too many cheats, haha c. DirectX version issue

d. Your machine configuration issue Some machines cannot simultaneously act as both client and server.

(Thunderbird 1200xp+, 512M DDR memory, runs quite sluggish) e. Map issue (the maps in server\Mir200\Map should be copied to the client)

4. Screen flashes and exits after entering:

a. Have you replaced the mir.dat in c:\mirserver\mir200? Just replace it back. Remember to replace it with the one that's around 700k.

5. Can't find BDEadministrator in the control panel:

a. A method newly discovered by friend zigzag from Uprising Community, thank him for his efforts.

Find BDEADMIN.EXE in C:\Program Files\Common Files\Borland Shared\BDE and run it directly:)

If you can't find it, search for BDEADMIN.EXE in drive C.

b. The reason is that you have installed Office, which contains something like DB data source that conflicts with this.

Uninstall Office, and it's said that some drawing or image software also conflicts, so please uninstall them as well.

Also, remember a small issue: after installing database software, you must use a registration tool to register, otherwise it won't work.

The registration tool is available for download in that post. If none of the above methods work, you may need to reinstall the operating system.

c. DB2000 seems to come in two versions. One is about 3M, the other is about 7M. The 3M one is probably version 3.5.

The 7M one is version 5.x, look for the 7M one. No need to download the 3M one.

6. Can't create a character:

a. It must be a client issue. Replace the client's mir.dat with the one in c:\mirserver\mir200 that's around 700k,

Regarding the IP address issue, carefully modify the IP addresses in mir.ini and mirsetup.ini. If it still doesn't work, find a clean Legend Client 1.0 and reinstall it.

As for why the UNKOWN error occurs, it's recommended to change the name to English.

b. RUN30 file didn't start properly, causing this error. Please restart your computer.

7. Compilation of problems running private servers under XP:

a. Can't run M2Server.exe Solution: Stop the SSDP Discovery Service in Administrative Tools - > Services

b. After running M2Server.exe using the above method, it keeps showing "Not ready 127.0.0.1" Solution: Unknown, please help experts

c. Solution for black screen after creating character and entering the game: I think it's caused by M2Server.exe not reading the game information correctly, which is the previous problem.

Please help experts solve this. Apparently, everyone is encountering the above problems currently. If the above methods don't work,

then use the methods for Win98. Compilation of issues after entering the game:

1. GM operation issues:

a. First, enter GM mode by typing @gamemaster. To clone items, use @make,

for example, to clone Dragon Slayer use @make dragonslayer. To clone monsters, use @mob, for example, to clone Evil Snake use @mob evilsnake. To teleport someone,

use @recall, for example, to teleport someone named king to your side, type @recall king. The commonly used commands are the ones mentioned above.

b. In C:\mirserver\Mir200\Envir\AdminList, the names of the GMs are recorded. You can modify them yourself!

Then enter the game with the modified name and type @GameMaster to enter GM mode.

GM commands are located in C:\mirserver\Documents\COMMANDS. We need to study them together. I have many things I don't understand. Please experts give more guidance!

c. Reason for not being able to enter GM mode: In Mir200\Envir\AdminList.txt, fill in the character name, not the login ID.

2. No monsters spawn, no monsters, or monsters don't attack:

a. Place the monster refresh file Mongen.txt in c:\mirserver\mir200\envir directory. Is Mongen.txt empty?

Use the content from mongen******.txt files instead, don't use the empty one.

b. After changing the file, monsters still don't spawn, or they appear but don't move: 555555, this is due to your machine being too slow. I couldn't spawn with MongenHighlevel.txt,

and with MongenOriginal.txt, the monsters spawned but didn't move. I opened it up and made changes,

deleting unnecessary spawn locations. Now there are only pigs, Zuma, Wuma, Bai*men with monsters, this is already my p3 733 limit, adding more will spawn frozen monsters.

c. New solution for monsters not attacking and not spawning! Cover the private server file with the latest mir.dat file from the official game!

Enter the game, if the monsters don't attack you, just exit and re-enter, the monsters will automatically respawn!

I did this on a computer with not-so-good configuration, and it felt fine.

d. The reason monsters disappear after a few minutes in single-player mode is due to insufficient memory. The solution is to increase memory to 256M.

3. Cannot input Chinese, cannot switch input methods. Using paste method, the input bar turns into Korean.

But my machine works perfectly normal. It's estimated to be a Windows language code issue, because even when my friend copies the entire Legend client over, it's still like this.

4. Solutions for lag in private servers:

a. Open and modify the !setup file under mirserver\Mir200. Modify TestServerUserLimit= item,

this is the number limit on the server side. If it's too high, it will cause lag. The default value is around 5600, which is very laggy.

b. Delete small monsters such as chickens, sheep, dogs, deer, orcs, cats, snakes, spiders, horn flies, and turn off worms and lower level monsters.

Also, close Half-Orc Tomb, Centipede Cave (both), Mine Cave, Connection Tunnel.

This should greatly improve performance. As for how to modify, look for it in forums. Method for Chinese localization of private servers

2. Copy the files in markit_def/other inese to markit_def, then the NPC dialogue interface will be in Chinese.

(not all) 3. In Mir200\Envir\MapInfo.txt, change the English map names to Chinese. Also, here you can modify the paths to Pig Seven and Zuma Seven.

For example, D71601 37,35 -> D716 36,17 `````````` Stone Tomb Entrance, Pig Six, entering the Stone Tomb through 37 35 entrance will directly lead to Pig Six.

4. Modify the monster names in Mongen.txt to Chinese (not sure, haven't done it yet, should work)

5. Use a Chinese client. I have a half-moon penetration version with jump checks. However, the server isn't good, not sure if it can be downloaded.

Common Issues:

A bundle of medicine or random items disappears upon opening... Super monsters originally supposed to drop items don't drop.

In mir200\Envir\MonItems, these text files correspond to the monster drop rates.

You changed the monster names but didn't update these text files, so they definitely won't drop.

There might be a text file related to medicine packs, which gives 6 medicines when you open it.

The reason might also be due to incomplete modifications. Change the items in the monster drop files (MonItems directory) to Chinese.

IP addresses found online don't work. Due to the illegality of private servers, most private server IPs are changed regularly,

so generally avoid joining those personal servers that change locations frequently, as you won't enjoy playing. Try to find one or a few fixed,

larger-scale private server providers and keep their IP addresses for long-term use.
