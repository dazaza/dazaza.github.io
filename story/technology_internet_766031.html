JavaScript and Finite State Machines

by anonymous on 2013-09-04 14:12:00

A Finite-state machine (FSM) is a very useful model that can simulate most things in the world.

In short, it has three characteristics:

The total number of states (state) is finite.

At any given moment, it exists in only one state.

Under certain conditions, it transitions from one state to another.

Its significance for JavaScript lies in the fact that many objects can be written as finite state machines.

For example, there is a menu element on a webpage. When the mouse hovers over it, the menu displays; when the mouse moves away, the menu hides. If described using a finite state machine, this menu has only two states (display and hide), and the mouse triggers state transitions.

The code can be written as follows:

```javascript

var menu = {

// Current state

currentState: 'hide',

// Bind events

initialize: function() {

var self = this;

self.on("hover", self.transition);

},

// State transition

transition: function(event){

switch(this.currentState) {

case "hide":

this.currentState = 'show';

doSomething();

break;

case "show":

this.currentState = 'hide';

doSomething();

break;

default:

console.log('Invalid State!');

break;

}

}

};

```

It can be seen that the finite state machine approach provides clear logic, strong expressiveness, and facilitates event encapsulation. The more states an object has and the more events occur, the more suitable it becomes to use the finite state machine approach.

Additionally, JavaScript is a language with a lot of asynchronous operations, and the common solution is to specify callback functions. However, this can lead to messy code structure, making testing and debugging difficult. Finite state machines offer a better solution: linking asynchronous operations to changes in object states. When an asynchronous operation completes, the corresponding state change occurs, which then triggers other operations. This approach is logically superior to callback functions, event listeners, or publish/subscribe solutions, and helps reduce code complexity.

Below, I will introduce a finite state machine library called Javascript Finite State Machine. This library is easy to understand, helps deepen our understanding, and is quite powerful.

The library provides a global object `StateMachine`, and by using its `create` method, you can generate instances of finite state machines.

```javascript

var fsm = StateMachine.create();

```

When generating, you need to provide a parameter object that describes the properties of the instance. For example, a traffic light can be described as follows:

```javascript

var fsm = StateMachine.create({

initial: 'green',

events: [

{ name: 'warn', from: 'green', to: 'yellow' },

{ name: 'stop', from: 'yellow', to: 'red' },

{ name: 'ready', from: 'red', to: 'yellow' },

{ name: 'go', from: 'yellow', to: 'green' }

]

});

```

The initial state (initial) of the traffic light is green, and the `events` property lists the various events that trigger state changes, such as the `warn` event changing the green state to yellow, the `stop` event changing the yellow state to red, etc.

After creating the instance, you can query the current state at any time.

`fsm.current`: Returns the current state.

`fsm.is(s)`: Returns a boolean indicating whether state `s` is the current state.

`fsm.can(e)`: Returns a boolean indicating whether event `e` can trigger in the current state.

`fsm.cannot(e)`: Returns a boolean indicating whether event `e` cannot trigger in the current state.

Javascript Finite State Machine allows you to specify two callback functions for each event. Taking the `warn` event as an example:

`onbeforewarn`: Triggered before the `warn` event occurs.

`onafterwarn` (can be abbreviated as `onwarn`): Triggered after the `warn` event occurs.

Similarly, it allows you to specify two callback functions for each state, taking the `green` state as an example:

`onleavegreen`: Triggered when leaving the `green` state.

`onentergreen` (can be abbreviated as `ongreen`): Triggered when entering the `green` state.

Assuming the `warn` event causes the state to change from `green` to `yellow`, the order of triggering for the four types of callback functions mentioned above is as follows: `onbeforewarn` → `onleavegreen` → `onenteryellow` → `onafterwarn`.

Besides specifying separate callback functions for each event and state, you can also define generic callback functions for all events and states.

`onbeforeevent`: Triggered before any event occurs.

`onleavestate`: Triggered when leaving any state.

`onenterstate`: Triggered when entering any state.

`onafterevent`: Triggered after any event ends.

If there are asynchronous operations (such as Ajax communication with the server) inside an event's callback function, you may want to wait until the asynchronous operation finishes before changing the state. This requires the use of the `transition` method.

```javascript

fsm.onwarn = function(){

light.fadeOut('slow', function() {

fsm.transition();

});

return StateMachine.ASYNC;

};

```

In the code above, the callback function contains an asynchronous operation (`light.fadeOut`). To prevent the state from changing immediately, the callback function should return a `StateMachine.ASYNC` object, indicating that the state will not change temporarily. Once the asynchronous operation completes, call the `transition` method to make the state change.

Javascript Finite State Machine also allows you to specify error-handling functions that automatically trigger when an event occurs that is impossible in the current state.

```javascript

var fsm = StateMachine.create({

// ...

error: function(eventName, from, to, args, errorCode, errorMessage) {

return 'event ' + eventName + ': ' + errorMessage;

},

// ...

});

```

For example, if the current state is `green`, theoretically only the `warn` event can occur. If a `stop` event happens at this point, the error-handling function above will be triggered.

The basic usage of Javascript Finite State Machine is outlined above. For more detailed information, refer to its homepage.
