Six Steps to Ensure the Security and Stability of WEB Servers

by qingian007 on 2012-02-29 16:27:52

Maintaining the security of WEB servers in information security is one of the most challenging tasks. You need to find a balance between conflicting roles, allowing legitimate users access to network resources while preventing human sabotage. Two-factor authentication, such as the RSA SecurID authentication system, can ensure high confidence, and you might even consider it, but it may not be practical or economical for all website users. Despite conflicting goals, there are still six steps to help secure Web servers.

Assuming the use of both internal and external applications on a single server, organizations have two types of web applications: services facing external users and those serving internal users. It's important to deploy these applications on different servers. This can reduce the risk of malicious users breaking through the external server to gain access to sensitive internal information. If you don't have deployment tools available, at least consider using technical controls (such as process isolation) so that internal and external applications do not interfere with each other.

Use a separate development server for testing and debugging applications. Testing applications on a separate Web server sounds like common sense – indeed. Unfortunately, many organizations do not follow this basic rule, instead allowing developers to debug code or even develop new software directly on production servers. This is disastrous for security and reliability. Testing code can lead to failures on production servers, and developers submitting untested code can easily introduce security vulnerabilities. Advanced version control systems (such as Microsoft’s Visual SourceSafe) can help automate the coding/testing/debugging process.

Review website activities and securely store logs. Every security professional knows the importance of maintaining server activity logs. Since most Web servers are public-facing, auditing all internet services is crucial. Auditing will help you detect and combat attacks, and enable you to resolve server performance issues. In advanced security environments, ensure your logs are stored in physically secure locations – the safest (but least convenient) technique involves creating paper records of logs that cannot be altered by intruders unless they have physical access. You may want to use electronic backups, such as logging to a secure host, using digital signatures and encryption to prevent logs from being stolen and modified.

Train developers to write reliable and secure code. Software developers focus on creating application software that meets business needs, but often overlook the fact that information security is also an important business requirement. As a security professional, you have the responsibility to provide security training for Web server developers. You should educate developers about network security mechanisms to ensure their software does not violate these mechanisms. Training concepts such as memory leak attacks and handling isolation can greatly assist in writing secure applications.

Patch operating systems and Web servers. This is another piece of common sense, but it is often overlooked when administrators are overwhelmed by other tasks. Advisories from sources like CERT or Microsoft Security Bulletins remind people about software vendor announcements and more frequent patch releases for certain security vulnerabilities. Some tools, such as Microsoft’s Software Update Service (SUS) and RedHat’s update service, help automate this task. In summary, once a vulnerability is released, if it is not addressed, it will eventually be discovered and exploited.

Use application software scanners if necessary. If the burden is too great, you might consider using application software scanners to verify internal coding. Tools like AppScan from Watchfire help ensure that the code in the production environment has no vulnerabilities. Remember to stay security-conscious. A well-designed Web server structure should be based on sound security policies.

Implementing these six methods will help you establish a solid foundation.
