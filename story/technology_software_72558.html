PHP Version of the Exploit Program for the phpwind Management Permission Leakage Vulnerability

by xloong on 2008-06-02 22:22:10

Nightfire: There have been a lot of vulnerabilities these past two days. Yesterday, I released a 0day vulnerability for phpWind involving administrative privilege leakage along with an exploit program. The exploit provided was in Python. Today, there's one in PHP.

The root cause of the vulnerability is that MySQL discards ASCII characters 129-255 if they are deemed useless. After researching, I couldn't find why this happens. The biggest mistake PhpWind made wasn't in the code itself, but rather in not setting the username field as unique.

An attacker can create a new ID with the same name as an existing administrator and append an invalid ASCII character during registration. This results in two identical usernames appearing in the database, while the `forums.forumadmin` field still stores the original username. This allows the attacker to gain full front-end privileges.

### Explanation:

1. **Vulnerability Overview**: The vulnerability arises because MySQL ignores certain invalid ASCII characters (129-255), leading to duplicate usernames in the database.

2. **Exploit Process**:

- An attacker registers a username identical to an admin's username, appending an invalid ASCII character (e.g., ASCII 193).

- Due to MySQL's behavior, the username appears duplicated in the database.

- However, the system still grants administrative privileges based on the original username stored in the `forums.forumadmin` field.

3. **PHP Code**:

- The script automates the registration process by appending the invalid ASCII character (`chr(193)`) to the chosen username.

- It sends a POST request to the `/register.php` endpoint with the crafted data.

This exploit highlights the importance of proper input validation and unique constraints in database design.
