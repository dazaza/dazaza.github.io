5 Things iOS Can Learn From OS X

by geekzhang on 2013-05-06 20:47:44

Although Apple removed the word "Mac" from its desktop operating system and renamed it OSX without any clear purpose, we have indeed seen a large number of features borrowed from iOS. OSX and iOS share the same origin, so connecting their experiences is an important and crucial step to drive the sales growth of Mac computers through the popularity of iOS devices.OSX is hailed as Steve Jobs' true legacy, but are there any characteristics in OSX that iOS could learn from? Gigaom has written an article listing a few points, let's discuss them.Global SearchOne of the biggest differences between OSX and iOS is the communication rules between applications. We've talked about iOS's sandbox mode many times, and also mentioned that if files could be shared freely like in Android, it would feel better. This time, we're talking about improving the experience without modifying the core rules of iOS.In OSX, after selecting text characters, you can directly look them up in the dictionary using the right-click menu, or open the browser to search directly. iOS6 solved the global dictionary issue, but now it's not possible to directly select characters for searching. The only exception is Apple's own iBooks. In iBooks, you can not only look up characters in the dictionary, but also directly select them to search on search engines and Wikipedia.We know that achieving an experience like OSX on a phone is not very realistic, adapting user interfaces and opening communication between applications are all challenges. However, if iOS7 or future versions could provide global character web search, it would be good news.Multi-user AccountsWithout mentioning the history of adding multi-user account functionality in MacOS9, most people already know that having multi-user accounts is a basic function of desktop operating systems. But currently, this feature is only available on tablets with Android 4.2. Unlike iOS where logging into different accounts uses different iCloud services, Android's multi-user accounts truly support managing multiple user accounts.Google believes that tablets are different from phones, and compared to phones, tablets are more likely to have multiple users. In reality, phones also have scenarios where they need to manage all settings, applications, and experiences like a computer does. At least, iPads need this.Slideshow WallpaperAutomatically switching desktop wallpapers has become a built-in feature on both Mac and Windows. Wouldn't it be great if iOS could automatically change wallpapers too? Looking back, iOS didn't even allow you to change wallpapers initially. It was said that Steve Jobs learned of the demand for changing wallpapers from many users and thus made this compromise. Automatically changing wallpapers seems highly unlikely, but at least, can the lock screen wallpaper change automatically?Multitasking ControlIn OSX, Exposé, Spaces, and Dashboard are integrated into the Mission Control feature. Through these three functions, OSX can quickly manage windows and workspaces. I'm not saying I hope such a function will be realized on iOS, phones don't need it, nor is it very likely. What I'm talking about is changing the way multitasking switches.Do you remember that legendary test video of iPhone4 from many years ago? In the video, the way multitasking switched in that test version of iOS was different from now. Instead of showing a bunch of icons, it displayed the real-time status of background-running apps in a full-screen grid layout, which looked somewhat similar to HTC Sense 5's multitasking interface today.The greatest advantage of this method isn't necessarily how much it speeds up identifying background apps, but rather how it allows quick management of these apps. Currently, iOS's multitasking bar can only display four apps at a time in sequence, often requiring multiple swipes. Using the entire screen to show multitasking wouldn't have this problem. Apparently, the design of that test version was abandoned.Now, we can use the jailbreak plugin Multifl0w to achieve this function. But looking at other mobile platforms, Android, Windows Phone, BlackBerry, WebOS, and even Symbian3 all use preview methods to switch multitasking. Everyone would probably be happy if Apple brings back the abandoned design in the next version.VersionsThe old version retrieval feature was added in OSX 10.7 Lion. Similar to Time Machine, this feature allows users to retrieve past document versions. However, clearly, even if iOS adds an auto-save function for documents, an interface like Time Machine is not very realistic. Nevertheless, currently, the most commonly used text input tools on iOS, such as cloud note-taking tools like Evernote, already have a real-time save function. Apple could certainly add an auto-save plus trace-back function to its own iWork suite.
