Alibaba Tech Carnival - Exclusive Interview with Jiang Feng: Big Data Practices in Wireless Client

by anonymous on 2013-07-24 18:08:44

**Translation:**

On July 13th, during the Alibaba Technology Carnival event, senior technical expert Haiyuan Ning (alias: Jiang Feng) from Alibaba's Wireless Division shared insights on "Big Data Practices in Wireless Client Applications" (Alibaba refers to "mobile" as "wireless"). As one of the media outlets present, Blog园 was fortunate to conduct a special interview with Jiang Feng, discussing topics related to user behavior data collection and analysis in mobile client applications. Below, we share our key takeaways from the interview.

For mobile app developers, collecting and analyzing user behavior data while they use the app is a crucial step in improving the product. However, this process brings about three challenging issues: 1. How to reasonably design the format for data collection; 2. How to embed collection code into apps across different mobile platforms; 3. How to statistically analyze large amounts of collected data on the server side. Jiang Feng's wireless data team is dedicated to solving these problems. Initially, they addressed these challenges for various mobile apps within Alibaba Group. Once their solution matured to a certain stage, they made it freely available to the public, leading to the creation of —— Taobao Wireless Data Insights (http://wdm.taobao.com).

Mobile app developers can simply call the corresponding SDK for Taobao Wireless Data Insights on their respective mobile platforms to collect data based on their needs. The statistical analysis results of the collected data can be easily viewed on the Taobao Wireless website. Data collection, transmission, computation, and analysis are all automatically handled by Taobao Wireless Data Insights, so app developers don’t need to worry about them. This is similar to how Google Analytics is used for website traffic statistics —— embedding JavaScript code into web pages and then viewing the analytics results.

The biggest technical challenge in this process is handling big data. According to Jiang Feng, the number of log data entries uploaded daily from client devices has reached as high as 6 billion. Such an enormous volume of data is processed using Alibaba's Cloud Ladder system, which essentially is a Hadoop cluster. Alibaba has now achieved cross-data center Hadoop clusters. The ability to provide this service externally is based on years of experience in big data processing.

After the exclusive interview, what struck me most was not Alibaba's technical prowess in big data processing, but rather the way services evolve from internal solutions to external public services —— moving from addressing internal issues to offering public services. This represents a trend in the industry: repetitive work among different IT companies will eventually evolve into shared public services enjoyed by everyone.
