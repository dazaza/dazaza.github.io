### **Linux Server Setup Part 1: Installing MySQL**

**Version:** Mysql5.1.22-rc.tar.gz

After copying the installation package to `/usr/local`, execute the following commands:

```bash

groupadd mysql

useradd -g mysql mysql

cd /usr/local/mysql

scripts/mysql_install_db --user=mysql

chown -R mysql:mysql .

cp /usr/local/mysql/support-files/my-medium.cnf /etc/my.cnf

cp /usr/local/mysql/support-files/mysql.server /etc/rc.d/init.d/mysqld

chmod +x /etc/rc.d/init.d/mysqld

ln -s /usr/local/mysql/bin/mysql /usr/sbin/mysql

chkconfig --add mysqld

```

**Start MySQL:**

```bash

service mysqld start

```

**Enter MySQL:**

```bash

mysql

```

---

### **Linux Server Setup Part 2: Installing Apache**

**Version:** httpd-2.0.61.tar.gz

```bash

./configure --prefix=/usr/local/apache --enable-module=so

make

make install

cd /usr/local/apache/bin

./apachectl start

```

*Note:* If GCC (the compiler tool) is not installed, you will receive a prompt during installation. You can find it on the installation CD!

---

### **Linux Server Setup Part 3: Installing PHP**

**Version:** php-4.4.7.tar.gz

```bash

./configure --prefix=/usr/local/php \

--with-apxs2=/usr/local/apache/bin/apxs \

--with-config-file-path=/usr/local/lib \

--enable-track-vars \

--with-xml \

--with-mysql=/usr/local/mysql \

--with-mbstring \

--enable-mbstring=all

make

make install

```

**Explanation:**

- `--with-apxs=dir`: Compile as an Apache module.

- `--with-config-file-path=dir`: Path for `php.ini`.

- `--enable-track-vars`: Track variables.

- `--with-xml`: XML support.

- `--with-mysql=dir`: MySQL support.

---

### **Linux Server Setup Part 4: Testing MySQL, PHP, and Apache**

Copy `php.ini-dist` to `/usr/local/lib`:

```bash

cp /php_directory/php.ini-dist /usr/local/lib

cd /usr/local/lib

mv php.ini-dist php.ini

vi php.ini

```

Modify:

```ini

register_globals=On

```

Save and exit (`:wq`).

Edit Apache configuration:

```bash

cd /usr/local/apache/conf

vi httpd.conf

```

Find and modify:

```apache

DirectoryIndex index.html

```

to:

```apache

DirectoryIndex index.php index.html

```

Change:

```apache

Options Indexes FollowSymLinks MultiViews

```

to:

```apache

Options FollowSymLinks MultiViews

```

Add:

```apache

AddType application/x-httpd-php .php

AddType application/x-httpd-php-source .phps

```

Save and exit (`:wq`).

Restart Apache:

```bash

cd ../bin

apachectl restart

```

```bash

cd /usr/local/apache/htdocs

vi test.php

```

```php

```

Save and exit (`:wq`).

Open IE and enter:

```

http:///test.php

```

If you see the PHP test page, the setup is complete!

### **Linux Server Setup Part 5: Installing Zend**

**Version:** Zend Optimizer v3.0.gz

```bash

./install.sh

```

This is simple; during installation, you will be prompted to specify the paths for `php.ini` and `apache/bin`. Adjust accordingly.

Edit `php.ini`:

- Change optimization value from `15` to `1023` to enable all optimization threads.

- Add:

```ini

zend_optimizer.enable_loader=0

```

Save and restart Apache.

**Install phpMyAdmin**

**Version:** phpMyAdmin v2.10.0.gz

Extract and copy to the `htdocs` directory of Apache.

Access via:

```

http://localhost/phpmyadmin/index.php

```

### **Linux Server Setup Part 6: Installing Discuz**

**Version:** Discuz 6.0

**Steps:**

1. Extract the files.

2. Upload all files and directories from the `upload` folder in `Discuz!6` to a web-accessible directory (e.g., using SSH Secure Shell Client).

3. Set directory permissions to `777` for:

```

./templates

./templates/default

./templates/default/*.*

./attachments

./customavatar

./forumdata

./forumdata/cache

./forumdata/templates

```

4. Configure database information by editing `config.inc.php`:

```php

$dbhost = 'localhost'; // Database server

$dbuser = 'dbuser'; // Database username

$dbpw = 'dbpw'; // Database password

$dbname = 'discuz'; // Database name

$adminemail = 'admin@your.com'; // Admin email

$dbreport = 0; // Send DB error report? 0=no, 1=yes

```

*Note:* The database does not need to be created beforehand; it will be automatically created during installation.

5. Execute the installation script by entering:

```

http:////install.php

```

After successful installation, delete `install.php` to prevent reinstallation.

*Note:* Installation issues are often caused by missing compilation tools. All operations have been tested.
