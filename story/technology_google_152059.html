The Crazy World of SEO Hidden Links and Malware

by logncn on 2009-05-22 09:18:21

Recently, many of my friends' websites have been compromised and had malicious code injected, causing them a lot of trouble with no place to complain. Alas, these days, the methods used for injecting malicious scripts are becoming more sophisticated, and dealing with them is quite challenging. While helping a friend deal with such an issue, I accidentally discovered a very unfamiliar CSS syntax in the code: "left:expression(1-900); top:expression(3-999);". Below this, there were countless links pointing to other websites. However, when I checked in the browser, I found nothing at all. I immediately remembered a familiar concept: [strong]hidden links[/strong].

It has to be said that this is a clever way of writing code. At least, after seeing it, I learned that "expression" could be used in this way. Of course, you can also laugh at my lack of knowledge if you wish. We all know that "expression" is an IE-only CSS property. Let's take a closer look at the official definition:

IE5 and later versions support the use of "expression" in CSS, which allows associating CSS properties with JavaScript expressions. The CSS property can either be an inherent property of the element or a custom one. This means that behind a CSS property, there can be a JavaScript expression, and the value of the CSS property will equal the result of the JavaScript expression calculation. Within the expression, you can directly reference the element's own properties and methods, as well as other browser objects. It’s as if the expression is a member function of the element.

From the definition, I roughly understand that this property can be compared to a JS function. And since we know that search engines currently cannot read JS, can we conclude that this method of hidden links is effective? Additionally, because it is an IE-only CSS property, it has no effect in other browsers, meaning the hidden links would become visible. If so, search engines would be even less likely to identify this as a method of hidden link cheating. What do you think?
