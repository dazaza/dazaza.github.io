CSS Chinese Font Unicode Encoding Scheme

by diglog on 2010-09-22 19:57:16

Using Chinese fonts in CSS typically involves directly setting the font name, such as setting the font to "SimSun" (宋体): `font-family: '宋体';`. However, this can lead to a problem where if the CSS encoding is incorrect, it may cause garbled Chinese characters in the font after the page loads. See the following example:

To solve this issue, you can use Unicode encoding for Chinese fonts. For example, the characters "宋体" (SimSun) become `\5b8b\4f53` after being encoded in Unicode. As an example, let's look at Taobao’s CSS: `font: 12px/1.5 tahoma, arial, \5b8b\4f53;` Here, the preselected Chinese font is the Unicode-encoded SimSun.

There is a quick way to get the Unicode encoding of a font. If you use the Firebug plugin in Firefox, you can directly type `escape('黑体')` in the console to obtain the Unicode encoding for "黑体" (Heiti). However, some processing is required before it can be used in a CSS file. See the following example:

Here we get `%u9ED1%u4F53`, which can be rewritten as `\9ED1\4F53` and then written into the CSS file.

This process might still seem a bit troublesome, but don't worry — here is a relatively complete table for reference:

| Chinese Name | Unicode Encoding |

|--------------|------------------|

| New Fine Ming | `\65B0\7EC6\660E\4F53` |

| Fine Ming | `\7EC6\660E\4F53` |

| Standard Kaiti | `\6807\6977\4F53` |

| Heiti | `\9ED1\4F53` |

| Songti | `\5B8B\4F53` |

| New Songti | `\65B0\5B8B\4F53` |

| FangSong | `\4EFF\5B8B` |

| Kaiti | `\6977\4F53` |

| FangSong_GB2312 | `\4EFF\5B8B_GB2312` |

| Kaiti_GB2312 | `\6977\4F53_GB2312` |

| Microsoft JhengHei | [...] |

You can refer to this table when setting Chinese fonts in your CSS files.
