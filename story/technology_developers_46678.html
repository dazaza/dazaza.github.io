Batch Processing

Batch processing, also known as a batch script or batch file, gets its BAT extension from the first three letters of "batch." There is no fixed format for creating a batch file, but it must follow this rule: each line can be considered a command, and each command can contain multiple sub-commands. Execution begins with the first line and continues until the last line, running on the DOS platform. One prominent feature of batch processing is its convenience, flexibility, powerful functionality, and high degree of automation.

To master batch processing, a solid foundation in DOS is essential! Of course, being mentally agile is also an important aspect.

Example 1: Here's a very simple batch script to get you familiar with it. Save the following commands as `name.bat` and execute it (in the future, only the code will be provided; saving and execution methods are similar):

```batch

@echo off

ping sz.tencent.com > a.txt

ping sz1.tencent.com >> a.txt

ping sz2.tencent.com >> a.txt

ping sz3.tencent.com >> a.txt

ping sz4.tencent.com >> a.txt

ping sz5.tencent.com >> a.txt

ping sz6.tencent.com >> a.txt

ping sz7.tencent.com >> a.txt

echo Press any key to exit...

pause >nul

```

Can you understand everything? Is it easy? But its function is very practical. After executing this batch file, a file named `a.txt` will be created in your current drive. The information recorded inside can help you quickly find the fastest QQ server, thus avoiding the painful process of "server relay." Here, `>>` means appending the results of the preceding command to the location specified afterward. The difference between `>>` and `>` is that `>>` appends the result to the next line while retaining the previous line's result, making the `a.txt` file grow larger over time (are you thinking of how to cause some trouble??). By the way, this batch file can also be combined with other commands to create a fully automated system for judging server speed, displaying the IP address of the fastest server after execution. More details will be introduced later.

Example 2: Here's another outdated example (`a.bat`):

```batch

@echo off

if exist C:\Progra~1\TencentAD*.gif del C:\Progra~1\TencentAD*.gif

echo Press any key to exit...

pause >nul

```

Why is this an outdated example? It’s simple—almost no one uses QQ with ads anymore, so it’s almost useless now. However, its past function was significant: deleting QQ advertisements to keep the chat window clean. The path used here is QQ's default installation directory, which you can modify according to your situation. This script uses the `if` command, allowing it to make real-time judgments and delete advertisement images effectively. As long as you don't close the DOS window after execution or forcibly terminate the command using `CTRL+C`, it will continue monitoring for ad images (QQ also continuously checks if its ads have been deleted). Of course, this script consumes a small amount of memory, hehe.

Example 3: Using a batch script to check if the IceRiver Trojan is present. The script content is as follows:

```batch

@echo off

netstat -a -n > a.txt

type a.txt | find "7626" && echo "Congratulations! You have been infected by GLACIER!"

del a.txt

pause & exit

```

Here, the `netstat` command is used to check all network port statuses. If you know the common ports used by trojans, it's easy to determine whether IceRiver has been installed. Of course, this isn't definitive because the default IceRiver port (7626) can be modified. What's introduced here is just the method and approach. With slight modifications, this script can be adapted to check for other types of trojans. Further modifications, adding parameters, port numbers, and an information list file, can transform it into an automatic detection script for all types of trojans. Hehe, sounds fun, right? The script also utilizes compound commands `&&` and the pipe command `|`, which will be explained in more detail later.

Example 4: Use a batch file to automatically clear system junk. The script is as follows:

```batch

@echo off

if exist c:\windows\temp*.* del c:\windows\temp*.*

if exist c:\windows\Tempor~1*.* del c:\windows\Tempor~1*.*

if exist c:\windows\History*.* del c:\windows\History*.*

if exist c:\windows\recent*.* del c:\windows\recent*.*

echo " Cleanup complete! Press any key to exit"

pause & exit

```

Save the above script content into `autoexec.bat`. Every time you boot up, it will automatically delete system junk. Two points to note: 1) DOS does not support long filenames, hence the appearance of `Tempor~1`; 2) You can modify it based on your actual needs to suit your requirements.
