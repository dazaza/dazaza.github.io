Wired: Google's Chrome OS Is Too Aggressive, Maybe It Should Take a Step Back

by a9107857 on 2009-11-22 22:36:53

Introduction: Although the idea of Google venturing into the operating system is beyond reproach, its strategy is overly aggressive. Since the wireless infrastructure and web applications have not yet been perfected, it will be difficult to completely replace the desktop system with Chrome OS. A more prudent approach would be for Chrome OS to coexist with existing systems and gradually capture the market. Below is the full article:

Extreme Concept

Sometimes, a step back can actually be an advance. Google should have adopted this strategy during the unveiling of its open-source Chrome OS on Thursday. Instead, Google positioned this operating system, which will be pre-installed on netbooks next year, as a challenger to Microsoft. Chrome OS is essentially an improved browser that allows netbooks to handle everyday computing through web-based applications. There's no desktop software, only the web.

The concept behind Chrome OS is extremely radical: go all-in on the web, moving everything online. As Ben Parr of the tech blog Mashable said, "This is a shift in thinking, aiming to make the web equal to the computer." However, due to current limitations in wireless networks and web-based applications, this vision appears overly idealistic and may take a few more years to fully realize. Google is aware of this and initially plans to promote this concept only on netbooks, hoping the general public will gradually accept this radical idea.

When discussing future development, Google stated that they plan to expand Chrome OS to more powerful devices, including laptops and desktops. However, we doubt whether users' enthusiasm for Chrome OS netbooks will truly meet Google's expectations, especially in the enterprise sector.

Unattainable

Chrome OS will offer fewer functions than even low-end netbooks. For instance, web applications cannot handle Microsoft Word documents, synchronize iTunes music libraries, or edit images using Photoshop. While netbooks are not suitable for software like Photoshop and Word due to their smaller keyboards and screens, they still became popular last year because of other advantages. On the contrary, Chrome OS aims to provide users with an operating system that cannot run such software.

Of course, as Google mentioned, all software can theoretically be replaced by web applications. For example, cloud storage can be used for backups, internet services can stream music and videos, and Google Docs can meet spreadsheet and word processing needs. The list goes on.

Google's idea is: abandon the computing experience we've grown accustomed to over the past decade, and let the Google browser handle everything. But why would users adopt this idea today, tomorrow, or even when Chrome OS launches next year?

Market researcher Interpret's Vice President of Strategy and Analysis, Michael Gartenberg, wisely summarized the current state of the computer market: "Most users today are looking for a combination of rich desktop applications and satisfactory web applications."

He further added: "The argument that users might give up rich desktop applications and handle all daily computing through a web browser alone has been discussed endlessly, and it will never happen."

Flaws Present

Although Chrome OS does offer certain benefits, these do not outweigh the losses incurred by eliminating all desktop software via the browser. The advantages include: increased security thanks to Google's monitoring of malicious software in Chrome OS applications; cost savings from cloud storage eliminating the need for external hard drives; and finally, as Google put it in its promotional video on Thursday: "You no longer have to worry about your computer."

No need to worry about our computers? But we must start worrying about you, Google. T-Mobile Sidekick users certainly understand this. The trio of Microsoft, T-Mobile, and Danger provided cloud computing services for T-Mobile Sidekick users, but a recent outage caused all data to be lost.

Nothing is perfect, and neither is Chrome OS. After all, Google's Gmail service experienced two outages in February and September this year. Although no data was lost, it caused hours of anxiety for users who increasingly depend on this email service.

In addition, there's the issue of cost. While desktop software fees can be saved, overall costs will rise when using Chrome OS. To keep this 24/7 online computer running, which relies solely on web applications, users must purchase data packages from carriers. In other words, monthly expenses will occur. Google stated that Chrome OS will offer caching capabilities, so not all tasks need to be conducted online, but the offline experience provided by caching cannot compare to fully functional desktop software.

Of course, if one could always access open Wi-Fi hotspots wherever they are, the wireless network problem would be solved. However, areas like Mountain View, California, where free Wi-Fi hotspots are common, are rare.

Taking a Step Back to Move Forward

In conclusion, Google's Chrome OS has great potential to gain widespread favor, and I am open-minded about it. However, for Chrome OS to succeed, Google should take a step back to move forward. Google should first develop Chrome OS into a "mini operating system" that coexists with other operating systems like Windows and target netbooks.

For example, Phoenix Technologies launched a mini operating system called HyperSpace, which has already been adopted by some netbooks. HyperSpace runs parallel to Windows, providing an instant-on environment that offers many web-centric features without needing to fully boot Windows. These features include multimedia players, browsers, VoIP, email, and instant messaging software.

This sounds very similar to Chrome OS, doesn't it? That's because the ideas are almost identical, except Phoenix Technologies is less well-known and takes a milder approach. HyperSpace is just an option, supplementing Windows rather than replacing it entirely. If a giant like Google were to adopt this method, it would yield greater returns. By the way, Phoenix Technologies is currently the leader in the BIOS (Basic Input Output System) market.

If Chrome OS were developed as a free mini operating system that users could download themselves and use alongside fully functional operating systems, Google could continuously integrate Chrome OS into more hardware, not just netbooks. Users could benefit from Chrome OS and its cloud computing services at convenient times, while using the main operating system for data backup or document handling at other times, resulting in a richer mixed experience.

Unfortunately, current web applications still lack many features necessary to meet all user needs. This situation cannot change quickly within a year, even for Google.
