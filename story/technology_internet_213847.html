The function of authentication ensures that only routers within the same area that have been authenticated can exchange routing information.

By default, OSPF does not use area authentication. Authentication functionality can be enabled using two methods: plain text authentication and message digest (MD5) authentication. Plain text authentication transmits the authentication password in clear text, which can be detected by network sniffers, making it insecure and not recommended for use. On the other hand, message digest (MD5) authentication encrypts the password before transmission, making it generally recommended to use this method for authentication.

When using authentication, all router interfaces within an area must use the same authentication method. To enable authentication, a password must be configured for each router interface in the router's interface configuration mode.

Task Command

Specify authentication: `area area-id authentication [message-digest]`

Use plain text authentication: `ip ospf authentication-key password`

Use message digest (MD5) authentication: `ip ospf message-digest-key keyid md5 key`

Below are examples of both types of authentication settings. The network distribution and address allocation environment in these examples are the same as the basic configuration example provided earlier, except that authentication functionality is used on Area 0 of Router1 and Router2.

Example 1: Using plain text authentication

Router1:

```

interface ethernet 0

ip address 192.1.0.129 255.255.255.192

!

interface serial 0

ip address 192.200.10.5 255.255.255.252

ip ospf authentication-key cisco

!

router ospf 100

network 192.200.10.4 0.0.0.3 area 0

network 192.1.0.128 0.0.0.63 area 1

area 0 authentication

!

```

Router2:

```

interface ethernet 0

ip address 192.1.0.65 255.255.255.192

!

interface serial 0

ip address 192.200.10.6 255.255.255.252

ip ospf authentication-key cisco

!

router ospf 200

network 192.200.10.4 0.0.0.3 area 0

network 192.1.0.64 0.0.0.63 area 2

area 0 authentication

!

```

Example 2: Message digest (MD5) authentication

Router1:

```

interface ethernet 0

ip address 192.1.0.129 255.255.255.192

!

interface serial 0

ip address 192.200.10.5 255.255.255.252

ip ospf message-digest-key 1 md5 cisco

!

router ospf 100

network 192.200.10.4 0.0.0.3 area 0

network 192.1.0.128 0.0.0.63 area 1

area 0 authentication message-digest

!

```

Router2:

```

interface ethernet 0

ip address 192.1.0.65 255.255.255.192

!

interface serial 0

ip address 192.200.10.6 255.255.255.252

ip ospf message-digest-key 1 md5 cisco

!

router ospf 200

network 192.200.10.4 0.0.0.3 area 0

network 192.1.0.64 0.0.0.63 area 2

area 0 authentication message-digest

!

```
