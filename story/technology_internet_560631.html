A Collection of Linux-.Htaccess File Usage Examples

by far9999999 on 2011-11-15 21:12:12

Given the many opportunities to use .htaccess files on Justhost and their wide range of applications, I have specifically translated a related article for your reference.

1. Time Zone Settings

Sometimes, when using the `date` or `mktime` functions in PHP, due to differences in time zones, strange information may be displayed. Below is one way to solve this issue by setting the server's time zone. You can find a list of all supported time zones here.

```

SetEnv TZ Australia/Melbourne

```

2. Search Engine-Friendly 301 Permanent Redirect Method

Why is this search engine-friendly? Because many modern search engines have the ability to update their existing records by checking for 301 permanent redirects.

```

Redirect 301 http://www.cometao.net http://www.gotao.org

```

3. Blocking Download Dialog Boxes

Typically, when downloading something, you will see a dialog box asking whether you want to save the file or open it directly. If you don't want to see this, you can add the following code to your .htaccess file.

```

AddType application/octet-stream .pdf

AddType application/octet-stream .zip

AddType application/octet-stream .mov

```

4. Removing the 'www' Prefix

An SEO principle is to ensure that your website has only one URL. Therefore, you need to redirect all visits through 'www' to non-'www', or vice versa.

```

RewriteEngine On

RewriteBase /

RewriteCond %{HTTP_HOST} ^www.cometao.net [NC]

RewriteRule ^(.*)$ http://cometao.net/$1 [L,R=301]

```

5. Customizing Error Pages

Customize personalized error pages for each error code.

```

ErrorDocument 401 /error/401.php

ErrorDocument 403 /error/403.php

ErrorDocument 404 /error/404.php

ErrorDocument 500 /error/500.php

```

6. Compressing Files

Optimize website access speed by compressing file sizes.

```

# Compress text, html, javascript, css, xml:

AddOutputFilterByType DEFLATE text/plain

AddOutputFilterByType DEFLATE text/html

AddOutputFilterByType DEFLATE text/xml

AddOutputFilterByType DEFLATE text/css

AddOutputFilterByType DEFLATE application/xml

AddOutputFilterByType DEFLATE application/xhtml+xml

AddOutputFilterByType DEFLATE application/rss+xml

AddOutputFilterByType DEFLATE application/javascript

AddOutputFilterByType DEFLATE application/x-javascript

AddOutputFilterByType DEFLATE image/svg+xml

```

7. Caching Files

Caching files is another good way to improve your website's access speed.

```

Header set Cache-Control "max-age=2592000"

```

8. Disabling Caching for Certain File Types

On the other hand, you can also customize to disable caching for certain file types.

```

# Explicitly disable caching for scripts and other dynamic files

Header unset Cache-Control

```

Security Issues

The following htaccess code can enhance the security level of your web server. Hotlink protection is very useful as it prevents others from stealing images hosted on your server.

1. Preventing Hotlinking via .htaccess

Hate those who steal links to images on your web server and exhaust your bandwidth? Try this; you can prevent such occurrences.

```

RewriteBase /

RewriteCond %{HTTP_REFERER} !^$

RewriteCond %{HTTP_REFERER} !^http://(www.)?cometao.net/.*$ [NC]

RewriteRule .(gif|jpg|swf|flv|png)$ /feed/ [R=302,L]

RewriteEngine on

RewriteBase /

RewriteCond %{HTTP_REFERER} !^$ [NC]

RewriteCond %{HTTP_REFERER} !cometao.net [NC]

RewriteCond %{HTTP_REFERER} !google.com [NC]

RewriteCond %{HTTP_REFERER} !gotao.org [NC]

RewriteCond %{HTTP_REFERER} !google.cn [NC]

RewriteCond %{HTTP_REFERER} !baidu.com [NC]

RewriteRule .*.(gif|jpg|jpeg|bmp|png|zip|pdf|doc|swf|flv)$ – [F]

```

2. Anti-Hacking

If you want to increase the security level of your site, you can remove the following lines of code, which can prevent some common malicious URL matching hacking techniques.

```

RewriteEngine On

# proc/self/environ? No way!

RewriteCond %{QUERY_STRING} proc/self/environ [OR]

# Prevent scripts attempting to modify mosConfig values via URL

RewriteCond %{QUERY_STRING} mosConfig_[a-zA-Z_]{1,21}(=|%3D) [OR]

# Prevent scripts passing base64_encode garbage via URL

RewriteCond %{QUERY_STRING} base64_encode.*(.*) [OR]

# Prevent scripts containing tags in the URL

RewriteCond %{QUERY_STRING} (<|) [NC,OR]

# Prevent scripts attempting to set PHP GLOBALS variables via URL

RewriteCond %{QUERY_STRING} GLOBALS(=|[|%[0-9A-Z]{0,2}) [OR]

# Prevent scripts attempting to set PHP _REQUEST variables via URL

RewriteCond %{QUERY_STRING} _REQUEST(=|[|%[0-9A-Z]{0,2})

# Redirect all blocked requests to the 403 forbidden page!

RewriteRule ^(.*)$ index.php [F,L]

```

3. Blocking Access to Your .htaccess File

The following code can block others from accessing your .htaccess file. Similarly, you can also set blocks for multiple file types.

```

# Protect your htaccess file

order allow,deny

deny from all

# Block viewing specific files

order allow,deny

deny from all

# Multiple file types

Order Allow,Deny

Deny from all

```

4. Renaming the htaccess File

You can protect the htaccess file by renaming it.

```

AccessFileName htacc.ess

```

5. Disable Directory Browsing

Prevent the server from displaying directory structures, or enable it.

```

# Disable directory browsing

Options All -Indexes

# Enable directory browsing

Options All +Indexes

```

6. Changing the Default Index Page

You can change the default index.html, index.php, or index.htm to another page.

```

DirectoryIndex business.html

```

7. Blocking Unwelcome Visitors via Referrer Information

```

RewriteEngine on

RewriteCond %{HTTP_REFERER} cometao.net [NC,OR]

RewriteCond %{HTTP_REFERER} cometao.net [NC,OR]

RewriteRule .* - [F]

```

8. Blocking Requests Based on Browser Header Information

This method can save bandwidth by preventing certain bots or spiders from crawling your site.

```

SetEnvIfNoCase ^User-Agent$ .*(craftbot|download|extract|stripper|sucker|ninja|clshttp|webspider|leacher|collector|grabber|webpictures) HTTP_SAFE_BADBOT

SetEnvIfNoCase ^User-Agent$ .*(libwww-perl|cometao_net_spiderman) HTTP_SAFE_BADBOT

Deny from env=HTTP_SAFE_BADBOT

```

9. Disabling Script Execution to Strengthen Directory Security

```

AddHandler cgi-script .php .pl .py .jsp .asp .htm .shtml .sh .cgi

Options -ExecCGI

```

10. Preventing Image Hotlinking

```

RewriteEngine on

RewriteCond %{HTTP_REFERER} !^$ [NC]

RewriteCond %{HTTP_REFERER} !google.com [NC]

RewriteCond %{HTTP_REFERER} !google.cn [NC]

RewriteCond %{HTTP_REFERER} !cometao.net [NC]

RewriteCond %{HTTP_REFERER} !baidu.com [NC]

RewriteRule .*.(jpg|gif|png|bmp|jpeg)$ xxx/forbid.png [R,NC,L]

```

11. Hiding Server Signature

```

ServerSignature Off

```

Hiding the server signature makes it harder for intruders to find security vulnerabilities because they won't know what server is behind it.
