How to Choose the Right Website Program

by sword500 on 2008-08-04 19:06:29

1. If you want to create a personal blog to publish your articles, photos, etc., in this case, you can directly choose ASP because it has been around longer and offers more complete functions for simple websites compared to PHP. Moreover, ASP is easy to learn; just read some books, and you can find good blog source codes online such as PJBLOG and Z-BLOG to build your own blog.

2. If you are only interested in making money online and hope to create a site to place ads and earn some money, I think ASP is still the best choice. First, ASP is easy to learn and takes less time; secondly, there are more ready-made ASP source codes than PHP, which makes it easier for you to choose.

3. If you want to create a website to introduce your company, school, etc., I still think ASP is a better choice for reasons similar to those mentioned above. Additionally, such sites don't have very high requirements (if you always feel that someone will try to mess with your website, then choose JSP).

4. If you want to create a URL navigation site, job recruitment site, or any other type of site, I still think ASP is better. The source codes for such sites in ASP are very abundant, and you can easily find one that suits you. You can quickly set up your site, and ASP's speed and security are sufficient for these types of websites.

5. If you want to create a medium-sized trading platform or a medium-sized corporate website (involving online product sales), I think PHP is better. It's fast enough and offers more functionality. You don't need to worry about PHP's scalability because this kind of site doesn't require it. PHP’s CMS systems are also quite useful.

6. If you want to create a dating site, chat site, or advanced forum, PHP is a great choice, especially when it comes to functionality and speed, where PHP far exceeds ASP.

7. If you want to create a software download site, movie streaming site, or music listening site, I think PHP is better. Although ASP has more source codes for these types of sites, PHP offers more applicable functions.

8. If you want to create a novel site, image site, or news site, I think PHP is still better. These sites have a large amount of data, which takes up a lot of space and puts significant pressure on the database. Using Access isn't ideal, whereas MySQL works much better. If you're using MySQL, why not choose PHP? Besides, PHP is significantly faster than ASP in terms of data collection.

9. If you want to create an enterprise system or business system, then use JSP.

10. If you plan to create illegal websites, I suggest you invent your own language, that way it would be the safest.

Conclusion: I believe webmasters are most concerned about what language is suitable for their site rather than what language they should learn to build their site. Trust in scientific development—everything now could eventually become obsolete, and it's impossible to keep up with every trend.
