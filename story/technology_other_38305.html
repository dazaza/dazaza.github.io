16. ||

The usage of this command is almost the same as &&, but its function is exactly opposite: using this method when executing multiple commands, it will exit the command combination upon encountering a correctly executed command and no longer continue executing subsequent commands. Title: Check if there are any exe files starting with 's' in the current directory; if so, then exit. Example 19:

@echo off

dir s*.exe || exit

In fact, this example has a flaw, can you see it? It's actually quite simple, try it yourself and you'll know: if the exe file exists, it exits; if the exe file does not exist, it also exits! Why? Because if the .exe file does not exist, the previous command dir s*.exe must definitely fail to execute, so it continues to execute exit, naturally exiting, hehe. So how do we solve the problem presented in the title? See Example 20:

@echo off

dir s*.exe || echo Didn’t exist file s*.exe & pause & exit

The result of this execution meets the requirements of the question; whether or not s*.exe exists, two results will appear. The reason for adding a pause here, of course, is to allow you to see the content output by echo, otherwise the window would flash by and the echo would be wasted.

For

Runs the specified command on each file in a set of files.

You can use the for command in a batch program or directly from the command prompt.

To use the for command in a batch program, use the following syntax:

for %%variable in (set) docommand [command-parameters]

To use the for command at the command prompt, use the following syntax:

for %variable in (set) do command [command-parameters]

Parameters

%%variable or %variable

Represents a replaceable parameter. The for command replaces %%variable (or %variable) with each text string specified in the set until the command (specified in command-parameters) processes all the files. Use %%variable to run the for command within a batch program. Use %variable to run the for command from the command prompt. Variable names are case-sensitive.

(set)

Specifies one or more files or text strings to process with the specified command. Parentheses are required.

command

Specifies the command to execute on each file contained in the specified set.

command-parameters

Specifies any parameters or switches to use with the specified command (if the specified command requires any arguments or switches).

If command extensions are enabled (the default setting in Windows 2000), additional forms of the for command are supported.

Additional Forms of the For Command

If command extensions are enabled, the following additional formats of the for command are supported:

Directories Only

for /D [%% | %]variable in (set) docommand [command-parameters]

If set contains wildcards (* and ?), it specifies matching directory names rather than filenames.

Recursive

for /R [[drive :]path] [%% | %]variable in (set) docommand [command-parameters]

Enters the root directory tree [drive:]path and executes the for statement in each directory in the tree. If no directory is specified after /R, the current directory is assumed. If set is just a period (.) character, it enumerates the directory tree.

Iterative

for /L [%% | %]variable in (start, step, end) do command [command-parameters]

The set is a sequence of numbers ranging from start to end divided by step size. Thus, (1,1,5) generates the sequence 1 2 3 4 5, and (5,-1,1) generates the sequence (5 4 3 2 1).

File Parsing

for /F ["options"] [%% | %]variable in (filenameset) do command [command-parameters]

for /F ["options"] [%% | %]variable in ("literal string") do command[command-parameters]

for /F ["options"] [%% | %]variable in (’command’) do command [command-parameters]

Or, if the usebackq option is present:

for /F ["options"] [%% | %]variable in (filenameset) do command [command-parameters]

for /F ["options"] [%% | %]variable in (’literal string’) do command [command-parameters]

for /F ["options"] [%% | %]variable in (`command`) do command [command-parameters]

The filenameset parameter specifies one or more file names. Each file is opened, read, and processed before moving on to the next file in filenameset.

The process involves reading the file, splitting it into separate text lines, and parsing each line into zero or more tokens. Then, the for loop body is called using the variable value(s) set to the found token string(s) (or multiple values). By default, /F passes the first blank-separated token from each line of each file.

Blank lines are skipped. You can override the default parsing behavior by specifying optional “options” parameters. This is a quoted string containing one or more keywords specifying different parsing options. Keywords are:

Keyword Description

eol=c Specifies the end-of-line comment character (only one character)

skip=n Specifies the number of lines to skip at the beginning of the file.

delims=xxx Specifies the delimiter set. This replaces the default delimiter set of spaces and tabs.

tokens=x,y,m-n Specifies passing tokens from each line to each iteration of the loop body. This causes other variable names to be assigned. m-n format is a range, specifying tokens from mth to nth. If the last character in the tokens= string is an asterisk, additional variables will be allocated, receiving the remaining text on the line after the last token is parsed.

usebackq Specifies executing the quoted string as a command, treating single-quoted strings as literal string commands, and allowing double quotes to include filenames in filenameset.

Variable Substitution

Additionally, enhancements have been made to the substitution modifiers for for variable references. Now you can use the following optional syntax (for any variable I):

Variable (with modifier) Description

%~I Expands %I removing any surrounding quotes (")

%~fI Expands %I to a fully qualified path name

%~dI Expands %I to a drive letter only

%~pI Expands %I to a path only

%~nI Expands %I to a file name only

%~xI Expands %I to a file extension only

%~sI Expands path to contain short names only

%~aI Expands %I to file attributes

%~tI Expands %I to date/time of file

%~zI Expands %I to size of file

%~$PATH:I Searches directories listed in PATH environment variable and expands %I to the first found fully qualified name. If the environment variable name is undefined, or no match is found after searching, this modifier expands to an empty string.

Modifiers can be combined for complex results:

Variable (with combined modifiers) Description

%~dpI Expands %I to drive letter and path only

%~nxI Expands %I to file name and extension only

%~fsI Expands %I to a fully qualified path name containing short names only

%~dp$PATH:I Searches for %I in directories listed in PATH environment variable and expands to the drive letter and path of the first found result

%~ftzaI Expands %I to a dir-like output line

Note

In the examples above, %I and PATH can be replaced with other valid values. Terminate the %~ syntax with a valid for variable name.

Using uppercase variable names (e.g., %I) makes the code more readable and avoids confusion with case-insensitive modifiers.

Shift

Changes the position of replaceable parameters in a batch file.

shift

When command extensions are enabled (the default setting in Windows 2000), the shift command supports the /n switch, which notifies the command to start changing from the nth parameter, where n can be any value from 0 to 8. For example,

SHIFT /2

changes %3 to %2, %4 to %3, etc., while leaving %0 and %1 unchanged.

Filter Commands

Filter commands can help you sort, view, and select parts of command output.

Passing Information Through Filter Commands

Filter commands can divide, rearrange, and extract some of the passed information. Windows 2000 has three filter commands:

more command displays file contents or command output one screen at a time.

find command searches for specific characters in files and command output.

sort command sorts files and command output alphabetically.

To send input from a file to a filter command, use the less-than symbol () and the filename to store the output. For example, the following command finds all occurrences of the string "Pacific Rim" in the Trade.txt file and saves the results in the Nwtrade.txt file:

find "Pacific Rim" nwtrade.txt

Sorting Text Files

The sort command sorts text files or command output alphabetically. For example, you can use the following command to sort the contents of the List.txt file and display the results on the screen:

sort ) and the filename in the command. For example, you can use the following command to sort the lines of the List.txt file alphabetically and save the results in the Alphlist.txt file:

sort alphlist.txt

To sort the output of a command, type the command followed by the pipe (|) and the sort command. For example, the following command sorts the output of the find command:

find "Jones" maillst.txt | sort

When typing this command, Windows 2000 lists alphabetically all lines in which "Jones" appears.

Merging Commands with Redirection Symbols

You can merge filter commands, other commands, and filenames to generate custom commands. For example, you can use the following command to store filenames containing the "LOG" string:

dir /b | find "LOG" > loglist.txt

Windows 2000 sends the output of the dir command through the find filter command and stores filenames containing the string "Log" in the Loglist.txt file. The results are stored as a list of filenames (such as A.log, Logdat.svd, and Mylog.bat).

To use multiple filters in the same command, separate the filters with pipes (|). For example, the following command searches every directory on the C drive for filenames containing the "Log" string and displays one screen at a time:

dir c:\ /s /b | find "LOG" | more

Because pipes (|) are used, Windows 2000 sends the output of the dir command through the find command. The find command selects only filenames containing the string "Log". The more command displays the filenames selected by the find command one screen at a time.
