Basic Mechanism of JavaScript Prototype Inheritance

by diglog on 2011-08-12 13:22:55

Due to the design of the language, JavaScript does not have a true concept of "classes." The typical method of using the `new` command to instantiate objects is actually an instantiation of prototype objects. The essence of this language feature relies on JavaScript's unique prototype chain (prototype chain) pattern. Therefore, in a strict sense, JavaScript is a prototype-based object-oriented language. This means that each instance object has a prototype. Objects inherit properties and methods from this prototype.

1. **Constructor Functions**

Using constructor functions, you can easily create objects. The `this` keyword within the constructor function points to the instance object itself:

```javascript

function People(name){

this.name = name;

}

```

Use the `new` operator and the constructor function to create instance objects:

```javascript

var people = new People('小明');

console.log(people.name); // 小明

```

However, if you create two instances, these instances cannot directly share properties and methods:

```javascript

var people1 = new People('小明');

var people2 = new People('小王');

people1.sex = 'male';

console.log(people2.sex); // undefined

```

In other words, once an object is instantiated, its properties and methods exist independently, and modifying one property will not affect other instances.

2. **Prototype**

Thus, there is the `prototype` attribute, which is automatically created when generating instance objects. It is itself an object that contains properties and methods that can be shared between instances. The instance's own properties and methods are included in the constructor function. In other words, the properties and methods inside the constructor function become local properties and methods after instantiation, while the properties and methods in the prototype (prototype) are only referenced in the instance, so they can be shared by multiple instances.

Let's add a `prototype` attribute to the previous constructor function:

```javascript

People.prototype.sex = [...]

```
