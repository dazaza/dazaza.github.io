Writing Dynamic Mobile Pages Using WML and ASP

Author: wap3g Release Time: 2008-02-24 12:13:34 Views: 12

The combination of ASP and WAP can generate rich, dynamic WML web pages, offering many interesting features to WAP-enabled phones. So how do we organically combine these two technologies to create dynamic web pages? In this article, I want to introduce how to use ASP to develop a dynamically generated WML application. For instance, mobile banking is currently very popular, so how would we build such an application? Here, I don't want to discuss the entire construction of a mobile bank because that content is too extensive and beyond the scope of what I can introduce. Instead, I will only introduce how to use ASP to build an application that checks a user's account balance. Considering its security, we need to use WTLS, but this issue is quite complex and can only be discussed in more detail later. We should focus on how to use ASP to achieve dynamic web pages.

After all this preamble, let's begin! Let's assume that the bank account information is stored in an SQL database, and if the appropriate network connection is used, access to our system is possible. In this example, for simplicity, we make the account information very small (which is not actually the case): it only includes the account number, customer name, security password, and account balance status, all stored in a table called `tbl_account_info` with four fields corresponding to `accountno`, `custname`, `accountpin`, and `accountbal`.

Next, I want to briefly talk about the programming approach: First, users enter our system interface and input their account number and security password (as shown in Figure 1). After verifying this login information, we allow the user to log in to the system, where we will display a welcome message along with the username and account balance status!

Figure 1

This is the general development approach. Now, we can start programming based on this approach. I have a habit: when writing dynamic program code, I first write the static code and then try to write the dynamic code. In this article, I will also adopt this method.

First, in the first part of the code, we will create a page to receive the user's login information (as shown in Figure 1).

Please note the following code, which consists of basic WML code:

```asp

// Note this line: To display Chinese characters, you must add encoding="gb2312"

Welcome to the WAP Electronic Bank

Enter Account Number:

Security Password:

```

After building the page to receive the user's login information, we need to validate whether the information entered by the user is invalid, such as incomplete account numbers or missing security passwords. These steps are usually performed on the client side, minimizing processing via wireless connections to reduce communication interruptions caused by congestion. After all, mobile phones are different from ordinary PCs; they have smaller bandwidths and slower connections.

Next, we will truly connect to the database. We will send a standard SQL query request, and the server will return a record set. From this record set, we extract the information we need and display it in the corresponding fields on the next page. All these contents are dynamic: obtaining request information from the user and then displaying the dynamic information they want to see, thus achieving "personalized mobile information." Please see the code below:

```asp

Response.Write " "
```

That's it! A simple electronic banking inquiry system has been completed. From the above example, we can see that when designing WAP pages, we need to pay attention to the following issues:

1. Keep the content concise and focused. The information provided based on user requests must be clear and quickly satisfy their needs!

2. Using ASP can very quickly complete the creation of dynamic WML pages. It is basically similar to using ASP to create regular pages, with only minor changes, such as adding WML page identification markers. You can also try writing other dynamic content.
