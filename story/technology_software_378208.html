So what exactly is going on? After some exploration, Xiao Wang finally discovered the hidden dirty tricks - rogue applications are to blame!

### 1. What Are Rogue Applications?

Simply put, rogue applications are akin to the notorious rogue software that gained attention a few years ago. They are developed or modified by a small group of developers/interest groups for illegal profits, utilizing the open characteristics of platforms like Android. Once these applications enter the user's phone, they will secretly carry out unauthorized and illegal operations in the background without permission, causing direct or indirect losses to the user.

Typically, rogue applications have the following characteristics:

- Forced or bundled installation

- Hijacking system resources/services

- Interfering with the normal operation of other software or system functions

- Difficult to uninstall completely using normal means

If rogue applications are installed, the most direct loss could be secret charges. Additionally, personal information privacy could also be threatened.

### 2. How Do Rogue Applications Secretly Charge?

By analyzing multiple APK packages of rogue applications on the Android platform, we can understand that rogue applications primarily go through the following process to charge:

Developers implant malicious charging code into the application during the development stage (usually at the initial entry point of the program), and eventually package the developed application into an APK and distribute it through various channels.

Users download the APK package of the application and successfully install it locally.

Thereafter, after each launch, the program automatically runs the malicious code, which may secretly collect the user's phone number, IMSI, ICCID, IMEI, etc., and send this information to a dedicated server (some even encrypt it to prevent being caught).

After receiving the information, the server returns instructions informing the application to execute a segment of malicious charging code, sending text messages to specified phone numbers (or secretly installing software, opening special URLs with the browser, etc.).

After sending the text message, the charging number generally returns a charging prompt. At this point, the malicious code forcibly intercepts the return receipt SMS, so the user receives no prompt and unknowingly pays the money.

The figure shows the charging code obtained by reverse engineering a certain application using technical means (click to enlarge).

In addition, some rogue applications themselves bundle additional third-party applications. According to the promotion effect of these third-party applications, they earn commissions from the bundled parties. Typically, for every successful promotion of a bundled application, the commission ranges from a few cents to one yuan. With the huge volume of bundled applications (especially those embedded in popular third-party ROMs), these rogue applications themselves also generate high profits.

Some rogue applications use both of the above methods to make excessive profits, earning income from both users and the bundled application parties, making them extremely malicious.

### 3. How Rogue Applications Succeed

- Unscrupulous developers/publishers implant rogue codes during the development phase to pursue short-term benefits.

- A large number of underground publishing channels for (cracked, beautified, localized, third-party) applications provide fertile ground for rogue applications.

- The review mechanisms for application uploads/publication on legitimate channels are not stringent enough.

- Users lack professional knowledge, identification skills, and vigilance, making them easy to deceive.

- There is a lack of corresponding detection and reporting tools on the market.

### 5. How to Deal with Rogue Applications

The most basic precaution comes from the user's conscious vigilance. Please obtain applications through legitimate channels and avoid unreliable third parties as much as possible. For uncertain quality applications, check their user reputation online first and compare against the rogue application blacklist. If already installed, it should be uninstalled quickly (before doing so, necessary evidence can be retained).
