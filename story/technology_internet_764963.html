What's going on with Github recently?

by anonymous on 2013-08-08 10:56:33

Recently, when visiting the Github.com website, there have occasionally been error pages like the following.

According to the Github status page, Github has been hit by DDoS attacks for several consecutive days. As of now, apart from some issues with the availability of the Github API, most other services have largely returned to normal, such as the Github website, storage services, Page services, and code download services.

Let’s take a look at what Github has been through in recent days.

7:36 UTC (15:36 Beijing Time): We are investigating abnormal traffic within unicorns.

7:40 UTC (15:40 Beijing Time): Currently, we are working to resolve an issue with increased database cluster load.

7:51 UTC (15:51 Beijing Time): The site is exiting maintenance mode.

7:56 UTC (15:56 Beijing Time): GitHub is available again, and we will continue to closely monitor its performance.

August 4, 2013

0:04 UTC: We are investigating an issue where Github.com is unreachable.

0:18 UTC: Github.com is recovering from a DoS attack. We will continue to monitor.

0:24 UTC: The DoS attack on Github.com continues, making most services unavailable. We are actively working to mitigate the attack.

0:28 UTC: Github.com is available again.

0:33 UTC: Github.com is experiencing a series of DoS attacks. We will suspend some services while continuing efforts to mitigate the attack.

1:33 UTC: Everything is running normally.

August 2, 2013

20:06 UTC: We are investigating an issue where Github.com is unreachable.

20:12 UTC: The site is recovering. We are investigating database issues.

20:22 UTC: Everything is running normally.

July 29, 2013

15:05 UTC: We are currently experiencing a significant DDoS attack, causing heavy data packet loss when accessing the site, and most services are unavailable. We are actively working to mitigate the attack.

16:10 UTC: We have weakened the DDoS attack, and the site should be responding normally. We are still investigating anomalies with the GitHub API.

22:23 UTC: Everything is running normally.
