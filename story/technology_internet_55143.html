Some Problem Solutions Collected Personally

1. When the C drive space is insufficient, delete temporary files on the C drive. The path is: C:\WINDOWS\Temp\Temporary Internet Files\Content.IE5.

2. When restoring a database from a backup file transferred from another server, after restoration is complete, you need to create a new database username in the database; otherwise, it will show that the password cannot be synchronized.

3. If the system slows down, you can recycle the application pool or restart IIS.

4. You should regularly clean up .rar/.mp3/.rm and other download and audio/video files on the server; otherwise, it may affect the server's operation.

5. If the website frequently encounters problems due to ASP virus infections, re-uploading might fix the issue. The main reason is the poor service provided by the rented hosting provider. They should perform regular virus scans. Also, there might be vulnerabilities in your ASP code.

6. If [url]www.xx.com[/url] opens slowly or shows a busy server while xx.com is fast, it’s because www cannot use intelligent parsing and needs to use alias parsing.

7. To change the owner of a database table, use the query analyzer: EXEC sp_changeobjectowner 'p_splitpage', 'dbo'.

8. Database error occurs, unable to connect. Check the settings in the backend website management.

9. When ASP.NET 1.0 and 2.0 are placed together, a "Service Unavailable" error may occur.

10. FTP upload can reach the DB folder but not the WEB folder. This indicates insufficient WEB space.

11. If the server can be pinged but the website cannot be opened, it might be that IIS has crashed. Restarting it should solve the problem.

12. Microsoft JET Database Engine Error '80040e09': Cannot update. The database or object is read-only.

- **Reason**: Read/write permissions for the database directory are not enabled.

- **Solution**: Ensure that the entire website has "Write" permissions enabled in IIS, not just the database file. Also, check if the website directory in Windows Explorer grants full permissions to the Everyone user. Specifically, right-click on the website folder, select "Properties," go to the "Security" tab, and add all permissions for the Everyone user.

- **Note**: If your system is XP, sequentially select "Tools" > "Folder Options" > "View," uncheck "Use simple file sharing," confirm, and then the "Security" tab will appear in the folder "Properties" dialog box.

13. Component-free upload doesn't work well. An ASP upload error may occur if an additional call code is added during the upload process.

14. If you don't know the backend password of the website, you can check the admin table in the database. If you see MD5 encryption, go to an MD5 website to find the corresponding value and overwrite the original MD5 in the table.

15. ASP page error: "Response object: 006~ASP 0251~Exceeds response buffer limit~This ASP page execution causes the response buffer to exceed its configured limit."

- **Solution**: Set the buffer before any other RESPONSE commands are run. Use `response.Buffer = True` at the beginning of the file.

16. Forum installation path: http://www.xxx.com/install.php

17. Microsoft OLE DB Provider for SQL Server Error '80004005':

- **[DBNETLIB][ConnectionOpen (Connect())]** SQL Server does not exist or access is denied.

- **Solution**: In local security settings under IP security policies, set "EP-sql" to unassigned, check if port 21 is open, and open port 1433. Change the SqlLocalName="(local)" to the IP address. In SQL Server properties > Security options, enable "Allow cross-database ownership chaining" (requires restarting SQL Server).

18. Problems encountered during database backup in the backend. Error number: 70, Error description: Permission denied, Error source: PE_Admin6.

- **Solution**: Log in via FTP to the server, find the admin directory, open it, and delete the Databackup directory. Then log back into the website backend to perform the backup.

19. For pseudo-static configuration in ASP.NET, right-click on the site - Properties - Main Directory - Configuration - Insert, then select c:\windows\microsoft.net\framework\v2.0.50727\aspnet_isapi.dll, and do not check [Detect if file exists].

20. Discuz staticization — PHP installation instructions:

- Download the Rewrite file from DZ, extract the compressed package to any directory (e.g., C:\Rewrite). In the IIS Information Service Manager - Website - Properties, click "Add" under the "ISAPI Filters" section. Enter "Rewrite" as the filter name, and the executable file is C:\Rewrite\Rewrite.dll. Restart IIS to make it effective.

- Similarly, configure in Web Extensions. Add everyone with Run and Read permissions for Rewrite.

21. Time zone setting: In the php.ini file, set date.timezone = PRC.

22. Changing database information: In the table of Enterprise Manager, where username='name'.

23. Image GD support: If not supported, and the PW forum image watermark feature is missing, search for gd in the PHP.INI file under C/WIN. Find ";extension=php_gd2.dll", remove the semicolon, and restart IIS.

24. If the website display is abnormal, it might be because short_open_tag = On in C:/WIN/PHP.INI.
