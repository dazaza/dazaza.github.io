MIX08入场券-微软推出一系列Windows Live API | 歪博2.0

by yetilfx on 2008-03-01 00:40:30

今天微软宣布推出一系列API ,在下周的MIX08活动之前。其中关键细节在David Treadwell（Windows Live 副总裁）的Blog中有披露。今天的新闻中谈到了其中的5个Live API、开发工具对web API的支持、以及Live 家族API 的整体技术架构（目前有超过20个API）。在过去的两三年里，微软越来越多的将MIX作为其在先技术平台的基础。

* AtomPub: 最值得注意的官方声明“微软斥巨资为开放服务制定统一的开发平台协议，基于Atom format (RFC4287) 和 the Atom Publishing Protocol (RFC 5023)标准.” 他们从Windows Live Spaces 照片API开始，到他们基于应用的存储服务，到Astoria ADO.Net数据服务，以及“在MIX领域带来的一些更多的惊喜”。自从Google’s GData（同样基于Atom）以来，这意味着最大的两个网络API提供商的网络服务均基于REST模型。当然，魔鬼在细节中，随着时间的推移，我们将看到如何实现，供应商将不可避免地有所不同。

* Windows Live Messenger Library (WLML): 这条新闻中最重要的信息是一个新的建立在Windows Live Messenger（原来的MSN，有3亿多用户)平台之上的JavaScript API，其允许网站定制自己个性化的聊天体验。你可以从这里获得这个API的更多细节。下面的屏幕截图，右侧展示了一个客户化的应用程序与左侧的标准Messager一样展示一样聊天内容。这一API支持 Messager所有的核心功能，像文本聊天、用户状态、图片和联系人列表管理。通过Windows Live ID进行登录的安全模型，使得网站上应用将不可能获得户的密码。这个函数库完全适用于运行在Windows和Mac平台上的IE和Firefox。那么有什么限制呢？它允许没月有一百万用户免费试用。注意这是一个更加强大的接口，不同于现在的Windows Live IM Control。

* Windows Live ID 代理认证: 这个最新的API和web控件给第三方应用程序一个安全的方式访问用户的Windows Live的数据，按照用户细粒度的权限定义，及什么可以访问和访问多久（参见这里）。想要使用这个服务，开发人员必须先注册从而获得一个应用程序ID。当用户想访问他们的Windows Live数据时，应用程序ID提供运行时支持（“请求应答”过程）。这会给程序一个“应答令牌”，返回一个“代理令牌”用来让应用程序访问给定的Live API和方法。这些令牌到期后需要被应用程序刷新。这将是一个重要的API在Live平台中，很多服务都会依赖于它。下面的幻灯片展示了这个服务和一个照片API间的交互。

* Windows Live Photo API: 这是 Windows Live Spaces Photo API一个重大的升级。这个API让第三方网站在Windows Live Spaces上创建照片应用程序。因为Live Space有庞大的用户基础，这个API让开发人用可以访问四十亿张图片。这个API现在支持WebDAV和AtomPub模型。这个API让你上传、下载和删除照片；创建、编辑和删除相册。这个API的变化之一和图片寄存模型是第三方开发人员不再需要保管或代管图片，但仍可以管理图片的元数据，抛开了存储并使用微软的服务。注意它使用Windows Live代理认证，用户通过代理授权许可来读或读/写照片。

* Application Based Storage API: 这个细节将在下周被发布在MIX,但是这是“一个实验性的API，允许应用程序开发人员存储少量状态/配置数据在Windows Live数据中心”。这个API具有一个AtomPub服务，使开发人员可以调用通过ADO.NET数据服务或其他AtomPub兼容的工具。这个的实际价值就是如果一个应用程序有成千上万的用户，那么就可以甩给Windows Live。”

* 最后但不是最不重要的更新是Silverlight流包含了对WebDAV的支持，也提供了新的示例代码在 mslivelabs，来帮助开发人员来开始，包括更新其现有的Tafiti应用 ( PW 聚合档案）连同源。
