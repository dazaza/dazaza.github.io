Windows Phone 7 Faces Six Enterprise Issues

[IT168 Commentary] In the coming weeks, Microsoft will reveal the features, services, and shortcomings of Windows Phone 7 in enterprise environments. This news is both positive and negative for enterprise IT departments.

Windows Phone 7 is a redesigned mobile operating system. At the recently concluded MIX Web Developer Conference, Microsoft detailed the Windows Phone 7 platform and related development tools. However, Microsoft executives have been evading questions about how Windows Phone 7 performs in enterprise mobility.

Microsoft's senior director of mobile communications, Todd Brix, said, "It’s not time to disclose everything yet; more information will be released this spring."

Based on reviews, responses, and hints, we've identified six issues that enterprises should consider regarding Windows Phone 7:

1. **Platform Shift**

- Microsoft's mobile platform is no longer enterprise-centric. The company now prioritizes delivering an attractive user experience for consumers.

- The marketing slogan is "Anyone can use Windows Phone 7 for both personal and business tasks." Brix stated, "With Windows Phone 7, our goal is 'to create truly appealing user experiences.' As such, we’ve overhauled many aspects of the operating system. While it meets enterprise needs, Windows Phone 7 focuses not just on enterprise applications but also on enhancing the user experience."

- The success of the iPhone and Google Android-based phones proves one point: user experience is the key focus for mobile operating systems, and enterprise users should be the target of independent software vendors (ISVs).

2. **Complete Break from Previous Windows Mobile Applications**

- Microsoft has confirmed that applications previously running on the Windows Mobile platform cannot directly run on Windows Phone 7. Although some applications may be adjusted to migrate to the new platform, the workload would be significant due to fundamental changes in application mechanisms.

- In older versions of Windows Mobile, applications ran like those installed on PCs, all atop the operating system kernel. In contrast, Windows Phone 7 offers two runtime environments for applications: XNA for games and Silverlight for other applications.

- Therefore, backward compatibility does not exist. You cannot directly install Windows Mobile applications written in C/C++ or other languages onto Windows Phone 7. They must be redesigned and rewritten. However, thanks to Microsoft's development strategy, much of the previous code can be reused.

3. **Existing Skills and Updated Microsoft Toolkits**

- Enterprise software developers and ISVs can leverage their existing skills and expertise widely applicable to Windows Phone application development, but they must adapt to the new mobile user interface.

- Few remember the launch of the first-generation iPhone, when Apple provided web developers with only the built-in Safari browser as a development tool. Yet, who would have thought that App Store applications would exceed 200,000? Microsoft holds an advantage here, as a considerable number of developers can pick up Microsoft's tools to write applications for Windows Phone, reusing much of the existing code—this highlights the power of the user base.

- Microsoft's latest core development tools—Visual Studio 2010, Silverlight 4, Expression Blend 4, and XNA Game Studio 4—all support Windows Phone development. These include a Windows Phone emulator running the complete Windows Phone 7 operating system. Visual Studio developers can design Windows Phone application interfaces through drag-and-drop controls, use existing Silverlight libraries or Azure cloud projects, making the process similar to developing Windows PC applications.

4. **Customized Windows Phone Marketplace for Enterprises**

- For mobile users, the redesigned Windows Phone Marketplace will be the sole destination for finding and downloading apps. At the MIX conference, Microsoft executives promised an "alternative" software distribution solution this spring, potentially including privacy protections on the Marketplace website. Employees could browse and download company-approved software or set up a private Marketplace on internal servers, providing IT departments with more control and security.

5. **Security and Management Infrastructure**

- Previously, Windows Mobile was tightly integrated with Microsoft's System Center Mobile Device Manager 2008. Microsoft Exchange ActiveSync allowed Windows Mobile devices to synchronize with Exchange Server, receive push emails, and offer security options like remote data wiping and encrypted connections. ActiveSync is the synchronization tool licensed to other platforms like the iPhone. In fact, the iPhone extended ActiveSync support, aiding its enterprise adoption, though Apple neither provides nor plans to offer an enterprise-grade security and management framework. Microsoft must address this gap.

- The question now is the extent of Windows Phone 7's support in this area. Based on Brix's hints, many functions may disappear, at least in the first release. Mobile VPN functionality might no longer be available.

6. **Microsoft Office and SharePoint Integration**

- From a user perspective, the most apparent enterprise connection in Windows Phone 7 is the "Office Hub," which allows users to create and edit Office documents, including Word, PowerPoint, and Excel. It also supports Microsoft's enterprise collaboration platform, SharePoint.

- Charlie Kindel, a project manager for Windows Phone application platforms and developer experience, stated, "The Office Hub is central, and SharePoint is a part of it."

- Windows Phone users can access SharePoint sites using their Exchange Server credentials. This means mobile groups can create Windows Phone 7 live tiles based on SharePoint documents and other information, opening, commenting, and editing them. Brix noted, "You can do all this within the SharePoint environment without email exchanges or internet browsing."

- Mobilizing SharePoint is a priority for Microsoft because it can generate strong enterprise interest. In this regard, Windows Phone 7 focuses on integrating Office OneNote 2007, a "digital notebook" designed to help users quickly create, collect, and search digital handwritten notes, along with other text, images, audio, and video content.

Additionally, Windows Phone will enable two-way synchronization with PCs via Wi-Fi, mobile networks, or USB cables.
