Method for configuring http.conf in Apache to bind multiple domains and subdomains

by dianke99 on 2008-08-05 17:09:44

Apache is one of the most popular HTTP server software, known for its speed and reliability (stability), and can be extended through a simple API. Perl/Python interpreters can be compiled into the server. It is completely free, and all source code is open-source. If you have your own server or VPS and don't want to buy an expensive control panel for convenience, then configuring Apache yourself becomes a required skill. Below is a brief introduction on how to bind multiple domains and their associated subdomains by setting the httpd.conf file in Apache (assuming the domains we want to bind are dianke99.com and xdbhvip.cn, with the subdomain blog.dianke99.com, and the independent IP is 72.167.11.30).

### How to Bind Multiple Domains in Apache

1. Open the `httpd.conf` file.

2. Change `ServerName 127.0.0.1` to `ServerName 72.167.11.30`.

3. Change `#NameVirtualHost *` to `NameVirtualHost 72.167.11.30`.

4. At the end of the file, add the virtual host configuration format:

```apache

ServerAdmin webmaster@dianke99.com

DocumentRoot "/path/to/dianke99.com"

ServerName dianke99.com

ServerAlias www.dianke99.com

ErrorLog "logs/dianke99.com-error_log"

CustomLog "logs/dianke99.com-access_log" common

ServerAdmin webmaster@xdbhvip.cn

DocumentRoot "/path/to/xdbhvip.cn"

ServerName xdbhvip.cn

ServerAlias www.xdbhvip.cn

ErrorLog "logs/xdbhvip.cn-error_log"

CustomLog "logs/xdbhvip.cn-access_log" common

ServerAdmin webmaster@dianke99.com

DocumentRoot "/path/to/blog.dianke99.com"

ServerName blog.dianke99.com

ErrorLog "logs/blog.dianke99.com-error_log"

CustomLog "logs/blog.dianke99.com-access_log" common

```

### Explanation:

- **ServerName**: Specifies the main domain name.

- **ServerAlias**: Allows additional domain names (such as `www.dianke99.com`) to point to the same site.

- **DocumentRoot**: The directory where the website files are stored.

- **ErrorLog & CustomLog**: Define the log files for error and access logs.

After completing these steps, restart the Apache service to apply the changes:

```bash

sudo service apache2 restart # For Debian/Ubuntu systems

sudo systemctl restart httpd # For CentOS/RHEL systems

```

This will allow Apache to serve multiple domains and subdomains using the same IP address.
