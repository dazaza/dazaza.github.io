When using the Windows XP operating system, you may notice that your system slows down. Upon checking the [Task Manager], you find that the CPU usage has reached 100%. What's going on here? Is it a virus, a hardware issue, or a problem with system settings? In this article, I will explain why system resources might be fully utilized from three aspects: hardware, software, and viruses.

Frequent occurrences of 100% CPU usage can mainly be attributed to some of the following issues:

Nine Possible Causes of High CPU Usage

1. Fault caused by anti-virus software

New versions of KV, Kingsoft, and Rising have added random monitoring for web pages, plugins, and emails, which undoubtedly increases the system load. Solution: There is no ideal solution except to use as few monitoring services as possible or upgrade your hardware configuration.

2. Unverified drivers causing 100% CPU resource usage

A flood of beta drivers online can lead to hard-to-detect fault causes. Solution: Pay special attention to graphics card drivers. It is recommended to use Microsoft-certified or officially released drivers and strictly verify the model and version.

3. Viruses and Trojans

Large amounts of worm viruses rapidly replicating inside the system can cause persistently high CPU resource usage. Solution: Thoroughly clean the system memory and local hard drive with a reliable antivirus software, and check for any abnormally started programs through system settings. Regularly update and upgrade antivirus software and firewalls, strengthen virus prevention awareness, and acquire correct knowledge about preventing and eliminating viruses.

4. Right-click on Control Panel - Administrative Tools - Services - RISING REALTIME MONITOR SERVICE, change it to manual.

5. Start -> Run -> msconfig -> Startup, disable unnecessary startup items, then restart.

6. Check the [svchost] process.

svchost.exe is a core process of the Windows XP system. svchost.exe not only appears in Windows XP but also exists in all Windows systems using the NT kernel. Generally, there are two svchost.exe processes in Windows 2000, while in Windows XP, there are four or more.

7. Check network connections. Mainly the network card.

8. Check network connections

When a computer with Windows XP installed acts as a server, upon receiving connection requests on port 445, it will allocate memory and a small amount of CPU resources to serve these connections. When overloaded, the CPU usage rate may become too high because there is an inherent trade-off between the number of work items and response capability. Determine the appropriate MaxWorkItems setting to improve system responsiveness. If set incorrectly, the server's response capability may be affected, or one user may monopolize too many system resources.

To resolve this issue, we can modify the registry: Open the Registry Editor and sequentially expand [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver]. In the right-hand window, create a new DWORD value named [maxworkitems]. Then double-click the value and enter the following number based on your computer's memory size: if the computer has more than 512MB of memory, enter [1024]; if less than 512MB, enter [256].

9. Check if Windows XP uses the mouse right-click causing 100% CPU usage

Recent reports indicate that using the mouse right-click in Explorer can lead to 100% CPU usage. Let's see what's happening?

Symptoms:

In Explorer, when you right-click on a directory or file, you may encounter the following problems:

Any copy operation at that time may stop responding.

Network connection speed significantly decreases.

All stream input/output operations, such as listening to music with Windows Media Player, may result in distorted sound.

Cause:

When you right-click on a file or directory in Explorer, and the shortcut menu appears, the CPU usage will increase to 100%, returning to normal levels only after closing the shortcut menu.

Solution:

Method One: Disable "Use transition effects for menus and tool tips"

1. Click [Start] -- [Control Panel]

2. In [Control Panel], double-click [Display]

3. In [Display Properties], click the [Appearance] tab

4. In the [Appearance] tab, click [Effects]

5. In the [Effects] dialog box, clear the checkbox before "Use the following transition effects for menus and tool tips (U)" and click [OK] twice.

Method Two: When right-clicking on files or directories, first use the left mouse button to select your target file or directory, then use the right mouse button to bring up the shortcut menu.

Solutions for 100% CPU Usage

In general, if the CPU is consistently at 100%, our computers tend to slow down. Often, we can solve this by making minor changes without needing expert help.

When the machine slows down, the first thing we think of is the Task Manager to see which program is consuming a high percentage. If it's a large program, that's understandable; after closing the program, if the CPU returns to normal, there's no problem. If not, you need to identify the program. If you cannot determine what the process is, search for it on Google or Baidu. Sometimes simply ending the process isn't enough. In Windows XP, we can combine the startup items in msconfig and turn off unnecessary ones. In Windows 2000, you can use winpatrol.

For commonly used software like browsers that consume a lot of CPU, consider upgrading the software or replacing it with similar alternatives. Sometimes software may be incompatible with the system, so try enabling the compatibility mode under the properties of the .exe file.

svchost.exe can sometimes be problematic. When you see a particular svchost.exe consuming a lot of CPU, you can download aports or fport to check its corresponding program path, i.e., what is invoking this svchost.exe. If it's not located in c:\Windows\system32 (XP) or c:\winnt\system32 (2000), it's suspicious. Upgrade your antivirus software and perform a virus scan.

Right-clicking on files leading to 100% CPU usage is another common issue. Sometimes pausing after a right-click might be due to this. The official explanation suggests selecting with the left mouse button first, then right-clicking (though this isn't entirely clear). Non-official methods suggest resolving it by disabling "Use the following transition effects for menus and tool tips" via Desktop -> Right-click -> Properties -> Appearance -> Effects. Additionally, certain antivirus software monitoring files can impact performance, so consider disabling file monitoring. Monitoring web pages, plugins, and emails works similarly.

Some driver programs may also exhibit such phenomena. It's best to choose Microsoft-certified or officially released drivers. Sometimes upgrading drivers can help, though remember the latest isn't always the best.

CPU cooling software utilizes all idle CPU time to cool down. Since Windows cannot differentiate between regular CPU usage and cooling instructions, the CPU always shows 100%. This does not affect normal system operation.

When handling large Word files, spell and grammar checks can strain the CPU. To alleviate this, go to Word's Tools -> Options -> Spelling and Grammar and uncheck "Check spelling as you type" and "Check grammar as you type."

High CPU usage after clicking on AVI video files occurs because the system scans the file, checks all parts, and builds an index. Solution: Right-click on the folder containing video files -> Properties -> General -> Advanced, and uncheck "Allow Indexing Service to index this folder for faster searches."

Case Analysis of 100% CPU Usage

1. dllhost process causing 100% CPU usage

Characteristics: A server's normal CPU consumption should be below 75%, and the CPU consumption should fluctuate. However, in this case, the CPU remains at 100% and does not decrease. Checking the Task Manager reveals that DLLHOST.EXE consumes all CPU idle time. Administrators often have to restart the IIS service. Strangely, after restarting IIS, everything seems normal, but the problem reoccurs after some time.

Direct Cause:

One or more ACCESS databases are damaged during multiple read/write operations. Microsoft's MDAC system writes into these damaged ACCESS files, causing ASP threads to enter a BLOCK state. Consequently, other threads must wait, resulting in an IIS deadlock, with all CPU time consumed within DLLHOST.

Solution:

Install the [First-class Information Monitoring Interception System] and use the [Chief File Inspector IIS Health Inspector] software.

Enable the "Find Deadlock Module," with the following settings:

--wblock=yes

Monitor the directory, please specify the directory where your host files are located:

--wblockdir=d:\test

The log file generated by the monitor is saved in the log directory of the installation directory, with the filename: logblock.htm

Stop IIS, then start the [Chief File Inspector IIS Health Inspector], and then restart IIS. The [Chief File Inspector IIS Health Inspector] will record the last written ACCESS file in logblock.htm.

After some time, when the problem reappears (e.g., the CPU remains at 100%), stop IIS and check the last ten files recorded in logblock.htm. Note that the most problematic ones are often counter-type ACCESS files, such as: **COUNT.MDB**, **COUNT.ASP**. You can move the last ten files or any suspected files to the Recycle Bin, then restart IIS and see if the problem reappears. We believe that after careful searching, you will definitely find the file that has been troubling you for a while.

Once you find the file, you can delete it or download it and repair it with Access 2000, solving the problem.

2. svchost.exe causing 100% CPU usage

In the win.ini file, under [Windows], [run=] and [load=] can be pathways for loading Trojan programs, so they must be carefully examined. Normally, there is nothing after their equal signs. If you find paths and filenames that aren't familiar startup files, your computer may be infected with a Trojan. Be cautious, as many Trojans, like the AOL Trojan, disguise themselves as command.exe files, making them hard to detect unless carefully inspected.

In the system.ini file, under [BOOT], there is a [shell=filename]. The correct filename should be [explorer.exe]. If it is not [explorer.exe] but instead [shell= explorer.exe program name], then the program following is a Trojan program, meaning your computer is infected.

The situation in the registry is more complex. Open the registry editor via the regedit command and navigate to [HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run]. Check if there are unfamiliar automatic startup files with EXE extensions. Remember, some Trojan programs generate files resembling system files, trying to pass unnoticed, like the Acid Battery v1.0 Trojan, which changes the Explorer key value under [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run] to Explorer=[C:\Windows\expiorer.exe]. The difference between the Trojan and the real Explorer is just the letters [i] and [l]. Of course, there are many places in the registry where Trojans can hide, such as [HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run] or [HKEY_USERS\****\Software\Microsoft\Windows\CurrentVersion\Run]. The best method is to find the Trojan's filename under [HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run] and then search the entire registry.

Let's first look at how Microsoft describes svchost.exe. In Microsoft Knowledge Base 314056, svchost.exe is described as the generic host process name for services running from dynamic-link libraries (DLL).

Actually, svchost.exe is a core process of the Windows XP system. svchost.exe doesn't just appear in Windows XP; it exists in all Windows systems using the NT kernel. Typically, there are two svchost.exe processes in Windows 2000, and in Windows XP, there are four or more. So seeing several svchost.exe processes in the system's process list shouldn't be alarming.

What exactly does svchost.exe do?

First, understand that Windows system processes are divided into two types: independent processes and shared processes. As Windows services continue to grow, Microsoft has implemented many system services in shared mode to save limited system resources. So what role does svchost.exe play in this?

svchost.exe serves as the host for these services, meaning it starts these services. svchost.exe merely provides the conditions for starting these services; it itself cannot implement any service functions or provide any services to users. svchost.exe starts system services by calling dynamic-link libraries (DLL).

Why is svchost.exe considered a virus?

Because svchost.exe can act as a host to start services, virus and Trojan writers exploit this feature to confuse users and achieve their goals of intrusion and destruction.

How can we distinguish between normal svchost.exe processes and virus processes?

The key values for svchost.exe are located at [HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\Svchost], as shown in Figure 1. Each key value represents an independent svchost.exe group.

Microsoft also provides a way to view the services running under svchost.exe in the system. For example, in Windows XP: in [Run], input: cmd, then in the command-line mode, input: tasklist /svc. The system lists the services as shown in Figure 2. The red-boxed area in Figure 2 shows the services started by svchost.exe. If you are using Windows 2000, replace [tasklist /svc] with [tlist -s]. If you suspect your computer might be infected with a virus and svchost.exe's services are abnormal, searching for the svchost.exe file can reveal abnormalities. Normally, you would only find one svchost.exe program in the [C:\Windows\System32] directory. If you find svchost.exe in other directories, it is likely infected.

Another way to confirm whether svchost.exe is infected is by checking the execution path of the process in the Task Manager. However, since the built-in Task Manager in Windows cannot display process paths, third-party process viewing tools are required.

The above briefly introduces the related situations of the svchost.exe process. In summary, svchost.exe is a core system process, not a virus process. Due to the special nature of the svchost.exe process, viruses will try every means to invade svchost.exe. By checking the execution path of the svchost.exe process, you can confirm whether it is infected.

4. Normal Software Causing 100% CPU Usage

First, if the situation persists from booting up until shutting down, it may be caused by a software program that logs in simultaneously with the system. You can open the [System Configuration Utility] by typing [msconfig] in the run command, then go to the [Startup] tab. Next, cancel the checkmarks before the suspicious options one by one, then restart the computer. Test repeatedly until you find the software causing the problem. Alternatively, you can achieve this purpose through optimization software like [Optimization Master]. Note: If a key on the keyboard is stuck, it could also cause this problem upon booting up.

If this problem occurs during computer use, you can bring up the Task Manager (WINXP: CTRL+ALT+DEL, WIN2000: CTRL+SHIFT[ESC]), then go to the [Processes] tab, and look at the [CPU] column. Find the program with higher resource usage from there (note that SYSTEM IDLE PROCESS is normal, its value is usually high, and its function is to tell you how much available CPU resource you currently have, so the higher its value, the better). Use the search function to find out which software this process belongs to. Then, the problem can be solved by upgrading, closing, uninstalling this software, or finding a similar software to replace it.

5. Viruses, Trojans, Spyware Causing 100% CPU Usage

The failure of 100% CPU usage is often caused by viruses and trojans, such as the Sasser virus. First, update the virus database and perform a full system scan. Next, use the anti-spyware software Ad-Aware to check for spyware. Many friends on forums have encountered the issue of svchost.exe taking up 100% CPU, which is often a sign of infection.

svchost.exe Windows system services are implemented in the form of dynamic link libraries (DLL). Some of them point the executable program to svchost.exe, which calls the corresponding service dynamic link library and adds the corresponding parameters to start the service. Due to its uniqueness and importance, it is more likely to become the host of some viruses and trojans.

8. AVI Video Files Causing 100% CPU Usage

In Windows XP, clicking on a large AVI video file may cause the system to freeze and make the exploere.exe process consume 100% CPU. This happens because the system needs to scan the file, check all parts, and build an index. If the file is large, it will take longer and cause 100% CPU usage. Solution: Right-click on the folder containing video files, choose [Properties --> General --> Advanced], and uncheck the checkbox before "Allow Indexing Service to index this folder for faster searches".

9. Antivirus Software Causing 100% CPU Usage

Modern antivirus software generally includes real-time monitoring functions for web pages, emails, and personal privacy, which undoubtedly increases the system load. For instance, playing games may become very slow. The most direct solution is to close the antivirus software.

10. High CPU Usage When Handling Large Word Files

These issues typically cause the computer to freeze, all due to WORD's spell and grammar checks. Simply open WORD's [Tools --> Options], go to the [Spelling and Grammar] tab, and uncheck the checkboxes before "Check spelling as you type" and "Check grammar as you type".

11. Network Connections Causing 100% CPU Usage

When your Windows 2000/xp acts as a server, upon receiving connection requests on port 445, the system allocates memory and a small amount of CPU resources to serve these connections. When overloaded, the aforementioned situation arises. This issue can be resolved by modifying the registry: Open the registry, locate HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver, create a new DWORD value named "maxworkitems" on the right side. Then double-click the value and set it to "1024" if your computer has more than 512MB of memory, or "256" if less.

Some incomplete drivers can also cause high CPU usage.

Frequently using the standby function can also cause the system to automatically disable hard disk DMA mode. This not only drastically reduces system performance and slows down system startup but also makes the system pause and show 100% CPU usage when running large software or games.
