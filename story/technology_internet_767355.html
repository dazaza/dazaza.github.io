30 Common Reactions Programmers Have After Encountering a Bug

by anonymous on 2013-10-13 13:52:49

English Title: 30 Common Reactions Programmers Have When Things Go Wrong

Developing applications is a highly stressful task, especially when you finish writing your code and suddenly a bug pops up. It can evoke a wide range of emotions and make you feel utterly miserable. Thus, learning how to calmly address each bug is an essential skill for every programmer to master.

The following article will share the 30 most common phrases programmers say when encountering bugs. See if any of these sound familiar!

1. I don’t know whether I should delete it or rewrite it.

2. Before starting the project, I should have looked on GitHub for a suitable framework.

3. Why does this script need so many libraries?

4. There must be a solution online somewhere.

5. Is there a plugin with this functionality?

6. I wonder if IE supports it.

7. Logically, this seems very logical.

8. It took me 30 minutes to write a function and 2 hours to make it work.

9. After reading a few blogs, I realized what I did was completely wrong.

10. Someone on Stack Overflow will help me.

11. It turned out to be a missing right parenthesis.

12. Let me grab a cup of coffee, step away from the monitor, and break the routine.

13. Let me set it aside and come back later.

14. Will classical music stimulate my programming ability?

15. Now is the perfect time to test the Ballmer Peak theory.

16. Did someone mess with my source code?

17. I have no idea what this means?

18. Just Google the error message directly.

19. It’s time to call it a day, but I really want to figure it out.

20. Oh my God! Why didn’t I add any comments?

21. It was working fine 20 minutes ago.

22. All because of a missing semicolon.

23. How much would it cost to hire someone to fix it?

24. A quick browse through Hacker News to boost my productivity.

25. This API doesn’t have any documentation.

26. I wish I had backed up this data in the database.

27. What’s the fastest solution to make this feature work?

28. I bet updating will solve this problem.

29. I’ll reorganize and study Git next week.

30. Forget it, let’s start over.

Dear readers, what do you say when you encounter bugs?
