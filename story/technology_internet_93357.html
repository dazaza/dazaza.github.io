Standardization of the Robots File for Search Engines - Age1983

by hexiaoyao on 2008-09-19 20:52:08

The three major search engines are fiercely competing, but they occasionally collaborate as well. Last year, Google, Yahoo, and Microsoft worked together to adhere to a unified Sitemaps standard. Just the other day, the trio announced their joint compliance with the robots.txt file standard. Google, Yahoo, and Microsoft each posted an article on their respective official blogs, revealing the standards for robots.txt files and Meta tags that all three support, along with some unique standards specific to each company. Below is a summary.

Records in the robots file supported by all three include:

**Disallow** - Instructs crawlers not to fetch certain files or directories. For example, the following code will prevent spiders from fetching all website files:

```

User-agent: *

Disallow: /

```

**Allow** - Informs crawlers which files should be fetched. Using Allow and Disallow together allows you to specify that most of the content in a directory should not be fetched, except for a specific part. For instance, the following code will prevent spiders from fetching other files under the "ab" directory but allow them to fetch files under "cd":

```

User-agent: *

Disallow: /ab/

Allow: /ab/cd

```

**$ Wildcard** - Matches characters at the end of a URL. For example, the following code will allow spiders to access URLs ending with ".htm":

```

User-agent: *

Allow: .htm$

```

**\* Wildcard** - Instructs spiders to match any sequence of characters. For example, the following code will disallow spiders from fetching all htm files:

```

User-agent: *

Disallow: /*.htm

```

**Sitemap Location** - Informs spiders where your sitemap is located. The format is:

tags users
