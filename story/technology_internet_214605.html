The PIX Firewall not only provides a scalable security solution but also offers failover support for certain models to ensure maximum reliability. The PIX Firewall uses a dedicated operating system, making it more secure and easier to maintain compared to software firewalls that often face threats and attacks due to their use of general-purpose operating systems. In this chapter, we will discuss methods for using the PIX Firewall to protect network assets and establish secure VPN access. This chapter includes the following sections: Controlling Network Access Preventing Attacks Launching Proprietary Protocols and Applications Establishing Virtual Private Networks System Management of Firewalls Failover for Firewalls Controlling Network Access (Controlling Network Access) This section will introduce the network firewall capabilities provided by the PIX Firewall, including the following content: How the PIX Firewall Works Adaptive Security Algorithms Multiple Interfaces and Security Levels Data Flow in the PIX Firewall Translation of Internal Addresses Cut-through Proxy Access Control How the PIX Firewall Works (How the PIX Firewall Works) The role of the PIX Firewall is to prevent unauthorized users from external networks (such as the public internet) from accessing the internal network. Most PIX Firewalls can selectively protect one or more peripheral networks (also known as demilitarized zones, DMZ). There are minimal restrictions on accessing external networks, moderate restrictions on accessing peripheral networks, and the highest restrictions on accessing internal networks. Connections between internal, external, and peripheral networks are controlled by the PIX Firewall. To effectively utilize firewalls within an organization, it is necessary to implement security policies to ensure that all traffic from protected networks only passes through the firewall to reach unprotected networks.
