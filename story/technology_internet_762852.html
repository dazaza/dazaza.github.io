Alibaba Tech Carnival - Exclusive Interview with Jiang Feng: Big Data Practices in Wireless Client

by anonymous on 2013-07-24 17:39:14

At the Alibaba Technology Carnival event on July 13th, senior technical expert Haiyuan Ning (alias: Jiang Feng) from Alibaba's Wireless Division shared insights on "Wireless Client Big Data Practices" (Alibaba refers to "mobile" as "wireless"). As one of the media outlets, Blog园 was fortunate enough to conduct an exclusive interview with Jiang Feng, discussing topics related to user behavior data collection and analysis on mobile clients. Below, we share our key takeaways from the interview.

For mobile app developers, collecting and analyzing user behavior data while using the app is a crucial step in improving the product. This process brings about three challenging issues: 1. How to reasonably design the format for data collection; 2. How to embed collection code into apps for different mobile platforms; 3. How to statistically analyze large amounts of collected data on the server side. Jiang Feng's wireless data team addresses these very problems. Initially, they solved this issue for various mobile apps within Alibaba Group. Once their solution matured to a certain extent, it was made freely available to the public, leading to the creation of Taobao Wireless Data Analysis (http://wdm.taobao.com).

Mobile app developers can simply call the corresponding platform's Taobao Wireless Data Analysis SDK to collect data according to their needs, and the statistical analysis results can be easily viewed on the Taobao Wireless website. The processes of data collection, transmission, computation, and analysis are all automatically completed by Taobao Wireless Data Analysis, leaving no concerns for app developers. It’s akin to using Google Analytics for website visit statistics—embedding a JS code snippet into the page and then viewing the results.

The biggest technical challenge faced here is big data processing. According to Jiang Feng, the volume of log data uploaded daily from client devices reaches as high as 6 billion records. The massive data computation is handled by Alibaba's Cloud Ladder, which is essentially a Hadoop cluster. Alibaba has now achieved cross-data center Hadoop clusters. Their ability to provide this service externally is based on years of experience accumulated in big data processing.

After the exclusive interview, what left the deepest impression on me was not Alibaba's technical prowess in big data processing, but rather the way services evolve from internal solutions to external public services—moving from solving internal problems to offering public services. This represents a trend in industry development: repetitive work among different IT enterprises will eventually evolve into a public service enjoyed by everyone.
