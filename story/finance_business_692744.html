Development Exercise of Notepad Application (Part I)

by dream2012 on 2012-03-06 11:22:43

In this exercise, you will learn how to add a second Activity to your Notepad application. This Activity will allow users to create, edit, and delete notes. It assumes that new notes are created in response to user input, and it packages itself into a return value Bundle provided by an intent. The goal of this exercise is:

- Build a new Activity and add it to the Android manifest.

- Call another Activity using the `startActivityForResult()` method asynchronously.

- Pass data between Activities using various Bundle objects.

- Learn how to use a more advanced screen layout.

### Step 1

Create a new Android project using the resources in the Notepadv2 directory within the NotepadCodeLab folder, similar to what was done in Exercise One. If you see an error regarding AndroidManifest.xml or the android.zip file, right-click on the project and select Android Tools > Fix Project Properties to correct the error.

Open the Notepadv2 project and review the following:

- Open and examine the strings.xml file under res/values for several new strings we'll use for the new features.

- Open and examine the top of the Notepadv2 class where you'll notice a new field pointer `mNotesCursor` defined along with several new constants.

- Note that the `fillData()` method has additional comments and now stores the note pointer in the newly defined field. The `onCreate()` method remains unchanged from Exercise One. The class member variable storing the note pointer is named `mNotesCursor`. The leading 'm' indicates that this is a member variable, conforming to Android coding style standards.

- There are also a couple of new overridden interfaces to implement (`onListItemClick()` and `onActivityResult()`).

### Step 3

Add a new branch statement for parameter `DELETE_ID` in the `onMenuItemSelected()` interface:

```java

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

```

Here, we delete the specified ID note through the `deleteNote()` interface. To get the specified note ID, we call `getListView().getSelectedItemId()`. Next, we call `fillData()` to keep the data updated. The full definition of the `onMenuItemSelected()` interface is as follows:

```java

@Override

public boolean onMenuItemSelected(int featureId, MenuItem item) {

switch(item.getItemId()) {

case INSERT_ID:

createNote();

return true;

case DELETE_ID:

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

}

return super.onMenuItemSelected(featureId, item);

}

```

### Step 4

Implement the `createNote()` interface:

Create a new Intent for the class `NoteEdit` to create a note (ACTIVITY_CREATE). Then call the `startActivityForResult()` interface to start the Intent:

```java

Intent i = new Intent(this, NoteEdit.class);

startActivityForResult(i, ACTIVITY_CREATE);

```

This form of Intent invocation is specific to the particular class in our Activity, which in this case is `NoteEdit`. Since the Intent class needs to send request information through the Android operating system, we must also provide a Context (`this`). The `startActivityForResult()` interface is called after the new Activity completes, calling back to our Activity. The callback interface in our Activity is called `onActivityResult()`, which we will implement in subsequent steps. Another interface to invoke the other Activity is `startActivity()`, which is a one-time invocation method (fire-and-forget). In this method, the current Activity does not receive a notification that the other Activity has completed, and we cannot obtain any result information from the `startActivity()` interface of the invoked Activity.

Don't worry about the fact that `NoteEdit` does not currently exist; we will address this soon.

### Step 5

Implement the overridden interface `onListItemClick()`:

`onListItemClick()` is a callback function we will override. It is called when the user clicks on an item in the note list. It has four parameters: the ListView object invoking the interface, the View clicked (exists in ListView), the position clicked (the position of the item in the list), and the row index of the clicked item (`mRowId`). In this example, we ignore the first two parameters (we can only have one ListView at this time) and the row index parameter. Our focus is entirely on the position of the selected item in the list. Through this parameter, we can obtain the desired data from the corresponding row and package it to send to the `NoteEdit` Activity.

In the implementation of this callback function interface, we create an Intent with the class `NoteEdit` to modify the note content, then add data to the extras bundle of the Intent passed to the invoked Activity. We pass the title, body, and row index of the note being edited. Finally, it calls the `startActivityForResult()` interface to start the Intent. Here is the complete code for the `onListItemClick()` interface:

```java

super.onListItemClick(l, v, position, id);

Cursor c = mNotesCursor;

c.moveToPosition(position);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

startActivityForResult(i, ACTIVITY_EDIT);

```

`putExtra()` is an interface used to add data to the extras bundle for triggering the Intent. Here, we pass the title, body, and row index of the note we want to edit. When we locate the corresponding position of the selected object in the list, the details of the note are available through the query pointer using `moveToPosition()`.

By adding information to the Intent's extras and using `startActivityForResult()`, we trigger the Intent and request type code (the request type code will be returned as a parameter `requestCode` to the `onActivityResult()` interface) in the `NoteEdit` class.

### Step 6

Open the `note_edit.xml` file and browse through it. This is the code related to the note editing interface.

This is the most complex interface layout we've handled so far. The file provided here avoids issues with fading in and out during code formatting. (XML has very strict rules about case and structure, and these errors often originate from layout problems.)

A new parameter we haven't seen before is used here: `android:layout_weight` (in this case, set to 1 in each instance). `layout_weight` assigns importance values to views within a linear layout. All views have a `layout_weight` value, defaulting to zero, meaning they occupy screen space according to their size requirements. If a value greater than zero is assigned, the available space in the parent view is divided based on the ratio of each view's `layout_weight` value to the overall `layout_weight` value in the current screen layout.

Example: Suppose we have a text label and two text editing elements horizontally aligned. The text label has no specified `layout_weight`, so it occupies the minimum required space. If both text editing elements have a `layout_weight` value of 1, they split the remaining width equally in the parent layout (because we declared them to have equal importance). If the first text editing element has a `layout_weight` of 1 and the second has a `layout_weight` of 2, the first gets one-third of the remaining space, and the second gets two-thirds (because we declared the second to have higher importance).

This layout also demonstrates how multiple layouts can be nested within others to achieve more complex and aesthetically pleasing layouts. In this example, a horizontal linear layout is nested within another vertical layout to align the title label and text field horizontally.

### Step 7

Create a `NoteEdit` class that extends `android.app.Activity`.

This is the first time we're creating an Activity without the help of the Android Eclipse plugin. When created this way, the `onCreate()` method is not automatically overridden. It's hard to imagine an Activity without overriding its `onCreate()` method, so this is the first task you need to complete.

### Step 8

Implement the definition of the `onCreate()` method for the `NoteEdit` class:

This method sets a title "Edit Note" (a string defined in the `strings.xml` file) for our new Activity and sets the note content view using the `note_edit.xml` layout file. We can obtain handles for the note title and text body views, as well as the confirm button. Thus, we can set the note title and body through these handles and bind the confirm button to respond to user actions when the button is pressed.

Next, we can unpack the parameter values transmitted through the extras bundle in the Intent and pre-fill the title, body, and view of the note for the user to edit. We then obtain and store the note row index (`mRowId`) to know which note the user is currently editing.

In the `onCreate()` method, establish the layout:

```java

setContentView(R.layout.note_edit);

```

Find the necessary editing and button components:

These resources can be found via the associated IDs declared in the `R` class, then converted to the correct view type (two text views are of type `EditText`, and the confirm button is of type `Button`):

```java

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

```

Note that `mTitleText` and `mBodyText` are class member variables (you need to declare them in the class before using them here).

Declare a private class member variable `Long mRowId` in the class to store the current row index value `mRowId` (if any).

Continue adding code in the `onCreate()` method to initialize the variables for the note title, body, and row index ID using the Intent extras data bundle (if present):

```java

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

```

We extract the title and body values set in the extras data bundle triggered by the Intent.

We also protect against assigning empty strings to the text fields.

Create an `onClickListener()` method for the button:

Listeners might be one of the most confusing aspects of UI implementation, but in this example, the effect we want to achieve is simple: call the `onClick()` method when the user presses the confirm button. Use it to do something and return some values of the edited note to the Intent caller. We achieve this through an anonymous inner class, which may seem confusing if you haven't seen it before. However, you should set aside confusion for now because there will be opportunities to refer back to this code segment and understand how to create a Listener and associate it with a button (Listener is a common term in Java development, especially in UI design and implementation). Below is an empty Listener:

### Step 9

Complete the implementation of the `onClick()` method in our Listener:

When the user presses the confirm button, this interface runs. We hope that this method can retrieve the title and text fields of the edited note and place these data in a return-value package so that these data can be passed back to the Activity that initially triggered this `NoteEdit` Activity. If the interface responds to an edit rather than a creation operation, we also need to place the row index ID in the return-value package so that the `Notepadv2` class can save the edits made to the note.

Create a data package (Bundle) as the return value. Pass the note title and body values, defined as constant keys in `Notepadv2`, to it:

```java

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

```

Pass this data package into a new Intent and end the Activity:

```java

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

```

This new Intent serves only as a simple data carrier (including the note title, body, and row index ID we want to pass).

The `setResult()` method sets the result type code and passes the Intent back to the caller of this Intent. In this example, everything works fine, so we return the result code `RESULT_OK`.

Calling the `finish()` method signals that the Activity is complete (equivalent to a return call). Any interfaces set in the result are returned together with execution control information to the result caller.

The complete implementation of the `onCreate()` method (plus some supporting fields for the class) is as follows:

```java

private EditText mTitleText;

private EditText mBodyText;

private Long mRowId;

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.note_edit);

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

}

});

}

```

### Step 10

Finally, define the new Activity in the manifest file:

Before the new Activity is recognized by the Android operating system, it needs its own Activity entry in the AndroidManifest.xml file. This way, the Android operating system knows that the Activity exists in the AndroidManifest.xml file and can invoke it. We can also determine which IntentFilters the Activity implements here, but we will skip this part for now and just let the Android operating system know that the Activity is defined.

The Manifest Editor included in the Eclipse plugin provides a convenient way to edit the AndroidManifest file, and we will use this plugin. If you prefer to directly edit the file or not use the Eclipse plugin, refer to the information below on how to edit the Manifest file without the new Manifest Editor.

Double-click the AndroidManifest.xml file in the folder tree to open it. Click the Application tab at the bottom of the Manifest Editor. In the application section, click Add.... If you see a radio button at the top of the dialog box, select the one labeled "Create a new element at the application level". Before clicking "OK", ensure that the "Activity" option in the dialog box tab is selected. In the Node part of the application, click the new "Activity", enter `.NoteEdit` in the Name* field, and press Enter.

The Android Manifest editor allows you to add more complex Activity entries in the file while showing other available options (but don't choose them unless you want to add them to your Manifest file). As you progress to more advanced Android applications, this editor helps you understand and modify the AndroidManifest.xml file.
