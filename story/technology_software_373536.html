A popular and practical compression tool that supports mouse drag-and-drop and shell extension, perfectly supports ZIP archives, and has built-in programs that can unpack various types of compressed files such as CAB, ARJ, LZH, TAR, GZ, ACE, UUE, BZ2, JAR, ISO, etc.; It has an estimated compression function, through which you can get the approximate compression ratio under three different compression methods using both ZIP and RAR compression tools before compressing a file; It also features history record and bookmark functions; The compression ratio is quite high, while resource usage is relatively low, with fixed compression, multimedia compression, and multi-volume self-extracting compression features that are not available in most compression tools; It is very simple and convenient to use, with not too many configuration options, allowing you to accomplish all your tasks directly within the File Explorer; For self-extracting archive files in ZIP and RAR formats (both DOS and WINDOWS formats), clicking on properties easily reveals the compression attributes of this file. Version 4.00 beta2:

1. When extracting or opening an encrypted archive, the password dialog box does not display the re-enter password verification field. This situation only occurs when the password is hidden.

2. If you enable the option for all archives, and input a default password (Ctrl+P) where the specified password is empty, then the extract and test commands will skip all encrypted archives.

3. If you prefer to use Install instead of Extract button in self-extracting modules but do not want to install any program, you can add the Setup= command in the archive comment, or enter it as the installation name in the advanced self-extraction options dialog box. We replaced Setup= from beta1 because in the advanced self-extraction options dialog box, specific parameters entered as installation names allow enabling this mode.

4. Bug Fixes:

a) If several selected archives are processed at once via the explorer context menu, the operation may fail;

b) When browsing disk root folders, the WinRAR window title did not show the current disk name;

c) The explorer context menu did not show decompression items related to 7-Zip self-extracting files;

d) Sorting by type does not work in archives opened with WinRAR.
