Wired: Google's Chrome OS is too aggressive, it should advance by retreating

by b48560406 on 2009-11-22 22:26:56

Introduction: The US magazine Wired wrote an article today stating that although there is nothing wrong with Google's foray into operating systems, its strategy is overly aggressive. Since the wireless infrastructure and web applications are not yet fully developed, it will be difficult to completely replace the desktop system with Chrome OS. Instead, Google should adopt a "retreat to advance" strategy, allowing it to coexist with existing systems and gradually capture the market. Below is the full text of the article:

Extreme Concept

Sometimes, retreating to advance is the best strategy. Google should have adopted this approach during the launch of the open-source Chrome OS on Thursday, but instead, Google positioned this operating system, which will be pre-installed on netbooks next year, as a challenger to Microsoft. Chrome OS is like an improved browser, enabling netbooks to handle daily computing through web-based applications. Yes, no desktop software, just through the web.

The concept behind Chrome OS is extremely radical: take the network path and move everything online. As Ben Parr, editor of the tech blog Mashable, said: "This is a shift in thinking aimed at making the internet equal to computers." However, due to current limitations in wireless networks and web-based applications, this idea seems overly idealistic for now and may take several more years to realize. Google is aware of this, so they initially targeted netbooks with this concept, hoping that the general public would gradually accept this radical idea.

When talking about future development, Google stated that they plan to expand Chrome OS to more powerful devices, including laptops and desktops. But we doubt whether users will really embrace Chrome OS netbooks as Google hopes.

Difficult to Achieve

Chrome OS offers fewer features, even less than low-performance netbooks. For example, web applications cannot handle Microsoft Word documents, synchronize iTunes libraries, or edit images with Photoshop. Due to their smaller keyboard and screen size, netbooks are not suitable for office software like Photoshop and Word, but they still gained popularity last year because of other advantages. In contrast, Chrome OS aims to provide users with an operating system that cannot use these softwares.

Of course, as Google says, all software can find alternative web applications. For instance, using cloud storage for backup, obtaining streaming music and video services through the internet, and Google Docs can meet spreadsheet and word processing needs. There are countless examples like these.

Google's idea is: abandon the computing experience we've grown accustomed to over the past decade and let the Google browser handle everything. But why would users adopt this idea today, tomorrow, or when Chrome OS launches next year?

Market research company Interpret's Vice President of Strategy and Analysis, Michael Gartenberg, gave a very insightful summary of the current state of the computer market: "Most users today seek a combination of rich desktop applications and satisfactory web applications."

He further added: "The debate about users possibly abandoning rich desktop applications and handling daily computing entirely through a web browser has been discussed repeatedly, and it will never happen."

Flaws Exist

Chrome OS does offer some benefits, but compared to the losses incurred by eliminating all desktop software through the browser, this method is not worthwhile. The benefits include: enhanced user security thanks to Google's monitoring of malicious software in Chrome OS applications; cost savings due to cloud storage eliminating the need for external hard drives; and finally, as mentioned in Google's promotional video on Thursday: "No longer worry about your computer."

No longer worry about our computers? But we need to start worrying about you, Google. T-Mobile Sidekick users surely understand this. Three companies—Microsoft, T-Mobile, and Danger—provided cloud computing services for T-Mobile Sidekick users, but recently experienced a service outage that resulted in the loss of all data.

There is no such thing as perfection, and Chrome OS is no exception. After all, Google's Gmail service also experienced two outages in February and September this year. Although no data was lost, it caused hours of anxiety for users who increasingly depend on this email service.

In addition, financial considerations must be taken into account. Although desktop software costs can be saved, the overall cost of using Chrome OS will increase. To keep a 24/7 online computer running that only uses web applications, users need to purchase data packages from operators. In other words, there will be monthly fees. Google states that Chrome OS will offer caching functions, so not all tasks need to be run via the internet, but the offline experience provided by caching cannot match feature-complete desktop software.

Of course, if open Wi-Fi hotspots are available everywhere, then the wireless network problem could be solved. But there aren't many places like Mountain View, California, where free Wi-Fi hotspots are readily available.

Retreat to Advance

In conclusion, Google's Chrome OS has great potential to gain widespread favor, and I am open-minded about it. But to make Chrome OS successful, Google should retreat to advance. Google should first develop Chrome OS into a "mini operating system" that coexists with other operating systems like Windows and target netbooks.

For example, Phoenix Technologies has launched a mini operating system called HyperSpace, which has already been adopted by some netbooks. HyperSpace is deployed alongside Windows, providing an instant-on environment that allows netbooks to access many web-centric features without fully booting Windows. These features include multimedia players, browsers, VoIP, email, and instant messaging software.

Doesn't this sound very similar to Chrome OS? This is because the ideas are almost identical, except that Phoenix Technologies is less well-known and their approach is more moderate. HyperSpace is just an option, complementing Windows rather than completely replacing it. If a giant like Google could adopt this method, it would yield greater benefits. By the way, Phoenix Technologies is currently the leader in the BIOS (Basic Input Output System) market.

If Chrome OS were developed into a free mini operating system that users can download themselves and use alongside feature-complete operating systems, Google could continuously integrate Chrome OS into more hardware beyond just netbooks. Users could benefit from Chrome OS and its provided cloud computing services when convenient, and at other times, they could use the main operating system to back up data or process documents, thereby gaining a richer mixed experience.

Unfortunately, current web applications still lack many functions and cannot meet all user needs. This situation cannot change quickly within a year, even for Google.
