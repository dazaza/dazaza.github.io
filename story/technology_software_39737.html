Kaspersky v7.0.1.321 Release Candidate MP1 Change Summary

by xloong on 2007-12-18 21:33:22

From: Yashi's Blog

-------------------------------------------------

Today, Kaspersky KAV 7.0.1.321 Release Candidate (final release candidate) has been released, and the official release of MP1 is getting closer. I would like to talk about the changes brought by MP1.

I have read some of the changes summarized by wangjay1980, and I will briefly summarize a few points he mentioned.

1. Further strengthening of active defense

We all know that Kaspersky's active defense is very excellent and highly effective in preventing unknown viruses. Many viruses and malware boast about being able to bypass Kaspersky's active defense. MP1 enhances the active defense; for example, methods such as bypassing Kaspersky through the HIVE registry or hooking into R0 (loading drivers) have been blocked by MP1. This makes it even more difficult to bypass the main active defense of Kaspersky 7.

2. Continuous improvement of heuristic scanning

In fact, Kaspersky has dedicated engineers responsible for the heuristic engine, who are constantly improving Kaspersky's heuristic scanning capabilities, whether it be detection rates or scanning speed. Additionally, MP1 will specifically add a heuristic component named kimul.dat, which is a hardware acceleration module. Of course, these features will only be available after the official release of MP1.

3. A humanized change in automatic updates

People often complained before, "Why does Kaspersky automatically update as soon as the computer starts up when the network isn't even connected yet?" After using MP1, you will find that Kaspersky now automatically checks if the network is connected before updating. If it is connected, then it will update. If you are offline, Kaspersky will not automatically update.

4. Resolution of system time issues

This may be one of the most headache-inducing problems for many Kaspersky users. Viruses and malware modify the system time, causing Kaspersky protection to stop, thus easily infecting the system. This method was one of the simplest and most effective ways to bypass Kaspersky's main defense. However, with the arrival of MP1, such viruses will become history. MP1 modifies the KEY logic, so no matter how you modify the system time, Kaspersky will not stop protecting; it will just prevent virus database updates.

5. Improvements in anti-spam, parental control, and firewall

This is also one of the key areas where MP1 aims to improve and enhance. Anti-spam reportedly uses Kaspersky's latest technology. Parental control focuses on improving detection rates and reducing false positives, solving legacy issues from version 125. The firewall mainly improves its efficiency (impact on internet speed), adds support for IP6, and includes some packet filtering rules.

6. Further enhancement of stability and smoothness

Stability generally refers to the likelihood of causing a blue screen or system crash. The chances of the program itself (AVP) freezing or crashing.

Smoothness generally refers to the smoothness of system operations, web browsing, and resource usage of the program itself (AVP).

MP1 has successfully achieved most of the above development goals. Whether it is good to use remains to be seen.
