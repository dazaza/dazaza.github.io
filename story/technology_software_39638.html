The Bandwidth Scheduler AppBand can allocate network bandwidth for Windows applications and specify the priority of data transmission and reception, achieving simple QOS scheduling for desktop applications through software.

(1) P2P download software (BT/eMule/Thunder etc.) or online playback occupies most of the single-machine bandwidth, while browsers and game programs often fall behind in the competition for bandwidth. It's very common to be unable to open web pages or have extremely high game ping while doing P2P downloads. AppBand can lower the bandwidth priority of P2P software, thus leaving enough bandwidth for other regular software, allowing users to browse the web smoothly and play online games with low latency.

(2) AppBand implements dynamic and accurate speed limitation. Not every P2P software can set speed limits for download tasks, and users cannot fully control their bandwidth usage.

(3) The bandwidth scheduling of AppBand cannot increase physical bandwidth, but only optimizes the allocation of limited bandwidth resources among different programs. P2P software that occupies bandwidth for a long time and has low real-time requirements can reasonably use the remaining bandwidth, while other key applications are not restricted at all.

Software functions:

... Static speed limit for application groups;

... Static reservation of minimum bandwidth for application groups;

... Setting low priority for application groups;

... Setting high priority for application groups and dynamically reserving minimum bandwidth;

... Overall speed limit for a single machine

This version is only applicable to WinXP SP2
