Every designer or front-end developer has their own effective method for dealing with IE. Each of these techniques has its pros and cons, so let’s take a look.

**CSS Hacks**: This is essentially a dirty approach that works but doesn't conform to standards, and it can be headache-inducing to look at. Most people are left helpless when faced with it, yet they have no choice but to use it. The previous article "CSS Hacks for IE – Making IE Perfect" already discussed CSS hacks for IE. Now, let's briefly revisit some commonly used methods:

```css

_selector { property: value; } /* IE6 */

*selector { property: value; } /* IE6 & IE7 */

selector { property: value\9; } /* IE6 & IE7 & ... */

```

These hacks allow you to target specific versions of IE, though they may not adhere to web standards. While they solve problems, they often require additional maintenance and can make your CSS harder to read and manage.
