Command Line Interfaces Making a Comeback? — Don Norman's Prediction for the Next Generation of User Interfaces_Human-Computer Interaction Observations

by cacard on 2007-06-16 01:58:23

In the third issue of the renowned human-computer interaction magazine *Interactions* in 2007, user experience master and cognitive psychologist Don Norman made a startling statement: the future of user interfaces will return to an era where command lines are the mainstream (the online version of this article can be found here). He said that although the advantages of graphical user interfaces (GUIs) are obvious, circumstances have changed. Due to the explosion of information, the sheer volume of items users face makes it increasingly time-consuming and labor-intensive to visually select and operate on targets directly. However, adopting new types of command-line interfaces represented by search boxes could significantly improve efficiency when people deal with massive amounts of information. Of course, Norman emphasized that the future command-line interface would not simply repeat the command-line interfaces from decades ago; it would have better error tolerance, stronger robustness, and even the ability to understand natural language. Moreover, the future command-line interface would not equal today's search engines (search engine), but rather would be a question-and-answer engine (answer engine).

Norman gave examples showing that some innovative products already exhibit a tendency to return to command lines. For instance, Gmail’s powerful search function and Vista’s desktop search feature. The use of these search functions weakens the previously widely used hierarchical file structure and may gradually change user habits. In his view, users will no longer get accustomed to opening folders level by level to find target files, but instead will directly input the characteristics of the target file into the search bar to retrieve the file, because under conditions where the number of files is excessively large, the latter is clearly more efficient than the former.

However, Norman seems to only present one side of the coin. Indeed, search-style new command-line interfaces do have direct and fast advantages, especially reducing the burden on the eyes for visual searches, but the weaknesses of command-line interfaces still stand out. I’d like to discuss two aspects from the perspective of human cognitive features:

Firstly, command-line interfaces may increase the difficulty for users to extract memories. Regardless of the learning process required for users to remember individual commands, even in terms of target searching and task operations themselves, command-line interfaces also increase the memory load on users. Because, when users look for a certain file (whether through searching or opening a hierarchical file system), they must first extract some characteristics of the file (file name, file type, path, etc.) from their long-term memory. There are two ways humans extract information from long-term memory: recall and recognition. For example, if you try to remember the appearance of a high school classmate, that is recall; if someone shows you a photo of a high school classmate and asks if you recognize them, that is recognition. Another example: answering essay questions in exams is recall, while multiple-choice questions are recognition. Clearly, recognition is far easier than recall, and the amount of information that can be recognized is much greater than what can be recalled. Under graphical user interfaces, users mostly extract information from long-term memory via recognition, whereas under command-line interfaces, users need to recall the characteristics of the file to input appropriate search keywords, which increases the difficulty of extracting memories. As Norman mentioned, in his personal computer, a single folder often contains thousands of files, so performing manual visual searches among these files is very time-consuming. But isn’t it even more difficult to remember the names of these thousands of files and recall them when needed?

Secondly, hierarchical file structures match the way knowledge is stored in the human brain better. According to the operation method of command-line interfaces, users don't need to know the exact path of file storage, nor do they need to know which folder the file belongs to or what the parent folder of that folder is. That is to say, even if tens of thousands of files are chaotically stored in the computer, it won’t affect user operations. However, the knowledge system inside the human brain isn't like this. A dominant theoretical school in educational psychology is constructivism, which holds that the learning process and the establishment of a knowledge system is not a simple process of receiving information, but rather requires constructing newly received information onto the existing knowledge system. Each person's knowledge system must be structured and logically organized in a hierarchical system. If the information a person receives is scattered and unorganized, it will soon be forgotten. Similar to the operation of the human brain, personal computers also have a user-customized file hierarchy system, which reflects each user's organization architecture for the information in the computer. The result of information explosion is that we face more and more files, making them harder to manage. Of course, using search as a way to avoid this trouble is a solution, but on the other hand, using a more appropriate organizational method to arrange and classify files in a more logical manner seems to be a more proactive approach.

Historical experience shows that even though emerging things have great appeal, old things won’t disappear quickly, and may even survive more tenaciously. Although Google's stock price has reached $500, a single small search box will not truly satisfy everyone's taste, and command-line interfaces cannot replace graphical user interfaces. Let me give two real-life examples:

One, the field where humans first faced the problem of handling massive amounts of information was libraries. Libraries appeared much earlier than the Internet and search engines. So what insights can libraries offer us? Even before electronic retrieval systems existed, large libraries were able to manage vast collections of books and materials in an orderly manner and allowed borrowers to quickly find the books they needed, thanks to a powerful classification and cataloging system. Every book was reasonably placed in a specific category within a subcategory. Users could quickly find the book they needed, and more importantly, nearby books in the same category might spark their interest. Nowadays, all libraries are equipped with electronic retrieval systems, but classification and cataloging systems still play an important role. Classification and cataloging systems are not just methods for helping find books, but also ways to organize knowledge, making the entire library an organic whole. Often, after using a search engine and getting a bunch of disordered search results, such classification and cataloging systems become something people miss.

Two, why do women love shopping? The fun of shopping for clothes is not just about finding planned styles to buy, but also about unexpected surprises, such as discovering some beautiful clothes that were never seen before. The process of using a search engine, however, seems like a "ready-made" process. When you enter a keyword, the world unfolds before you but is artificially divided: what appears is the world related to that keyword, while the part unrelated to the keyword continues to vanish into darkness. Not going shopping might mean never discovering those fresh things, leaving one immersed only in the known world. Similarly, why do larger supermarkets attract more customers? One reason is the sense of surprise. Much of the pleasure of being online also lies in such unexpected surprises: aimlessly browsing, randomly clicking on links, hours go by effortlessly. All of these experiences are not brought by the one-question-one-answer interactive mode of the search box.

Therefore, Mr. Norman's prediction seems somewhat exaggerated. Of course, there is no denying that the new command-line interface will play a more important role than before, but the graphical user interface will remain the protagonist, and it doesn't seem that a new interaction method will replace it.

Reference:
