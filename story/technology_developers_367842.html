If you want to check the original content, please visit his website.Installing various application software for computers or users in an Active Directory domain is one of the daily tasks for network administrators. Performing "software deployment" through Group Policy editing in the Active Directory domain is an ideal way to quickly complete the task. Software deployment in the Active Directory domain mainly depends on three aspects: Windows Installer, Group Policy, and the installation file itself. It can deploy software for both domain users and domain member computers.MSI (Microsoft Software Installer) files are one of the only two file types that Windows Installer can deploy. Windows Installer is a Microsoft enterprise-standard installation file format. Installation files using this standard are packaged with all components being independent from each other. Users can inspect, streamline these installation files, and decide whether to install them locally or not. Unfortunately, this distribution method supports only MSI file extensions, which makes EXE files impractical. In the left pane, sequentially expand "User Configuration" -> "Software Settings", then right-click "Software Installation" and choose "New" -> "Package" command as shown in the figure.Tip: Both "Computer Configuration" and "User Configuration" directories contain the "Software Installation" option, and both are used for deploying software in the AD domain. If the software needs to be deployed to computers in the AD domain, settings should be made in the "Computer Configuration" directory; if the software is to be deployed to users in the AD domain, it should be set in the "User Configuration" directory.Step 4: In the "Open" dialog box, find the previously stored MSI package installation file picasa36-setup.MSI via Network Neighborhood, and click the "Open" button. The "Deploy Software" dialog box will open, select the "Published" radio button, and click the "OK" button as shown in the figure.The "Publish" method does not automatically install software on client machines within the domain; instead, it places the installation option in the "Add or Remove Programs" section for users to choose when needed. The "Assign" method directly installs the software in the Start menu program group of domain member computers. The "Publish" method is generally used to provide various software tools to users, allowing them to choose whether or not to install; the "Assign" method is used for mandatory software installation where users cannot uninstall the software themselves.Step 5: Return to the "Group Policy Editor" window, where you can see the name of the published software in the right pane. Close the "Group Policy Editor" window and the "Active Directory Users and Computers" window as shown in the figure.By now, we have completed the software distribution in AD. Let's take a look at the workstation.After completing the software publishing operation on the Active Directory domain controller, log in to the Active Directory domain as a system administrator group user on the Active Directory workstation. Open the "Control Panel" window, double-click the "Add or Remove Programs" icon to open the "Add or Remove Programs" window. Click the "Add New Program" button on the left side, and in the "Add Programs from the Network" list on the right, you can see the name of the published program. Click the "Add" button to start the installation, as shown in the figure.OK, that's about it. I hope everyone provides valuable feedback.
