WordPress Data Migration

by jimwang on 2011-08-21 12:44:18

Since January 2008, the North Street Blog has been co-renting a virtual host with EVSE, the administrator of Baijia Pinke. Recently, as the server lease contract is about to expire, EVSE has decided not to renew it, so North Street has no choice but to go independent. Through the introduction of my good friend Su Yang, I chose Host Gediao and prepared for data migration.

1. Back up the data on the old host

Before officially starting the migration, we need to back up the data first. Meanwhile, in order to ensure that the backup data is not accidentally changed during the migration process, it's best to keep two backups: one backup for migrating to the new host, and another backup to prevent the first backup from being damaged.

The backup content includes: data in the database, used themes, used plugins, used attachments (such as pictures used in your blog), corresponding configuration files (including wp-config.php, robots.txt, .htaccess files, etc.), and WordPress files you have modified.

2. Upload data to the new host

After backing up all the data on the original host, the next step is to upload these data to the new host. Except for the database file, other text files can be uploaded via FTP. Note that if necessary, modifications should be made to the database username and password in wp-config.php accordingly.

3. Import the database

After uploading all the data files and databases, we need to test the site on the new host. At this point, I found that all Chinese characters on the blog were showing garbled, which is due to character set issues. Two methods are provided to solve this problem:

1. Use Empire Backup King to back up and restore WordPress data (Empire Backup King tutorial)

2. Use PHPAdmin on the old host to back up data and restore it on the DA of the new host

4. Change the domain DNS to point to the new host IP

If the site on the new host has already started working normally, then we can modify the DNS of your domain to point it to the IP address of the new host.
