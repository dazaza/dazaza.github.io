Microsoft Outlook在全球范围内崩溃，出现0xc0000005错误

2020-07-16 02:09:37

当用户启动应用程序时，Microsoft Outlook立即在全球范围内崩溃，并在Windows事件查看器中显示0xc0000005错误。

这些撞车事故是在过去的一个小时里开始发生的，来自世界各地的用户的报告蜂拥而至。

检查其应用程序事件日志的用户将看到错误日志，指出OUTLOOK.EXE程序已崩溃，异常代码为0xc0000005。

蜂鸣计算机已经在测试中确认了崩溃，如下图所示。

故障应用程序名称：OUTLOOK.EXE，版本：16.0.13001.20266，时间戳：0x5ef262ee故障模块名称：mso98win32client.dll，版本：0.0.0.0。时间戳：0x5ef2aa2d异常代码：0xc0000005故障偏移量：0x000474b2故障进程ID：0x4cf0故障应用程序启动时间：0x01d65ac9b0e13874故障应用程序路径：C：\Program Files(X86)\Microsoft Office\root\Office16\OUTLOOK.EXE故障模块路径：C：\Program Files(X86)\Common Files。

微软365状态推特账号在推特上表示，这是由新的更新造成的，他们正在调查这一问题。

在Microsoft 365管理中心的一条名为“EX218604：用户遇到Outlook连接问题和崩溃”的事件状态消息中，Microsoft建议用户使用Web和移动客户端，直到问题得到解决。

用户影响：用户可能会遇到崩溃或无法通过Outlook访问Exchange Online。

更多信息：我们的分析表明Outlook on Web和移动客户端不受影响。在我们解决此问题时，用户可以利用这些协议作为访问电子邮件和服务功能的替代方法。

当前状态：我们对可用数据的初步审查表明，最近部署的更新可能是问题的根源。我们正在对所有最近的服务更新进行分析，以找出问题的根本原因，并确定恢复服务的最便捷方法。

影响范围：此问题可能会影响任何尝试使用Outlook的用户。

Microsoft Outlook用户已成功回滚到以前版本的Microsoft Outlook以解决此问题，下一节将对此进行说明。

如果Microsoft Outlook为您崩溃，您可以通过回滚到以前版本的Microsoft Outlook来解决此问题。

通过单击开始菜单并键入CMD打开命令提示符。当出现命令提示符结果时，单击它。

您现在将位于ClickToRun文件夹中。键入：officec2rclient.exe/update user updatetoversion=16.0.12827.20470并按Enter键。

Microsoft Office现在将下载指定版本的Microsoft Office并回滚到以前的版本。请耐心等待，因为这可能需要几分钟或更长时间，具体取决于您的计算机和Internet连接速度。

更新安装完成后，将显示一个屏幕，说明已安装更新。'；您可以按此屏幕上的关闭按钮。

Microsoft Outlook现在将回滚到以前的版本，您应该可以再次启动Microsoft Outlook。
