微软已经开始部署Exchange ActiveSync

by xjyzhenai on 2010-08-11 22:23:09

继7月微软完成了Windows Live Wave 4的部署工作后，本月初又紧锣密鼓地开始了Exchange ActiveSync的部署。

什么是Exchange ActiveSync？

　　Exchange ActiveSync 是一种 Microsoft Exchange 同步协议，该协议经过优化，以适用于高延迟和低带宽网络。该协议基于 HTTP 和 XML，使移动设备（例如支持浏览器的移动电话或 Microsoft Windows Mobile&；reg； 设备）可以访问运行 Microsoft Exchange 的服务器上的组织信息。Exchange ActiveSync 使移动设备用户可以访问其电子邮件、日历、联系人和任务，并且在脱机工作时仍可以继续访问这些信息。详情请参见百度百科>；>；

实测

　　JarryJarry今天下午测试了Exchange ActiveSync。使用的设置为日常使用的Windows Mobile6.5手机。

　　ActiveSync配置工作可在手机上完成，也可以将手机与电脑同步后，在电脑端的ActiveSync同步软件上完成。实测为直接在手机上配置。

　　配置完成后手机会自动联网并开始第一次同步。用手机测试解析m.hotmail.com域名正常，Ping测试响应正常，但同步过程始终无法完成。经过两天的实测，有时表现为不停的连接服务器，但永远也得不到服务器的响应，有时反应为要求重新输入密码。但不管你输入多少次依然无法开始同步。

　　可见Exchange ActiveSync服务并不能正常使用。这与livesino放出的消息不符。

6:00PM update

　　下班在公车上手机收到微软Hotmail工作组发来的电子邮件，主题为关于在移动设备上使用 ActiveSync 的重要信息。邮件称：

　　为了随时随地获取 Hotmail，您最近已尝试在移动设备上设置名为 ActiveSync 的服务。

　　我们非常高兴您对此感兴趣，但目前有太多的注册请求需要处理，不得不暂时停止接受新的注册。

　　我们正在不断改进，应该很快就能接受新的请求。请大约在两周后访问下列网站，查看新的变化：http://www.hotmail.com。

　　再次感谢您使用 Hotmail。对此延误，我们深表歉意。

此致！

Hotmail 工作组

　　微软最近连续的升级工作进行得并不干净利落。在Wave4部署的前期就因为遇到技术问题而放缓了进度。而在此番的Exchange ActiveSync部署进程中，据分析是遇到了新的技术问题，或者发现与生产环境的某些应用产生了冲突。可见，要在移动设备上正常使用Exchange ActiveSync服务尚需时日。

后话

　　其实Exchange ActiveSync早该部署了。Exchange ActiveSync是微软自家的技术，Exchange应用软件也非常成熟了。加上Windows Live系列应用中包括电子邮件、日历服务、联系人工具等。这些应用恰恰适用于Exchange ActiveSync的同步功能。而Windows Live发布以来，这些应用与移动设备竟然一直是相互独立的。反而Google的Exchange ActiveSync倒是用起来非常的便利。

　　除此之外，部署Exchange ActiveSync功能也可以增加Windows Live系列服务的产品粘度。依靠它与移动设备间的同步功能，可以牢牢的抓住一批使用支持Exchange ActiveSync同步协议的移动用户，并增强邮件、日历、联系人等可供移动设备同步的Live系列应用的价值。

　　最后，Exchange ActiveSync协议需要终端的支持。所幸现今流行的移动操作系统都已经支持Exchange ActiveSync，如Symbian、iPhone、iPad、Android、Windows Phone等。虽然微软应该尽早部署Exchange ActiveSync，但现在正是Exchange ActiveSync发展的黄金时期。
