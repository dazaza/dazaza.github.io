A Few Tricks for Repairing Hard Drives

As a member of storage devices, hard drives play an extremely important role. However, since hard drives are magnetic media, their lifespan and stability aren't as good as those of other devices like memory, so various problems may inevitably occur during use. Moreover, the situation is complicated by the fact that hard drives involve system-level settings, making it difficult to solve problems under the familiar Windows environment. Often, issues must be handled in DOS, which can leave many DIY beginners at a loss, especially since they didn’t experience the DOS era.

Common Signs Before Hard Drive Failures

If a hard drive fails, it's best to detect the issue early and take appropriate measures promptly. Waiting until it's too late could result in the loss of valuable data on the hard drive. Generally speaking, there are several signs before a hard drive failure:

1. SMART fault warning appears. This is due to the automatic detection function built into the hard drive by the manufacturer. Such warnings indicate potential physical faults with your hard drive, which will soon lead to irregular failures.

2. The system crashes while initializing Windows. This situation is more complex, first requiring the elimination of possibilities such as poor memory quality, overheating caused by fan stoppage, or virus damage. If it’s determined to be a hard drive fault, then handle accordingly.

3. You can enter the Windows system, but errors occur when running programs. Additionally, disk scanning cannot pass, often slowing down or even causing the system to crash during scanning. This phenomenon might be due to a hard drive issue or long-term software faults in Windows. After ruling out software setting issues, it can be confirmed that the hard drive has a physical fault.

4. You can enter Windows and the disk scanning program directly detects errors, even bad sectors. In this case, Windows' diagnostic tools will provide detailed reports.

5. The BIOS suddenly fails to recognize the hard drive entirely, or even if recognized, the operating system cannot find the hard drive. This is the most severe fault.

The Lucky Break - Partition Table Damage

First, we should confirm that the power interface and data cable of the hard drive have not come loose. Then, enter the BIOS and use "HDD Auto Detect" to check the hard drive. If the BIOS can correctly identify the hard drive at this point, then at least there is hope for repair; otherwise, with the tools available to ordinary DIYers, it's basically impossible to fix.

In UltraEdit, search for the "55aa" string.

After booting the system using a CD or floppy disk, you can attempt to access the C drive. If at this point you receive a message saying the C drive cannot be found, it is actually a good thing. This likely indicates that the partition table information of the hard drive has been damaged, or attacked by some kind of virus. If the data on the hard drive is not critical to you, you can use the FDISK/MBR command to unconditionally clear the partition table content, then re-partition and format using FDISK or similar partitioning software, which usually resolves the problem. If you still need the data on the hard drive, the process becomes more complicated. Ideally, you should have antivirus software or relevant software included with your motherboard. Refer to the help documentation, which generally includes commands to recover the hard drive's partition table, and these are very easy to use.

For users without an antivirus disk, you can use NDD from NU8.0 to repair, checking for errors in the partition table. If errors are found, NDD will ask if you want to make modifications, and answering YES repeatedly will correct the errors, or you can overwrite it with a backed-up partition table.

If the above methods fail, you can use FDISK to repartition, but the partition size must match the original partition. This is crucial. After partitioning, do not perform high-level formatting, but instead use NDD for repair. This ensures the hard drive can boot after repair, and the data on the hard drive won't be lost.

Surviving on the Edge - Physical Bad Sectors on Hard Drives

If DOS can switch to the C drive, but the hard drive doesn't work properly, it is likely that the hard drive has bad sectors. Generally, bad sectors can be divided into logical bad sectors and physical bad sectors. Logical bad sectors usually don't severely affect usage, so it is likely physical bad sectors.

Our approach to handling physical bad sectors is to isolate the clusters with bad sectors into a separate partition and hide them to prevent other programs from accessing them. This prevents the spread of bad sectors and minimizes further damage. For this purpose, we mainly use PartitionMagic 6.0, a DOS-based software.

After entering PQ6.0 in the DOS interface, we first use the Check For Errors command under the Operations menu to determine the location of the physical bad sectors, as PQ6.0 excels at this, unlike Scandisk in Windows which often misdiagnoses.

PQ6.0's real strength lies in its ability to mark all clusters containing bad sectors with special markers. You can select all these bad clusters and assign them to a new independent partition. This operation is graphical and very convenient. Remember to hide this problematic partition afterward to ensure it isn't accessed. Use the Hide Partition command under the Advance menu to achieve this.

With PQ6.0, physical bad sectors are not difficult to resolve, and it effectively prevents their spread. However, note that regardless of what, the hard drive is now in a sub-healthy state, and the data on it remains at risk. It's hard to say how much longer the hard drive will last after using PQ6.0 to handle physical bad sectors; it depends on luck.

Low-Level Startup Mistakes - Logical Locks

Logical locks on hard drives are a common prank method. After being locked logically, no device can normally boot the system, whether it's a floppy disk, optical drive, or dual hard drives.

To solve the logical lock issue, it's necessary to understand the root cause. During the DOS system startup, it searches for the sequence of all logical disks. First, it looks for the partition table information in the master boot sector, located at the OBEH address starting from the first sector of zero head and zero cylinder. When the start of the partition information is 80H, it indicates the main boot partition, others are extended partitions. The main boot partition is defined as the logical C drive, then it finds the logical drives in the extended partition, defined as D drive, and so on for E, F, G... Logical locks interfere here by modifying the normal main boot partition record, pointing the first logical drive of the extended partition to itself. As a result, DOS, upon finding the first logical drive, always loops back to itself, creating a dead loop.

For such issues, if you don't care about the data on the hard drive, the solution is straightforward. You can set all IDE interfaces to NONE in the BIOS, then boot the system with a floppy disk. At this point, the system can definitely boot because there is no hard drive. Next, use the classic hard drive management software DM. Since DM can recognize hard drives independently of the motherboard BIOS, you can use DM to partition and format, completely solving the problem and ensuring no mistakes. The only downside is that the data cannot be retained.

Another recommended method can preserve the data on the hard drive. Prepare a boot disk, then on a normal machine, use a familiar binary editing tool (UltraEdit, etc.) to modify the IO.SYS file on the floppy disk (remember to change the file attribute to normal before modification). Specifically, search for the first "55aa" string in this file and change it to any other value. With this modified system floppy disk, you can smoothly boot the locked hard drive. However, since the normal partition table of the hard drive has been damaged, you cannot use FDISK to delete or modify partitions, but you can handle it using the partition table recovery method mentioned earlier.

SATA Hard Drive Bad Sector Detection Method

Many people know to use Efficiency Source to check for bad sectors on hard drives, but today when I was detecting my SATA hard drive, I found that Efficiency Source couldn't recognize the SATA drive, so I searched online for solutions. Here's a method I'd like to share with you, which might help in future hard drive checks.

The method is: Enter the BIOS and set the SATA hard drive to Enhanced Mode, and set the SATA mode to Compatible Mode. After restarting, check if the hard drive is on the first IDE device. If not, plug the SATA interface into SATA0 (the first SATA interface), then use Efficiency Source, and it should recognize the hard drive.

Alternatively, MHDD can be used for repairs, as it supports SATA drives. Below is the usage method for MHDD.

MHDD

1. MHDD is a professional hard drive tool software produced by Maysoft, Russia, with many powerful functions unmatched by other hard drive tools. It is divided into free and paid full versions, and this article introduces the detailed usage of the free version.

2. Regardless of CHS or LBA mode, MHDD can access up to 128G super-large capacity hard drives (accessible sector range from 512 to 137438953472), even if you use a 286 computer, no BIOS support or interrupt support is required.

3. MHDD works best in a pure DOS environment.

4. MHDD can directly access the IDE port without relying on the motherboard BIOS, but it is recommended not to use original Intel brand motherboards.

5. Do not run MHDD on the hard drive you want to detect.

6. MHDD requires recording data during operation, so it cannot run on write-protected storage devices (such as write-protected floppy disks, CDs, etc.).

Detailed Commands for MHDD

EXIT (Hotkey Alt+X): Exit to DOS.

ID: Hard drive detection, including hard drive capacity, number of heads, number of sectors, SN serial number, Firmware version number, LBA value, supported DMA level, whether HPA and AAM are supported, SMART switch status, security level and switch status, etc.

INIT: Hard drive initialization, including Device Reset (hard drive reset), Setting Drive Parameters (setting hard drive parameters), Recalibrate (recalibration).

I (Hotkey F2): Simultaneously execute ID and INIT commands.

ERASE: Quick deletion function, each deletion unit equals 255 sectors (data recovery invalid).

AERASE: Advanced deletion function, thoroughly deletes data in specified sector segments one sector at a time (slower than ERASE, data recovery also invalid), each deletion unit equals 1 sector.

HPA: Hard drive capacity cutting function, reduces the capacity of the hard drive, reducing the BIOS detected capacity, but software like DM that independently detects hard drive capacity will still show the original capacity.

NHPA: Restores the hard drive capacity to the true capacity.

RHPA: Ignores capacity cutting, displays the true capacity of the hard drive.

CLS: Clear screen.

PWD: Adds a USER password to the hard drive, up to 32 characters, leaving it blank cancels the password. A locked hard drive cannot be read or written at all, all read/write operations including low-level formatting and partitioning are invalid. If the password is successfully added, pressing the F2 key will show red ON next to Security. Note that the password must be effective after shutting down and restarting the power.

UNLOCK: Unlocks the hard drive. First select 0 (USER), then correctly input the password. Note: Selecting 1 (Master) cannot unlock the password.

DISPWD: Removes the password, first select 0 (USER), then correctly input the password. Must use the UNLOCK command to unlock before using DISPWD. Note: Except for using UNLOCK and DISPWD commands, there is no way to unlock the password. Furthermore, if the password is forgotten (or entered incorrectly), there is no way to unlock it. If the password removal is successful, pressing the F2 key will show gray OFF next to Security. Note: Selecting 1 (Master) cannot unlock the password.

RPM: Measures hard drive rotation speed (very inaccurate, different measurement values each time).

TOF: Creates an image file for the specified sector segment (maximum 2G).

FF: Restores from an image file (maximum 2G) to a sector segment.

AAM: Automatic noise management. Use the AAM (automatic noise management) command to adjust the noise of the hard drive in a "what you hear is what you get" manner. After entering the AAM command, the current noise level of the hard drive will be displayed, and you can immediately hear the read/write noise of the hard drive. Note that the noise and performance of the hard drive are proportional, the higher the noise, the better the performance, and vice versa. After entering the AAM command, pressing the 0 key disables the AAM function, pressing the M key minimizes the noise (lowest performance), pressing the P key maximizes the noise (highest performance), pressing + and - freely adjusts the noise value of the hard drive (value range from 0 to 126), pressing the L key obtains the middle value of noise and performance (for some hard drives, if + and - are ineffective and you do not want the noise level to be maximum or minimum, press the L key to obtain the middle value), pressing the D key disables the AAM function, pressing ENTER ends the adjustment.

FDISK: Quickly divides the hard drive into one FAT32 formatted partition (actually just writes an MBR master boot record) and sets it as active, but complete formatting is still needed for use.

SMART: Displays SMART parameters and performs related SMART operations. SMART ON enables SMART functionality, SMART OFF disables SMART functionality, SMART TEST tests SMART functionality.

PORT (Hotkey Shift+F3): Displays hard drives on each IDE port, select the corresponding number to choose the hard drive on the respective port. Afterward, the port will be recorded in the MHDD.CFG file under the /CFG directory, 1 represents the primary IDE1 port, 2 represents the secondary IDE1 port, 3 represents the primary IDE2 port, 4 represents the secondary IDE2 port. The next time you enter MHDD, this port will become the default port. Editing the MHDD.CFG file changes this value, changing the default detection port of MHDD. Therefore, if after entering MHDD and pressing F2 prompts "Disk Not Ready," it means the current hard drive is not connected to the default port from the last MHDD session, at which point you can use the PORT command to reselect the hard drive (or modify the MHDD.CFG file).

CX: Performs seek testing on Quantum CX and LCT (including LA, LB, LC) series hard drives, testing the stability of the Philips TDA5247 chip on these two types of hard drives (because poor-quality 5247 chips are most prone to reveal flaws during frequent seeking). Press ESC to stop. This command can also be used on other hard drives, primarily increasing the temperature of the hard drive motor driver chip through frequent random seeking, thus testing the stability of the hard drive under heavy load.

WAIT: Waits for the hard drive to be ready.

STOP (Hotkey Shift+F4): Turns off the hard drive motor.

IBME: Views the defect list (P-LIST) of IBM hard drives. At this point, a large amount of data needs to be recorded, the larger the defect list, the larger the generated file (under the IBMLST directory), if MHDD exists on a floppy disk, there might be insufficient space;

FUJLST: Views the defect list (P-LIST) of Fujitsu hard drives. At this point, a large amount of data needs to be recorded, the larger the defect list, the larger the generated file (under the FUJLST directory), if MHDD exists on a floppy disk, there might be insufficient space;

MAKEBAD: Artificially creates bad sectors in a specified area. Note, the bad sectors generated by it are hard to repair.

RANDOMBAD: Randomly generates bad sectors across the hard drive, press ESC to stop generation. Note, the bad sectors generated by it are hard to repair.

BATCH (Hotkey F5): Batch processing.

R (Hotkey F3): Hard drive reset. For example, after using the PWD command to add a password, to make the password take effect immediately, this command can be used.

FUCKFUJ, KILLFUJ, AKILLFUJ: All are deliberate destruction commands for Fujitsu hard drives, use with caution, otherwise the hard drive will be irreparably damaged. A perfectly normal Fujitsu hard drive, after using the FUCKFUJ command, shows destruction success within one or two seconds, and after restarting, even the self-test action disappears, the motherboard cannot detect the hard drive, and the hard drive is completely scrapped.

SCAN (Hotkey F4): Disk surface scan, can repair bad sectors with specific modes, including:

[Scan in: CHS/LBA]: Scan in CHS or LBA mode. CHS is only effective for old hard drives below 500M.

[Starting CYL]: Set the starting cylinder for scanning.

[Starting LBA]: Set the starting LBA value for scanning.

[Log: On/Off]: Whether to write to log files.

[Remap: On/Off] (Re-remapping): Whether to repair bad sectors.

[Ending CYL]: Set the ending cylinder for scanning.

[Ending LBA]: Set the ending LBA value for scanning.

[Timeout(sec)]: Set the timeout value from 1 to 200, default is 30.

[Advanced LBA log] (Advanced LBA log): Unsupported.

[Standby after scan]: Turn off the hard drive motor after scanning, allowing the SCAN to end automatically and cut off power supply to the hard drive while the host remains powered (unattended feature).

[Loop the test/repair]: Loop testing and repair, mainly used to repeatedly repair stubborn bad sectors.
