Will Windows Phone be merged with Windows RT?

by anonymous on 2013-10-08 18:38:50

The iPad, which was released in 2010, directly adopted the iPhone's system, providing ammunition for some critics. The tech media heavily criticized the iPad at its release, deeply upsetting Jobs, who championed the use of the iOS system. However, after the official launch of the iPad, a buying frenzy ensued, once again proving Jobs' foresight: "People don't know what they want until you show it to them."

The obvious benefit of tablets and phones using the same system is that when the iPad first came out with a lack of applications, it could borrow from the iPhone apps to fill the gap. The same system also makes it easier for developers to create applications. Apple still retains this feature today; iOS 7 even defaults iPads to open iPhone apps at a higher resolution.

When Microsoft developed the tablet system Windows RT, it took a different approach from Apple. Windows RT used an entirely different NT kernel from the mobile system, while the then WP7 system continued to use the CE kernel. A year later, Microsoft changed its mind, and the WP8 released in June 2012 adopted the NT kernel, abandoning the outdated WP7.

This decision angered many people, with furious WP7 users criticizing Microsoft harshly, but the fact remained unchanged. Despite knowing it would provoke public anger, Microsoft still insisted on implementing the new system because the more powerful Windows.NT could achieve more content — including merging with Windows RT.

The Microsoft Financial Analyst Meeting (FAM) held at the end of September attracted numerous high-level executives such as Julie Larson-Green, Qi Lu, and Terry Myerson. In response to a question about the future of Windows RT, Terry Myerson, who oversees Windows and Windows Phone, gave a meaningful answer:

"Windows RT is Microsoft's first ARM tablet. As phones gradually infiltrate into tablets, we look forward to seeing more ARM tablets and Windows ARM tablets in the future."

"We really should develop a unified interface and API for all devices, and applications should be available to all end users."

It couldn’t be clearer. Media outlets believed that Myerson was hinting at the future merger of Windows Phone and Windows RT. ZDNet columnist Mary Jo Foley wrote:

"Windows Phone is based on the ARM architecture, and Windows RT is also based on the ARM architecture. Both systems use the NT kernel. And Microsoft is working hard to make both systems have the same programming interface, framework structure, and development tools. Although the application stores for Windows Phone and Windows RT have not yet been fully unified, there’s no reason to think they won’t merge in the future."

Mary Jo Foley's argument that the WP and RT app stores haven't been unified is almost becoming "a thing of the past," as rumors suggest that Windows Phone and Windows RT will soon merge their app stores. This can be seen as a prelude to the merger of the two systems, and Microsoft has already started to push this plan slowly. Rumors on Twitter also point out that Myerson plans to gradually merge Windows RT and Windows Phone over the next two years through three GDR updates per year.

It must be said that merging the two systems is a reliable plan, offering a more unified user interface, more consistent operation methods, and a richer app store—there’s nothing more enticing than that. Look at the already successful competitors; whether iOS or Android, none have created separate systems for tablets and phones—Android once did so, but the short-lived Honeycomb merely proved how unreliable independent tablet systems are.

Of course, the merger of the two systems will also bring problems. Apart from technical difficulties, some have raised these questions:

"The differences between the two systems are very clear: phone system and tablet system. No one wants to simulate iPhone apps on a 2012 iPad, and even fewer people would want to use tablet programs on a phone."

"From an engineering design perspective, RT is as close as possible to the X86 version, so the control panel, file manager, desktop version of IE, and Office suite all appear on RT."

"The initial design philosophy of Windows Phone was to help users quickly complete tasks and then return to life quickly, which differs greatly from RT... Whether it’s UI details or transition animations, RT and WP are two completely different products created by the same design language."

Indeed, that's true. However, Microsoft's biggest problem isn't the development differences between the two systems, but the misunderstanding of Windows RT — the concept of a 'productivity tablet' is fundamentally flawed. Tablets are more commonly used for content consumption rather than productivity tools. For example, we may read Shakespeare's *A Midsummer Night's Dream* on an iPad, but who would write a novel on it?
