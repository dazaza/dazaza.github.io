What is Android

by dream2012 on 2012-02-28 10:03:06

Android is a software suite specifically designed for mobile devices, which includes an operating system, middleware, and some key applications. The beta version of the Android SDK provides the necessary tools and API interfaces for developing Android applications using the Java language on the Android platform.

Features:

- Application Framework: Supports component reuse and replacement.

- Dalvik Virtual Machine: Optimized specifically for mobile devices.

- Integrated Browser: Based on the open-source WebKit engine.

- Optimized Graphics Library: Includes a custom 2D graphics library; the 3D graphics library is based on OpenGL ES 1.0 (with optional hardware acceleration).

- SQLite: Used as structured data storage.

- Multimedia Support: Includes common audio, video, and static image formats (such as MPEG4, H.264, MP3, AAC, AMR, JPG, PNG, GIF).

- GSM Telephony Technology (hardware-dependent).

- Bluetooth, EDGE, 3G, and WiFi (hardware-dependent).

- Camera, GPS, compass, and accelerometer (hardware-dependent).

- Rich Development Environment: Includes a device emulator, debugging tools, memory and performance analysis charts, and Eclipse IDE plugins.

Applications:

Android will be released with a series of core application packages, including email clients, SMS programs, calendars, maps, browsers, contact management programs, etc. All applications are written in the Java language.

Application Framework:

Developers also have full access to the API framework used by the core applications. The architecture design of the application simplifies component reuse; any application can publish its functional blocks, and any other application can use the published functional blocks (subject to the security restrictions of the framework). Similarly, this application reuse mechanism allows users to easily replace program components.

Behind each application is a series of services and systems, including:

- Rich and extensible Views that can be used to build applications, including lists, grids, text boxes, buttons, and even embeddable web browsers.

- Content Providers that allow applications to access another application's data (such as the contact database) or share their own data.

- Resource Manager that provides access to non-code resources such as local strings, graphics, and layout files.

- Notification Manager that allows applications to display custom notifications in the status bar.

- Activity Manager that manages the lifecycle of applications and provides common navigation back functionality.

For more details and how to write an application from scratch, please refer to "How to Write an Android Application."

Libraries:

Android includes some C/C++ libraries that can be used by different components in the Android system. They provide services to developers through the Android application framework. Some core libraries include:

- System C Library - A standard C system function library (libc) inherited from BSD, customized specifically for embedded Linux-based devices.

- Media Library - Based on PacketVideo OpenCORE; this library supports playback and recording of various common audio, video formats, and static image files. Supported formats include MPEG4, H.264, MP3, AAC, AMR, JPG, PNG.

- Surface Manager - Manages the display subsystem and provides seamless blending of 2D and 3D layers for multiple applications.

- LibWebCore - A latest web browser engine supporting the Android browser and an embeddable web view.

- SGL - Lower-level 2D graphics engine.

- 3D Libraries - Implemented based on OpenGL ES 1.0 APIs; this library can use hardware 3D acceleration (if available) or highly optimized 3D soft acceleration.

- FreeType - For bitmap and vector font display.

- SQLite - A powerful lightweight relational database engine available to all applications.

Android Runtime:

Android includes a core library that provides most of the functionality of the core library of the Java programming language.

Each Android application runs in its own process and has an independent instance of the Dalvik virtual machine. Dalvik is designed so that a device can efficiently run multiple virtual systems at the same time. The Dalvik virtual machine executes (.dex) Dalvik executable files, which are optimized for low memory usage. The virtual machine is register-based, and all classes are compiled via the Java compiler and then converted into .dex format by the "dx" tool in the SDK for execution by the virtual machine.

The Dalvik virtual machine relies on some features of the Linux kernel, such as threading and lower-level memory management mechanisms.

Linux Kernel:

The core system services of Android depend on the Linux 2.6 kernel, such as security, memory management, process management, network protocol stack, and driver model. The Linux kernel also serves as an abstraction layer between hardware and the software stack.
