How to Identify a Computer Virus by Its Name

by xloong on 2007-10-28 12:46:21

Night Fire: After the first two days of the Nimaya.AL variant panda worm event, I've been wanting to write an article about how to identify virus type names. This is for a friend who got infected so they can learn more about this field and be able to personally handle any future viruses they encounter. Today, I saw a related article on ZOL (Zhongguo Online), which was much more comprehensive than what I had in mind, so I'm sharing it here for my friend who got infected and others who may need it.

Many times people have already used antivirus software to detect viruses such as Backdoor.RmtBomb.12, Trojan.Win32.SendIP.15, etc., which are long strings of English letters with numbers. At this point, some people get confused: How do I know what kind of virus this is?

In fact, if we understand some of the rules for naming viruses, we can use the virus name that appears in the antivirus software report to determine some common characteristics of the virus.

There are so many viruses in the world, and anti-virus companies, for ease of management, categorize and name viruses according to their characteristics. Although each anti-virus company has different naming conventions, they generally follow a unified naming method. The general format is: ...

The virus prefix refers to the type of virus and is used to distinguish between virus families. Different types of viruses have different prefixes. For example, the prefix for common Trojan viruses is "Trojan," while the prefix for worm viruses is "Worm," among others.

The virus name refers to the family characteristic of a virus and is used to distinguish and identify virus families, such as the famous CIH virus whose family name is uniformly "CIH," and the recent Sasser worm virus whose family name is "Sasser."

The virus suffix refers to the variant characteristic of a virus and is used to distinguish specific variants within a virus family. Generally, the 26 letters of the English alphabet are used to indicate variants, such as Worm.Sasser.b, which refers to the B variant of the Sasser worm virus, hence often referred to as "Sasser Variant B" or "Variant B of Sasser." If there are a large number of variants of a particular virus (which also indicates the virus's resilience ^_^), numerical and alphabetical combinations can be used to denote the variant.

In summary, the virus prefix greatly helps us quickly determine the type of virus. By identifying the virus type, we can make a rough assessment of the virus (of course, this requires accumulating knowledge about common virus types, which is not within the scope of this article). Through the virus name, we can further research detailed features of the virus. The virus suffix allows us to know which variant is currently residing in your system.

Below is a list of explanations for some common virus prefixes (for the widely used Windows operating system):

1. System Viruses

The prefix for system viruses is: Win32, PE, Win95, W32, W95, etc. These viruses generally share the characteristic of being able to infect *.exe and *.dll files in the Windows operating system and spreading through these files, such as the CIH virus.

2. Worm Viruses

The prefix for worm viruses is: Worm. The common characteristic of these viruses is that they spread via networks or system vulnerabilities. A significant portion of worm viruses also have the characteristic of sending malicious emails and clogging networks, such as Blaster (network clogging) and Small Postman (sending malicious emails).

3. Trojan and Hacker Viruses

The prefix for Trojan viruses is: Trojan, and the prefix for hacker viruses is generally Hack. The common characteristic of Trojan viruses is that they enter users' systems via networks or system vulnerabilities and remain hidden, then leak user information externally. Hacker viruses, on the other hand, have a visible interface that enables remote control of the user’s computer. Trojans and hacker viruses often appear in pairs, where the Trojan is responsible for infiltrating the user's computer, and the hacker virus controls it through the Trojan. Nowadays, these two types are increasingly merging. Common examples include QQ message tail Trojans like Trojan.QQ3344, and game-targeting Trojans such as Trojan.LMir.PSW.60. Here's a tip: virus names containing PSW or PWD usually indicate that the virus has password-stealing capabilities (these letters are typically abbreviations for "password"). Some hacker programs include Network Overlord (Hack.Nether.Client).

4. Script Viruses

The prefix for script viruses is: Script. The common characteristic of these viruses is that they are written in scripting languages and spread through web pages, such as Code Red (Script.Redlof) — not our big brother Code though ^_^. Script viruses may also have the following prefixes: VBS, JS (indicating the type of script used), such as Happy Time (VBS.Happytime), Fourteenth Day (Js.Fortnight.c.s), etc.

5. Macro Viruses

Actually, macro viruses are a type of script virus but due to their special nature, they are categorized separately here. The prefix for macro viruses is: Macro, with a secondary prefix being Word, Word97, Excel, Excel97 (there might be others). Any virus that only infects WORD97 and earlier versions of WORD documents uses Word97 as the secondary prefix, in the format: Macro.Word97; any virus that only infects WORD97 and later versions of WORD documents uses Word as the secondary prefix, in the format: Macro.Word; any virus that only infects EXCEL97 and earlier versions of EXCEL documents uses Excel97 as the secondary prefix, in the format: Macro.Excel97; any virus that only infects EXCEL97 and later versions of EXCEL documents uses Excel as the secondary prefix, in the format: Macro.Excel, and so on. The common characteristic of this type of virus is that it infects OFFICE series documents and spreads through OFFICE universal templates, such as the famous Melissa (Macro.Melissa).

6. Backdoor Viruses

The prefix for backdoor viruses is: Backdoor. The common characteristic of these viruses is that they spread via networks, open backdoors in systems, and create security risks for user computers. An example many friends have encountered is the IRC backdoor Backdoor.IRCBot.

7. Virus Dropper Viruses

The common characteristic of this type of virus is that when it runs, it releases one or several new viruses into the system directory, and these newly released viruses cause damage. Examples include Ice River Dropper (Dropper.BingHe2.2C), MSN Shooter (Dropper.Worm.Smibag), etc.

8. Destructive Program Viruses

The prefix for destructive program viruses is: Harm. The common characteristic of these viruses is that they have attractive icons to entice users to click, and when users click on these viruses, they directly cause damage to the user's computer. Examples include Format C Drive (Harm.formatC.f), Killer Command (Harm.Command.Killer), etc.

9. Joke Viruses

The prefix for joke viruses is: Joke. Also known as prank viruses. The common characteristic of these viruses is that they have attractive icons to entice users to click, and when users click on these viruses, they perform various disruptive operations to scare users, but in reality, these viruses do not cause any actual damage to the user's computer. An example is the Girl Ghost (Joke.Girlghost) virus.

10. Bundler Viruses

The prefix for bundler viruses is: Binder. The common characteristic of these viruses is that the virus author uses a specific bundling program to bundle the virus with some applications such as QQ, IE, etc., appearing to be normal files. When users run these bundled viruses, they seemingly run these applications, but then secretly run the bundled virus, causing harm to users. Examples include Bundled QQ (Binder.QQPass.QQBin), System Killer (Binder.killsys), etc.

The above are relatively common virus prefixes. Sometimes we may also see some others, but they are less common, mentioned briefly here:

DoS: Targets a specific host or server for Denial of Service attacks;

Exploit: Automatically exploits system vulnerabilities on the target or its own system to spread itself, or it itself is a hacking exploit tool;

HackTool: Hacker tools, which may not destroy your machine themselves, but could be used by others to make you a proxy to destroy others.

You can use the methods described above after detecting a certain virus to initially assess the basic situation of the virus, achieving mutual understanding. When automatic virus removal fails and manual handling is considered, this information will be very helpful.

Night Fire: One more thing to add - viruses with the DownLoader or DL prefix are "downloaders." They themselves have no destructive capability, but once this virus runs, it will automatically download 1-N different types of viruses or Trojans from the internet and run them automatically, potentially causing even more harm than destructive viruses, somewhat similar to what was mentioned in point 7 above.
