Two Useful Tools in Sybase ASE - An Introduction to Sybase Database Repair

Tag: Sybase database repair

If you have performed a full installation of Sybase ASE 12.5 (or higher), or during installation, selected the jutils component, then you can use the two tools mentioned in this article: jisql and Ribo. They are located under `%SYBASE%\jutils-2_0` and are Java-based GUI tools.

### 1. jisql

This is a lightweight Sybase query tool specifically designed for Sybase ASE databases and Sybase ASA databases.

Navigate to `%SYBASE%\jtuils-2_0\jisql`, where you will find a batch file named `jisql.bat`. To run it, it's best to install a JDK, even an extracted version works. At the top of the file, add `JAVA_HOME` and `PATH`, setting the path to your Java installation. Additionally, set `JDBC_HOME` to point to the location of the Sybase ASE JDBC driver, jConnect:

Example:

```batch

Set JAVA_HOME=d:\shared\jdk1.5.0_12

Set PATH=%JAVA_HOME%\bin;%PATH%

Set JDBC_HOME=%SYBASE%\jConnect-6_0

```

(Note: The `jConnect*` directory contains a subdirectory named `classes` which holds the ASE JDBC driver.)

Once configured, running `jisql` will display the following interface:

We can set the language to Chinese, input the user, password, host, and port number to connect to the database. This tool not only connects to Sybase ASE databases but also to Sybase's mobile database SQL Anywhere (a global leader in mobile databases). After connecting to the database, you'll see the following interface:

The execution results are clear. Using some basic features is quite convenient.

### 2. Ribo

What is this tool used for? Everyone knows that the Sybase ASE database uses the TDS protocol (MS SQL Server also uses this protocol; the difference lies in the fact that MS SQL Server uses TDS 8.0). This tool is specifically used to listen and read TDS protocol packets when the client connects to the ASE database. It acts as a proxy server, receiving client requests, forwarding them to the server, intercepting the server’s response, obtaining all TDS packets, and then sending the appropriate results back to the client.

To use it, you need to slightly modify the `ribo.bat` file under `%SYBASE%\jutils-2_0\ribo`. Add the following lines below `@echo off`:

```batch

set JAVA_HOME=d:\shared\jdk1.5.0_12

set RIBO_HOME=%SYBASE%\jutils-2_0\ribo

```

After this, you can directly run `ribo`. I recommend running it in GUI mode, though if conditions do not allow, you can also run it without GUI (for example, via terminal remote login): `ribo –gui`, which will bring up the following interface:

The equivalent proxy port is 5005. When the client connects to this port, it will be monitored, and the TDS content will be analyzed.

At this point, don't rush, first set the "Configuration" under the "File" menu:

Check both "Translate to file" and "Display results in window", then confirm. Return to the main window and start capturing.

Finally, try opening a DOS window, run an `isql` connection, and perform a simple query:

Of course, first, you need to add a registry entry in your `interfaces` file, i.e., `%SYBASE%\ini\sql.ini`, adding the connection information for port 5005.

Here, I added:

```ini

[SEAN]

master=NLWNSCK,sean-laptop,5005

query=NLWNSCK,sean-laptop,5005

```

Then execute the command and enter the password:

```batch

C:\Documents and Settings\hex>isql -Uspring -Ssean

Password: iihero

1>

```

At this moment, we notice that Ribo pops up a result window: `cap1.tds`:

The content inside is quite interesting. Sometimes, when debugging complex programs, you may need to rely on Ribo. I once encountered issues while reading and writing data in bulk using Openclient ctlib, and ultimately resolved the problem with the help of Ribo.

These two tools are not large, compressed together they total about 4.5MB, and can be further compressed. They are very convenient for developers to use.
