Abstract: Through a full example of the test using the low power output test in digital mobile communication equipment, this paper introduces how to use Visual Basic programming environment with SCPI smart meters, programmable, to form an automatic test system and achieve GPIB interface.

Keywords: VB, GPIB, Automatic Test System, SCPI

1 Introduction

The rapid development of wireless communication testing and the increasing requirements for measurement techniques have brought about demands for high performance, precision, and speed in testing tasks. Traditional measurements often involve multiple instruments testing a single device under test, resulting in long test cycles and tedious processes. Additionally, manual data recording during measurements makes it difficult to effectively manage and replay the data, which affects future data processing to some extent. This has made traditional manual measurements inadequate for practical needs, creating an urgent requirement for automatic test systems. Typically, these systems can automatically measure with minimal human intervention and output processed measurement data. Generally, an automatic test system consists of five parts: the controller, programmable instruments, bus interface, test software, and the device under test. In the calibration of digital mobile communication comprehensive testers, many performance measurements are closely related to measuring the low-power receiver FSMR26. This method uses VB and GPIB interface-controlled smart meters to measure the low-power receiver FSMR26 as an example through a computer.

2 System Composition and Functionality

In the testing system, the computer connects to the instruments via a GPIB interface card and cable, forming the hardware platform based on the GPIB bus standard. Since each instrument on the GPIB standard bus has a unique GPIB address, the computer, acting as the core of the system, uses the GPIB interface card, GPIB cables, and connects to the GPIB instruments on the bus for operation handling and data communication, ultimately outputting the data in user-required formats. The GPIB interface card realizes the connection between the USB bus and the GPIB bus.

2.1 Remote startup and shutdown of instruments within the system, remote setting of GPIB instrument status parameters.

2.4 Measurement data storage in a time-sharing automatic process, enabling more flexible, convenient handling, analysis, application, and presentation of the data, solving issues that were previously difficult or impossible to resolve with original instrument hardware.
