Android development tools - Hierarchy Viewer

by dream2012 on 2012-03-08 10:47:34

The Android development tool, Hierarchy Viewer, provides an application that helps users adjust and optimize their user interface. It offers a hierarchical view of the overall interface layout and includes zoom display functionality (i.e., pixel optimization).

### How to Start Using Hierarchy Viewer:

1. **Connect your device** or launch the emulator.

2. On your terminal, navigate to the SDK /tools directory to access the Hierarchy Viewer.

3. In the opened window, you will see a list of devices. After selecting a device, the active windows on the right side of the interface will be listed.

4. The currently displayed window is the one in the foreground. If no window is selected, the default loaded window will be shown.

5. Select the window you want to inspect and click "Load View Hierarchy" to load the overall layout view.

6. To load the pixel-optimized interface, click the second icon at the bottom left corner of the window.

7. To operate another window on the device, use the "Refresh Windows" button to refresh the list of available windows on the right side of the interface.

### Overall Layout

Through the overall layout interface, you can view the layout and properties of the interface. It provides three types of views:

1. **Tree Diagram**: A hierarchical structure diagram as shown on the left side of the figure below.

2. **Property List**: Displays the properties of the selected view, as shown in the upper right corner of the figure below.

3. **Wireframe Structure Diagram**: Displays the layout content in wireframe form, as shown in the lower right corner of the figure below.

After selecting a node in the tree diagram, the properties of that element will be listed in the property list. When a node is selected, it will also be highlighted with a red rectangle in the wireframe structure diagram. Double-clicking a node in the tree diagram (or selecting it and then clicking "Display View") will pop up a new window showing a detailed perspective view of the element.

The overall layout interface also provides two methods to help you adjust the interface layout: Invalidate and Request Layout. After selecting an element in the interface, clicking these buttons will respectively call different interface operations: invalidate() and requestLayout(). Regardless of which view interface you are in, using both methods simultaneously while debugging your written application will yield better results.

Scrollbars are provided at the bottom of the window to adjust the size of these three views. You can also see the number of interface elements in the entire window here. You should try to reduce the number of interface elements in the window because fewer interface elements mean faster program execution speed.

If you connect a device and change the current interface, the view will not refresh immediately. You must click "Load View Hierarchy" to reload the overall layout interface.

### Pixel Optimization

The pixel optimization interface can enlarge the window interface of the currently connected device. It provides three types of views:

1. **Browser View**: Displays the hierarchy of the interface in list form, as shown on the left side of the figure below.

2. **Normal View**: Normal window display, as shown in the middle of the figure below.

3. **Zoomed View**: Enlarged pixel view of the current window, as shown on the right side of the figure below.

After selecting an element in the browser view, its position will be marked with a ruler in the normal view. The ruler uses different rectangles to mark the general range, padding part, and page blank space (if necessary). Purple or green rectangles are used to mark the general range (height and width) of the element. Inside the padding part, white or black rectangles are used to mark the padding content. Outside the green/purple rectangle, black or white rectangles are used to mark all blank parts. (Each type of rectangle has two colors to contrast sharply with the current background color.)

In the normal view and zoomed view, there is a very convenient function where you can overlay images on the interface for operation. For example, you can place an image on the interface to check the effect. In the normal view, select "Load..." to choose an image for preview on the interface. The selected image will be displayed in the lower left corner of the screen, and then you can adjust the image's blur level and adjust the entire interface layout to make the selected image harmonize better with the whole.

The normal view and zoomed view will refresh periodically (default every 5 seconds), but the browser view will not refresh. If you leave this view and select another view, you can refresh the browser hierarchy by clicking "Load View Hierarchy". Even if you open many different views in the same window, you need to use this method to refresh. If you do not use this method to refresh, although the preview image will still refresh, when you select a view in the browser view, the current layout structure will not be displayed in the normal view because the Hierarchy Viewer will still think you are in the previous view.

### Control Options Include:

- **Overlay**: Reads an image and overlays it on the view, allowing adjustment of the image's blur level.

- **Refresh Rate**: Sets the refresh time for the normal view and zoomed view.

- **Zoom**: Adjusts the zoom ratio of the zoomed view.
