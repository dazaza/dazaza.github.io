Development Exercise of Notepad Application (Part I)

by dream2012 on 2012-03-06 11:24:09

In this exercise, you will learn how to add a second Activity to your notepad application. This Activity allows users to create, edit, and delete notes. It assumes that new notes are created in response to user input and packages itself into a Bundle return value provided by an intent. The goal of this exercise is:

- Build a new Activity and add it to the Android manifest.

- Invoke another Activity using the `startActivityForResult()` method through asynchronous calls.

- Pass data between Activities using various Bundle objects.

- Learn how to use a more advanced screen layout.

If you see any errors related to `AndroidManifest.xml` or the `android.zip` file, right-click on the project, select `Android Tools > Fix Project Properties`, and the errors will be corrected.

- Note that the `fillData()` method has some comments added, and it now stores the note pointer in the newly defined field. The `onCreate()` method remains unchanged from Exercise One. Also, note that the class member variable used to store the note pointer is called `mNotesCursor`. The prefix 'm' indicates that the variable is a member variable, adhering to Android coding style standards.

- Implement the new overridden interfaces (`onListItemClick()` and `onActivityResult()`).

```java
mDbHelper.deleteNote(getListView().getSelectedItemId());
```

Here, we delete the specified ID note via the `deleteNote()` interface. To obtain the specified note ID, we call `getListView().getSelectedItemId()`. Next, we call the `fillData()` interface to keep the data updated.


### Step 4: Implement the `createNote()` Interface

Create a new Intent for the `NoteEdit` class to create a note (ACTIVITY_CREATE). Then call the `startActivityForResult()` interface to start the Intent:

```java

Intent i = new Intent(this, NoteEdit.class);

startActivityForResult(i, ACTIVITY_CREATE);

```

This form of Intent invocation is specific to a particular class within our Activity. In this example, the specific class is `NoteEdit`. Since the Intent class needs to send request information through the Android operating system, we also need to provide a Context (`this`).

The `startActivityForResult()` interface is triggered when the new Activity is completed. In our Activity, the interface for receiving callbacks is named `onActivityResult()`, which we will implement in subsequent steps. Another interface for calling an Activity is `startActivity()`, but this is a one-time call (fire-and-forget), where the current Activity does not receive notification that another Activity has been completed, and no result information can be returned from the `startActivity()` interface of the called Activity.

Do not worry about the fact that `NoteEdit` does not currently exist; we will soon implement it.

### Step 5: Implement the Overridden Interface `onListItemClick()`

`onListItemClick()` is a callback function we will override. It is called when the user clicks on an item in the note list. It has four parameters: the ListView object calling the interface, the clicked View (within the ListView), the clicked position (the position of the item in the list), and the row index (`mRowId`) of the clicked item. In this example, we ignore the first two parameters (we currently only have one ListView) and the row index. Our focus is entirely on the position parameter, which represents the location of the selected item in the list. Through this parameter, we can extract the desired data from the corresponding row and package it to send to the `NoteEdit` Activity.

In the implementation of this callback function interface, we create an Intent with the `NoteEdit` class to modify the note content, then add data to the extras bundle passed to the called Activity. We pass the title, body, and row index of the note being edited. Finally, it calls the `startActivityForResult()` interface to start the Intent. Here is the complete code for the `onListItemClick()` interface:

```java

super.onListItemClick(l, v, position, id);

Cursor c = mNotesCursor;

c.moveToPosition(position);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

startActivityForResult(i, ACTIVITY_EDIT);

```

`putExtra()` is an interface that adds data to the extras bundle when triggering an Intent. Here, we pass the title, body, and row index of the note we want to edit.

When we locate the corresponding position in the list for the selected object, all the details of the note are available in the query pointer through the `moveToPosition()` interface.

By adding information to the Intent and using `startActivityForResult()`, we trigger the Intent and request type code (which will be returned as the `requestCode` parameter to the `onActivityResult()` interface) in the `NoteEdit` class.

### Step 6: Open the `note_edit.xml` File and Browse Through It

This is the most complex interface layout we've dealt with so far. The file is provided here to avoid issues with fading in and out during code formatting. (XML has very strict rules regarding case and structure, and these errors are often the root cause of screen layout problems.)

A new parameter we haven't seen before is used here: `android:layout_weight` (in this example, it is set to 1 in each case). `layout_weight` assigns importance values to multiple views in a linear layout. All views have a `layout_weight` value, defaulting to zero, meaning they take up as much screen space as they need to display. If a value greater than zero is assigned, the available space in the parent view is divided according to the ratio of each view's `layout_weight` value to the total `layout_weight` value in the current screen layout.

For example, suppose we have a text label and two text editing elements horizontally. The text label has no specified `layout_weight` value, so it will take up the minimum required space. If both text editing elements have a `layout_weight` value of 1, they will evenly divide the remaining width of the parent view layout (since we declare them to be of equal importance). If the first text editing element has a `layout_weight` value of 1 and the second has a value of 2, one-third of the remaining space will be allocated to the first and two-thirds to the second (since we declare the second to be of higher importance).

This layout also demonstrates how to nest multiple layouts within other layouts to achieve more complex and aesthetically pleasing layouts. In this example, a horizontal linear layout is nested within another vertical layout to align the title label and text field horizontally.

### Step 8: Implement the Definition of the `onCreate()` Method for the `NoteEdit` Class

This method sets a title called "Edit Note" (a string defined in the `strings.xml` file) for our new Activity, and sets the note content view via the layout file `note_edit.xml`. We get handles for the note title and text body views, as well as the confirm button. This allows us to set the note's title and body and bind the confirm button to respond to user actions.

Next, we can unpack the parameter values passed in the extras bundle of the called Intent and pre-fill the title and note text body and view, allowing the user to edit them. We can also get and store the note row index (`mRowId`) to know which note the user is currently editing.

In the `onCreate()` method, establish the layout:

```java

setContentView(R.layout.note_edit);

```

Find the necessary edit and button components:

These resources can be found via the associated IDs declared in the R class, then converted to the correct view type (two text views are of type `EditText`, and the confirm button is of type `Button`):

```java

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

```

Note that `mTitleText` and `mBodyText` are class member variables (you need to declare them in the class before using them here).

Declare a private class member variable `Long mRowId` in the class to store the current row index value `mRowId` (if any).

Continue adding code in the `onCreate()` method to initialize the note title, body, and row index ID variables with the values from the Intent extras bundle (if present):

```java

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

```

We extract the title and body values set in the extras bundle when the Intent was triggered.

We also protect against assigning null values to the text fields.

Create an `onClickListener()` method for the button:

Listeners may be one of the most confusing aspects of UI implementation, but in this example, the effect we want to achieve is simple: invoke the `onClick()` method when the user presses the confirm button. Use it to do something and return some values to the Intent caller after the note has been edited. We achieve this purpose using an anonymous inner class, which might seem confusing if you haven't seen it before. However, you should set aside your confusion for now, because later you'll have the opportunity to revisit this code snippet and understand how to create a Listener and associate it with a button (Listener is a common term in Java development, especially in UI design and implementation).

### Step 9: Complete the Implementation of the `onClick()` Method in Our Listener

When the user presses the confirm button, this interface runs. We want this method to retrieve the title and text fields of the edited note and place these data into a package as a return value, so that these data can be passed back to the Activity that initially triggered this `NoteEdit` Activity. If the interface responds to an edit rather than a create operation, we also need to place the row index ID into the return value package so that the `Notepadv2` class can save the edits made to the note.

Create a data package (Bundle) as the return value. Pass the note title and body values, along with the constants defined in `Notepadv2`, to it:

```java

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

```

Pass this data package into a new Intent and end the Activity:

```java

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

```

The new Intent here simply acts as a carrier for the data package (including the note title, body, and row index ID we want to pass).

The `setResult()` method sets the result type code and passes the Intent back to the caller of this Intent. In this example, everything works, so we return the result code `RESULT_OK`.

Calling the `finish()` method signals that the Activity is complete (equivalent to a return call). Any interfaces set in the result are returned together with the execution control information to the result caller.

The complete implementation of the `onCreate()` method (plus some fields supporting this class) is as follows:

```java

private EditText mTitleText;

private EditText mBodyText;

private Long mRowId;

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.note_edit);

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

}

});

}

```

### Step 10: Define the New Activity in the Manifest File

Before the new Activity can be recognized by the Android operating system, it needs its own Activity entry in the `AndroidManifest.xml` file. This way, the Android operating system knows that the Activity exists in the `AndroidManifest.xml` file and can call it. We can also specify which IntentFilters the Activity implements here, but we will skip that part for now and just let the Android operating system know that the Activity has been defined.

The Manifest Editor included in the Eclipse plugin makes it easy to edit the `AndroidManifest.xml` file, and we will use this plugin. If you prefer to directly edit the file or not use the Eclipse plugin, refer to the information in the box below on how to edit the Manifest file without using the new Manifest Editor.

Double-click the `AndroidManifest.xml` file in the folder tree to open it.

Click the Application tab at the bottom of the Manifest Editor.

In the Application section, click Add....

If you see a radio button at the top of the dialog, select the top one labeled "Create a new element at the application level."

Before clicking "OK," make sure you have selected "(A)Activity" in the dialog options tab.

In the Node section of the Application, click the new "Activity," enter `.NoteEdit` in the Name* field, and press Enter.

The Android Manifest Editor allows you to add more complex Activity entries to the file and explore other available options (but don't choose them unless you want to add them to your Manifest file).
