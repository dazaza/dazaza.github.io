### 1. Issue: [A client is modifying the data, please try again later.] However, in reality, no one is making changes. What could be the cause?

**Solution:** Clear the content of the GL_lockrows table in the database; you can use a command to clear the contents of this table.

---

### 2. T6 Sales Accounting and Accounts Receivable Management: How are returns and refunds calculated and processed? How do you write off accounts?

**Solution:**

- For returns, create a return order in the sales module, review it, and generate a negative invoice. After rechecking, proceed to the accounts receivable module to review the receivables document and generate a voucher to complete the return process.

- For refunds, go to the payment entry interface, click the switch button to display the payment form, enter the data, review it, and generate a voucher to complete the refund operation.

---

### 3. Two warehouses were created: 1) Raw Material Warehouse 2) Consignment Processing Warehouse.

The procurement process is as follows: Purchase Request → Production Order → Purchase Order → Goods Receipt Note → Inspection Report → Generate Purchase Inbound Note.

During inbound, items are stored in both warehouses. Now, for returns, should we return only the goods stored in the consignment processing warehouse?

**Solution:**

- If you're using U8, fill out the Goods Receipt Return Order in the procurement management system, and select the consignment warehouse during the process.

- If you're using NC, fill out the outbound adjustment form and modify the receipt and issue category.

---

### 4. When setting up new free items for inventory, an error message appears when saving: "Key column information is insufficient or incorrect, update affects multiple rows."

**Solution:** There is an extra trigger in the userdef table, which does not exist in the T6 data structure. In the Query Analyzer, locate this table (userdef), right-click → All Tasks → Manage Triggers → Name → Select Trigger → Delete → Confirm.

---

### 5. Can T6 v6.0 install a client? After installing the client, I received error code 429 when connecting to the server.

**Solution:** Apply a patch. This is a significant oversight in T6 v6.0. It's unclear what the testing team at headquarters was doing. Both newly installed servers and clients have the 429 error, and the patch is mandatory with no other solution.

Patch Address: [http://www.chanjet.com/backserve](http://www.chanjet.com/backserve?pkid=T6060RTMPJP00)

---

### 6. What is the difference between original currency balance and local currency balance?

**Solution:** Relative to foreign currency accounting. The local currency refers to Renminbi (RMB), while the original currency refers to the foreign currency used in your transactions. For example, if the transaction is in US dollars, then USD is the original currency, and RMB is the local currency.

---

### 7. Failed to log into U6, cannot open the enterprise portal.

**Solution:** Navigate to C:/Windows/System32, find the client.dll file, and move it to C:/Windows/System. This should resolve the issue.

---

### 8. Forgot the password for T6 System Management. What should I do?

**Solution:** In the Enterprise Manager, locate the ufsystem database, find the ua_user table, open it to return all rows, locate the admin account, and press Ctrl+0 (zero) to clear the admin password.

---

### 9. Which accounting subjects can be set up for project accounting?

**Solution:** All subjects can be set up for project accounting, depending on your specific needs.

---

### 10. How to set up inventory accounting subjects and counterpart subjects in T6?

**Solution:**

- **Inventory Subject:** Corresponds to the subject for recording inventory itself, such as raw materials, finished goods, low-value consumables, etc.

- **Counterpart Subject:** The subject that records the opposite side after business transactions occur, typically set according to the receipt and issue categories:

- Procurement Inbound: Material Purchases or Accounts Payable.

- Sales Outbound: Cost of Main Business.

- Material Outbound: Production Cost.

- Finished Goods Inbound: Production Cost.

- Inventory Loss: Administrative Expenses.

---

#### 1. Procurement Inbound (Precondition: Single Arrival Backflush)

- **Procurement Inbound Document (Provisional Estimate):**

- Debit: Raw Materials

- Credit: Provisional Estimated Payable

- **Procurement Inbound Document (Reimbursement) Settlement Voucher (Checked):**

- Debit: Raw Materials, Taxes Payable

- Credit: Accounts Payable

- **Red Backflush Document:**

- Debit: Provisional Estimated Payable

- Credit: Raw Materials

- **Blue Backflush Document (Reimbursement) Settlement Voucher (Checked):**

- Debit: Raw Materials, Taxes Payable

- Credit: Accounts Payable

---

#### 2. Material Outbound Document

- Debit: Production Cost

- Credit: Raw Materials

---

#### 3. Sales Outbound Document

- Debit: Cost of Main Business

- Credit: Finished Goods

---

#### 4. Finished Goods Inbound

- Debit: Finished Goods

- Credit: Production Cost

#### 6. Outsourced Inbound Business

For provisional outsourced inbound documents:

- Debit: Inventory Subject

- Credit: Material Fee (for outsourced processing materials)

- Credit: Provisional Estimated Processing Fee

For settled outsourced inbound documents:

- Debit: Inventory Subject

- Credit: Material Fee (from counterpart subject based on receipt/issue category)

- Credit: Processing Fee (from counterpart subject based on receipt/issue category). If the material fee and processing fee subjects are the same, combine them into one entry during voucher creation.

After completing procurement settlement, enterprises can prepare vouchers based on the settlement documents.
