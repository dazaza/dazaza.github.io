Revealing the Most Expensive Line of JavaScript Code in History

by anonymous on 2013-07-29 16:31:59

The Origin of the Most Expensive JavaScript Code:

A while back, Opera Browser Company needed more and more servers. The company had foreseen explosive growth for Opera Mini, Opera Link, and Opera Turbo, so they planned to purchase N servers to alleviate server pressure. (In fact, Opera did experience explosive growth, as can be seen in the monthly global mobile internet reports that are published.)

Since these services are server-intensive, Opera Mini's servers were tasked with saving bandwidth and accelerating connections for tens of millions of mobile users worldwide. Opera Turbo was responsible for full-web acceleration on computers, and Opera Link ensured frequent synchronization to provide thousands of real-time updates for users. Therefore, the number of servers that needed to be purchased wasn't just a few; management approved a budget of several million Norwegian kroner for the entire order (at the current exchange rate, 1 Norwegian krone is roughly equal to 1 Chinese yuan).

The Course of the Most Expensive JavaScript Code:

The tech enthusiasts in the IT department were thrilled. Their job nature meant they spent more time on servers than with their friends of the opposite sex. After their efforts, the company established a process for collecting sample machines for testing. All suppliers sent their machines to the Oslo headquarters in Norway for testing and trial use.

The tech enthusiasts scrutinized the servers like a picky girlfriend until one server utterly defeated them. This server came from a globally recognized first-tier manufacturer (whose name we won’t mention). When Opera staff turned it on and accessed the management interface using Opera Browser, they were immediately redirected to an error page. This wasn't the whole issue; when they discovered the root cause, even the inventor of CSS (the current Opera CTO) and his entire team were stunned.

The reason for the error was that the server's management interface had been modified by a dealer who added a line of JavaScript code. This code detected when the management interface was accessed via Opera Browser and then brutally redirected the user to an error page.

Outcome:

Due to this line of JavaScript code, the company successfully angered all relevant personnel at Opera, losing a million-dollar order in the process. Opera was baffled: writing code specifically to target our browser and still expecting us to buy your servers?

The Culprit:

It was just one line of JavaScript code, with logic that simply and harshly handled users—detecting if the user was using Opera Browser and then redirecting them to an error page. This ranks as the most expensive JavaScript code in history.

```javascript

if (is.opera) { window.location.href="config/error.htm"; }

```
