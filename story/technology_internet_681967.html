### **I. Output Traps in Design and Layout**

When designing and laying out content, various software programs are used. Each software has different functions; some are powerful, while others are weaker. If weaker or less mature features are used, errors may occur during the output process. The pre-press file creation process follows certain rules, but without practical experience, using seemingly supported but actually not recommended features can cause issues during output or even when saving and opening files. Some foreign software is localized into Chinese for convenience, but this process can introduce functional errors. Every software has programming mistakes (bugs) that affect usability. Using text and images with hidden garbled characters or special symbols, or not designing according to printing requirements, can also lead to output errors. These situations can cause output errors if not promptly detected, leading to significant losses once the printing stage begins. However, most of these issues appear correct on the screen and may go unnoticed even by professional detailed checks. This is why they are called "output traps" – traps created by pre-press production personnel for the output process.

Only when designers understand the software they use and strictly follow the safest methods during the design and layout process, paying attention to avoiding output traps and carefully checking before output, can the correct output of pre-press files be ensured.

### **II. Common Output Traps**

These are traps to be mindful of when using most software:

#### **1. Font Issues**

- Using system fonts such as [Kaiti-GB2312], [Heiti], [FangSong-GB2312]. These fonts may exhibit hollowing at intersections when converted to curves. They also lack universality and should not be used in pre-press files.

- Using uncommon font libraries, causing font matching difficulties, missing characters, or misalignment.

- Using thin fonts for reverse white text (e.g., Songti), making registration difficult.

- Using very small colored fonts. Fonts below 6-point size can only be used if they are pure black.

- Using damaged fonts, resulting in invisible text in the document.

- Including unnecessary invisible text in the file.

- Randomly applying underline, hollow, bold, or italic effects to text. Bold effects are particularly problematic and should be replaced with thicker fonts.

#### **2. Other Issues**

- Files containing unnecessary objects, invisible extra objects, or extra pages.

- Keeping unused parts of the file inside or outside the page.

- Arbitrarily linking files from any directory on the disk, causing image path confusion or duplicate names. Images should be copied to the directory where the layout file is located before linking.

- Incomplete linked images, causing errors.

- Using RGB or Lab color modes instead of grayscale, CMYK, or black-and-white.

- Incorrect bleed settings or failure to note special bleeds, causing misalignment or incorrect dimensions.

- Large areas of pure black using K100 only, resulting in dull colors. Use a combination like C30M30Y30K100 for richer blacks. To achieve warm black tones, add magenta appropriately.

- Setting pure black as four-color black (e.g., C78M69Y69K95), which can cause excessive ink usage and smudging.

- Filling with very light background colors (e.g., under 5% dot gain), causing dot loss.

- Excessive use of four-color text, making registration difficult.

- Incorrect knockout settings for spot color text; choose based on process requirements. Knockout should not be used for foil stamping positions.

- Unreasonable gradients causing banding. Gradients should be created in Photoshop or Illustrator and slightly blurred.

- Using inkjet printer proofs as color correction references, leading to significant color discrepancies.

- Directly pasting text from web pages or Word documents into layout software, causing square boxes, missing characters, or other errors. Text should first be saved as TXT format and then imported or pasted into the layout software to filter out special characters, symbols, and garbled text.

- Directly using OLE objects (e.g., pasting Word tables) in design software, causing issues like missing table lines.

- Arbitrarily copying and pasting objects from other software.

- Randomly rotating or mirroring objects in design and layout software.

- Misusing DCS format, causing missing images or low-resolution images.

- Using overly thin lines, making registration difficult or causing them to disappear.

- Unreasonable gradient colors, such as magenta-to-black gradients set as M100*K100 instead of M100*M100K100.

- Determining size and position by eye rather than numerically, leading to inaccuracies.

- Importing many high-resolution images or resizing large images in layout software, causing sluggish performance or crashes. Images should be adjusted to appropriate resolutions and sizes in Photoshop before importing into layout software. This issue is especially evident in CorelDRAW software.
