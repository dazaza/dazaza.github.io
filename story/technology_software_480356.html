Mcu official website new version release: interpretation of 5 major characteristics: enhanced experience, practicality first (with photo)

by xdash on 2011-08-21 12:43:57

Mknote (http://mknote.com) is a personal cloud-based notebook developed by Shanda Innovation Institute, which is permanently free. It is similar to Evernote abroad. You can store various notes and documents stably and securely, and edit or view them at any time through web-based or mobile clients. If you are an old reader of Sync Control, it must be familiar to you.

Last night (or rather this morning), Mknote completed another important update this year, mainly redesigning and upgrading the official website and the Web version backend. After actual testing by XDash, the new version of Mknote Web has taken another step forward in terms of user guidance and functional practicality, and the effect after the redesign is very ideal. Here, I will interpret and recommend it with pictures and text for all readers.

1. A more concise and practical official website design

(The above figure: The home page of the new version of the Mknote official website after the redesign)

After the redesign, the home page of the new version of the official website uses a large area of VI main color - green as the background color, with a simple and generous page layout.

Clicking the prominent "Download PC Client" will directly download the installation package to your local machine. Compared with the previous design (after clicking the download button, it does not immediately start downloading, but instead redirects to the download list page, where users need to manually select what they want to download again), this version simplifies user operations to the extreme, aiming for the fastest speed to achieve user goals. (Seriously, the original design let the Mknote team suffer from my complaints quite a bit. This version finally fixed it, and it must greatly help improve the actual conversion rate of the Mknote official website.)

2. More complete new user guidance and user help

Despite the industry's general optimism about the future of "synchronization" boosted by Apple iCloud, user demand still needs to be explored, and the understanding of synchronization tools also needs to be popularized urgently (I can say that the readers of Sync Control are all at the forefront of the times, right? ^_^). This time, Mknote has added a more friendly user guidance interface.

(The above figure: New user help interface on the Mknote official website - usage guide)

(The above figure: New user help interface on the Mknote official website - usage scenarios)

On the feature page, you can learn about the most basic product functions of Mknote (save/sync, edit/organize, share/publish); on the usage guide page, there are illustrated interface annotations; on the usage scenario page, six common scenarios for using synchronization tools are listed, such as travel insights, meeting records, web clipping, etc.

In addition, when first entering the new version of the Mknote personal backend after the redesign, prompts for new features will also be provided:

(The above figure: Prompt for the new "drag-and-drop note" function)

3. Drag-and-Drop Operations

The new personal backend interface is cleaner and simpler, compared to previous versions where some visual elements could distract attention. This improvement helps users focus on "recording" itself. It can also be clearly seen that the new interface draws inspiration from the new version of Gmail, such as the "draggable" visual cue before the note list and the "upward" button shape above the content.

Support for drag-and-drop is one of the highlights of this redesign and is a reflection of user-friendliness. In the left classification list, you can drag one category into another to make it a subcategory; you can also directly drag a note from the right list into the left category (for example, quickly setting a private note as a shared note).

(The above figure: Dragging and dropping notes under a certain category)

When setting categories via drag-and-drop, two rules should be noted: 1) Categories that already have subcategories cannot be included in other categories (quite tongue-twisting); 2) Automatically created default categories cannot be placed under other categories. These points are not sufficiently prompted by the system during operation and can only be understood by users themselves. Hopefully, there will be clearer prompts in the future.

4. Practical New Features

This version of Mknote Web has also added a right-click menu (right-click on a category to directly create a note under that category, etc.), star marking (similar to Gmail, adding a prominent mark to a specific note), word count (no need to copy to Word to check the word count anymore), note view zooming, printing, optional auto-save, etc., making the overall experience smoother.

(The above figure: Add star marking to notes)

(The above figure: Word count function)

5. Social Sharing

For XDash, a microblog enthusiast, good note materials will definitely be shared with the appropriate audience in the shortest time possible, and hopes that more people's discussions will deepen the understanding of digested content. This update supports binding to commonly used social networks (currently supports Sina Weibo, Douban, Tencent Weibo, Sohu Weibo, Kaixin Network, QQ Zone, Renren Network). When editing a shared document, simply select the desired social network from the menu.

(The above figure: Share to commonly used social networks)

(The above figure: The default format for sharing to Sina Weibo is title + link, which can be edited manually)
