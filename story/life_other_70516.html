Explorer.exe has encountered a problem and needs to close.

by diglog2008 on 2008-05-23 21:59:39

1. Software issues (especially with third-party add-ons for right-click) - Installing the Tsinghua ZhengGuang Input Method version 3.0 sometimes causes an error in explorer.exe. If you cancel the Tsinghua ZhengGuang Input Method and use another input method without encountering any problems, it can be determined as an issue with the input method. Uninstalling the Tsinghua ZhengGuang Input Method will resolve the problem. No similar issues have been found in version 4.0 of the Tsinghua ZhengGuang Input Method. The recommended article on www.sq120.cocm suggests that the "Apple Beautification Edition of RAR" may cause such trouble. Uninstalling it temporarily eliminates this error. You can also check if the Apple Beautification Edition of RAR is installed; if so, uninstall it to see the effect. Installing Alcohol 120% or Alcohol 52% virtual optical drive in Tomato's system often leads to explorer.exe errors. Uninstalling these programs sometimes solves the problem.

2. Reasons related to the explorer.exe program itself - Check the file size of explorer.exe, which should normally display as 237k or 238k. If the size does not match, you can copy an explorer.exe file from another machine to your computer. Use Task Manager to terminate the explorer.exe process, then create a new task to copy the new file into the WINNT directory of the system drive.

3. Possible virus-related reasons (wc98pp.dll) - This is associated with the network protocol processor and eBook compilation tool Web Compiler. The wc98pp.dll file itself does not cause any issues and is present on many computers. However, when explorer.exe encounters an error, deleting this file can solve the problem. Then search for related registry keys and delete them. usign.dll has been mentioned by some users as being similar to wc98pp.dll. Deleting both files can eliminate constant pop-up ads in IE. This file was not found on company computers.
