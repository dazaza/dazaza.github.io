Comprehensive Analysis of Ports from 0 to 33600

by xloong on 2007-12-07 22:19:31

In network technology, a port (Port) generally has two meanings: one is the physical port, such as the interfaces used in ADSL Modems, hubs, switches, and routers to connect other network devices, like RJ-45 ports, SC ports, etc. The second meaning refers to logical ports, usually associated with TCP/IP protocols, where port numbers range from 0 to 65535, for example, those used for web browsing services.

### Viewing Ports

To view ports in Windows 2000/XP/Server 2003, you can use the Netstat command:

Click sequentially on "Start → Run," type "cmd" and press Enter to open the Command Prompt window. In the Command Prompt state, type "netstat -an," press Enter, and you will see the port numbers and statuses of TCP and UDP connections displayed in numeric form.

### Closing/Opening Ports

Before introducing the functions of various ports, here's how to close/open ports in Windows, as by default, many unsafe or useless ports are open, such as port 23 for Telnet service, port 21 for FTP service, port 25 for SMTP service, port 135 for RPC service, etc. To ensure system security, we can close/open ports using the following methods.

#### Closing Ports

For example, to close port 25 for SMTP service in Windows 2000/XP, you can do this: First, open the "Control Panel," double-click "Administrative Tools," then double-click "Services." Next, find and double-click the "Simple Mail Transfer Protocol (SMTP)" service in the opened Services window, click the "Stop" button to stop the service, select "Disabled" in the "Startup Type," and finally click the "OK" button. This way, closing the SMTP service is equivalent to closing the corresponding port.

#### Opening Ports

If you want to open that port, just first select "Automatic" in the "Startup Type," click the "OK" button, then open the service and click the "Start" button under "Service Status" to enable the port. Finally, click the "OK" button.

**Tip:** In Windows 98, there is no "Services" option; you can use the firewall's rule settings function to close/open ports.

### Port Classification

Logical ports have several classification standards. Below are two common classifications:

1. **Classification by Port Number Distribution**

(1) **Well-Known Ports**

Well-known ports are commonly known port numbers ranging from 0 to 1023, which are generally fixedly allocated to some services. For example, port 21 is assigned to FTP service, port 25 to SMTP (Simple Mail Transfer Protocol) service, port 80 to HTTP service, port 135 to RPC (Remote Procedure Call) service, etc.

(2) **Dynamic Ports**

Dynamic ports range from 1024 to 65535, which are generally not fixedly allocated to any specific service. That is, many services can use these ports. When a running program requests network access from the system, the system assigns one of these port numbers for the program to use. For example, port 1024 is assigned to the first program that requests it. After closing the program process, the occupied port number will be released.

However, dynamic ports are also often exploited by viruses and Trojan programs, such as IceRiver default connection port 7626, WAY 2.4 port 8011, Netspy 3.0 port 7306, YAI virus port 1024, etc.

2. **Classification by Protocol Type**

By protocol type, they can be divided into TCP, UDP, IP, and ICMP (Internet Control Message Protocol) ports. Below mainly introduces TCP and UDP ports:

(1) **TCP Ports**

TCP ports, or Transmission Control Protocol ports, require establishing a connection between the client and server, thus providing reliable data transmission. Common ones include port 21 for FTP service, port 23 for Telnet service, port 25 for SMTP service, and port 80 for HTTP service, etc.

(2) **UDP Ports**

UDP ports, or User Datagram Protocol ports, do not require establishing a connection between the client and server, thus lacking security assurance. Common ones include port 53 for DNS service, port 161 for SNMP (Simple Network Management Protocol) service, and ports 8000 and 4000 used by QQ, etc.

### Common Network Ports

#### Network Basics Port Correspondence

Port: 0

Service: Reserved

Description: Usually used for analyzing operating systems. This method works because on some systems, "0" is an invalid port, and attempting to connect to it using a regular closed port will produce different results. A typical scan uses an IP address of 0.0.0.0, sets the ACK bit, and broadcasts at the Ethernet layer.

Port: 1

Service: tcpmux

Description: This indicates someone is looking for SGI Irix machines. Irix is the main provider implementing tcpmux, which is enabled by default on this system. Irix machines come with several default accounts without passwords when released, such as IP, GUEST, UUCP, NUUCP, DEMOS, TUTOR, DIAG, OUTOFBOX, etc. Many administrators forget to delete these accounts after installation. Therefore, hackers search for tcpmux on the Internet and exploit these accounts.

Port: 7

Service: Echo

Description: You can see many people searching for Fraggle amplifiers by sending messages to X.X.X.0 and X.X.X.255.

Port: 19

Service: Character Generator

Description: This is a service that only sends characters. The UDP version will respond with packets containing junk characters upon receiving a UDP packet. During a TCP connection, it will send streams of junk character data until the connection closes. Hackers can use IP spoofing to launch DoS attacks by forging UDP packets between two chargen servers. Similarly, a Fraggle DoS attack broadcasts a packet with a forged victim IP address to this port, causing the victim to overload while responding to these data.

Port: 21

Service: ftp

Description: The port opened by FTP servers for uploading and downloading. It is commonly attacked to find methods for accessing anonymous FTP servers. These servers have readable/writable directories. Trojans Doly Trojan, Fore, Invisible ftp, WebEx, WinCrash, and Blade Runner open this port.

Port: 22

Service: Ssh

Description: A TCP connection established by PcAnywhere to this port may be for finding ssh. This service has many vulnerabilities, and if configured in a specific mode, many versions using the RSAREF library will have numerous loopholes.

Port: 23

Service: Telnet

Description: Remote login, intruders are searching for remote login UNIX services. Most cases involve scanning this port to identify the operating system running on the machine. Using other techniques, intruders can also find passwords. The Trojan Tiny Telnet Server opens this port.

Port: 25

Service: SMTP

Description: The port opened by SMTP servers for sending emails. Intruders search for SMTP servers to deliver their SPAM. When their accounts are closed, they need to connect to high-bandwidth E-MAIL servers to pass simple information to different addresses. Trojans Antigen, Email PassWord Sender, Haebu Coceda, Shtrilitz Stealth, WinPC, WinSpy all open this port.

#### Port: 31

Service: MSG Authentication

Description: Trojans Master Paradise and Hacker's Paradise open this port.

#### Port: 42

Service: WINS Replication

Description: WINS replication

#### Port: 53

Service: Domain Name Server (DNS)

Description: The port opened by DNS servers. Intruders might attempt zone transfers (TCP), DNS spoofing (UDP), or hiding other communications. Therefore, firewalls often filter or log this port.

#### Port: 67

Service: Bootstrap Protocol Server

Description: Through DSL and Cable modems, firewalls frequently observe large amounts of data sent to broadcast address 255.255.255.255. These machines request an address from DHCP servers. Hackers often enter them, allocate an address to themselves as local routers, and initiate numerous man-in-the-middle attacks. Clients broadcast configuration requests to port 68, and servers broadcast responses to port 67. This response uses broadcasting because the client does not yet know the IP address to send.

#### Port: 69

Service: Trivial File Transfer

Description: Many servers provide this service along with bootp, facilitating downloading boot code from the system. However, due to misconfiguration, intruders can steal any file from the system. They can also be used to write files to the system.

#### Port: 79

Service: Finger Server

Description: Used by intruders to obtain user information, query operating systems, detect known buffer overflow errors, and respond to Finger scans from their own machine to others.

#### Port: 80

Service: HTTP

Description: Used for web browsing. The Trojan Executor opens this port.

#### Port: 99

Service: Metagram Relay

Description: The backdoor program ncx99 opens this port.

#### Port: 102

Service: Message transfer agent (MTA) - X.400 over TCP/IP

Description: Message transfer agent.

#### Port: 109

Service: Post Office Protocol - Version 3

Description: POP3 servers open this port for receiving emails, allowing clients to access the mail service on the server. POP3 services have many recognized vulnerabilities. There are at least 20 buffer overflow vulnerabilities related to username and password exchanges, meaning intruders can enter the system before actual login. Successful login reveals additional buffer overflow errors.

#### Port: 110

Service: SUN Company's RPC Service All Ports

Description: Common RPC services include rpc.mountd, NFS, rpc.statd, rpc.csmd, rpc.ttybd, amd, etc.

#### Port: 113

Service: Authentication Service

Description: A protocol run on many computers, used to authenticate users of TCP connections. Using standard services can provide much computer information. However, it can act as a logger for many services, especially FTP, POP, IMAP, SMTP, and IRC services. If many clients access these services through a firewall, you will see many connection requests for this port. Remember, blocking this port may cause clients to experience slow connections to E-MAIL servers behind the firewall. Many firewalls support returning RST during the blocking of TCP connections, which stops slow connections.

#### Port: 119

Service: Network News Transfer Protocol

Description: NEWS newsgroup transfer protocol, carrying USENET communication. Connections on this port are usually people searching for USENET servers. Most ISPs restrict access to their news group servers to only their customers. Open news group servers allow anyone to post/read, access restricted news groups, post anonymously, or send SPAM.

#### Port: 135

Service: Location Service

Description: Microsoft runs DCE RPC end-point mapper on this port for its DCOM service. This is similar to the UNIX 111 port function. Services using DCOM and RPC register their locations with the end-point mapper on the computer. Remote clients connect to the computer and look up the end-point mapper to find the location of the services. Hackers scan this port on computers to determine if Exchange Server is running, what version, and some DOS attacks directly target this port.

#### Port: 137, 138, 139

Service: NETBIOS Name Service

Description: Ports 137 and 138 are UDP ports used when transferring files via the network neighborhood. Port 139: Connections entering through this port attempt to gain access to NetBIOS/SMB services. This protocol is used for Windows file and printer sharing and SAMBA. WINS Registration also uses it.

#### Port: 143

Service: Interim Mail Access Protocol v2

Description: Like POP3, many IMAP servers have buffer overflow vulnerabilities. Remember: A Linux worm (admv0rm) reproduces through this port, so many scans on this port come from unknowingly infected users. When RedHat enabled IMAP by default in their Linux releases, these vulnerabilities became popular. This port is also used for IMAP2 but is not widely used.

#### Port: 161

Service: SNMP

Description: SNMP allows remote management of devices. All configuration and operational information stored in databases can be obtained through SNMP. Misconfigurations by many administrators expose them to the Internet. Hackers attempt to access the system using default passwords public, private. They may try all possible combinations. SNMP packets may be mistakenly directed to the user's network.

#### Port: 177

Service: X Display Manager Control Protocol

Description: Many intruders access X-windows consoles through this, requiring opening port 6000 simultaneously.

#### Port: 389

Service: LDAP, ILS

Description: Lightweight Directory Access Protocol and NetMeeting Internet Locator Server share this port.

#### Port: 443

Service: Https

Description: Web browsing port, providing encryption and secure port transmission.

#### Port: 555
