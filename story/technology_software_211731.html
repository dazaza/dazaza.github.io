In the last article, I introduced some background information on server security. We learned why networks need security and why security issues are the foundation of the normal operation of the Internet. Starting from this issue, I will begin delivering technical content.

Let's start with a hypothetical proposition. Suppose you now have a server, internet access bandwidth, and a fixed IP address. Then, you can start providing Internet content services.

The first thing to do, of course, is to install an operating system on your server. Let's assume again: the installed operating system is the common Windows Server 2003.

Now, it seems like you're ready to start providing services. But if it were me, I wouldn't consider the server ready at all. Because currently, your server has almost no security measures in place. What? You set a password? NONONO. Don't think that setting a strong password for your computer administrator account guarantees safety. You should know that the default security settings of Windows Server 2003 are completely insecure. Very fragile, very weak. Therefore, we must carry out basic system security configurations first. You need to install the latest SP2 and all official patch packages. This doesn't need much explanation, but it does not fall under the scope of security configuration.

Basic Security Settings:

1. Adjust the test environment (set resolution, IP address, etc., to prepare for server setup. Before connecting to the network, you should disable the DCOM component service and install driver programs).

2. Configure the server. Use "Manage Your Server" to run the server configuration wizard and configure all required service functions, such as IIS.

3. Set up the input method for future maintenance when typing Chinese characters.

4. If you only partitioned the system drive before installing the system, you can now partition the other drives.

To enhance security, the server's file system format must be divided into NTFS (New Technology File System) format. It significantly improves security and space utilization compared to FAT16 and FAT32. Through NTFS, we can configure file security, disk quotas, EFS file encryption, etc. If your drive is already in FAT32 format, you can use CONVERT DriveLetter /FS:NTFS /V to convert FAT32 to NTFS format.

5. Set folder options to facilitate file settings, such as displaying system files and hidden files.

6. Set virtual memory. Do not put virtual memory on the C drive. You can move it to another drive. This saves valuable system drive space and reduces the number of read/write operations on the C drive.

7. Set power management. Of course, servers cannot shut down hard drives or go into automatic hibernation after XX minutes of inactivity, right?
