One, Parameter Default Interface

1. Function Description

Nowadays, many reports have parameters, and content is queried according to these parameters. This requires editing of the parameter interface. If the parameter interface can be automatically generated based on templates in the designer, it would save a lot of manual operations and greatly improve the efficiency of creating report templates.

Support Level: FineReport (hereinafter referred to as FR) fully supports;

Runqian (hereinafter referred to as RQ) does not support, the report file and the parameter query form are independent files.

Two, Parameter Definition and Reference

1. Function Description

There are multiple ways to define parameters, each with its own advantages and disadvantages. Common parameter definition methods include: defining within SQL statements, report parameter definitions, and direct references in cells. In FR tools, these forms of definition are mutually independent and can be defined as needed, whereas in RQ, parameters must first be defined in the report parameters before being referenced in SQL or cells.

Three, Predefined Parameter Editing Controls

1. Function Description

When making templates, sometimes we need to create multiple report templates that all use the same parameter controls. If we have to define each template individually, this undoubtedly makes the work quite tedious. If we define them beforehand and directly drag and drop when needed, it would be much more convenient.

Support Level: FR fully supports

RQ does not support.

FR Design:

Four, Drag-and-Drop Designed Report Parameters

1. Function Description

Directly click from predefined controls and drag into the corresponding cell, which can greatly reduce the time spent designing parameters. FR supports full drag-and-drop design, while RQ requires defining an editor for each cell. If there are many parameters, the workload is significant, which seriously affects work efficiency.

Five, Parameter Interface Status

The FR parameter interface has two states: pop-up and embedded. The parameter interface can be on the same interface as the report or the parameter interface can pop up first before displaying the report. In RQ, the parameter interface can only be displayed on the same interface as the report.

Six, Parameter Control Style

1. Function Description

The more styles available for parameter control editing, the greater the range of choices, and the more diverse and personalized the report design becomes.

FR supports over 20 editing styles; RQ only supports 10 editing styles.
