Methods for Installing the Android Software Development Kit (SDK)

by dream2012 on 2012-02-29 13:21:58

### I. System and Software Requirements

To develop Android applications using the Android SDK code and tools, you need an appropriate development computer and environment as described below:

1. **Required Operating Systems:**

- Windows XP or Vista

- Mac OS X 10.4.8 or higher (x86 only)

- Linux (Ubuntu Dapper Drake version has been tested)

2. **Required Development Environment:**

- Eclipse IDE:

- Eclipse 3.3 (Europa), 3.4 (Ganymede)

- Eclipse JDT Plugin (included in most Eclipse IDE packages)

- WST (optional but required for Android editor features, included in most Eclipse IDE packages)

- JDK 5 or JDK 6 (JRE alone is insufficient)

- Android Development Tools Plugin (optional)

- Not compatible with GNU Java Compiler (gcj)

- Other Development Environments or IDEs:

- JDK 5 or JDK 6 (JRE alone is insufficient)

- Apache Ant 1.6.5 or higher (Linux and Mac environments), 1.7 or higher (Windows environment)

- Not compatible with GNU Java Compiler (gcj)

**Note:** If JDK is already installed on your computer, ensure it matches the versions listed above. Some Linux distributions may include JDK 1.4 or the GNU Java compiler, neither of which are supported for Android development.

### II. Installing the SDK

After downloading the SDK, extract the `.zip` file to a suitable location on your computer. By default, the SDK files are extracted into the `android_sdk___` folder, which includes `tools/`, `samples/`, etc.

Take note of the name and location of the SDK folder after extraction, as you will need to reference this folder when installing the Android plugin and using SDK tools.

You can add the SDK `tools/` folder path to your environment variables. As mentioned, the `tools/` folder is located within the SDK folder.

- **Linux Environment:** Modify the `~/.bash_profile` or `~/.bashrc` file. Locate the environment variable settings and add the absolute path of `tools/`. If the setting cannot be found, add a new line: `export PATH=${PATH}:/tools`

- **Mac Environment:** Look for `.bash_profile` in your home folder and handle it similarly to Linux. If there is no `.bash_profile`, create a new one.

- **Windows Environment:** Right-click on "My Computer" and select Properties. In the Advanced tab, click "Environment Variables." When the dialog box appears, double-click Path under System Variables and add the full path of the `tools/` folder.

Adding `tools` to your environment variables allows you to run the Android Debug Bridge (adb) and other commands in `tools/` without entering the full path name. Note that if you upgrade your SDK, you will need to update your environment variables to the new location.

### III. Installing the Eclipse Plugin (ADT)

If you plan to use Eclipse IDE as your development environment for Android applications, you can install the Android Development Tools (ADT) plugin, which supports Android projects and tools. The ADT plugin includes powerful extensions that make creating, running, and debugging Android applications faster and easier.

If you do not use Eclipse IDE, you do not need to download or install the ADT plugin.

To download and install the ADT plugin, follow the steps specific to your version of Eclipse.

#### For Eclipse 3.3 (Europa) and Eclipse 3.4 (Ganymede):

1. Start Eclipse, then select Help > Software Updates > Find and Install...

2. In the dialog box that appears, select Search for new features to install and click Next.

3. Click New Remote Site.

4. In the dialog box, enter the site name (e.g., Android Plugin) and the URL: `https://dl-ssl.google.com/android/eclipse/`

5. Click OK. You should see the new site added to the search list (and checked), then click Finish.

6. In the search results dialog box, select the checkbox for Android Plugin > Developer Tools.

7. It will check the features: "Android Developer Tools" and "Android Editors."

8. The feature for Android Editors is optional but recommended. If you choose to install it, you need the previously mentioned WST plugin. Click Next.

9. Read the license agreement, then select Accept License Agreement and click Next.

10. Click Finish. Since the ADT plugin is unsigned, you can click "Install All" to install everything.

11. Restart Eclipse.

Alternatively, start Eclipse, then select Help > Software Updates.... In the dialog box that appears, click the Available Software tab. Click Add Site... and input the following address: `https://dl-ssl.google.com/android/eclipse/` Click OK. Return to the Available Software view, and you should see the plugin. Select Next to Developer Tools and click Install... In the next installation window, select "Android Developer Tools" and "Android Editors." Click Finish. Restart Eclipse.

After restarting, update your Eclipse preferences to point to the SDK folder:

- Select Window > Preferences... to open the properties panel. (Mac OS X: Eclipse > Preferences)

- From the left panel, select Android.

- In the main interface, locate the SDK. Click Browse... and navigate to the SDK folder.

- Click Apply, then click OK.

### IV. Installation Notes

1. **Ubuntu Linux Notes:**

- If you need help installing and configuring Java on your Ubuntu machine, the following resources might be helpful:

- https://help.ubuntu.com/community/Java

- https://help.ubuntu.com/community/JavaInstallation

- Follow these steps to install Java and Eclipse before installing the Android SDK and ADT plugin.

- If you are using a 64-bit version on your development machine, you need to install the `ia32-libs` package via `apt-get`.

- The Ubuntu package manager does not provide Eclipse 3.3 downloads, so it is recommended to use the Java or RCP version of Eclipse.

- Follow the steps provided earlier to install the SDK and ADT plugin.

2. **Other Linux Version Notes:**

- If you encounter the following error while installing the ADT plugin in Eclipse: "An error occurred during provisioning. Cannot connect to keystore. JKS," your development environment lacks the appropriate virtual machine. Installing Sun Java 6 should resolve this issue, then reinstall the ADT plugin.

- If JDK is already installed on your development computer, ensure its version matches those listed at the top of this page. Some Linux distributions include JDK 1.4 or the GNU Java compiler, neither of which are supported by Android.

### V. Troubleshooting ADT Installation

If you encounter issues downloading the ADT plugin following the steps above, here are some suggestions:

- In step four, try changing the remote update address URL to `http` instead of `https`.

- If you are behind a firewall (corporate firewall), ensure your Eclipse proxy settings are appropriate. In Eclipse 3.3/3.4, you can configure this from the main Eclipse menu: Window (on Mac, Eclipse) > Preferences > General > Network Connections.

- If you cannot install the downloaded ADT plugin into Eclipse, follow these steps to download and install the plugin from your computer:

- Download the ADT archive file (do not extract).

- Follow the default first two steps (as mentioned above).

- In Eclipse 3.3, click New Archive Site....

- In Eclipse 3.4, click Add Site..., then click Archive....

- Browse and select the downloaded archive file.

- Continue from step five onward.

To update your plugin, you must follow these steps instead of the default update instructions.

---

### VI. Updating the ADT Plugin

In certain cases, the SDK on your machine may be compatible with the ADT plugin, and you can update the ADT plugin from Eclipse using the following steps:

#### For Eclipse 3.3 (Europa) and Eclipse 3.4 (Ganymede):

1. Select Help > Software Updates > Find and Install....

2. Select Search for updates of the currently installed features and click Finish.

3. If ADT can be updated, select and install the updates.

Or:

1. Select Help > Software Updates > Manage Configuration.

2. Expand the navigation tree and select Android Development Tools.

3. Under Available Tasks, select Scan for Updates.

Alternatively:

1. Select Help > Software Updates....

2. Select the Installed Software tab.

3. Click Update....

4. If ADT allows updates, select it and click Finish.
