### 1. What Are Rogue Apps?

As the name suggests, rogue apps are akin to the notorious rogue software that garnered much attention a few years ago. They are applications developed or modified by a small group of unscrupulous developers or interest groups for illegal profit, taking advantage of the open characteristics of smartphone platforms like Android. Once they infiltrate a user's phone, they secretly perform unauthorized and illegal operations in the background without permission, causing direct or indirect losses to users.

Typically, rogue apps have the following characteristics:

- Forced or bundled installation

- System resource/service hijacking

- Interference with the normal operation of other software or system functions

- Difficult to completely uninstall using normal means

Once rogue apps are installed, the most immediate loss could be secret charges. Additionally, personal information privacy is also at risk.

### 2. How Do Rogue Apps Secretly Charge Fees?

By analyzing multiple APK packages of rogue apps on the Android platform, we can understand that rogue app charging mainly goes through the following process:

- Developers implant malicious charging code into the application during the development phase (usually at the initial entry point of the program). The final application is packaged into an APK and distributed through various channels.

- Users download the APK package of the application and successfully install it locally.

- Thereafter, each time the program starts, it automatically runs the malicious code, which may secretly collect the user's phone number, IMSI, ICCID, IMEI, etc., and send this information to a dedicated server (some even encrypt it to prevent detection).

- After receiving the information, the server returns instructions informing the application to execute a segment of malicious charging code, sending a text message to a specified phone number (or secretly installing software, secretly opening special URLs in the browser, etc.).

- After sending the text message, the charging number generally returns a charging notification. At this point, the malicious code forcibly intercepts the receipt SMS, leaving the user without any notification, and unknowingly pays the money.

In addition, some rogue apps themselves bundle additional third-party applications. Based on the promotion effectiveness of these bundled applications, they earn commissions from the bundled parties. Typically, for every successful promotion of a bundled application, the commission ranges from a few cents to one yuan. With the huge number of installations of bundled applications (especially those embedded in popular third-party ROMs), these rogue apps also gain substantial profits.

Some rogue apps use both of the above methods to make huge profits, earning income from both users and the bundled application parties, making them very malicious.

### 3. How Do Rogue Apps Succeed?

- Unscrupulous developers/publishers, seeking short-term benefits, implant rogue code during the development stage.

- A large number of underground publishing channels for (cracked, beautified, localized, third-party) applications provide fertile ground for rogue apps.

- Inadequate review mechanisms for application uploads/releases on legitimate channels.

- Users lack professional knowledge, identification skills, and vigilance, making them easy to deceive.

- There is a lack of corresponding tools for detecting and reporting rogue apps on the market.

### 5. How to Deal with Rogue Apps

The most basic precaution comes from users' awareness and vigilance. Please obtain applications through legitimate channels and avoid unreliable third parties as much as possible. For applications whose quality is uncertain, check their user reviews online first and compare them against the rogue app blacklist. If already installed, uninstall them promptly (before doing so, retain necessary evidence for future reference).

Additionally, there are some methods to help us deal with rogue apps:

#### 3. Request Your Carrier to Completely Disable SP Service Authorization You Don't Need

If you only use your phone to make calls and send texts, you can adopt a more extreme method and disable non-essential services. For example, Mobile phone users can directly call 10086, press 0 for customer service, and request to disable SP service access, including: all SP MobiDream services, SMS access points, subscription functions for this number, retaining only phone, SMS (and GPRS Internet) functions. If the operator refuses to disable, you can file a complaint with the industry and commerce department or the MIIT, which is your right.

According to netizen feedback, after applying to disable the above non-essential services, testing of some applications revealed that SP service SMS messages were sent (of course, prompting that authorization was not granted).
