A Brief Discussion on Oracle DBlink Construction [Shangguan World Original] iPhone Development

by tiantian2d87 on 2011-06-14 17:14:40

**iPhone Development**

If you want to establish access methods for a remote database, you can directly read or modify the data in a remote Oracle database. To achieve this, you can create a dblink. A dblink can be either a public connection (PUBLIC) or a private connection (PRIVATE). This is very similar to synonyms.

The syntax for creating a dblink is simple, but the user must have the permission to create a dblink. These permissions can be granted by the administrator.

The syntax for creating a dblink is as follows:

1. **Configured local service:**

```

CREATE DATABASE LINK database_link_name CONNECT TO username IDENTIFIED BY password USING local_configured_instance_name;

```

2. **Unconfigured local service:**

```

CREATE DATABASE LINK database_link_name CONNECT TO username IDENTIFIED BY password USING '(DESCRIPTION =

(ADDRESS_LIST =

(ADDRESS = (PROTOCOL = TCP)(HOST = ip)(PORT = 1521))

)

(CONNECT_DATA =

(SERVICE_NAME = xxx)

)

)';

```

- `host`: The IP address of the database.

- `service_name`: The SID of the database.

When querying a remote database using a dblink, it's best to create a synonym. This will make the query process much easier.

The method for creating a synonym is as follows:

```

CREATE SYNONYM synonym_name FOR table_name;

CREATE SYNONYM synonym_name FOR table_name@database_link_name;

```

It’s also worth mentioning the database's `global_name`.

When the value of `global_name` is `true`, the database link name must match the database's `global_name`; otherwise, accessing remote data via the dblink will fail.

For convenience, we usually set the boolean value of the database's `global_name` to `false`.

To check the value of the database's `global_name`, use the following command:

```

show parameters global_name

```

To set the value of `global_name` to `false`:

```

alter system set global_names=false;

```
