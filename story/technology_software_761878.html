JavaScript Memory Analyzer in Visual Studio 2012 Identifies Leaks in Windows Store Apps

Visual Studio 2012 Update 1 includes a JavaScript memory analysis tool for Windows 8, which helps developers understand the memory usage of JavaScript-based Windows Store applications. This tool is available in the Express, Professional, Premium, and Ultimate versions of Visual Studio 2012 for Windows 8.

The "Run Startup Project" option in the JavaScript memory tool allows developers to analyze the startup project within the current instance of Visual Studio. The "Launch Installed App Package" option runs the Windows Store app so that Visual Studio 2012 can initiate a diagnostic session. The final option, "Attach to Running App," enables developers to connect to an already running app to start a diagnostic session in Visual Studio.

After running the JavaScript memory tool, developers will see a process memory graph in the tool, which shows how much memory the application consumes over time. Through this graph, developers can observe memory usage trends and determine the appropriate time to capture a heap snapshot.

To inspect JavaScript memory usage, developers need to use the "Take a Snapshot" option to generate a summary tile. Besides a screenshot of the application, the summary indicates the size of the JavaScript heap at the time the snapshot was taken, as well as the number of allocated objects.

Developers can continue to capture snapshots at regular intervals and use the additional tiles generated for future comparisons. Additionally, a dropdown menu located in the upper-right corner of the detailed view allows developers to display built-in plugin IDs and object IDs.

Furthermore, developers can also capture snapshots programmatically using `console.takeHeapSnapshot` and `console.profileMark`. The former captures a snapshot and displays it in the tool, while the latter shows brief marks on the memory graph during a diagnostic session and supports custom annotations on the graph.

Sometimes, developers need to check the availability of commands before using them, as some commands are only available in the context of the memory profiler:

`if (console && console.takeHeapSnapshot) { console.takeHeapSnapshot(); }`

By default, snapshot collections are stored in the diagnostic session folder on the hard drive, which also includes .diagsession files. Snapshots can be renamed or shared among teams and reopened in compatible versions of Visual Studio 2012. Finally, developers can review snapshot tiles and detailed views when opening the files.
