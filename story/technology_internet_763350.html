New Features in Android 4.3 That Developers Care About

by anonymous on 2013-07-31 10:25:54

What’s New for Developers in Android 4.3

The latest version of Android Jelly Bean 4.3 did not introduce major changes to Google's mobile operating system like the Ice Cream version did, but instead focused on specific performance optimizations and included many small improvements, such as: restricted profiles, Bluetooth Low Energy (LE), Wi-Fi scan mode, OpenGL ES 3.0, new sensor types, and more — most of which can be found on the Android 4.3 APIs page. Below is an overview of these new APIs.

Restricted Profiles

Android 4.3 added an extension to the multi-user feature for tablets, allowing device owners to set up and manage multiple new profiles with fine-grained control over which apps are allowed to run and what content is available within each profile. This feature helps configure devices for use in various environments: for guests, digital kiosks, retail terminals, restricted access for children, etc.

Before claiming that your app supports Android 4.3, developers should review it because issues may arise due to restricted profiles. For example, an app might depend on a standard Android application that has been disabled in a certain profile. Additionally, users associated with restricted profiles do not have default account access, which could affect applications relying on account functionality.

OpenGL ES 3.0

Android 4.3 introduces support for Khronos OpenGL ES 3.0, though it requires specific hardware to run. The Nexus 4, Nexus 7 (2013 edition), and Nexus 10 are compatible with this feature, but not all devices have the necessary hardware. OpenGL ES 3.0 provides new texture formats, ETC2 texture compression, updated shading language, geometry instancing, GPU-accelerated clipping techniques, multiple render targets, and more, laying the foundation for higher-quality graphics.

Connectivity

A feature previously seen on some Android devices (e.g., Samsung S III/IV, Motorola Droid RAZR, etc.), called Bluetooth Smart Ready, has now entered the Android market. Devices supporting this feature can discover Bluetooth Low Energy (BLE) devices and query GATT services. These devices include specific watches, fitness shoes, heart rate monitors, and more.

Google also introduced a "scan-only" mode for Wi-Fi, which allows scanning for Wi-Fi access points without connecting, improving location accuracy while reducing power consumption.

User Interaction and Input

Developers can use the new ViewOverlay class to create transparent layers above views without affecting the layout structure. This can be used to create animations, such as sliding a view out of its container, without requiring focus or user input on that view.

The new TYPE_GAME_ROTATION_VECTOR can be used to detect motion without being affected by electromagnetic interference, as it does not rely on magnetic north.

The TYPE_GYROSCOPE_UNCALIBRATED and TYPE_MAGNETIC_FIELD_UNCALIBRATED sensors provide raw values, allowing apps to calibrate the sensors.

Testing

Developers can use the new UiAutomation class to simulate user activities by inspecting screen content and injecting keyboard and touch events. It can also be used to change screen orientation and capture screenshots.

Other improvements in Android 4.3 involve multimedia, user interface, notifications, contacts, location, accessibility, and security aspects.
