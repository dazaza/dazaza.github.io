The best that Microsoft's WP8.1 can do is these ten points.

by anonymous on 2013-11-17 20:27:52

Regarding Microsoft's Windows Phone operating system, the currently known upgrade information is as follows: In 2014, Microsoft will officially release WP8.1, and in 2015, Microsoft plans to merge the Windows Phone and Windows RT systems. It’s quite apparent that the upgrades of Microsoft's WP system seem to be accelerating. The joke that used to circulate among people—"the system grows by only 0.1 per year, and it will be upgraded to WP8.3 by 2016"—is becoming outdated.

According to developers' leaks, Nokia and Microsoft have already begun large-scale internal testing of the Windows Phone 8.1 (codenamed Blue) system. This is another major system upgrade after the WP8 GDR3 firmware update. Many WP phone users and Microsoft supporters are highly confident about this upgrade. Everyone hopes that through this upgrade, Microsoft can completely eliminate the obvious defects currently present in the WP system, making Microsoft phones more user-friendly.

Although a relatively large system upgrade has just been completed, there are still some noticeable defects in the current WP8 system that need to be corrected urgently. Some seemingly easy-to-fix "chronic problems" have existed since the WP7 era but have not been resolved for three years. Will the upcoming Windows Phone 8.1 meet everyone's expectations? Judging from the GDR3 upgrade, Microsoft does show sincerity in addressing the system defects that users have strongly called for recently. In my personal judgment, Windows Phone 8.1 will be the most satisfying system upgrade in WP history.

Below, let's take a look at the most anticipated upgrade points for the WP 8.1 system among Microsoft users:

First, ensuring that WP8 users can upgrade

The claim that "WP8 cannot upgrade to WP8.1" has appeared more than once in various so-called "industry insiders." However, based on the information we currently have, this claim should be a misunderstanding. From a system version perspective, W8.1 is considered an internal upgrade of the WP8 version. No matter how significant the changes are, it would be impossible to cause an inability to upgrade. Moreover, after experiencing two "upgrade gates" with WP7 and WP8, Microsoft could not possibly make such a foolish mistake again.

WP8.1 will be officially pushed in 2014, and this push will occur just one year before the merger of the Windows Phone and Windows RT systems. If older devices cannot be upgraded, the loss will not only affect consumers but also result in enormous losses for product manufacturers—something Microsoft cannot accept.

In addition, a senior executive responsible for the Microsoft WindowsPhone team personally confirmed that there has been no news internally at Microsoft regarding WP8 being unable to upgrade to WP8.1. All claims stating otherwise can be regarded as rumors.

Second, separate volume control for system/phone

Separate volume control and gravity lock are evident functional defects that have existed in the WP system since its inception. After years of complaints from users, the gravity lock feature finally appeared in the recent GDR3 update, but the separate volume control function still remains only in the deep minds of consumers.

What is separate volume control? Currently, the WP operating system uses a unified volume control mode. If a user sets the phone ringer to maximum, it means that the volume for various games, movies, and music applications will also be set to maximum. Conversely, if a user reduces the volume while playing a game, the phone's ringtone will also decrease accordingly...

Volume separation should be a piece of cake for any software company, and for Microsoft itself, even back in the Windows Mobile 5 era, phones could separately adjust system and phone volumes. However, three years have passed, and Microsoft has yet to address this issue seriously. So, will next year's WP8.1 fulfill users' small requests?

Third, more convenient background management

Before the GDR3 update, the background management interface of the Microsoft WP system did not allow users to close visible programs. Of course, the "tombstone-style" background of the WP system does not carry the same significance as Android when it comes to closing programs. However, in actual use, many programs do indeed run in the background, so many users have a strong desire to close background programs.

In the GDR3 update, Microsoft added the program-closing function to the system, but the method of closure was not the mainstream swipe operation. Instead, a close icon was added to the upper-right corner of each program thumbnail. This method is both ugly and difficult, resulting in poor user feedback.

We hope that in the WP8.1 system, there will be a more convenient way to manage background programs. After all, for a smartphone, managing background programs is a very basic function. Even Apple's iOS, which also belongs to the "tombstone" system, has seen its background management methods improve year by year.

Fourth, notification center

After the appearance of six-column icons in GDR3, the importance of the notification center function has become even more prominent. Of course, the "notification center" has always been one of the features most anticipated by Windows Phone users. This feature is equivalent to the pull-down menu on Android and iOS, mainly providing users with a convenient way to view various types of notifications and news updates.

Actually, before the GDR3 update, there were reports that Microsoft would add the notification center function, and even screenshots were leaked. Unfortunately, Microsoft failed to deliver, which was expected.

Will WP8.1 include the notification center in the system? We hope so, but I wouldn't be surprised if it doesn’t. Imagine WP phones with six columns of icons and full-screen information prompts; would that drive users crazy?

Fifth, voice assistant “Cortana”

In any mobile operating system, voice assistants are still somewhat redundant. The anticipation for this feature mainly stems from many users expecting Microsoft to demonstrate its strength by developing a better-functioning alternative to Siri.

There are reports that Microsoft is already developing a digital voice assistant similar to Apple's Siri. According to internal information, this voice assistant will be named “Cortana,” originating from a character in the game "Halo 2." It is said that Cortana will be fully integrated into Microsoft's TellMe service, offering functions similar to Apple's Siri.

In addition, Microsoft plans to integrate this voice assistant into most of its major products, such as Xbox and Windows-based devices. If everything goes according to plan, this voice assistant will first be integrated into the Windows Phone 8.1 update early next year.

Cortana will initially be available only to U.S. users, while users outside the U.S. will have to wait until the end of 2014 to use this virtual assistant. At present, Cortana may just be a codename, and some people have already requested that Microsoft choose a more branded name for this assistant.

Sixth, battery percentage display

This small feature can currently be solved using third-party software, but it can only display the battery data as a tile on the desktop. Displaying the battery percentage isn't suitable for everyone; for someone like me who suffers from obsessive-compulsive disorder (OCD), watching the battery percentage drop bit by bit gives a feeling akin to despair.

However, not being suitable for everyone is not a reason. After all, many users have complained about this for years. Currently, both Android and iOS allow users to turn the battery percentage display on or off, fully considering the needs of ordinary people and those with OCD. Among the three major systems, only Microsoft lacks this functionality.

Similarly, the battery percentage was predicted as one of the features before the GDR3 update, and screenshots were exposed. Although it didn't arrive as scheduled, I believe it will definitely be included in WP8.1.

Seventh, call duration display

Existing WP phones cannot check the call duration in the call log—this design is simply unbelievable. Anyway, hopefully, this will be possible in WP8.1.

Eighth, more aesthetically pleasing UI

Friends who like the WP system generally do so because of their admiration for Microsoft's unique UI style. However, after prolonged use, they discover that the chances of having a "screen collision" with others are too high. All WP users have black-and-white square blocks, which, for a company that constantly promotes personalization, is undoubtedly ironic.

If you have a basic understanding of color composition and spend some time arranging things, the WP8 interface can actually be made fairly pleasant. However, in reality, not many users have an aesthetic foundation or the necessary skills, and Microsoft's rigid limitations on system colors further exacerbate the problem. As a result, most WP users' desktops currently suffer from chaotic color combinations.

For WP8.1, we hope that Microsoft can design several additional colors for WP icons. Although they remain solid-color squares, the tiles on Windows 8 look much better because different icons use different colors, significantly enhancing visual appeal.

In addition, if the icons could be reduced in size and a desktop background added, that would be even better (though the likelihood is low). Fans of solid colors can continue using a black desktop, while others can choose their preferred images.

Ninth, opening more APIs

Microsoft's stinginess regarding WP system APIs is truly surprising. Although this is a restriction aimed at developers, it indirectly affects ordinary users during usage. Many veteran WP users have long awaited Microsoft to open up more APIs to developers. For consumers themselves, all browsers on WP phones use the IE kernel, there are no third-party input methods available, and no call firewalls...

Microsoft behaves similarly to Apple in this aspect, and Apple is equally stingy. We hope Microsoft can change its mindset, especially since Apple is thriving while Microsoft is not.

API (Application Programming Interface) is essentially a calling interface left by the operating system for applications. Applications use the API to invoke commands (actions) from the operating system. Different operating systems have varying degrees of API openness.

Tenth, customizing the system appearance

The concept of customization originates from the six-column small icon interface brought by GDR3. Before the official system push, many old users hoped to use this cool new UI. It wasn’t about whether six columns were more practical—it was just about bringing some change to the monotonous system. Unfortunately, GDR3 imposed restrictions on screen size, preventing older models with smaller screens from using the new interface.

Just as everyone was about to lose hope, the Samsung i8750, after being jailbroken, managed to achieve the mysterious six-column icon display of GDR3 within seconds by modifying the registry. This small example clearly demonstrates that achieving personalized appearances on WP phones is extremely simple for Microsoft.

Since it's as simple as modifying the registry, why not add more user options in WP8.1, allowing users to decide how their phones display? Users with smaller screens could enlarge the icons, and users with good eyesight—could they use six-column icons on a 4.5-inch screen?

Small changes can bring a lot of convenience and fun to users. Stubbornness either makes someone a genius or utterly incurable. In 2014, WP8.1 is under active development by Microsoft, and we will watch closely.
