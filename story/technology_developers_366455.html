Virus confrontation under Linux platform

by fdisk54 on 2010-09-28 18:01:42

Author: Anonymous Source: Reprinted from Hacker Animation Bar

Compared to the Windows system, Linux is stable, cost-effective, and very secure. If you are not a superuser, malicious programs that aim to infect system files have a hard time succeeding, and it's rare to see viruses spreading on Linux. However, this does not mean that Linux is completely invulnerable.

　　Nowadays, more and more servers, workstations, and personal computers are starting to use Linux, and computer virus creators are also beginning to attack this system. Although the powerful computer viruses we know may not damage Linux servers, they can spread to Windows-based computers that come into contact with them. In practical work, viruses often reside in Linux systems, causing no harm to the system but acting as a command center for Windows system viruses, making it extremely difficult to detect and eliminate Windows viruses in networks.

　　One, Types of Viruses under the Linux Platform:

　　Currently, viruses under the Linux platform are mainly divided into the following four categories:

(1) Executable file viruses, which refer to viruses that parasitize in files and primarily infect files.

(2) Worm (worm) viruses, worm viruses under the Linux platform are rampant, such as Ramen, Lion, and Slapper, which exploit system vulnerabilities for propagation. These viruses have infected a large number of Linux systems, causing significant losses.

(3) Script viruses, mostly written using shell script languages, these viruses are relatively easy to write but equally destructive. For example, .sh ending script files, a short shell script of just dozens of lines can quickly infect all script files on the hard drive.

(4) Backdoor programs, ranging from simple backdoors that add system superuser accounts to advanced techniques like system service loading, shared library injection, rootkit toolkits, and even kernel module (LKM) loading, backdoor technology under the Linux platform has developed very maturely. They are highly concealed, difficult to remove, and have become a major headache for Linux system administrators.

　　Two, Antivirus Software under the Linux Platform:

　　Antivirus software under Linux is mainly divided into two parts: open-source antivirus software and commercial antivirus software. The former includes German SEBASTIAN and H+BEDV AntiVir/X company's Anti Vir Linux, while the latter includes GeCAD Software's RAV Anti Virus Desktop For Linux v8, among others.

　　Additionally, although Linux desktop applications are developing rapidly, the command line (shell) still remains strong in Linux because launching an X-Window manager consumes a lot of system resources. Therefore, I will focus on introducing how to use Anti Linux to detect and eliminate viruses in the command line.

　　Small Tip:

Anti Linux is an antivirus software by German SEBASTIAN and H+BEDV AntiVir/X company specifically designed for Linux. It can detect and eliminate various types of worms, boot sector viruses, and StarOffice macro viruses. The latest version is 2.0.9.

　　Three, Environment Introduction for Detection and Removal:

　　Hardware: Central Processing Unit, compatible with Intel X86 processors Pentium 200 and above, 32MB (recommended 64MB) memory, 100MB hard disk space, 4MB video memory.

Software: Kernel version 2.2 and above, GCC version 2.95 and above.

　　Four, Installation of Detection and Removal Software:

　　Below, I will guide IT168 network administrator readers step-by-step through installing antivirus software on the LINUX operating system via the command line.

　　Step One: First, check the version number of the glibc file in the Linux server to download the corresponding software. You can use the following command:

#rpm -qa | grep glibc

　　Step Two: After determining the glibc version number, download the corresponding version.

　　Step Three: Install the software using the following commands:

#gunzip avlxsrv.tar.gz

#tar vxf avlxsrv.tar

#cd avlxsrv

#./install

An installation wizard will then appear, allowing you to easily install the software according to system prompts.

　　Step Four: Software Usage, AntiVir Linux is a command-line-based tool that can be used with some parameters to more effectively detect and eliminate Linux and Windows viruses. Its basic operation format is AntiVir [options][path].

　　Small Tip:

AntiVir Linux has many parameters that can be used. If you want to know more detailed parameter descriptions, you can use #antivir --help to view them. Additionally, here we need to mention CRC files. For the Linux system, for security reasons, system administrators generally rely on timestamps and system checksum programs to identify whether a binary file has been altered. Thus, Linux's sum program is based on CRC verification.

　　Five, Practical Virus Removal:

After installing the antivirus software, manual virus removal is required.

　　Step One: Check all files under the /usr directory using the following command

#antivir -s /usr. Here's another practical scenario —— our allocated /usr directory size is 2.4GB, and the file format is ReiserFS.

　　Step Two: Check the Windows partitions loaded by the system. If checking drive C, input

#antivir -s /mnt/winc.

　　Step Three: Software Update, first connect the computer to the Internet and execute the command:

#antivir -update, so the system will perform virus scans at set times daily.

　　Additionally, there is a scheduled task function in the Windows system, which can complete certain tasks at set times according to user requirements, which is very useful for network administrators. Linux also has similar functionality, executed by Cron. Cron is a daemon that provides timer functions. As long as the user defines the timer settings file, Linux can complete scheduled tasks.

(1) First, check if the Cron service is enabled:

#chkconfig -list | grep crond

(2) If it is not turned on, run the following command:

#ntsysv

(3) Set the crond service option to "*" (use the space key), then restart the system, and the system will start the Cron service.

(4) Create a timer settings file named mytype (name self-defined).

#crontab -e

(5) Modify the file content to 15 09 * * * antivir, then save and exit using vi or other editors.

(6) Use the crontab command to add to the task list:

#crontab -u mytype

This way, at 9:15 every day, the system will automatically perform a virus scan to detect and eliminate viruses on Windows platform computers in the network.

　　Six, Using Samba Technology to Assist in Virus Removal:

　　We usually use Samba for file sharing, which works well with computers using WINDOWS operating systems. From the perspective of a computer with Windows, the Samba server appears as a server in the Network Neighborhood, where we can save Windows files on this server. Moreover, we can mount shares from other Windows computers under the Samba directory to conduct virus checks together. Specific steps are as follows:

(1) Map Network Drives

We know that in Windows, shared directories can be mapped as network drives, allowing us to treat shared directories as local folders. A similar function exists in Linux, which can be achieved using the smbmount command.

First, create a directory under /mnt, assuming it is /mnt/smb/A1, then proceed to detect and eliminate computer viruses as if it were a local file system.

#smbmount //A1/d /mnt/smb/A1

#cd /mnt/smb/A1

#antivir

(2) To unmount the mapped directory, use the umount command, then mount other computers for virus detection, the method is the same, details omitted here.
