Oracle Critical Patch Update Released: 127 Patches, 51 for Java

by anonymous on 2013-10-22 19:19:59

A new season of Oracle's Critical Patch Update (CPU) has been released. This update includes 127 security patches, 51 of which are Java-related. The number of patches in this release is indeed significant.

This marks the first time that Java patches have been released alongside other Oracle products. Previously, Java patches were updated every four months. In this release, other Oracle products receiving updates include the E-Business Suite, MySQL, and the Primavera product suite.

Oracle’s October CPU release contains fixes for various software applications, most of which are enterprise server-related product lines:

- Oracle Database

- Oracle Fusion Middleware

- Oracle Enterprise Manager

- Oracle Applications - E-Business Suite

- Oracle Applications - Oracle Supply Chain, PeopleSoft, Siebel, and iLearning Enterprise Product Suite

- Oracle FLEXCUBE Product Suite

- Oracle Health Sciences Product Suite

- Oracle Retail Product Suite

- Oracle Primavera Product Suite

- Oracle Java

- Oracle MySQL

Among these updates, the Java patch update is arguably the most important, given the widespread use of Java software by consumers worldwide. In addition to the 51 patches specifically for Java, there are another 50 patches related to Java applets and Java WebStart. Many security experts believe that while Java is a useful application, users should disable it in their browsers due to its inherent security risks.

Of particular concern, 12 of the patched vulnerabilities are considered highly critical and scored a maximum of 10 under the CVSSv2 standard. This means these vulnerabilities could be exploited remotely without authentication, allowing unauthorized access to your computer over a network. As warned by Wolfgang Kandek, CTO of cloud security company Qualys:

Kandek wrote:

"Most of the vulnerabilities affect the Java client, which is typically deployed on desktops or laptops. The common attack vector here is web browsing and malicious websites. However, two very critical vulnerabilities, CVE-2013-5782 and CVE-2013-5830, also apply to server installations. You need to install the latest Java 7 Update 45 as soon as possible on all your desktops and laptops."

While some versions of Java automatically update, others do not. It is worth checking which version of Java your operating system is running. Despite the confusion, Oracle "strongly recommends that customers apply the CPU fixes as soon as possible."

Chester Wisniewski, senior security advisor at UK-based security firm Sophos, commented:

"If you already have a bad reputation and expose vulnerabilities to over a billion users, you need to respond faster. Microsoft and Adobe issue monthly patches, with an average of fewer than 50 security patches per quarter combined. It's time for Oracle to catch up."

The next Oracle CPU update will be released on January 14, 2014.
