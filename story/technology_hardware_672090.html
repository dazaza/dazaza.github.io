Create your personal website with ease (Part 1)

by qqlove99 on 2012-02-22 15:30:58

For someone who loves and is passionate about the internet, there must have been such an idea (and for many, it has already been realized): to own a personal website. Indeed, managing a website and being able to modify it freely is a wonderful thing. This article will introduce how to create a personal website by setting up a virtual server locally. The software we will use here is WampServer, which can simulate a server environment locally. After the website is created, you can experiment with it as much as you want, and the software itself also provides a portable version (Wampee), which can be run after extracting.

In this article, I'll demonstrate how to use WampServer by installing WordPress, which is currently the most popular blogging platform in the world. It has abundant themes and plugins, and using these resources creatively can turn WordPress into more than just a blogging platform—such as creating a portfolio or an e-commerce site. Let's get started!

### Preparations

First, you need to install WampServer or Wampee on your system. During installation, you may encounter some setup windows; just follow the default settings. Second, prepare the WordPress installation package (WordPress Chinese official website: http://cn.wordpress.org/).

Next, extract the downloaded WordPress package to the "www" folder under the WampServer directory and run WampServer.

### Creating a Database

Left-click the WampServer icon in the system tray, then select "Localhost" from the menu to enter the host control panel. In the opened page, click "phpmyadmin" to enter the PHP control panel. If the interface is in English, you can change the language to "Chinese Simplified" at the "Language" option. Here, we will create a database for the WordPress site (used to store site data).

Click "Databases," and at the bottom of the page, fill in a database name (e.g., "wp") in the "Create Database" field, then click "Create." This way, the database is easily set up (do not close the control panel yet), as shown below.

### Installing WordPress

Through the previous steps, we have created a new database. Now let's start installing WordPress on the server.

In your browser, replace "phpmyadmin" in the control panel address with "WordPress" (if you directly extracted the WordPress package to the "www" folder, change it to "WordPress"; otherwise, adjust according to your actual situation), then press Enter to access the WordPress installation page.

Click "Create a configuration file," then click "Let’s go!" on the next page. On the new page, fill in the database name (e.g., "wp"), username (you can directly input "root," leave the password blank), keep the database host and table prefix as default, then click "Submit," as shown below.

On the new page, click "Run the installation." You can now set the title and administrator account for this site based on your needs. Fill in the required information, then click "Install WordPress."

After that, there's nothing left to do—you can log in to the site now! After logging into WordPress, you will enter its backend (dashboard), where you can make all the settings and modifications for the WordPress site. To browse the site, simply input "localhost/wordpress" in your browser (without quotes, the specific address might vary), or delete "wp-admin" from the console page address and press Enter.

Now, we can freely "play around" with this website. How to play around is beyond the scope of this article, so please explore on your own.

If you want to create a forum or similar application in WampServer, the basic method is the same, so I won’t elaborate further.

### Conclusion

The website created according to this article can only run locally. If you want to access the site via the internet, you will need to install WordPress (or other types of websites) on a remote server. How to use a remote server to create a website will be introduced in the next article.

As mentioned in this article, the purpose of creating a website locally is generally for testing and learning, as it only occupies local storage space, responds quickly, and performs similarly to a real "website" in terms of content presentation.
