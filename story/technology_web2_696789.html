Android Development Tool Hierarchy Viewer

The application provided by the Hierarchy Viewer can help users adjust and optimize their user interfaces. It provides you with an overall interface layout diagram in hierarchical form (i.e., overall layout) and has a zoom display function (i.e., pixel optimization).

Getting started with Hierarchy Viewer: Connect your device or log in to the emulator. On your personal terminal, access the Hierarchy Viewer through the SDK /tools directory. In the opened window, a list of devices is displayed. After selecting a device, the currently active windows will be listed on the right side of the interface. The one that is currently displayed in the foreground is the current window; if you haven't selected a window, then the default loaded window will be displayed. Select the window you want to view, click Load View Hierarchy, and it will load the overall layout view. If you want to load the pixel-optimized interface, you can click the second icon at the bottom left of the window.

If you want to operate another window on the device, use the Refresh Windows button to refresh the list of available windows on the right side of the interface.

* Overall Layout

Through the overall layout interface, you can view the layout and properties of the interface. It provides three views: Tree diagram: i.e., hierarchical structure diagram, as shown on the left side of the figure below. Property list: displays the properties of the selected view, as shown in the upper right corner of the figure below. Wireframe structure diagram: displays the layout content in wireframe structure form, as shown in the lower right corner of the figure below.

After selecting a node in the tree diagram, the properties of that element will be listed in the property list. After selecting a node, the wireframe structure diagram will also mark that element with a red rectangle. Double-clicking a node in the tree diagram (or selecting it and then clicking Display View) will pop up a new window to display a detailed perspective view of that element.

The overall layout interface also provides two additional methods to help you adjust the interface layout: Invalidate and Request Layout. After selecting an element in the interface, clicking these two buttons will call different interface operations, invalidate() and requestLayout(). Regardless of which view interface you are in, using both methods simultaneously while debugging the applications you have written will achieve twice the result with half the effort.

Scroll bars are provided below the window to adjust the size of these three views. You can also see the number of interface elements in the entire window here. You should try to reduce the number of interface elements in the window because the fewer the number of interface elements, the faster the program's execution speed.

If you connect a device and change the current interface, the view will not refresh immediately. You must click Load View Hierarchy to reload the overall layout interface.

* Pixel Optimization

The pixel optimization interface can enlarge the window interface of the currently connected device. It provides three views: Browser view: displays the interface hierarchy in list form, as shown on the left side of the figure below. Normal view: normal window display, as shown in the middle of the figure below. Zoomed view: pixel view of the current window enlarged, as shown on the right side of the figure below.

After selecting an element in the browser view, the selected element's position will be marked with a ruler in the normal view. The ruler will use different rectangles to mark the common range, padding part, and page blank space (if needed). Purple or green rectangles are used to mark the common range of the element (height and width). In the padding part, white or black rectangles are used internally to mark the padding content. Outside the green/purple rectangle, black or white rectangles are used to mark all blank parts. (Each type of rectangle has two colors to represent it, so it can contrast sharply with the current background color.)

In the normal view and zoomed view, there is a very convenient function that allows images to be overlaid on the interface for operation. For example, you can place an image on the interface first to see the effect. Select Load... in the normal view to choose an image for preview on the interface. The selected image will be displayed in the lower-left corner of the screen, and then you can adjust the blur level of the image and adjust the entire interface layout to make the selected image more harmonious with the whole.

The normal view and zoomed view will refresh periodically (default is 5 seconds), but the browser view will not refresh. If you leave this view and select another view, you can refresh the browser hierarchy structure through Load View Hierarchy. Even if you open many different views in the same window, you need to use this method to refresh. Without using this method to refresh, although the preview picture will still refresh, when you select a view in the browser view, the current layout structure cannot be displayed in the normal view because the hierarchy viewer still thinks you are in the previous view.

Control options include: Overlay: reads an image and overlays it on the view, allowing adjustment of the image's blur level. Refresh Rate: sets the refresh time for the normal view and zoomed view. Zoom: adjusts the magnification ratio of the zoomed view.
