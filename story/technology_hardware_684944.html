Tutorial: Build your own mobile app using tools like the Android framework

by dream2012 on 2012-03-01 11:41:58

Starting from a pre-configured project file, this tutorial guides you through the complete development process of a simple notepad application, supplemented by detailed examples throughout. It teaches you how to set up your project, organize application logic and UI, as well as compile and run the executable program afterward. The tutorial treats the development process of this notepad application as a series of exercises (see below), each consisting of several steps. You can follow along step-by-step to build and refine your own application. These exercises provide specific example code for every step needed to implement this application. By the time you finish this tutorial, you will have created a functional Android mobile application with practical use, and you will also have a deeper understanding of some extremely important concepts in Android application development. If you want to add more complex features to your simple notepad application, you can compare your exercise code with another version of the notepad program implemented differently. For details, refer to the Sample Code document section.

**· Target Audience of This Tutorial**

This tutorial is mainly aimed at developers with some experience, especially those who have knowledge of the Java programming language. If you have never written a Java application before, you can still use this tutorial, but your learning progress may be slightly slower. This tutorial assumes that you are familiar with some basic Android application concepts and terminology. If you are not yet proficient in these, you should review "Overview of an Android Application" before continuing with the lessons. Additionally, please note that the integrated development environment for this tutorial is Eclipse with the Android plugin pre-installed. If you do not use Eclipse, you can still perform these exercises and build the application, but you will need to figure out how to handle steps involving Eclipse in a non-Eclipse IDE.

**· Preparations Before Exercises**

Information about setting up an Android application project related to this tutorial has been detailed in the documents "Installing the SDK" and "Hello Android." These two documents thoroughly explain the knowledge required to set up your development environment for building Android mobile applications. Before starting this tutorial, make sure you have fully prepared the two documents, installed the SDK, and set up the development environment.

Preparations needed for this training:

1. Download the project exercises archive (.zip)

2. Extract it to a directory on your local machine

3. Open the NotepadCodeLab folder

The NotepadCodeLab folder contains six project files: Notepadv1, Notepadv2, Notepadv3, Notepadv1Solution, Notepadv2Solution, and Notepadv3Solution. Among them, directories like Notepadv# serve as the starting point for each exercise, while Notepadv#Solution provides the corresponding answers. If you encounter any problems during an exercise, you can resolve them yourself by comparing the code of your local project with the corresponding answer.

**· Exercises**

The table below lists some exercises from this tutorial and describes the development training content covered in each exercise. Each exercise builds upon the premise that you have completed one of the previous exercises.

Exercise 1:

Starts with the development process of a simple notepad application. Build a simple note list that can add new notes but cannot edit them. Demonstrates the basic aspects of ListActivity, how to create and control menu items, and how to store note content using an SQLite database.

Exercise 2:

Add a second Activity to the notepad application. Demonstrates how to construct a new Activity within the Android operating system, transfer data between different Activities, use more advanced screen layouts, and shows how to activate another window and return a result by calling the API startActivityForResult().

Exercise 3:

Add control over the Event lifecycle to the application to obtain the state of the application throughout its entire lifecycle.

Additional Learning:

Demonstrates how to use Eclipse's debugger and view the entire lifecycle after an Event occurs via the debugger. This section is not mandatory but highly recommended.

**· Other Resources and Further Learning**

1. Refer to Common Android Tasks for slightly broader concepts not covered in this tutorial.

2. The Android SDK includes excellent, fully-featured examples suitable for further study. Specific examples can be found in the samples directory of the SDK you downloaded.

3. Although this tutorial is not entirely consistent with the complete notepad application in the SDK samples directory, it is derived from it. While studying this tutorial, it is strongly recommended that you carefully examine the notepad application in the SDK Sample directory. It includes some additional information more interesting than personal learning applications, such as:

* Building a custom striped note list

* Creating a custom note text editing window that overrides the draw() method, making it look like a real underlined note pad

* A full implementation of the ContentProvider() method

* Implementing undo and discard editing operations instead of automatically saving the results of the editing operation
