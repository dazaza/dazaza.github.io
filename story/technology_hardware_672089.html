Create your personal website with ease (Part 1)

by qqlove99 on 2012-02-22 15:29:47

For someone who loves and is passionate about the internet, there must have been such an idea (and for many, it may already have been realized): to own a personal website. Indeed, managing a website and being able to modify it at will is a wonderful thing. This article will introduce how to create a personal website by simulating a server locally. The software we will use here is WampServer, which can simulate a server environment locally. The created website can be freely modified, and the software itself also provides a portable version (Wampee), which can run directly after extraction.

Here, we'll take installing WordPress as an example to show how to use WampServer. WordPress is currently the most popular blogging platform in the world, with no close competitor. It has rich themes and plugin resources, and utilizing these resources diversely can make WordPress more than just a blog-writing platform. So let's dive into the process!

### Preparation

First, you need to install WampServer or Wampee on your system. During the installation process, you will encounter some setup windows; just follow the default settings. Second, prepare the WordPress installation package (WordPress Chinese official website: http://cn.wordpress.org/).

Next, extract the downloaded WordPress installation package to the "www" folder in the WampServer directory and start WampServer.

### Creating a Database

Left-click on the WampServer icon in the system tray, then choose "Localhost" from the menu to enter the host control panel. In the opened page, click "phpmyadmin" to enter the PHP control panel. If the interface is in English, you can change the language to "Chinese simplified" under "Language." Here, we will create a database for the WordPress site (to store the site's data).

Click on "Databases," and at the bottom of the page, fill in a database name in the "Create database" field (e.g., wp), then click "Create." This way, a database is easily created (please do not close the control panel), as shown in the figure below.

### Installing WordPress

Through the steps above, we've created a new database. Now let's start installing WordPress on the server.

In the browser, replace "phpmyadmin" in the control panel address with "WordPress" (if you extracted the WordPress package directly into the "www" folder, then replace it with "WordPress"; otherwise, adjust according to your actual situation). Press Enter to access the WordPress installation page.

Click "Create a configuration file," and in the next page, click "Let’s go!" On the new page, fill in the database name (such as the one created above: wp), username (you can directly fill in root, leave the password blank), keep the database host and table prefix as default, then click "Submit," as shown in the figure below.

On the new page, click "Run the installation." Now you can set the title and administrator account for this site based on your actual situation and needs. After filling in the content, click "Install WordPress." That's it—now you can log in to the website!

After logging into WordPress, you'll enter its backend (dashboard), where you can perform all the settings and modifications for this WordPress site. To browse the site, type "localhost/wordpress" in the browser (without quotes, the specific address may vary), or delete "wp-admin" from the console page address and press Enter.

Now, we can freely "play around" with this website. As for how to play, that is beyond the scope of this article, so please explore on your own.

If you want to create a forum or similar using WampServer, the basic method is the same, so I won't elaborate further.

### Conclusion

The website created according to this article can only run locally. If you want to access the site we created through the internet, we need to install WordPress (or other types of websites) on a remote server. How to create a website using a remote server will be introduced in the next section.

As mentioned in this article, the purpose of creating a website locally is generally for testing and learning, as it only occupies local storage space, responds quickly, and performs similarly to a real "website" in terms of content presentation.
