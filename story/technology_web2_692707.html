Development Exercise of Notepad Application (Part I)

by dream2012 on 2012-03-06 11:14:22

In this exercise, you will learn how to add a second Activity to your Notepad application. This Activity allows users to create, edit, and delete notes. This Activity assumes that a new note is created in response to user input, and it packages itself into a Bundle return value provided by an intent. The objectives of this exercise are:

- Build a new Activity and add it to the Android manifest.

- Invoke another Activity using the asynchronous method `startActivityForResult()`.

- Pass data between Activities using several Bundle objects.

- Learn how to use a more advanced screen layout.

**Step 1:** Create a new Android project using the resources in the NotepadCodeLab folder's Notepadv2 directory, similar to what was done in Exercise One. If you see an error related to AndroidManifest.xml or the android.zip file, right-click on the project and select Android Tools > Fix Project Properties to resolve the issue. Open the Notepadv2 project and examine the following:

- Open and review the strings.xml file under res/values, which contains several new strings for the new features.

- Open and review the top of the Notepadv2 class where you'll notice a new field pointer `mNotesCursor` and some newly defined constants below it.

- Note that the `fillData()` method has been commented and now uses the newly defined field to store the note pointer. The `onCreate()` method remains unchanged from Exercise One.

- Pay attention to the new overridden interfaces we will implement (`onListItemClick()` and `onActivityResult()`).

**Step 3:** In the `onMenuItemSelected()` interface, add a new branch statement for the parameter `DELETE_ID`:

```java

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

```

Here, we delete the specified ID note via the `deleteNote()` interface. To obtain the ID of the specified note, we call `getListView().getSelectedItemId()`. Next, we call `fillData()` to keep the data updated. The complete definition of the `onMenuItemSelected()` interface is as follows:

```java

@Override

public boolean onMenuItemSelected(int featureId, MenuItem item) {

switch(item.getItemId()) {

case INSERT_ID:

createNote();

return true;

case DELETE_ID:

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

}

return super.onMenuItemSelected(featureId, item);

}

```

**Step 4:** Implement the `createNote()` interface:

Create a new Intent with the class `NoteEdit` to create a note (ACTIVITY_CREATE). Then invoke the Intent using `startActivityForResult()`:

```java

Intent i = new Intent(this, NoteEdit.class);

startActivityForResult(i, ACTIVITY_CREATE);

```

This form of Intent invocation is specific to a particular class in our Activity; in this case, the class `NoteEdit`. Since the Intent class needs to send requests through the Android operating system, we must also provide a Context (`this`). The `startActivityForResult()` interface is invoked after the completion of the new Activity, returning to our Activity. The interface in our Activity that receives the callback is named `onActivityResult()`, which we will implement later. Another interface to invoke an Activity is `startActivity()`, but this is a fire-and-forget call where the current Activity does not receive a notification when the other Activity completes, and no results can be returned from the `startActivity()` interface.

**Step 5:** Implement the overridden interface `onListItemClick()`:

`onListItemClick()` is a callback function we will override. It is called when the user clicks on an item in the note list. It has four parameters: the ListView object invoking the interface, the View clicked (within the ListView), the position (the position of the item in the list), and the row index (`mRowId`) of the clicked item. In this example, we ignore the first two parameters (since we only have one ListView) and the row index. Our focus is entirely on the position parameter, which tells us the location of the selected item in the list. Using this parameter, we can retrieve the desired data from the corresponding row and package it to send to the `NoteEdit` Activity.

In the implementation of this callback function, we create an Intent with the class `NoteEdit` to modify the note content, then add data to the extras bundle passed to the invoked Activity. We pass the title, body, and row index of the note being edited. Finally, it invokes `startActivityForResult()` to start the Intent. Here is the complete code for implementing the `onListItemClick()` interface:

```java

super.onListItemClick(l, v, position, id);

Cursor c = mNotesCursor;

c.moveToPosition(position);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

startActivityForResult(i, ACTIVITY_EDIT);

```

`putExtra()` is an interface that adds data to the extras bundle for triggering the Intent. Here, we pass the title, body, and row index of the note we want to edit.

When we locate the corresponding position in the list of the selected object, all details about the note are within the query pointer via the `moveToPosition()` interface. By adding information to the Intent extras and invoking `startActivityForResult()`, we trigger the Intent and request type code in the `NoteEdit` class (the request type code will be returned as the parameter `requestCode` to the `onActivityResult()` interface).

**Step 6:** Open the file `note_edit.xml` and browse through it, which contains the code for the note editing interface.

This is the most complex interface layout we've dealt with so far. The file is provided here to avoid issues with code formatting. (XML has strict rules regarding case sensitivity and structure, and these errors often stem from layout problems.)

A new parameter we haven't seen before is used here: `android:layout_weight` (set to 1 in this example). `layout_weight` assigns importance values to views within a linear layout. All views have a `layout_weight` value, defaulting to zero, meaning they take up only as much space as needed. If a value greater than zero is assigned, the available space in the parent view is divided based on each view's `layout_weight` value relative to the overall `layout_weight` value in the current screen layout.

For example, if we have a text label and two text editing elements horizontally aligned, and the text label does not specify a `layout_weight`, it will occupy the minimum required space. If both text editing elements have a `layout_weight` value of 1, they will equally share the remaining width in the parent view layout (since their importance is declared equal). If one text editing element has a `layout_weight` of 1 and the other has 2, the first gets one-third and the second gets two-thirds of the remaining space (since the second is declared more important).

This layout demonstrates how to nest multiple layouts within others to achieve more complex and aesthetically pleasing layouts. In this example, a horizontal linear layout is nested within another vertical layout to align the title label and text field horizontally.

**Step 7:** Create a `NoteEdit` class extending `android.app.Activity`.

This is the first time we're creating an Activity without the help of the Android Eclipse plugin. When created this way, the `onCreate()` method isn't automatically overridden. It's hard to imagine an Activity without overriding its `onCreate()` method, so this is the first task you need to complete.

Right-click the `com.android.demo.notepad2` folder in the Eclipse folder tree, and in the context menu, select New > Class. Name the class `NoteEdit`: dialog box field. In the Superclass class, input `android.app.Activity`, or use the shortcut (Ctrl+Space on Windows/Linux, Cmd+Space on Mac) to trigger code suggestions in the IDE to find the appropriate folder and class. Click Finish.

In the generated class, right-click in the editor window and select Source > Override/Implement Methods... Scroll through the checkbox list in the dialog box to locate `onCreate(Bundle)` and check the adjacent checkbox. Click OK. This places the method in your class.

**Step 8:** Implement the definition for the `onCreate()` method of the `NoteEdit` class:

This method sets a title "Edit Note" (a string defined in the `strings.xml` file) for our new Activity and sets the note content view via the layout file `note_edit.xml`. We get handles for the note title and text body views, as well as the confirm button. With these handles, we can set the title and body of the note and bind the confirm button to respond to user actions.

Next, we can unpack the parameter values passed through the extras bundle in the invoked Intent and pre-fill the title and note text body and views, allowing the user to edit them. We also obtain and store the note row index (`mRowId`) to know which note the user is currently editing.

Set the layout in the `onCreate()` method:

```java

setContentView(R.layout.note_edit);

```

Find the necessary editing and button components:

```java

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

```

Note that `mTitleText` and `mBodyText` are class member variables (you need to declare them in the class before using them).

Declare a private class member variable `Long mRowId` in the class to store the current row index value `mRowId` (if any).

Continue adding code in the `onCreate()` method to initialize the note title, body, and row index ID variables using the Intent extras data bundle (if present):

```java

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

```

We extract the title and body values set in the extras data bundle when the Intent was triggered. We also protect against assigning empty strings to the text fields.

Create an `onClickListener()` method for the button:

Listeners might be one of the most confusing aspects of UI implementation, but in this example, the effect we want to achieve is simple: calling the `onClick()` method when the user presses the confirm button. Use this to do something and return some values to the caller Intent after the note is edited. We achieve this using an anonymous inner class, which may seem a bit confusing if you haven't seen it before. However, you need to set aside any confusion because you will have the opportunity to revisit this code later and understand how to create a Listener and associate it with a button (Listener is a common term in Java development, especially in UI design and implementation). Below is an empty Listener:

```java

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

}

});

```

**Step 9:** Complete the implementation of the `onClick()` method in our Listener:

When the user presses the confirm button, this interface runs. We want this method to retrieve the title and text fields of the edited note and place these data into a bundle as a return value, so that these data can be passed back to the Activity that initially triggered this `NoteEdit` Activity. If the interface responds to an edit operation rather than a creation operation, we also need to place the row index ID into the bundle as a return value, so that the `Notepadv2` class can save the edits made to the note.

Create a data bundle (Bundle) as a return value. Pass the note title and body values along with the constants defined in `Notepadv2`:

```java

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

```

Pass this bundle into a new Intent and end the Activity:

```java

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

```

This new Intent simply acts as a carrier for the data bundle (including the note title, body, and row index ID) we want to pass. The `setResult()` method sets the result code and passes the Intent back to its caller. In this example, everything works, so we return the result code `RESULT_OK`.

Calling the `finish()` method signals that the Activity is complete (equivalent to a return call). Any interfaces set in the result are returned together with the execution control information to the result caller.

The complete implementation of the `onCreate()` method (along with some supporting fields for the class) is as follows:

```java

private EditText mTitleText;

private EditText mBodyText;

private Long mRowId;

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.note_edit);

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

}

});

}

```

**Step 10:** Finally, define the new Activity in the manifest file:

Before the new Activity is recognized by the Android operating system, it needs its own Activity entry in the AndroidManifest.xml file. This way, the Android operating system knows that the Activity is defined in the AndroidManifest.xml file and can invoke it. We can also determine which IntentFilters the Activity implements here, but we'll skip that part for now and just let the Android operating system know that the Activity is defined.

The Manifest Editor included in the Eclipse plugin makes it easy to edit the AndroidManifest file, and we will use this plugin. If you prefer to directly edit the file or don't use the Eclipse plugin, refer to the information in the box below on how to edit the Manifest file without using the new Manifest Editor.

Double-click the AndroidManifest.xml file in the folder tree to open it. Click the Application tab at the bottom of the Manifest Editor. In the application section, click Add... If you see a radio button at the top of the dialog box, select the one at the top labeled "Create a new element at the application level." Before clicking "OK," ensure that the "(A)Activity" option in the dialog box tab is selected. In the Application Node section, click the new "Activity," enter `.NoteEdit` in the Name* field, and press Enter.

The Android Manifest Editor allows you to add more complex Activity entries in the file while also exploring other available options (but don't select them, otherwise these options will be added to your Manifest file). As you progress to more advanced Android applications, this editor helps you understand and modify the AndroidManifest.xml file.
