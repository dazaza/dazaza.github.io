N Possible Reasons for High CPU Usage

by cqe on 2009-08-03 17:36:58

1. **Antivirus Software Causing Issues**

The new versions of KV, Kingsoft Antivirus, and Rising have added random monitoring for web pages, plugins, and emails, which undoubtedly increases the system load. **Solution**: There is no perfect solution; try to use the minimum necessary monitoring services or upgrade your hardware.

2. **Unverified Drivers Causing 100% CPU Usage**

A large number of beta drivers available online can cause hard-to-detect issues. **Solution**: Pay special attention to graphics card drivers. It's recommended to use Microsoft-certified or officially released drivers, and carefully verify the model and version.

3. **Viruses and Trojans**

Numerous worm viruses replicate rapidly within the system, causing high CPU resource usage. **Solution**: Use a reliable antivirus program to thoroughly clean the system memory and local hard drive. Open the system configuration tool to check for any abnormally starting programs. Regularly update and upgrade antivirus software and firewalls, enhance virus prevention awareness, and acquire proper knowledge about antivirus measures.

4. **Control Panel - Administrative Tools - Services - RISING REALTIME MONITOR SERVICE**: Right-click and change it to "Manual."

5. **Start - Run - msconfig - Startup**: Disable unnecessary startup items and restart.

6. **Check the "svchost" Process**

svchost.exe is a core process in Windows XP. It appears not only in Windows XP but also in all Windows systems using the NT kernel. Typically, there are 2 svchost.exe processes in Windows 2000, while in Windows XP, there are 4 or more.

7. **Check Network Connections** (Mainly the network card).

8. **Network Connection Monitoring**

When a computer running Windows XP acts as a server and receives connection requests on port 445, it will allocate memory and a small amount of CPU resources to service these connections. When overloaded, the CPU usage may be excessively high due to the inherent trade-off between the number of work items and response capability. You need to determine the appropriate MaxWorkItems setting to improve system responsiveness. If the value is set incorrectly, the server's response capability may be affected, or a single user may monopolize too many system resources.

To resolve this issue, you can modify the registry: In the registry editor, navigate to [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\lanmanserver], create a DWORD value named "maxworkitems" in the right-hand window. Double-click the value, input the following number, save, and exit:

- If the computer has more than 512MB of memory, enter "1024".

- If the computer has less than 512MB of memory, enter "256".

9. **Check if Mouse Right-Click in Windows XP Causes 100% CPU Usage**

Recent reports indicate that using the mouse right-click in Explorer may lead to 100% CPU usage. Let's explore why.

**Symptoms**:

- In Explorer, when you right-click on a folder or file, the following problems may occur:

- Any copy operation may stop responding at that time.

- Network connection speed significantly decreases.

- All stream input/output operations, such as playing music with Windows Media Player, may result in distorted sound.

**Cause**:

When you right-click on a file or folder in Explorer, the CPU usage increases to 100% when the context menu is displayed, and it returns to normal only after closing the context menu.

**Solutions**:

- **Method 1**: Turn off "Use transition effects for menus and tooltips"

1. Click "Start" -> "Control Panel"

2. Double-click "Display" in "Control Panel"

3. In the "Display Properties", click the "Appearance" tab

4. In the "Appearance" tab, click "Effects"

5. In the "Effects" dialog box, uncheck the box for "Use transition effects for menus and tooltips" and then click "OK" twice.

- **Method 2**: First, select the target file or folder with the left mouse button before using the right mouse button to bring up the context menu.

Generally, when the CPU usage reaches 100%, the computer slows down. Often, minor adjustments can solve the problem without needing expert advice.

When the computer slows down, we usually think of the Task Manager first to see which program is consuming a high percentage of resources. If it's a major program, that's understandable, and closing the program will resolve the issue if the CPU usage returns to normal. If not, check what program it is. If you can't identify the process, search Google or Baidu.

For commonly used software like browsers that consume a lot of CPU, consider upgrading the software or switching to another similar application. Sometimes software and the system may not be fully compatible; in this case, try the compatibility option in Windows XP by right-clicking the .exe file and selecting "Compatibility."

svchost.exe can sometimes be problematic. When you notice a specific svchost.exe process consuming a lot of CPU, download tools like aports or fport to check the corresponding program path, i.e., what is calling this svchost.exe. If it's not from c:\Windows\system32 (XP) or c:\winnt\system32 (2000), it's suspicious. Upgrade your antivirus software and run a scan.

Right-clicking files leading to 100% CPU usage can also happen. Sometimes a pause during a right-click might be related to this issue. Official explanation: Select the file/folder with the left mouse button first, then right-click (though this isn't entirely clear). Non-official: Resolve by unchecking "Use the following transition effects for menus and tooltips (U)" through a right-click on the desktop -> Properties -> Appearance -> Effects. Additionally, some antivirus software's file monitoring may affect performance; you can disable file monitoring in the antivirus software. Similarly, monitoring web pages, plugins, and emails can have the same effect.

Some driver programs may also exhibit this behavior. It's best to choose Microsoft-certified or officially released drivers. Occasionally, updating the driver is appropriate, but remember that the latest isn't always the best.

**CPU Cooling Software**: Since the software uses all idle CPU time for cooling and Windows cannot distinguish between regular CPU usage and cooling instructions, the CPU will always show 100%. This doesn't affect normal system operation.

**Word Processing**: When handling large Word documents, spell and grammar checks can strain the CPU. To alleviate this, open Word, go to Tools -> Options -> Spelling and Grammar, and uncheck "Check spelling as you type" and "Check grammar as you type."

**High CPU Usage When Clicking AVI Video Files**: This happens because the system scans the entire file, checks all parts, and creates an index. **Solution**: Right-click the folder where video files are stored -> Properties -> General -> Advanced, and uncheck "Allow Indexing Service to index this folder for faster searches."
