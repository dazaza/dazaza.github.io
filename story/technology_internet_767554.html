Comparison of Community Activity for the 10 Hottest JavaScript Open-source Frameworks

by anonymous on 2013-10-22 19:23:00

The 10 hottest JavaScript framework projects

Each open-source project has its own community, and the level of activity within the community indirectly reflects the activity level of the project itself. This article highlights the currently most popular 10 JavaScript open-source frameworks. Let's take a look at which one is the most active.

Community strength is also a major factor in determining whether a JavaScript framework will be adopted by developers. When considering whether a framework meets project needs, what factors should developers consider? This article will delve into an investigation of the top 10 most active JavaScript frameworks. The data in this article comes from Ohloh.net and GitHub.

In recent years, a large number of UI libraries and frameworks have emerged for developers, and most of them are open source. Their usage and completeness are described online. However, when people make choices, they often overlook some key indicators, such as strength, community capacity, and the entire ecosystem surrounding the open-source project, not realizing that these indicators play very important roles.

Top 10 Projects

Based on the aforementioned factors, we have conducted an in-depth exploration of the community to uncover the top 10 most popular open-source JavaScript frameworks today. These frameworks include UI libraries, full-stack frameworks, and more. The hot projects are:

- AngularJS

- Ember.js

- SproutCore

- Backbone.JS

- Knockout.js

- SpineJS

- CanJS

- Meteor.JS

- Derby.js

- Yahoo Mojito

We will statistically analyze the scale of the communities around these 10 projects, their growth trends, and the entire ecosystem surrounding each project.

Monthly Code Contributions

The chart below reflects the amount of code contributed by all participants for each project each month, which directly reflects the activity level of the project.

From the chart above, it is evident that the contributions to Ember.js and Google’s AngularJS are growing rapidly. As an early leader, SproutCore has shown a significant decline in contributions since December 2011. Besides Ember.js and AngularJS, Backbone.JS and MeteorJS also demonstrate substantial contribution levels.

Project Contributors

Counting the number of contributors over time can reveal many issues, such as the governance style of the project. Many projects are managed by small teams. Of course, the more contributors there are, the faster issues can be resolved, and the documentation may be more comprehensive.

As shown in the chart above, AngularJS and Ember.js have the largest number of contributors, with rapid growth recently.

Project Ecosystem

In addition to contributors, the projects built based on the library and the entire ecosystem surrounding the project are clear manifestations of community strength. This also indicates that a core project is not always independent; having a broad ecosystem is also a key to success for a project.

The robustness of an open-source project can, to a certain extent, be measured by the projects surrounding it. Without a doubt, Backbone.JS boasts a very extensive ecosystem and is widely used across the industry.

Full-Stack Solutions

Since the applicability of full-stack solutions may differ from other projects, we conducted a separate study on them. Full-stack solutions include both a client-side framework and a server-side framework; the amount of code involved should be greater, and the number of contributors is expected to increase over time.

Rise of JavaScript

An increasing number of developers are willing to invest time and create structured methods to use JavaScript. The chart below shows the growth in the number of developers using the top 10 frameworks since the beginning of 2011. It also illustrates that JavaScript is attempting to dominate the world, with the average number in 2013 being almost double that of 2011.

Lines of Code Contributed

The chart shows that there are differences between all contributing participants and the lines of code submitted, indicating that these projects are easier for developers to contribute to and can sustain long-term development.

Community Size and Growth in Contributors is a very important project indicator. While these metrics do not tell the whole story, they provide developers with excellent insights and reflect current investment in the project. Based on feasibility and sustainability, these data help developers make decisions and assist in finding the tipping point throughout the project lifecycle.
