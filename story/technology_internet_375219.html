Magical CSS Element Transparency Method

by diglog on 2010-12-19 12:12:37

1. **HTML Element Transparency**

In reality, achieving element transparency with CSS is quite straightforward. Here's the code:

```css

opacity: 0.5;

```

The `opacity` property refers to the level of opacity, with values ranging from 0 to 1. A value of 1 indicates full opacity, while 0 indicates full transparency. Most modern browsers (A-grade) support the `opacity` property, but when it comes to IE, things tend to get complicated. However, IE has filters that can help us achieve transparency indirectly:

```css

filter: alpha(opacity=50);

```

Note: In practice, you may encounter transparent background layers. The `opacity` property is inherited, so to avoid this issue, positioning techniques need to be combined.

In CSS3, you can also use HSLA (Hue, Saturation, Lightness, Alpha) or RGBA (Red, Green, Blue, Alpha) to achieve element transparency. For example:

```css

background: hsla(0, 100%, 50%, 0.5);

background: rgba(0, 0, 0, 0.5);

```

2. **Transparent Background Images**

One important issue to note is that PNG8 format images only support full transparency in IE6. This means that using PNG8 in IE6 will not allow for semi-transparent backgrounds. While transparency might not always matter, the problem arises when full transparency causes unpleasant jagged edges around the image. Although this can be mitigated by selecting a dithering color or removing jagged edges when exporting PNG8, the solution is not perfect.

Fortunately, PNG24 supports full transparency, making it a powerful choice for image transparency. In terms of transparency solutions, PNG24 ranks highest in terms of effectiveness.

However, when dealing with IE, another issue arises—IE6 does not support semi-transparent PNG24 images (which might sound a bit confusing).

What to do? Filters! Here's the code:

```css

_background: none;

_filter: progid:DXImageTransform.Microsoft.AlphaImageLoader(src='mangguo.png', sizingMethod='crop');

```
