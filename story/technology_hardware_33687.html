Nine Tips to Boost XP's Running Speed

by pengcqu on 2007-11-08 14:06:10

During the use of XP, the system's operating speed will gradually slow down over time. Some friends might consider reinstalling the system, but after reinstallation, so many application software programs also need to be reinstalled. How can you improve the running speed of XP without reinstalling the system? As long as you follow these nine aspects, your XP will run at high speed again.

1. Speed up the startup and shutdown process

When shutting down Windows XP, the system sends messages to the running programs and remote servers, telling them that the system is about to shut down, and waits for a response before starting the shutdown process. To speed up the startup process, you can first set the system to automatically end tasks. Find HKEY_CURRENT_USERControl PanelDesktop, set the AutoEndTasks key value to 1; then under this branch, find "HungAppTimeout" and change its value to "4000 (or less), default is 50000"; finally, find HKEY_LOCAL_MACHINESystemCurrentControlSetControl, and set WaitToKillServiceTimeout to "4000". Through such settings, the shutdown speed will be significantly faster.

2. Increase broadband speed

The Professional Edition of Windows XP defaults to reserving 20% of the bandwidth, which is actually useless for individual users. It’s better to fully utilize it rather than let it sit idle.

In "Start → Run", input gpedit.msc to open the Group Policy Editor. Find "Computer Configuration → Administrative Templates → Network → QoS Packet Scheduler", select "Limit reservable bandwidth" on the right side, choose "Properties" to open the Limit reservable bandwidth properties dialog box, select "Enable", and change the original "20" to "0", thus releasing the reserved bandwidth.

3. Optimize network neighborhood

When using Windows XP network neighborhood, the system searches for shared directories, printers that can be used as network shares, and network-related scheduled tasks in the task scheduler, and then displays them. This process is obviously much slower. If these functions are not very useful to us, we can remove them. In the registry editor, find HKEY_LOCAL_MACHINEsofewareMicrosoftWindowsCurrent VersionExploreRemoteComputerNameSpace, delete "(Printer)" and "{D6277990-4C6A-11CF8D87- 00AA0060F5BF}"(Task Scheduler). Restart the computer, and when you access the network neighborhood again, you'll find it much faster.

4. Accelerate startup speed

To speed up the startup of Windows XP, you can achieve this by modifying the registry. In the registry editor, find HKEY_LOCAL_MACHINESYSTEMCurrentControlSetControlSession ManagerMemory ManagementPrefetchParameters, find the EnablePrefetcher main key on the right, change its default value of 3 to 1, which will reduce the time the progress bar takes to scroll.

5. Automatically close non-responsive programs

Sometimes, XP will prompt you that a certain program has stopped responding, which can be quite annoying. By modifying the registry, we can make it close automatically. In HKEY_CURRENT_USERControl PanelDesktop, change the numerical data of the character key with the value AutoEndTasks to 1. Then log off or restart.

6. Speed up menu display

To speed up the display of menus, we can follow the method below: In HKEY_CURRENT_USERControl PanelDesktop, find the "MenuShowDelay" main key and change its value to "0" to speed up the menu display.

7. Clear unused DLL files from memory

In the registry, HKKEY_LOCAL_MACHINESOFTWAREMicrosoftWindowsCurrentVersion, add an item AlwaysUnloadDLL under Explorer, and set the default value to 1. Note: if the default value is set to 0, it means disabling this function.

8. Improve pre-read capability to enhance boot speed

Windows XP pre-read settings can increase system speed and speed up booting. Modifying according to the following method can further optimize CPU efficiency: In HKEY_LOCAL_MACHINESYSTEMCurrentControlSetControlSessionManagerMemoryManagement], in the right window under PrefetchParameters, change the numerical data of EnablePrefetcher as follows: For CPUs above PIII 800MHz, it is recommended to change the numerical data to 4 or 5, otherwise, it is recommended to keep the numerical data as the default value, which is 3.

9. Reduce the number of projects loaded at startup

Many applications will arbitrarily add themselves to the system startup group during installation, and they will automatically run every time the system starts, not only prolonging the startup time but also consuming system resources after startup!

Launch the "System Configuration Utility", in the "Startup" item, it lists the projects loaded during system startup and their sources. Carefully check whether you need them to load automatically. Otherwise, clear the checkbox before the project. The fewer projects loaded, the faster the startup speed naturally becomes. This requires a restart to take effect.
