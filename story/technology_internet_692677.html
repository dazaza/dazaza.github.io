Development Exercise of Notepad Application (Part I)

by dream2012 on 2012-03-06 11:09:16

In this exercise, you will learn how to add a second Activity to your Notepad application. This Activity allows users to create, edit, and delete notes. It assumes that a new note is created in response to user input and packages itself into a Bundle return value provided by an intent.

### Objectives:

1. Build a new Activity and add it to the Android manifest.

2. Call another Activity using the `startActivityForResult()` method asynchronously.

3. Pass data between Activities using various Bundle objects.

4. Learn how to use a more advanced screen layout.

### Step 4: Implement the `createNote()` Interface

Create a new Intent for the `NoteEdit` class to create a note (ACTIVITY_CREATE). Then, call `startActivityForResult()` to launch the Intent:

```java

Intent i = new Intent(this, NoteEdit.class);

startActivityForResult(i, ACTIVITY_CREATE);

```

This form of Intent invocation is targeted at a specific class within our Activity (`NoteEdit`). Since the Intent class requires requests to be sent through the Android operating system, we must also provide a Context (`this`).

The `startActivityForResult()` interface is called when the new Activity completes, returning control to our Activity. The callback interface in our Activity is named `onActivityResult()`, which we will implement later. Another interface, `startActivity()`, is a one-time invocation (fire-and-forget), where the current Activity does not receive notification of completion from the other Activity, and no results are returned via `startActivity()`.

Don't worry about the fact that `NoteEdit` does not yet exist; we will address this shortly.

### Step 5: Implement the Overridden `onListItemClick()` Interface

`onListItemClick()` is a callback function we will override. It is invoked when the user clicks an item in the note list. It has four parameters: the ListView object invoking the interface, the clicked View (contained within the ListView), the position of the clicked item (its location in the list), and the row index (`mRowId`) of the clicked item. In this example, we ignore the first two parameters (since we can only have one ListView) and the row index. Our focus is entirely on the position parameter, which tells us the location of the selected item in the list. Using this parameter, we can retrieve the desired data from the corresponding row and send it to the `NoteEdit` Activity.

In the implementation of this callback interface, we create an Intent using the `NoteEdit` class to modify the note content, then add data to the extras package passed to the called Activity. We pass the title, body, and row index of the note being edited. Finally, we call `startActivityForResult()` to launch the Intent. Below is the complete code for the `onListItemClick()` interface:

```java

super.onListItemClick(l, v, position, id);

Cursor c = mNotesCursor;

c.moveToPosition(position);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

startActivityForResult(i, ACTIVITY_EDIT);

```

`putExtra()` is an interface used to add data to the extras package for triggering the Intent. Here, we pass the title, body, and row index of the note we want to edit.

When we locate the details of the selected note using `moveToPosition()`, they are available in the query pointer.

By adding extras to the Intent and calling `startActivityForResult()`, we trigger the Intent in the `NoteEdit` class along with the request type code (which will be returned as the `requestCode` parameter in `onActivityResult()`).

### Step 6: Open the `note_edit.xml` File

This is the most complex layout we've handled so far. The file provided here avoids issues with fading in and out during code formatting. (XML is very strict about case sensitivity and structure, and these errors often stem from layout problems.)

A new parameter is introduced here: `android:layout_weight` (set to 1 in all cases). `layout_weight` assigns importance to views within a linear layout. All views have a `layout_weight` value, defaulting to zero, meaning they occupy only the space they need. If a value greater than zero is assigned, the available space in the parent view is divided according to the ratio of each view's `layout_weight` value to the total `layout_weight` value in the current screen layout.

For example:

- If we have a text label and two text editing elements horizontally aligned, and the text label has no specified `layout_weight`, it will occupy the minimum required space.

- If both text editing elements have a `layout_weight` value of 1, they will evenly divide the remaining width in the parent view layout (since their importance is declared equal).

- If the first text editing element has a `layout_weight` value of 1 and the second has a value of 2, one-third of the remaining space will be allocated to the first, and two-thirds to the second (since the second is declared more important).

This layout demonstrates how to nest multiple layouts within others to achieve more complex and aesthetically pleasing designs. In this example, a horizontal linear layout is nested within a vertical one to align the title label and text field horizontally.

### Step 8: Implement the Definition of the `onCreate()` Method in the `NoteEdit` Class

This method sets a title "Edit Note" (a string defined in the `strings.xml` file) for our new Activity and sets the note content view using the `note_edit.xml` layout file. We can obtain handles for the note title, text body views, and the confirm button. With these handles, we can set the note's title and body and bind the confirm button to respond to user presses.

Next, we can unpack the parameters passed to the Activity via the extras bundle of the calling Intent and pre-fill the title, note text, and view with these values so the user can edit them. We also retrieve and store the note row index (`mRowId`) to know which note the user is currently editing.

Set up the layout in the `onCreate()` method:

```java

setContentView(R.layout.note_edit);

```

Find the necessary edit and button components:

```java

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

```

Note that `mTitleText` and `mBodyText` are class member variables (you must declare them in the class before using them here).

Declare a private class member variable `Long mRowId` in the class to store the current row index value `mRowId` (if any).

Continue adding code to the `onCreate()` method to initialize the note title, body, and row index ID variables using the Intent extras bundle (if present):

```java

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

```

We extract the title and body values set in the extras bundle when the Intent was triggered.

We also protect against assigning null values to the text fields.

### Step 9: Complete the Implementation of the `onClick()` Method in Our Listener

When the user presses the confirm button, this interface will run. We want the method to retrieve the title and text fields of the edited note and place these data into a package as a return value, so these data can be passed back to the Activity that initially triggered the `NoteEdit` Activity. If the operation is an edit rather than a creation, we also include the row index ID in the return value package so the `Notepadv2` class can save the edited note.

Create a data package (Bundle) as the return value and pass the note title and body values using the constants defined in `Notepadv2`:

```java

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

```

Pass this data package into a new Intent and end the Activity:

```java

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

```

This new Intent serves as a simple data carrier (including the note title, body, and row index ID we want to pass).

The `setResult()` method sets the result type code and passes the Intent back to its caller. In this case, everything works, so we return the result code `RESULT_OK`.

Calling the `finish()` method signals that the Activity is complete (equivalent to a return call). Any interfaces set in the result are returned together with the execution control information to the result caller.

The complete implementation of the `onCreate()` method (along with some supporting fields for the class) is as follows:

```java

private EditText mTitleText;

private EditText mBodyText;

private Long mRowId;

@Override

protected void onCreate(Bundle savedInstanceState) {

super.onCreate(savedInstanceState);

setContentView(R.layout.note_edit);

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

}

});

}

```

### Step 10: Define the New Activity in the Manifest File

Before the new Activity can be recognized by the Android operating system, it needs its own Activity entry in the `AndroidManifest.xml` file. This way, the Android operating system knows about the Activity in the `AndroidManifest.xml` file and can call it. We can also determine which IntentFilters the Activity implements here, but we will skip that part for now and simply let the Android operating system know that the Activity is defined.
