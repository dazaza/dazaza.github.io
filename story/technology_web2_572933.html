Shengda Innovation Institute launches a free and open-source service platform: TeamHost

by ybcz0519 on 2011-11-20 19:10:27

Teamhost.org is a public, free open-source service platform for developers dedicated to open source projects. It provides open source project hosting services similar to Google Code and Github. This is just the first phase of what Teamhost aims to achieve. In this stage, apart from a few features still under development, the platform has been largely completed and officially entered the internal testing phase on October 31st! The next phase will focus on establishing a more agile interactive platform service centered around social networking (SNS).

Teamhost is based on Redmine with secondary development, providing a version control system (Git/SVN) that can be set up instantly without the need for approval. It supports new features such as directly reading readme files. In fact, it can help developers better manage all aspects of a project, including developer communication, release management, bug management, and version control.

Unlike other project management platforms, Teamhost is very user-friendly. Important functions are provided with online guidance and prompts, and there are comprehensive user documents available. All major functions can be smoothly operated step by step under the guidance of the user instruction module. On every important function page, when users use it for the first time, a detailed user guide for that function will appear, allowing even first-time users to get started quickly.

In terms of code and site security, Teamhost offers full HTTPS SSL secure channel support. Whether accessing the web site or submitting code, a secure channel is established using 2048-bit RSA encryption to ensure your safety while using the platform.

For the version control system of the project, Teamhost currently supports instant creation and use of Git and SVN. The readme file you submit can be displayed directly on the project's homepage. Additionally, the readme file in any directory of the code repository will be directly displayed on the Web source code page, making it easier for your team members to understand the content of the directory.

The roadmap feature allows you to better plan the development direction and goals of your project. It also enables users who follow your project to clearly understand the development vision of your project and get excited about it.

Through the "Issues" page of the project, errors and feature suggestions for the project can be quickly collected.

By combining the use of download and roadmap functions, after uploading the packaged project version to the download page, a download link for the latest released version will appear on the project's homepage.

Teamhost's Wiki supports multiple formats (markdown, textile, mediawiki, rdoc). Developers can choose the Wiki format they are familiar with to write the project wiki.

Advanced Usage: For some projects beyond issues, there may be other communication needs. Teamhost can provide a separate discussion area for each project. Even projects can publish news, making it convenient for developers to announce project information.

There are also some other convenient features for developers to explore during use. According to reports, more interesting functions will be added in the future. For example:

More social functions: Users can follow interesting projects and developers to keep track of their latest updates.

Fork project function: Create your own sub-branch from a project you follow.

Sync function: Supports automatic synchronization with version management platforms like Github and GoogleCode, reducing the time developers spend on code synchronization.

Teamhost launched its internal testing on October 31st and has now completed basic functionality testing. Interested individuals can send relevant project information (the more detailed the better) to xuwen02@snda.com to possibly obtain internal testing access in the first round.

Any issues or suggestions encountered during use can be reported in the dedicated "Support" section on Teamhost (https://www.teamhost.org/projects/support/issues), or you can check the current development progress.
