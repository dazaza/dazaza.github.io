Microsoft Developer Explains Why Windows Kernel Trails Linux

by geekzhang on 2013-05-13 10:08:38

It is a widely acknowledged fact that Microsoft's Windows operating system lags behind Linux in performance under complex workloads. With each new version of the Linux kernel, we can see continuous improvements in I/O scheduling, process scheduling, file system optimization, wireless networking optimizations in the TCP/IP stack, and more. A developer from Microsoft’s Windows NT kernel team admitted that the performance gap between the Windows kernel and the Linux kernel is growing wider. He posted anonymously on Hacker News via Tor (for obvious reasons) and later deleted his post, pointing out that the root cause of the issue is not technical but social. Very few developers at Microsoft are motivated to improve the kernel for personal satisfaction or glory, whereas such phenomena are common in the Linux world. In a large corporation like Microsoft, those who direct developers to improve system performance under specific conditions usually do so out of commercial intent. However, performance has never been considered critical to survival in a business sense, so there is no formal or informal process for improving system performance. Microsoft only began paying attention to security before Windows SP3 because they viewed it as an existential threat to their business goals, while performance was not seen as such a threat. The people in charge of different components of the kernel are generally hostile to external patch submissions, as unplanned changes may affect predefined objectives, angering leadership, the testing team, and product managers alike. As a result, the kernel development team has little incentive to accept externally submitted patches. There are always reasons to say no, but rarely any motivation to say yes. In contrast, in the Linux world, if you can improve the performance of a certain function by 5%, you will be widely recognized.

This highlights the differing cultures and motivations within the development ecosystems of Windows and Linux, where Linux encourages contributions and recognizes performance improvements, while Windows' corporate structure limits such incentives.
