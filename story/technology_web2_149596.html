canonical attribute - Strong web weight

by qionglian on 2009-05-16 10:47:29

Google, Yahoo, and live search have announced support for a new attribute of Link, Canonical, that specifies authoritative|canonical urls for web pages to address duplicate content.

Copying content or copying a web page is when the same or very similar content has multiple different urls, such as a print page or a URL with a sessionid. Webmasters can now add a link tag to the head of a page with the same content, specifying the preferred URL, in order to declare an authoritative page as the standard version of the copied page, so that links and search results are concentrated on this preferred version.

< link rel= quot; canonical" href= "quot; authoritative | canonical URL" />

* href can be absolute or relative; But do not point to a different domain name, otherwise it will be ignored; Allows points to different subdomains;

*link rel= quot; canonical" Attributes can be passed, that is, page A declares B to be an authoritative link, and B declares C to be an authoritative page, then SE will consider C to be the preferred authoritative version of both A and B.

* If the authority page points to a 404 page, SE will still index the contents of that page and try to find a standard page.

When is it appropriate to use cononical: In the production of the website still need to reduce the copy of the page or use 301 to go to the standardized URL, if not use link rel= "Link rel=". canonical" This label. In google's blog, you also see that if a specified authoritative page is not indexed, it may be indexed immediately, so this tag will be the best way to point out authoritative pages when no other method can eliminate copied pages.
