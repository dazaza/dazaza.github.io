Even when writing code alone, it is suggested to use a version manager. A long time ago, version management software was not widely used, and we experienced the primitive state of copying files back and forth. Nowadays, I have developed a good habit: even when working alone, I use a version manager.

There are several benefits:

1. Sometimes, you may accidentally overwrite your own program, especially due to operational errors, which can result in losing valid code and being unable to find the original correct code. This can be very painful, especially when you lose complex, time-consuming code that contains technical challenges. Losing such code might even make you feel like giving up.

2. If peers see that you manage your code using version control software, they will perceive you as professional, enhancing your reputation. In the future, when leading a team, this will make things easier.

3. In the future, if someone needs to help modify your code, having used a version manager from the start will make collaboration much smoother and more efficient when adding a few more people to the project.

Even personal code requires offsite disaster recovery. I have learned this lesson the hard way. Around 2005, I participated in a group activity and forgot to lock my hotel room door, resulting in the loss of a brand-new IBM laptop. At the time, it contained nearly a month's worth of written code, which had not been backed up. The consequences were devastating—everything I had written in that month was wasted, and I had to rewrite all the code, including addressing many details. Rewriting everything would take at least another month, with no creativity involved—it felt like taking a step backward.

While losing a laptop is a small matter compared to losing code, the laptop itself cost over 13,500 yuan, which is not easy to earn even now. Losing the code was one thing, but losing the computer was worse—the code at the time wasn't as valuable as the laptop.

Later, around 2008, another incident occurred: my external hard drive failed, and all the code backups and related software were lost. This dealt a significant blow and caused substantial losses.

Over the years, I've witnessed numerous incidents of lost laptops, damaged external hard drives, and disk crashes. As a result, I've developed the habit of always protecting critical data and program code.

---

1. I purchased a server on the public network. After making changes to the code, I upload it immediately and keep the server code synchronized almost daily. This prevents issues like theft, disk failure, or other hardware failures.

2. I share core code (such as the C#.NET development framework) with absolutely trusted friends and maintain synchronization with their computers, such as through SVN. They can fix errors in the code and upload them to the server, allowing me to stay updated with their progress.

3. My home has a 2M ADSL connection, making it convenient to update the code anytime.

4. When traveling, I use 3G wireless networks to conveniently update and synchronize the code.

5. I purchased an external hard drive and periodically back up the code, replacing the hard drive every 3-4 years to prevent major failures.

6. Periodically, I compress the latest code and store it on QQ Network Drive.

7. Periodically, I back up the latest code in my email.

---

It's now 2011. Back in 2005, the laptop was indeed more valuable than the code because earning money was difficult, and my skills were not yet sufficient. Now, while the laptop is still more valuable than the code, there has been a change: time is more valuable. Losing a month’s worth of improved and refined code would be devastating. A month's worth of life is far more valuable than 10,000+ yuan, and I cannot tolerate the idea of losing six months' worth of improvements.
