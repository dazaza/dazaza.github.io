Audacity 的新所有者正在与开源社区进行另一场斗争

2021-07-22 22:36:18

流行的音频编辑应用 Audacity 的所有者 Muse Group 再次与开源社区陷入困境。这一次，争议不在于 Audacity——而是关于 MuseScore，这是一个开源应用程序，允许音乐家创建、共享和下载乐谱（特别是但不仅限于乐谱形式）。 MuseScore 应用程序本身已获得 GPLv3 许可，这使开发人员有权分叉其源代码并对其进行修改。一位这样的开发人员，Wenzheng Tang（GitHub 上的“Xmader”）比修改应用程序更进一步——他还创建了旨在绕过 MuseScore Pro 订阅费的单独应用程序。在彻底审查了双方在 GitHub 上发表的公开评论后，Ars 与 Muse 集团战略负责人 Daniel Ray（在 GitHub 上以绰号“workedintheory”而闻名）进行了详细交谈，以找出争议的根源。在我们谈论 Muse Group 如何让自己陷入困境之前，我们必须先谈谈 MuseScore 应用程序本身是什么——什么不是。 MuseScore 应用程序提供对乐谱的访问，包括合法访问受版权保护并归迪士尼等大型团体所有的乐谱。需要注意的是，应用程序本身和它提供访问权限的乐谱不是一回事，它们不是在同一许可下提供的。该应用程序本身是 GPLv3，但它允许通过 musescore.com 访问的音乐作品具有多种许可，包括公共领域、知识共享和完全商业。对于商业保留所有权利的乐谱，Muse Group 通常不是受版权保护作品的权利人——Muse Group 是一个中介机构，通过 MuseScore 应用程序获得了分发该作品的权利。据 Muse Group 称，MuseScore 是同类应用程序中最受欢迎的——它声称每天有超过 200,000 名音乐家从超过 1,000,000 个公开可用乐谱的存储库中找到乐谱。它还声称每天有超过 1,000 个新分数上传到该服务。实际上，虽然 Xmader 确实对 MuseScore 进行了分叉，但这并不是争议的根源。 Xmader 于 2020 年 11 月对 MuseScore 进行了分叉，并且似乎完全放弃了该分叉；它总共只有 6 次提交——都是微不足道的，而且都是在创建分叉的同一周完成的。 Xmader 目前也是原始 MuseScore 项目存储库背后的 21,710 次提交。

Muse Group 与 Xmader 的合作来自另外两个存储库，专门为绕过订阅费而创建。这些存储库是 musescore-downloader（创建于 2019 年 11 月）和 musescore-dataset（创建于 2020 年 3 月）。 Musescore-downloader 简洁地描述自己：“从 musescore.com 免费下载乐谱，无需登录或 MuseScore Pro。” Musescore-dataset 几乎同样简单：它宣称自己是“musescore.com 上所有乐谱和用户的非官方数据集”。简单来说：musescore-downloader 让你从 musescore.com 下载你不应该下载的东西； musescore-dataset 是那些文件本身，已经下载。对于公共领域的乐谱或用户根据知识共享许可上传的乐谱，这不一定是问题。但是，许多乐谱只能通过乐谱所有者和 Muse Group 本身之间的安排才能获得——这有几个重要的含义。仅仅因为您可以通过应用程序或网站访问乐谱并不意味着您可以在任何地方自由访问它，无论如何，或者自己重新分发该乐谱。 Muse Group 与权利人之间的分发协议允许合法下载，但仅限于按预期使用网站或应用程序时。这些协议并没有让用户全权绕过对这些下载施加的控制。此外，这些下载通常会花费发行商真钱——由商业权利人（例如，迪士尼）免费下载许可给 Muse Group 的乐谱对于 Muse Group 本身来说通常不是“免费”的。该网站必须为分发该分数的权利付费——在许多情况下，基于下载的数量。绕过这些控制使 Muse Group 陷入困境，要么因为它无法货币化的成本（例如，通过免费用户的广告）或违反自己与权利人的分销协议（未能正确跟踪下载）。 2020 年 2 月，MuseScore 开发人员 Max Chhistyakov 向 Xmader 发送了一个删除请求，Xmader 在 GitHub 上将其重新发布为 musescore-downloader。他宣称 Xmader “非法使用 [s] 我们的私有 API 和许可的音乐内容。” Chistyakov 继续表示，大部分有问题的内容是由 EMI 和 Sony 等主要出版商授权给 Muse Group 的，而 Xmader 的下载器侵犯了这些权利人的权利。

奇斯蒂亚科夫然后威胁说，如果相关存储库没有关闭，他将不得不“将有关您的信息传输给我们的律师，他们将与 Github.com 和中国政府合作，以实际找到您并阻止非法使用许可内容。” （这个对中国政府的神秘提及将在稍后再次出现。）2020 年 6 月，MuseScore 的 Daniel Ray（又名 workintheory）回应了 GitHub 问题，“看看我们是否能够在不需要进一步流程的情况下解决这种情况。”几个月来，Ray 与 Xmader 和各种 Github 用户讨论了版权和分发的法律问题。在大多数情况下，这些讨论没有激烈的争论。 2020 年 10 月，Ray 宣布他“给了足够的时间做出回应，但现在必须继续请求 GitHub 下架”。不幸的是，事实证明这并不像 Ray 想象的那么简单——虽然 musescore-downloader 促进了 DMCA 保护作品的未经许可下载，但它本身并不包含这些作品，这意味着 GitHub 本身可以忽略 DMCA 删除请求。这导致 Github 的删除工作停滞不前，并且在长达数月没有 Muse Group 持续反馈的情况下，GitHub 帖子上的评论者宣布自己获胜，并且该帖子从 2020 年 12 月到 2021 年 5 月一直保持不变。 2021 年 5 月，对 GitHub 的兴趣返回的问题，可能是由于 GitHub 用户“marcan”从 Audacity 存储库（该存储库也归 Muse Group 所有）的遥测拉取请求中交叉引用所致。 6 月，由于商标声明，谷歌 Chrome 浏览器的 musescore-downloader 扩展从 Chrome Web Store 中删除，7 月，自由记者 Arki J. Kirwin-Muller（又名“kirwinia”）请求所有相关人员允许引用Github 发帖。 Kirwin-Muller 的请求让 Ray 再次从木制品中脱颖而出，进一步解释了 Muse Group 的争议。 Ray 指出，musescore-downloader 和 musescore-dataset 违反了美国法典第 17 条，该法典规范了美国的版权执法，直接链接到 § 1201（版权保护系统的规避），更严重的是，链接到 § 506（刑事犯罪）。 Ray 继续表示，他在起诉这些所谓的罪行时“犹豫了一年多”，部分原因是 Xmader 的个人身份。除了与 Title 17 本身相关的潜在严厉的法律处罚外，Ray 还担心刑事起诉可能会导致 Xmader 被驱逐出他目前的居住国。对于 Xmader 来说，驱逐出境也可能比大多数人更糟糕——他高度公开批评中国政府，并且在另一个 Github 回购中，他自己指出，他有朝一日可能会因这种批评而被捕。

雷最终直接对 Xmader 讲话，说他“年轻，显然很聪明，但非常天真”，并问道，“你真的想冒你的整个生命危险，这样一个孩子就可以下载你非法盗版的加勒比海盗主题曲子吗？双簧管？”有两种明显的方式来解释雷的结束问题。它是一种真诚的呼吁，还是一种毫不掩饰的公开威胁？大多数社区似乎都选择了后者。在写这篇文章之前，Ars 通过电话与 Ray 本人进行了交谈。在我们的谈话中，Ray 对音乐和开源软件都充满热诚和热情。他在不经意间明确表示，Muse Group 对代码本身没有问题——事实上，该公司鼓励这样做； Ray 表达了对分叉的毫无冲突的理解和欣赏，认为它是“自由软件是如何实现的——我是一个自由软件专家，我怀疑你知道其中的区别——是如何完成的。” Ray 继续指出，当 Muse Group 首次收购 MuseScore 时，没有任何内容获得适当的许可——简而言之，MuseScore 是一个盗版中心。根据雷的说法，最初的 MuseScore 在被 Muse Group 收购时“处于被音乐出版商和版权组织关闭的边缘”。这对于解释 Muse Group 在回应 musescore-downloader 时的必要尽职调查以及他对 Xmader 笨拙地表达的担忧都变得很重要——即使 Muse Group 忽略了 musescore-downloader，但索尼、迪士尼和 BMI 等版权所有者也有可能忽略它一旦引起他们的注意，似乎几乎为零。我们就许可问题向雷施压。我们想更好地了解他——以及 Muse Group 的——真正的开源善意。 Muse Group 最近收购开源音频编辑器 Audacity 的一个有争议的方面涉及许可变更——从 GPLv2 到 GPLv3。 Ray 解释说，GPLv3 许可更改是必要的，以允许合并 VST3 数字信号处理库，该库本身已获得 GPLv3 许可。 Ray 还解释说，Muse Group 联系了 Audacity 项目的所有 117 位个人贡献者，请求许可更改许可。他说，这些贡献者中有 90 多个做出了回应，并且每个回应都是“是”——其余的贡献很容易进行简单的重构。使用 git-blame 进行快速“嗅探检查”使这听起来合理——粗略地说，Audacity 99% 的总代码仅来自 30 人。与许多开源项目的情况一样，大多数个人贡献者都是“路过”，他们编写了几行代码来解决眼前的问题，然后就消失了。此外，Audacity 最多产的贡献者——他独自负责了其总代码行的 28% 和过去两年对该项目的承诺的 50% 以上——是现任 Muse Group 的全职员工。

我们无法对 Ray 或 Muse Group 的真实意图做出绝对的陈述。我们只能评论他们的行为。也就是说，我们花了数小时审查公司与开源社区的互动，并直接与雷本人交谈——似乎很难为恶意辩护，而不是简单的笨手笨脚的公共关系。 Ray（为 MuseScore 效力）和 Tantacrul（Audacity 的设计负责人）各自花费了大量时间，耐心地与心烦意乱的开源社区直接互动，试图解释 musescore-downloader 的删除请求以及在 Audacity 中添加基本遥测的提议。 Tantacrul 本人是著名的作曲家和软件设计师（例如，他为 Ubuntu Touch 做出了重大贡献），而 Ray 显然对开源软件既热情又知识渊博。 Muse Group 试图关闭 musescore-downloader 的最糟糕的方面是它讨论了 Xmader 作为中国侨民的身份，并警告如果诉讼开始，他可能面临的严重后果。从表面上看，很容易将其解释为隐蔽的敲诈企图——但考虑到 Muse Group 多次尝试在直接的个人层面上与社区互动，我们认为这不太可能。似乎更可能的是，Ray 的陈述应该完全按照表面价值来看待——因为认真，如果笨手笨脚，关心一个光明的年轻开发商的未来，以及在行使 Muse Group 自己必要的尽职调查过程中避免伤害他的愿望.假设是这样的话，缪斯集团的下一次收购应该是一家公关公司，而不是一个软件项目。
