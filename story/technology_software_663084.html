Chapter 3

The relationship and differences between processes and programs:

1) A program is an ordered collection of instructions. It has no inherent meaning of execution on its own; it's a static concept.

2) A process is one execution of a program on a processor; it’s a dynamic concept.

3) A process has concurrent characteristics and can operate concurrently with other processes.

4) A process is the unit for resource allocation and processor scheduling.

5) One program can correspond to multiple processes (handling different data sets such as I, C, P).

6) A process consists of a program segment, a data segment, and a process control block.

4. States and transitions of a process:

A process is created when initiated and terminates when canceled. During its lifecycle, there are generally three states (as shown in Figure 3-4). In different systems, more states can be set up.

(Familiarize with commonly used primitives) Commonly used primitives in processes: (Primitives form the core of the operating system)

Creation, cancellation, waiting (blocking), wake-up, process delay primitive and delayed wake-up primitive, process suspension primitive and process activation primitive, P operation, V operation. Process control is a function of the kernel, but not entirely.

Multiple processes need to coordinate and solve four problems:

1. Synchronization and mutual exclusion of processes

(Their concepts)

The concept of mutual exclusion. Mutual exclusion is commonly solved using locks (for understanding) and signals, PV operations (focus). A semaphore represents: a pair (s, q) where: s is an integer variable with a non-negative initial value representing the entity of resources or the state of concurrent processes; q is an initially empty wait queue. A semaphore can only be changed by P and V operations. The initial value of the semaphore is usually 1, commonly represented by mutex.

Process mutual exclusion

1. Concept of mutual exclusion

Mutually exclusive shared resources: physical devices and software resources

Related concepts: critical section and critical resources

1) P operation: Subtract 1 from the semaphore, then handle the process calling the P operation according to the value of the semaphore.

2) V operation: Add 1 to the semaphore

Synchronization: Mostly implemented using P and V operations. Set private semaphores, others can only execute V operations and cannot execute P operations. (Focus on readers, writers, producer-consumer problems.)

2. Process communication

Types: Shared memory system; Message system; Communication using shared files (pipes).

Causes of deadlock: 1) Insufficient resources; 2) Improper progress sequence of processes

Four necessary conditions for deadlock: Mutual exclusion; Non-preemption condition; Request and hold condition; Circular wait.

Methods to resolve deadlocks: Prevention: Avoidance (safety checks); Detection (monitoring programs); Recovery (such systems take no measures, once a deadlock occurs, user intervention resolves it, such as ending front-end applications, restarting, etc. Unix and Windows both adopt this method.)

3. Processor scheduling

Three-level processor scheduling: Job scheduling, process scheduling, and intermediate scheduling

4. Threads

3.3.1 Concept of threads

A thread is a smaller activity unit than a process, being an execution path of a process. (A single execution of a process without its own resources or memory.)

Threads can be described as:

1) A thread is an execution path within a process.

2) It has its own private stack and processor execution environment (especially processor registers).

3) It shares the main memory allocated to the parent process.

4) It is one of many threads created by a single process.

Comparison between threads and processes:

1. Scheduling: Use threads as the basic unit of scheduling, and use processes as the basic unit of resource ownership.

2. Concurrency: Not only can processes execute concurrently, but multiple threads within a process can also execute concurrently.

3. Resource ownership: A process is an independent unit that owns resources, while a thread does not own system resources but can access all resources belonging to the process.

4. System overhead: When creating or canceling a process, the system needs to allocate or reclaim resources, leading to significantly higher overhead compared to creating or canceling threads.

I/O system mainly includes various channels, device controllers, and devices, and four major modules' functional software (device management module software) - hardware & software.

Functions of device controllers:

1. Receive and identify commands sent from CPU or channel.

2. Implement data exchange: including data exchange between CPU, memory, and devices.

3. Identify and report device status.

4. Device address recognition: used to identify the address of each device it controls.

(Multi-channel mode, microcomputer system controls one device per control mode, but one controller can control multiple devices.)

Concept of channels:

A channel is a dedicated processor that can read, write, reverse-read, test device status, control devices (such as magnetic tape reverse winding, page swapping, etc.), control channel program jumps, so there are six types of instructions accordingly.

Channels complete input/output tasks by executing channel programs composed of channel instructions.

Classification:

Typical I/O channels have three types: byte multiplexing channel, array selection channel, and array multiplexing channel.

Connection methods:

Typical channel connection methods have two types: single-channel I/O system and multi-channel system.

Concept of device independence: (User programs are independent of specific physical device characteristics.)

To enhance the adaptability and scalability of OS, almost all operating systems implement device independence. The basic idea is that users do not directly use physical device names or physical addresses, but logical device names. Logical device names abstract physical device attributes and are not limited to specific physical devices, such as in DOS, the logical device name con corresponds to basic input/output devices, including keyboard, monitor, etc., printer logical name is PTR or LPT. Users do not care if the printer is laser or inkjet, and when one breaks, the system automatically detects and selects another good one, which is the manifestation of device independence.

Characteristics and functions of device drivers:

Set a process for each type of device;

Set an I/O process throughout the system;

Do not set a dedicated device processing process;

Functions:

Receive commands and parameters sent by the I/O process, convert abstract requirements in the command to specific requirements.

Check the legality of user I/O requests. For example, if the disk is reading, writing requests are not allowed, judged and prevented by the driver program.

Read out and check the status of I/O devices, such as busy or idle, whether damaged, etc. If ready, start the device controller.

Pass relevant parameters to the device controller.

File and file system concepts:

A file is composed of bytes, which is an unstructured file, also known as a stream file. Currently, UNIX operating systems and MS-DOS systems use this type of file.

A file is composed of records, a record is a collection of related information, a record can consist of several data items, data item types can be character type, numeric type, Boolean type, etc.

Windows supports the following files:

Concepts of logical structure and physical structure of files:

Logical structure: Record file, unstructured file.

The physical structure of a file refers to the storage structure form of a logical file on a physical storage device.

File access methods:

Sequential access; Random access;

Disadvantages of physical structure: Continuous files - like partition allocation in main memory, fragmentation problems exist.

Main disadvantage of linked files - suitable only for sequential access; relatively slow access speed;

Cannot perform random access because to read the information on a certain block, you must sequentially read all preceding physical blocks and follow pointers to find the required block.

Record storage formula (calculation):

Method to determine the physical block address and relative address within the physical block of any logical record:

Determine the relative block number of the record based on the logical address. The calculation formula is still RBN=[LBA/PBL].

Look up the index table to determine the physical block number PBN.

Calculate the relative address within the physical block. The calculation formula is still: PBO=[LBA mod PBL]

Calculate the logical byte length ALBL in that physical block.

The calculation formula is as follows: ALBL=min[LBL, (PBL-PBO)]

Calculate the remaining byte length LBL and make a judgment, if it is 0 return, if not 0 proceed to the next step.

The calculation formula is: LBL=LBA-ALBL

Calculate the logical address of the remaining part of the record, the calculation formula is: LBA=LBA+ALBL

Structure of FCB: File name; Internal name; User name

Writing of relative paths and absolute paths, especially relative paths. (Page 181)

Protection measures for file systems and files: Passwords, encryption technology, access control.

Common access control technologies: (1) Access control matrix (2) Access control list

Management of disk free space: Bit map method (hard drive, given number of pages, tracks, sectors, ask for the number of words, bits, may further ask: which physical block number corresponds in the bit map?)

Bit maps are also called bit vectors, a mapping structure.

The method adopted is to regard a certain agreed area in memory as a vector, representing a certain file storage device, each bit in the vector corresponds to a physical block, the length of the vector equals exactly the number of blocks owned by this file storage device, agreed that "0" in a certain bit of the vector indicates that the block is free, otherwise "1" indicates it is occupied.

Chapter 2 Job Management and User Interface

1. User interface: Program interface; Command interface

Program interface: The interface used by programmers when writing source programs.

It consists of a set of system call commands (shortened as system calls). This is provided for programmers to interact with the operating system through assembly programs.

Assembly source programs can directly use these commands, high-level language source programs can only use procedure call statements, translated by the compiler into system call commands.

System calls are essentially some subroutines provided by the operating system.

Command interface: The interface used for job control.

That is, the job control interface, which is further divided into online user interface and offline user interface.

Online user interface: Keyboard operation commands; Menu-driven method; Icon-driven method; Graphical user interface.

Offline user interface: Composed of a set of job control commands. Operated by the system's command interpreter program interpreting and executing the commands in the operation manual one by one.

2. Jobs, job steps, job flow

Job: A collection of work that users require the computer system to perform during a single problem-solving process or a transaction handling.

Job step: A relatively independent task that requires the computer system to perform is called a job step.

Job flow: A batch of jobs input and run in a certain sequence.

Steps in job processing: Editing, compilation, linking, running.

3. Input and output of jobs:

A. Early online input and output: Input or output runs under direct control of the CPU.

Jobs are loaded onto input devices (such as card readers, paper tape machines), then sent by the CPU running the supervisory program to auxiliary storage (early magnetic tapes), and then selected by the scheduling program from the magnetic tape for execution.

B. Offline input and output: Consists of a host and satellite machine.

The satellite machine does not connect directly to the host, but only interacts with external devices. The satellite machine transfers jobs from input devices to large-capacity backup storage (magnetic tapes, disks).

C. Spooling system: Refers to offline I/O no longer using a separate satellite machine, but completed by the host's channel and can work concurrently with the host. Mainly includes: Input program module and output program module (software)

The channel is the hardware foundation of the Spooling system.

4. Job scheduling

A. Four states of job status: Submission state, standby state, running state, completion state

B. Control block (JCB, Job Control Block)

The system sets up a job control block for each job, recording relevant information about the job.

JCB is a sign of the existence of a job.

C. Job scheduling

The program that completes job scheduling is called a job scheduling program.

Measures of scheduling performance: Average turnaround time, average weighted turnaround time

Calculation:

Turnaround time Ti for job i is defined as:

Ti=Tei-Tsi (where Tei is the completion time of job i, Tsi is the submission time of job i);

Average turnaround time T for n jobs is:

T=(T1+ T2+...+ Tn)/n

Weighted turnaround time Wi for job i is defined as:

Wi= Ti/Tri (where Tri is the actual runtime of job i)

Average weighted turnaround time W for n jobs is:

W=(W1+ W2+...+ Wn)/n

Master various job scheduling algorithms.
