Architecture Analysis of Secure Networks for IPv6

by lk0pcee on 2009-07-24 16:47:40

IPv6 first addresses the issue of IP address shortages, and secondly, makes significant changes to many imperfections in the IPv4 protocol. The most notable change is integrating IPSec (IP Security) into the protocol itself. From this point on, IPSec will no longer exist independently but will be an inherent part of the IPv6 protocol, permeating all aspects of IPv6.

### IPv6 Security Mechanism

The security mechanism of IPv6 mainly manifests in the following areas:

1. The original independent header authentication and encapsulation of security information outside the IPv4 protocol suite are integrated as an extension header within the IPv6 basic protocol. This provides a protocol-level guarantee for implementing full-network security authentication and encryption encapsulation in IPv6 networks.

2. Address resolution is placed at the ICMP (Internet Control Message Protocol) layer, which reduces its coupling with the medium compared to ARP (Address Resolution Protocol). It also allows the use of standard IP authentication and other security mechanisms.

3. For some operations in the protocol that may pose security risks to the network, the IPv6 protocol itself has implemented good protection measures. For example, regarding the potential link congestion caused by multiple interfaces starting up simultaneously on one link and sending neighbor request messages, IPv6 adopts a method of random delay transmission within a certain range to reduce the possibility of link congestion. This also reduces the likelihood of multiple nodes competing for the same address at the same time.

4. In addition to the security measures provided by IPSec and IPv6 itself, other security protection mechanisms remain effective in IPv6. For instance, NAT-PT (Net Address Translate - Protocol Translate) can provide the same protective function as NAT in IPv4; extended ACLs (Access Control Lists) on IPv6 can implement all the security protections provided by IPv4 ACLs. Additionally, secure tunnels based on VPLS (Virtual Private LAN Segment), VPWS (Virtual Private Wire Service), and VPN (Virtual Private Network) technologies can be fully realized on IPv6.

Of course, the large-scale use of IPSec inevitably affects the forwarding performance of network devices, so higher-performance hardware is needed to ensure proper operation. Overall, IPv6 significantly improves the current state of network security.

### Architecture of IPv6 Secure Networks

The security of IPv6 networks is mainly achieved through three layers: protocol security, network security, and secure encryption hardware. Below, using ZTE Corporation's ZXR10 series of IPv6 routers as an example, we introduce how to achieve IPv6 network security at these three levels.

#### Protocol Security

The combination of extended headers in IPv6's AH (Authentication Header) and ESP (Encapsulating Security Payload) with diverse encryption algorithms can provide security guarantees at the protocol level. As shown in the actual networking solution in Figure 1, ESP encryption encapsulation is used for routing protocol messages, while AH authentication is employed for protocol messages such as IPv6 neighbor discovery and stateless address configuration to ensure the security of protocol interactions.
