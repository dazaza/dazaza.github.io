在我使用Linux的20年中，我曾说过并听到过许多人们发音的方式，并认为我对大多数公认的发音和说话方式有了很好的认识在处理Linux系统时。当您在工作中进行电话会议或与同事面对面交流时，您要确保您知道他们在说什么。前几天，我听到两个同事（乔和斯科特）的讲话。乔说＆＃34; cat proc cpuinfo＆＃34;。斯科特（Scott）没明白，所以乔（Joe）说不。不，键入cat slash proc slash cpuinfo＆＃34;最终自己上了键盘。当要求某人做某事时..您不要说斜线..例如＆＃34;列出etsy主机文件＆＃34;或＆＃34; tail var日志消息＆＃34;您要做的一件事是学习如何在Linux中发音每个顶级目录。我收到的与众不同的一些信息是：/ etc：etsy（例如网站）不是＆amp; E T C＆＃34;。或“等等” / lib：lib ..押韵。不放纵。 / lib64：参见上述/ mnt：挂载/ proc：prock ..带有摇滚的押韵/ sbin：s bin / selinux：SE Linux / tmp：temp / var：带有jar的押韵/ usr：user我听说过的其他东西的发音与大多数人的发音不同：Linux：大多数人说的简短，我喜欢旋转。 MySQL：大多数人都说“我的S Q L”，而不是我的续集。 fstab：F stab（有时称为F S tab）initrd：init R D fsck：F S C K-或FS check chown：作为一个音节。 chmod：作为一个音节。韵律与＆nod＆＃39; ls：L S（ell ess）sudo：我读它时与伏都教押韵，但有人说苏面团。 vim：带健身房的韵律vi：vee eye emacs：带垃圾的韵律在下面列出您的添加内容/经验！ （注意：这篇文章是从我最初发布它的linuxforum.com中移出的）
