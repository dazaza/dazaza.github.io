Ten PHP Programming Tips You May Not Know

by billychow on 2007-12-28 22:05:09

When I used to develop with PHP, I found that PHP is both an "infuriating" and enjoyable language. The reason it is "infuriating" mainly stems from its inconsistent function naming. In fact, even though I use it every day, I still have to consider questions like "Is it str_pos or strpos? str_split or strsplit?" On the other hand, sometimes I find that a single line of code can perfectly solve the problem at hand.

Here is a short list (which might be very useful for you):

1. Use the `ip2long()` and `long2ip()` functions to convert IP addresses from strings to integers for storage in the database. This reduces storage space by about one quarter (15 bytes of characters versus 4 bytes of integers), makes it easier to calculate the valid range of the IP address, and speeds up search and sorting efficiency.

2. Use the `checkdnsrr()` function to verify whether the domain part of an email address exists. This built-in function checks if the specified domain resolves to a certain IP address to validate its existence. You can find a user-defined function in the "User Contributed Notes" section of the PHP documentation that uses this function to validate email addresses.

3. If you are using PHP5 and MySQL version 4.1 or higher, it is recommended to use the improved `mysqli_*` functions instead of the `mysql_*` functions. One excellent feature is that if your website heavily relies on databases, you can use `mysqli_stmt_prepare` to improve query efficiency. Note: Some tests.

4. Get familiar with, and love, the ternary operator.

5. Sometimes in a project, you may need to rewrite a module. Before rewriting, check out PEAR. PEAR is a very important resource; many PHP programmers know it, but even more don't. It is an online repository containing over 400 reusable code snippets that work well with PHP programs. Unless your project is really "unique," you should be able to find a PEAR package that can save time. (Reference: PECL)

6. Use `highlight_file()` to output a formatted and highlighted copy of the page source code. This function is very practical, especially in applications like guestbooks or IRC. Obviously, source files containing sensitive information such as database connections and passwords need attention.

7. Use the `error_reporting(0)` function to avoid displaying potential sensitive information to users. Ideally, error reporting should be completely disabled by modifying the `php.ini`. However, if you're on a shared virtual host, you won't have the necessary permissions, so the best solution is to add `error_reporting(0)` at the beginning of each script (or include it via `require_once()`). This way, when errors occur, potential SQL queries and path information will not be exposed to users.

8. Before storing a large string in the database, compress/decompress it using `gzcompress()` or `gzuncompress()`. These built-in functions use the gzip algorithm, which can achieve a plain text compression rate of up to 90%. Almost every time I read/write a BLOB field with PHP, I use them (unless full-text indexing is required).

9. Return multiple values from a function by passing parameters by reference. Like the ternary operator, many programmers with a programming background already know this. However, programmers who know HTML better than Pascal might be confused about "How do I return multiple values from a function? Can I only return one value?" This issue can be resolved by adding "&" before the variable to pass it by reference.

10. Fully understand the dangers of "magic quotes" and SQL injection. I hope that many developers were already familiar with SQL injection before reading this article. However, I list it here because its importance absolutely needs to be known! If you've never heard of it, spend the rest of the day Googling and learning about it!
