[Case Sharing] Ding Yu: My Mobile Mac Solution

by xdash on 2011-09-27 20:41:51

# Editor's Note: The author of this article, @FelixDing, is a project manager at Shanda Innovation Institute and the founder of the Human-Centered Design Salon.

Because of cervical spondylosis, I could no longer endure the daily long commute to and from work carrying a MacBook Pro weighing 3 kilograms. Therefore, I applied for an iMac at the company. But how can I keep the data synchronized between the two computers? For someone with a bit of data洁癖 (compulsion), this is quite important.

By data synchronization, I don't just mean keeping certain folders or program data in sync; rather, when I need it, I should be able to fully use all the data on my MBP. For example, my photos, music, emails, Adium chat records, as well as various system settings, custom-written cron scripts, and software packages installed via Port. Simply put, the data on both computers must remain strictly consistent.

Some people say this isn't difficult, there are many ready-made solutions on the market. Initially, I thought so too, but after researching extensively and practicing each one, I found that almost none of them were feasible. Fortunately, I eventually found my own solution, and this little article might be called a chronicle of blood and tears.

Dropbox/SugarSync

This was the first solution I thought of. They are easy to set up, and because they support incremental backups, I don't need too much storage space or high-speed internet. Although the cost is a bit high, it’s still acceptable.

However, after trying it out, I quickly discovered two problems: first, I severely underestimated the amount of data produced by daily computer use. While backing up large amounts of data is fine, restoring it would be unbearably slow; secondly, for some files related to startup, since the system has already started by the time restoration occurs, restoring them at this point would cause them to fail or conflict, and fixing this issue is very troublesome.

Therefore, not only are Dropbox/SugarSync unfeasible, but other network synchronization methods also don't work. In that case, the only way to transfer data is through external storage. USB speed isn't fast enough, eSATA doesn't have an interface on Macs, so the only viable option is FireWire 800, which is 1394 B. Next, we need to consider how to use the external hard drive to synchronize data.

Apple Backup, Time Machine, and Carbon Clone Copy (CCC)

These three are all beautiful and reliable data backup solutions (Apple Backup cannot back up all data). The first two are official software, and the veteran CCC has a good reputation. So, my initial idea was: could I use these to back up data onto a hard drive and then restore it on another Mac?

After consulting materials, I abandoned this type of solution: none of them support incremental restoration, and spending hours restoring data every time before using the computer would drive me crazy.

rsync and similar real-time synchronization tools

rsync is still a fairly reliable solution. After simple testing, rsync combined with a cron script can relatively well accomplish real-time synchronization between two hard drives.

The only downside of rsync is: it's a black box, you don't know which files encountered problems during synchronization, and even if you're willing to look at the log, the cost of manually handling issues is so high that you wouldn't be able to do it.

However, since I'm already using an external hard drive and can tolerate the decrease in disk performance caused by rsync, why not directly...
