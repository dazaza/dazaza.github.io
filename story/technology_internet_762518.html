Visual Studio 2013 Preview Introduces New Diagnostics Tools for Windows Store Apps

Pratap Lakshman, as a senior program manager, demonstrated and introduced various diagnostic tools included in the Visual Studio 2013 preview at the Build conference held in San Francisco. These tools can help you measure all aspects involved when developing XAML-based Windows Store applications.

According to Pratap's introduction, "Application not responding" is the main reason for about 75% of apps receiving poor reviews, followed by "application crashes," which account for 70%. About 55% of apps receive poor reviews due to "slow response," and 50% of poor reviews are attributed to significant battery drain. He stated that users look for high-quality and stable performance applications.

Pratap further explained that to make mobile applications perform well, they should be fast, smooth, and efficient. "Fast" refers to application startup and page navigation, while "smooth" focuses on scrolling, panning, and animation-related aspects. "Efficient" pertains to battery life and memory usage. He recommended that developers generate detailed inspection reports through data analysis.

As described by Pratap, the Performance and Diagnostics Hub included in Visual Studio 2013 will serve as a single entry point for analyzers. Additionally, you should create a visual tree for touch testing and rasterization checks, displaying visual objects as basic units. He also suggested reducing program complexity and updating the content displayed on the screen.

The conference also extensively discussed a new energy consumption tool that can identify parts of a scenario where there is excessive energy use. However, it does not require power metering hardware but uses a software metering model to map resource usage and evaluate the energy consumption of applications. It roughly estimates energy consumption by detecting major system resources like the CPU, display, and network, helping you optimize resource usage to make the application more energy-efficient.

Pratap pointed out during the conference that an application's energy consumption depends on the type of device screen, such as LCD, LED, OLED, and design choices, like color content and animations. It also varies with the type of network on the device, such as wired, wireless, 2G, 3G, and 4G.
