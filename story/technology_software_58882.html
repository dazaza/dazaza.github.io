World Window 2.1 Final (2.1.2.4) Released

by spmobile on 2008-03-31 16:41:21

Version: The Window to the World 2.1 Final (2.1.2.4)

Date: March 31, 2008

Now, clicking the left mouse button on the URL icon in the address bar will focus the address bar, and double-clicking will bring up a drop-down list.

For efficiency considerations, the blacklist general rules section no longer relies on HTTP protocol layer filtering.

Additions:

- Added saving of the position and size of the page element viewer window.

- Added support for system BHO plugins, and added the TWBHO type in the plugin types, with the same functionality as the system BHO.

- Added the function to display web page icons, HINT tips in the recovery list and window list menu, and show URLs in the status bar.

Improvements:

- Improved the auto-fill form feature, now it can correctly fill in checkboxes with the same name.

- Improved the auto-fill form repeat save function, existing values can now be cleared.

- Improved the middle mouse button always opening a new window function, now it unconditionally opens a new window.

- Improved the filtering of one type of pop-up window.

- Improved the logic for downloading web page icons.

- Improved the compatibility of several issues related to the boss key and various settings.

- Improved the handling of pressing Ctrl+Tab when splitting tabs.

- Improved the logic for dealing with pseudo-freezing and automatic closing after not closing web pages, now it won't lose the URL of the tw:home page.

- Improved the built-in download function, supporting more website types.

- Improved the judgment logic of the status bar only showing the URL function.

Fixes:

- Resolved the issue where the new window option in the menu was invalid under certain circumstances.

- Fixed the problem where the download prompt window always showed up when using the boss key.

- Fixed the issue where the focus setting was incorrect when switching from other programs to the Window to the World window under certain circumstances.

- Fixed the problem where the status bar displayed incorrectly under certain circumstances.

- Fixed the issue where links could still be opened after clicking on blank areas of the page when the left mouse button always opens a new window function was enabled.

- Fixed the problem where the page element viewer couldn't save properly after sorting.

- Fixed the issue where some links failed to download using the built-in download function.

- Resolved some problems where web page frames were displayed abnormally.

- Fixed the issue where the tab list menu sorted incorrectly under certain circumstances.
