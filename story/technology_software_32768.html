A Complete Summary of Windows Autostart Methods

by xloong on 2007-10-30 17:05:09

This was seen on "Professional Debt Collection & zj1244," and is translated for archival purposes. In the future, if you encounter stubborn viruses or malware, you can refer to this guide.

I. Startup Items:

Start --- Programs --- Startup, add some applications or shortcuts.

This is the most common and simplest startup method in Windows. If you want certain files to start when your computer boots up, you can drag them into the folder or create a shortcut and drag it in. Nowadays, most viruses do not use this startup method, though some may still do so.

Path: C:\Documents and Settings\Owner\「Start」 Menu\Programs\Startup

II. Second Startup Item:

This is a very obvious but often overlooked method, with the same usage as the first startup directory. Simply find the directory, drag the necessary startup files into it, and they will be started automatically.

Path: C:\Documents and Settings\User\「Start」 Menu\Programs\Startup

III. System Configuration File Startup:

Many people are unfamiliar with system configuration files, yet many viruses use this method to start.

1) WIN.INI Startup:

Startup location (xxx.exe is the name of the file to be started):

[windows]

load=xxx.exe [This method runs the file in the background]

run=xxx.exe [This method runs the file under default conditions]

2) SYSTEM.INI Startup:

Startup location (xxx.exe is the name of the file to be started):

Default:

[boot]

Shell=Explorer.exe [Explorer.exe is the Windows Program Manager or Windows Explorer, which is normal]

With a starting file:

[boot]

Shell=Explorer.exe xxx.exe [Many viruses now use this startup method, starting with Explorer, providing good concealment]

Note: SYSTEM.INI and WIN.INI are different; SYSTEM.INI can only start one specified file. Do not change Shell=Explorer.exe xxx.exe to Shell=xxx.exe, as this will cause Windows to crash!

3) WININIT.INI Startup:

WinInit is the Windows Setup Initialization Utility, also known as the Windows Installation Initialization Tool.

It allows the system to execute commands before loading Windows, including copying, deleting, and renaming files, to complete file updates.

File format:

[rename]

xxx1=xxx2

This means copying the xxx2 file as the xxx1 file, effectively overwriting xxx1.

To delete a file, use the following command:

[rename]

nul=xxx2

All filenames must include their full paths.

4) WINSTART.BAT Startup:

This is a batch file used during system startup, mainly for copying and deleting files. For example, after uninstalling software, there may be residual items in the system, at which point its function becomes useful.

Example:

“@if exist C:\WINDOWS\TEMPxxxx.BAT call C:\WINDOWS\TEMPxxxx.BAT”

This executes the xxxx.BAT file.

5) USERINIT.INI Startup [Supplement 2/2]:

Some viruses also use this startup method, similar to SYSTEM.INI.

6) AUTOEXEC.BAT Startup:

This is a commonly used startup method. Viruses can use it to perform actions, such as including malicious code in the AUTOEXEC.BAT file. For example, "format c: /y" or other commands.

V. Other Startup Methods:

(1). C:\Explorer.exe Startup Method:

Few people know about this startup method.

In Win9X, since SYSTEM.INI only specifies the name of the Windows shell file Explorer.exe, without specifying an absolute path, Win9X will search for the Explorer.exe file.

The search order is as follows:

(1). Search the current directory.

(2). If Explorer.exe is not found, the system will obtain relative path information from

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Executive\Path].

(3). If no file is found, the system will obtain relative path information from

[HKEY_CURRENT_USER\Environment\Path].

The relative path values stored in [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Executive\Path] and [HKEY_CURRENT_USER\Environment\Path] are: “%SystemRoot%System32;%SystemRoot%” and empty.

Therefore, when the system starts, the "current directory" is definitely %SystemDrive% (the system drive), so the system's search order for Explorer.EXE should be:

(1). %SystemDrive% (e.g., C:\)

(2). %SystemRoot%System32 (e.g., C:\WINNT\SYSTEM32)

(3). %SystemRoot% (e.g., C:\WINNT)

At this time, if a file named Explorer.EXE is placed in the system root directory, then every time the system starts, it will automatically start the Explorer.exe in the root directory instead of the one in the Windows directory.

In WinNT series, WindowsNT/Windows2000 pays more attention to the placement of the Explorer.exe filename, putting the name of the shell file (Explorer.exe) to be used during system startup in:

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\WindowsNT\CurrentVersion\Winlogon\Shell], and in Windows 2000 SP2, Microsoft has changed this method.

(2). Screen Saver Startup Method:

A Windows screen saver program is a *.scr file, which is an executable PE file. If the screen saver program *.scr is renamed to *.exe, this program can still start normally, and similarly, *.exe files renamed to *.scr can also start normally.

The file path is saved in the SCRNSAVE.EXE= line in System.ini. For example: SCANSAVE.EXE=/%system32% xxxx.scr

This startup method is somewhat dangerous.

(3). Scheduled Task Startup Method:

The scheduled task feature in Windows refers to a program starting at a specific time. This startup method has excellent concealment.

[Start]---[Programs]---[Accessories]---[System Tools]---[Scheduled Tasks], follow the step-by-step instructions.

(4). AutoRun.inf Startup Method:

The Autorun.inf file appears when loading a CD. When inserting a CD, the CD drive will determine whether to open the content inside the CD based on the contents of this file.

The content of Autorun.inf is usually:

[AUTORUN]

OPEN=filename.exe

ICON=icon(logo file).ico

1. For example, if a Trojan horse is xxx.exe, then Autorun.inf can be as follows:

ōPEN=Windows\xxx.exe

ICON=xxx.exe

At this time, each double-click on drive C: will run the Trojan horse xxx.exe.

2. If Autorun.inf is placed in the root directory of drive C:, then the content inside is:

ōPEN=D:\xxx.exe

ICON=xxx.exe

At this time, double-clicking drive C: will run xxx.exe on drive D.

(5). Extension Change Startup Method:

Change the extension: (*.exe)

For example, *.exe files can be changed to extensions like *.bat, *.scr to start.

VI. VXD Virtual Device Driver Startup Method:

Applications dynamically load VXD virtual device drivers to gain control of the Windows 9X system (VXD virtual device drivers are only applicable to Windows 95/98/Me).

They can manage system resources such as hardware devices or installed software using 32-bit executable programs, allowing several applications to use these resources simultaneously.

VII. Service Startup Method:

[Start]---[Run]---input "services.msc" (without quotes)---to operate on service items.

Under the "Service Startup Method" option, you can set the system's startup method: automatic operation when the program starts, manual operation, permanent stoppage, or pause (still starts after reboot).

Registry location: HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services

Programs started via services all run in the background, such as the domestic Trojan "Gray Pigeon" which uses this startup method to achieve background startup and steal user information.

VIII. Driver Program Startup Method:

Some viruses disguise themselves as hardware driver programs to achieve startup.

1. System-bundled driver programs. [Directly using standard programs bundled with the operating system to start]

2. Hardware-bundled driver programs. [Using standard programs bundled with the hardware to start]

3. Virus-disguised driver programs. [Using standard programs disguised by the virus itself to start]
