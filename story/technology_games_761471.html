Analysis of the HTML5 Game Market

by geekzhang on 2013-05-03 14:18:34

Frequently, I am asked: What is the best HTML5 game engine? What is the best HTML5 game so far? What is the best library that supports HTML5? I have answered these questions before, but until now, my responses were tailored to each individual without providing specific data.

To obtain concrete data about the HTML5 gaming industry, I created a web crawler that searches for HTML5 games online. So far, it has discovered 1,319 HTML5 games, and there may be even more that the crawler hasn't found yet. To make data collection easier, it only looks for games made using the canvas element, not the Document Object Model (DOM). Of course, not all HTML5 games are available on websites; many HTML5 applications can only be found in the iOS, Android, and Windows app stores.

Below is the data gathered by the crawler.

Most Commonly Used Game Engines:

From all the games analyzed, most were made using Construct2 (52%). When narrowing down to the top 250, top 100, top 50, and top 10, here's how the results compare:

GameMaker and Construct2 lower the difficulty of game development—you don’t need to know how to code in JavaScript, which is why these two engines are widely used in popular games. Games using the ImpactJS engine tend to be of higher quality; within the top 50 ranked games, the number using it is the same as those using Construct2. You can check out a game made with Construct2 called Not Spartan, and one using the ImpactJS engine called Z-Type.

Some well-funded game companies didn't appear on the list: GameClosure and GameSalad. This is because they primarily focus on the mobile market, publishing HTML5-based games to app stores.

Good news for HTML5 games is becoming a reality: Firefox OS, IE11 supporting WebGL, and improvements in JavaScript engine performance (especially on mobile platforms). It remains an open market, and in the coming months, we will see more tools emerge, including both tools and distribution platforms.

Annotation One: Both Construct2 and ImpactJS have active forum communities, which might explain why there are more games made with these two engines. Scirra (the developer of Construct2) has a game app store on their website, mainly featuring games built with Construct2, which also influenced the accuracy of the results. These results are not perfect.

Annotation Two: Clay.io is both an app marketplace and an API, so many games in the marketplace use its API. Kongregate operates similarly.
