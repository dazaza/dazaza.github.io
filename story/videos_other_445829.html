The biggest difference between system upgrades in Linux and Windows

by tiantian2d87 on 2011-06-14 17:01:52

For proprietary software users, OCP certification does not bring much excitement when it comes to upgrading to a new version. This is because such upgrades typically require paying for patch fixes or minor feature updates.

Even without considering the financial aspect, these upgrades can result in significant time wastage, such as backing up files, uninstalling the old version, and installing the new one. In contrast, Linux users look forward to upgrade times just like they look forward to their birthdays. For example, with a new version of KDE or Gnome, the cooler new features and user experiences are all free. Unlike upgrading Windows software, Linux program upgrades are usually handled by the operating system itself.

Linux users can usually sit back and relax while the upgrade manager completes the task on its own. The upgrade time for Linux applications is often an opportunity for users to unwind. Open-source software developers give their users a gift that keeps surprising them.

### 1. User-driven development is the biggest advantage of open source

Dave Roberts, Vice President of Strategy and Marketing at Vyatta, stated that most open-source projects release new versions very quickly. This process is community-driven, though sometimes commercial companies also participate. Therefore, users often see the expected new features. However, they usually don't feel too surprised by new features. If a developer frequently discusses with his users, new features are rarely a secret. This process is highly transparent.

For an open-source project, there is usually ongoing discussion or communication between users and the development team, which prepares product users to decide whether they need to upgrade. In comparison, proprietary software companies develop products based on their own will, rather than direct user participation.

It usually takes 12 to 18 months for a new version of proprietary software to enter the market. However, open-source software may see several different version upgrades every month.

In the open-source world, users are closer to the development process. Upgrades are usually more refined, said Roberts. The cost of releasing a new version is much lower, usually just posting the download link for the new version, without needing to burn it onto a CD or package it.

### 2. Potential impact of frequent upgrades

Although users usually expect to see more frequent new feature gifts in their favorite applications, software developers often have mixed feelings about this process.

Frequent release of new versions is a double-edged sword. Software developers are continuously making corrections and applying patches to their applications. On the downside, developers do not want to be constantly upgrading. Barry Klawans, CTO of JasperSoft and one of the founders of the Open Solutions Alliance (OSA), stated that commercial entities within the open-source community usually do not prefer frequent upgrades.

The upgrade process in Linux is generally considered quite perfect. In many Linux distributions, the operating system upgrade is automatically completed through an upgrade manager similar to Microsoft's. Software applications can be tracked via the upgrade manager, so users usually don't need to do anything other than click the OK button and input a password.

In most Linux distributions, open-source software (commonly referred to as packages) has already been included in the software repository by the community in the required format. Its installation script can be automatically processed by the operating system and added to the upgrade manager program list to automatically check for available updates.

However, for application packages outside the community software repository, users must manually run the installation script. The initial installation and future upgrades may only be possible by entering commands in the terminal window. This somewhat reminds us of the DOS era of Microsoft.
