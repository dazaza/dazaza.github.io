Super Policeman Virus Analysis Tool (File Format Identifier) v1.1 New Virtual Unpacking Function

This tool is an auxiliary tool for virus analysis, which includes various file format identification functions. It uses part of the code from Super Policeman's format recognition engine and integrates shell detection, PE file editing, MD5 calculation, and quick use of third-party tools, making it suitable for systematic processing of some virus and Trojan samples during virus analysis.

This software product is free software, and users can download, install, copy, and distribute this software product non-commercially. If commercial sales, copying, and distribution are required, such as antivirus companies using it for batch Trojan analysis, authorization and permission from DSWLAB must be obtained. Commercial companies and teams using this software must also obtain authorization and permission from DSWLAB.

New Features in V1.1:

★ Added unpacking function using VMUnpacker's unpacking engine. Identified shells can be directly unpacked by clicking the Unpack button, facilitating the analysis of packed Trojans. The unpacking capability of this version's unpacking engine is equivalent to VMUnpacker V1.4.

★ Added handling of appended data, allowing direct deletion or saving of appended data as a file for further analysis.

★ Added address conversion function for PE files, enabling convenient conversion between RAVRAW.

One, Shell Detection Function:

Supports file dragging, directory dragging, and setting right-click shell detection functions for files and directories. In addition to the built-in shell library unpack.avd of FFI, extended shell libraries can also be used (must be named userdb.txt, compatible with PEID library format, enhancing shell detection by adding your collected userdb.txt).

Note: If a shell is detected using features from the extended library, there will be a * mark after the shell information.

Two, PE Editing Function:

The main interface of this program can display the entry point/physical offset of the entry point and section information of the inspected program.

The button after PE Section can edit the section table of the current file, and clicking it will bring up the Sections Editor window.

Main functions include:

★ Display detailed section information

★ View and edit section name, size, execution attributes, and other related information.

★ Clear the selected section name

★ Automatically repair the section

★ Load sections from disk

★ Save sections to disk

★ Add a new section

★ Delete sections from the file

★ Delete sections from the PE header (section content still exists)

★ Fill sections with specified data

The button after SubSystem can display detailed information about the PE file, supporting detailed editing of Dos headers, NT headers, etc., and viewing export/import table information of PE files. Please refer to the interface for specific details.

Three, Appended Data Detection:

Can scan whether the application contains appended data and provides detailed starting position and size of the appended data.

Four, Third-Party Tool Support:

In the Options button, click the Manage Tools button to add/delete third-party tools like IDA/OllyDBG via the right-click menu. This allows you to directly open the current file in OllyDBG, IDA, etc., within FFI for disassembly.

Note: After adding third-party tools, click the Plugin>> button to see the added tool information, and clicking it will open the current file with the selected tool.
