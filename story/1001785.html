发布Proxmox VE 6.2

2020-05-12 18:27:05

奥地利维也纳-2020年5月12日-Proxmox Server Solutions GmbH今天宣布开放源码虚拟化管理平台的最新版本Proxmox VE 6.2正式上市。Proxmox VE 6.2包括旨在解决现代数据中心管理员和IT团队面临的问题的新功能。新版本的虚拟化管理解决方案提供了许多新功能、显著改进，以及许多针对基于Web的用户界面的高级选项。它基于Debian Buster 10.4和5.4长期Linux内核，包括对领先的开源虚拟化技术QEMU 5.0、LXC 4.0、Cave Nautilus(14.2.9)和ZFS 0.8.3的最新版本的更新。

Proxmox VE除了现有的基于HTTP的验证模式外，还通过基于DNS的质询机制实现了对let加密TLS证书的域的内置验证。

完全支持多达8条CorSync网络链路。使用的链路越多，群集可用性就越高。

在存储内容视图中，管理员现在可以使用新的“创建日期”列筛选存储的数据，例如，这简化了搜索某个日期的备份。

无需重新启动会话即可无缝更改Web界面的语言。添加了阿拉伯语翻译，因此Proxmox VE总共支持20种语言。

集成容器技术已更新到lxc 4.0.2和lxcfs 4.0.3。Proxmox VE 6.2现在允许为基于目录的存储上的容器创建模板。

用于Ubuntu20.04、Fedora32、CentOS8.1、Alpine Linux和Arch Linux的新LXC模板。

集成的备份管理器支持快速高效的无损数据压缩算法ZStandard(Zstd)。

Proxmox VE对所有对象(如虚拟机、存储、节点等)使用基于角色的用户和权限管理。新的LDAP同步支持将LDAP用户和组同步到Proxmox用户和组权限框架。

添加了对API令牌的完全支持和集成，允许另一个系统、软件或API客户端对REST API的大部分进行无状态访问。可以为单个用户生成API令牌，并且可以选择使用单独的权限和到期日期进行配置，以限制访问的范围和持续时间。如果API令牌受到危害，则可以撤销它，而不必禁用用户本身。

现在可以从https://www.proxmox.com/downloads下载Proxmox VE6.2。使用APT可以将Proxmox VE 4.x或5.x版本的分发升级到6.x。

Proxmox VE在自由软件许可证GNU Affero GPL v3下获得许可。企业支持可从Proxmox Server Solutions以订阅方式获得，起步价为每年85欧元和中央处理器，请参阅https://www.proxmox.com.。

开源项目Proxmox VE在全球拥有超过350,000台主机的庞大用户群。虚拟化平台被翻译成20多种语言。在支持论坛上，超过6万名活跃的社区成员相互参与。通过使用Proxmox VE作为专有虚拟化管理解决方案的替代方案，企业能够集中化和现代化其IT基础设施，并将其转变为基于最新开源技术的经济高效、灵活的软件定义数据中心。来自不同部门、规模或行业的数以万计的公司客户都依赖Proxmox VE支持订阅，这是Proxmox Server Solutions GmbH提供的一项服务。

Proxmox VE是面向包罗万象的企业虚拟化的领先开源平台。通过中央Web界面，您可以轻松运行虚拟机和容器、管理软件定义的存储和网络功能、高可用性群集以及多种集成的开箱即用工具，如备份/恢复、实时迁移、复制和防火墙。企业使用功能强大且易于管理的一体化解决方案Proxmox VE来满足当今现代数据中心的核心要求(复杂性更低、灵活性更强)，这要归功于灵活、模块化和开放的架构，从而确保保持对未来增长的适应性。

Proxmox成立于2005年，是一家功能强大但易于使用的开源服务器软件的全球提供商。任何规模、部门或行业的企业都可以使用稳定、安全、可扩展和开放的Proxmox解决方案来部署高效、灵活和简化的IT基础设施，最大限度地降低总拥有成本，并避免供应商锁定。Proxmox提供商业支持和培训，以确保其客户的业务连续性。Proxmox Server Solutions GmbH总部设在奥地利维也纳，在世界各地拥有数百个合作伙伴的全球合作伙伴网络。
