Issues to be mindful of when using tags during WML development

by wap3g on 2008-03-18 18:47:27

Under normal circumstances, the HTML code generated by this tag would look like:

```html

```

This code clearly does not conform to WML page standards because the `img` tag is not closed. To ensure that our pages can be viewed on all mobile devices, we must fully adhere to WML standards. In a WML page, the requirement is:

```html

```

(Note the closing slash here.)

But why doesn't the `` tag generate the closing slash? After examining the source code of `ImgTag -> BaseHandlerTag`, the reason becomes clear. This tag uses a parameter to control whether or not to generate the closing slash. However, this parameter is not controlled directly by the tag itself but rather by another tag, specifically ``. We must specify a parameter for this tag as follows: ``. Only after doing so will `` or other tags generate the closing slash.

However, there's a problem: the `` tag cannot be used in a WML page. Therefore, there are two solutions:

### Solution 1:

Based on the source code of `BaseHandlerTag`, as shown below:

```java

protected boolean isXhtml() {

String xhtml =

(String) this.pageContext.getAttribute(Globals.XHTML_KEY, PageContext.PAGE_SCOPE);

if ("true".equalsIgnoreCase(xhtml)) {

return true;

} else {

return false;

}

}

```

We just need to set a value of `true` for `pageContext`. Therefore, before using the `` tag, we can execute the following code:

```java

pageContext.setAttribute(Globals.XHTML_KEY, "true", PageContext.PAGE_SCOPE);

```

### Solution 2:

Override the `isXhtml` method. We can create a new tag library class called `NewImgTag` that extends `ImgTag`. The code is as follows:

```java

/**

* Image tag

* @author liudong

*/

public class NewImgTag extends org.apache.struts.taglib.html.ImgTag {

protected boolean isXhtml() {

return true;

}

}

```

Of course, you also need to define a new tag library definition file and copy the definition of the `img` tag from `struts-html.tld`, modifying the class name accordingly.

With these changes, the `img` tag can now be used in WML pages :)
