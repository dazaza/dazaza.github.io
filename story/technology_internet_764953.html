Microsoft has released the Windows Azure SDK 2.1 for .NET.

by anonymous on 2013-08-08 10:54:52

Windows Azure SDK 2.1 with VS 2013, VM, Notification Hub, Cloud Services, PowerShell cmdlets Support

Microsoft has released the Windows Azure SDK 2.1 for .NET, which supports virtual machine images and enhances Visual Studio Server Explorer, virtual machines, and cloud services. This SDK also supports Service Bus Notification Hubs, new PowerShell commands for automating Web sites, cloud services, and virtual machines, as well as improved Visual Studio tools. In addition to Visual Studio 2010 and 2012, this latest release now supports the Visual Studio 2013 Preview.

Microsoft has also added a new image to the Windows Azure Virtual Machine Library that comes pre-installed with the Visual Studio 2013 Ultimate Preview, SharePoint 2013, SQL Server 2012 Express, and the Windows Azure 2.1 SDK. This enables you to build applications in the cloud using the latest tools.

The Visual Studio Server Explorer has been redesigned so that all services are now contained under a single Windows Azure node, with the ability to manage certificates and subscription files. You can filter Windows Azure services in the Server Explorer based on subscriptions or regions, and access properties of cloud services, deployment slots, role instances, storage accounts, virtual machines, and web sites.

You can also directly start, stop, restart, and delete a virtual machine from within the Visual Studio Server Explorer via the Windows Azure SDK 2.1 for .NET. The stop operation shuts down the virtual machine and suspends billing when the VM is not in use. Additionally, restarting the virtual machine is made very easy. The SDK also provides support for paired namespaces, integrated subscription management, and integrates the Service Bus node into the Windows Azure node. The release includes new PowerShell commands in the following categories:

China instance and Windows Azure package support

Environment configuration, Virtual Machines, Cloud Services, Web Sites, Storage, SQL Azure, Service Bus

The latest release of the Azure SDK includes a new option called Emulator, which allows users to launch Visual Studio and run cloud services locally as a standard user without requiring administrative privileges. It also provides support for Windows Azure Notification Hubs, enabling you to create, update, and delete hubs using code. Additionally, you can manage device registrations and send push notifications to Windows Store, Windows Phone 8, iOS, and Android devices.

Microsoft has also updated the Visual Studio Team Foundation Service (TFS) to support the features included in the Windows Azure SDK 2.1, including automated builds and testing.
