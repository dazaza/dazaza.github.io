When you encounter AIX TELNET-related fault issues, do you feel it's very difficult to handle? Don't worry, we will address some of these fault issues. Here, we will share the experience of a netizen and see how they resolved the AIX TELNET problem. The issue was: "The paging space on AIX was full, and telnet could no longer be used. However, Xmanager could still connect."

```

-bash-3.00$ lsps -a

Page Space Physical Volume Volume Group Size %Used Active Auto Type

hd6 hdisk0 rootvg 512MB 91 yes yes lv

```

The machine has 8GB of memory and runs a DB2 database, but the swap partition is only 512MB. It is planned to expand it to 8GB. While the system is running business operations, I am unsure whether using `chps -s32` to expand the paging space would affect the business. Therefore, I decided to add a new paging space and then activate it.

**Add Another Paging Space**

Fill in or select values in the entry fields.

Press Enter AFTER making all desired changes.

[Entry Fields]

Volume group name rootvg

SIZE of paging space (in logical partitions) [32] #

PHYSICAL VOLUME name +

Start using this paging space NOW? yes +

Use this paging space each time the system is restarted? yes +

After pressing Enter, I checked with `lsps -s` and `lsps -a`, and found that the new paging space had been added successfully, and the usage rate dropped to 6%.

I thought the AIX TELNET issue was resolved. However, later, I discovered that telnet and FTP were still unable to connect. Does this mean a reboot is required? That would be troublesome as an application needs to be submitted.

Next, I checked the telnet service:

```

lssrc -t telnet

// Unable to view, roughly meaning the Subsystem is not started

lssrc -s inetd

-bash-3.00$ lssrc -s inetd

Subsystem Group PID Status

inetd tcpip 237684 inactive (approximately this term, anyway, it hasn’t started)

-bash-3.00$

-bash-3.00$

startsrc -t inetd

0513-059 The sshd Subsystem has been started. Subsystem PID is 585780

```

After checking again with `lssrc`, I found that the inetd, telnet, and FTP services were all functioning normally. Thus, the AIX TELNET issue was fully resolved.
