Ubuntu 8.04 LTS RC Upgrade Manual [Repost]

by xloong on 2008-04-17 23:21:08

Nightfire: There is only one week left until the official release of Ubuntu Linux 8.04 LTS. The Ubuntu 8.04 LTS RC was originally planned for release today, but due to a small issue with package packaging, it will be delayed by one day.

This applies only to users performing fresh installations. If you simply want to upgrade your existing Ubuntu system to the latest Ubuntu 8.04 LTS TS, you can proceed now! The network resources are already prepared! This article describes an upgrade process from Ubuntu 7.10; similar methods apply to Ubuntu 7.04, Ubuntu 6.10, etc.

### Preparation Work

When upgrading to Ubuntu 8.04 RC, please ensure that you have backed up important files to avoid potential losses caused by failed upgrades.

Also, make sure there is plenty of free space in the root directory (at least 2GB or more), as a large number of new software packages will be downloaded during the upgrade process (depending on the quantity of software packages in your original system). Lastly, ensure a stable internet connection.

### Starting the Upgrade

Open your terminal and type `update-manager -c -d` (no need for `sudo`). The "Update Manager" will then start and check for a new distribution. If everything goes well, it will display "New distribution '8.04 LTS' available." As shown below:

After clicking "Upgrade," a release note window will pop up, showing your upgrade target, the Release Candidate of Ubuntu 8.04 Hardy Heron. Confirm and click "Upgrade" again.

From this point onward, the upgrade process becomes mostly automatic with minimal user interaction.

Note: You can cancel the upgrade at any time before the installation begins after downloading the new software packages. Once the download completes and the upgrade starts, you cannot cancel anymore.

The update manager will first update the upgrade tool itself, downloading the latest version from the internet to ensure a smooth upgrade process.

#### Preparing for the Upgrade

If you haven't modified your `sources.list` beforehand, a dialog box will appear asking if you want to update the software sources to the hardy version. Choose "Yes" to update all software sources to the hardy version.

After setting up the sources automatically, the system will update the software channels, equivalent to running `apt-get update` in the terminal.

After updating, you will receive a prompt about package support.

After a short calculation, the Update Manager will display detailed upgrade information, including the list of packages to be removed, newly installed, and upgraded.

It's worth noting that due to what seems to be a Chinese translation issue, the estimated upgrade time is incorrect. It should show 2 hours and 0 minutes, but instead shows 2 days and 0 hours. This error has been carried over since version 6.10.

#### Downloading New Software Packages

This is your last chance to regret the upgrade. If you don't want to upgrade, click "Cancel." Beyond this point, saying goodbye to the old Ubuntu is inevitable, though you can still force a restart to cancel the upgrade.

After approximately "1 day and 28 hours," the software package downloads complete, and the formal upgrade begins. The displayed time estimate is now correct, depending on the number of packages, it may take between 30 minutes to 1 hour.

After another long wait, the installation is almost complete, and the final step is: cleaning up obsolete packages.

Ensure that none of the listed packages are needed, then remove them!

Don't hesitate—restart now!

If everything goes smoothly, after restarting and logging into your account, a brand-new Firefox 3 will automatically open, welcoming you with the homepage "Welcome to Ubuntu 8.04 LTS."

Congratulations on entering the world of Ubuntu 8.04!

### Postscript:

I started using Ubuntu since version 6.10, and every time a new version was released, I would use this method to upgrade. However, both upgrades from 6.10 to 7.04 and from 7.04 to 7.10 failed, forcing me to perform a complete reinstallation.

However, this time, the upgrade from 7.10 to 8.04 went very smoothly, resulting in a highly stable system post-upgrade. All previous applications were fully inherited (Apache2, Mysql, etc.).

Can this indicate that Ubuntu's version upgrade function is becoming increasingly perfect? That remains to be tested by everyone!
