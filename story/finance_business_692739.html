Development Exercise of Notepad Application (Part I)

by dream2012 on 2012-03-06 11:21:07

In this exercise, you will learn how to add a second Activity to your Notepad application. This Activity allows users to create, edit, and delete notes. It assumes that new notes are created in response to user input and packages itself into a Bundle return value provided by an intent. The objectives of this exercise are:

- Build a new Activity and add it to the Android manifest.

- Invoke another Activity using the `startActivityForResult()` method asynchronously.

- Pass data between Activities using several Bundle objects.

- Learn how to use a more advanced screen layout.

### Step 1: Create a New Android Project

Create a new Android project using the resources in the Notepadv2 directory inside the NotepadCodeLab folder, similar to what was done in Exercise 1. If you encounter any errors related to `AndroidManifest.xml` or the `android.zip` file, right-click on the project, select **Android Tools > Fix Project Properties** to resolve them.

Open the `Notepadv2` project and examine the following:

- Open and review the `strings.xml` file located under `res/values`, which contains several new strings for the new functionality.

- Open and review the top of the `Notepadv2` class where you'll notice a new field pointer `mNotesCursor` has been added, along with some newly defined constants below it.

- Note that the `fillData()` method now includes additional comments and uses the newly defined fields to store note pointers. The `onCreate()` method remains unchanged from Exercise 1.

- Observe the class member variable used to store note pointers, called `mNotesCursor`. The prefix 'm' indicates that this is a member variable, adhering to Android coding style standards.

- Implement two new overridden interfaces: `onListItemClick()` and `onActivityResult()`.

### Step 2: Add a Menu Item to Delete a Note

Add the following line in the `onCreateOptionsMenu()` interface:

```java

menu.add(0, DELETE_ID, 0, R.string.menu_delete);

```

The complete definition of the interface is as follows:

```java

@Override

public boolean onCreateOptionsMenu(Menu menu) {

super.onCreateOptionsMenu(menu);

menu.add(0, INSERT_ID, 0, R.string.menu_insert);

menu.add(0, DELETE_ID, 0, R.string.menu_delete);

return true;

}

```

### Step 3: Add a New Branch Statement for `DELETE_ID` in `onMenuItemSelected()`

Add the following code in the `onMenuItemSelected()` interface:

```java

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

```

Here, we delete the specified ID note through the `deleteNote()` interface. To get the ID of the specific note, we call `getListView().getSelectedItemId()`. Then, we call `fillData()` to keep the data updated.

Complete definition of the `onMenuItemSelected()` interface:

```java

@Override

public boolean onMenuItemSelected(int featureId, MenuItem item) {

switch(item.getItemId()) {

case INSERT_ID:

createNote();

return true;

case DELETE_ID:

mDbHelper.deleteNote(getListView().getSelectedItemId());

fillData();

return true;

}

return super.onMenuItemSelected(featureId, item);

}

```

### Step 4: Implement the `createNote()` Interface

Create a new Intent with the `NoteEdit` class to create a note (ACTIVITY_CREATE). Then, invoke the Intent using the `startActivityForResult()` interface:

```java

Intent i = new Intent(this, NoteEdit.class);

startActivityForResult(i, ACTIVITY_CREATE);

```

This form of Intent invocation is targeted at a specific class within our Activity, in this case, `NoteEdit`. Since the Intent class requires sending requests through the Android operating system, we must provide a Context (`this`).

The `startActivityForResult()` interface is invoked only after the new Activity completes execution. In our Activity, the callback interface is named `onActivityResult()`, which we will implement in later steps. Another interface, `startActivity()`, is a fire-and-forget approach where the current Activity does not receive a notification when the other Activity finishes, and no results can be returned via `startActivity()`.

Don't worry about the fact that `NoteEdit` does not yet exist; we will address this soon.

### Step 5: Implement the Overridden `onListItemClick()` Interface

`onListItemClick()` is a callback function we will override. It is invoked when the user clicks on an item in the note list. It has four parameters: the ListView object invoking the interface, the clicked View (within the ListView), the position (the location of the item in the list), and the row index (`mRowId`) of the clicked item.

In this example, we ignore the first two parameters (since we can only have one ListView) and the row index. Our focus is entirely on the position parameter, which tells us the location of the selected item in the list. Using this parameter, we can retrieve the desired data from the corresponding row and send it packed to the `NoteEdit` Activity.

In the implementation of this callback function, we create an Intent with the `NoteEdit` class to modify the note content, then add data to the bundle passed to the called Activity. We pass the title, body, and row index of the note currently being edited. Finally, it invokes `startActivityForResult()` to start the Intent.

Complete code for `onListItemClick()`:

```java

super.onListItemClick(l, v, position, id);

Cursor c = mNotesCursor;

c.moveToPosition(position);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

startActivityForResult(i, ACTIVITY_EDIT);

```

The `putExtra()` interface adds data to the bundle for triggering the Intent. Here, we pass the title, body, and row index of the note we want to edit.

When we locate the details of the selected object in the corresponding position in the list, all the information about the note is available in the query pointer through the `moveToPosition()` interface.

By adding information to the Intent and invoking `startActivityForResult()`, we trigger the Intent in the `NoteEdit` class and request a type code (which will be returned as the parameter `requestCode` to the `onActivityResult()` interface).

### Step 6: Implement the `onActivityResult()` Interface

`onActivityResult()` is an overridden interface that is only invoked when an Activity returns a result (remember! An Activity only returns a result if it was started with `startActivityForResult()`).

Parameters for the callback include:

- `requestCode`: The initial request type code determined when the Intent was triggered (for us, either `ACTIVITY_CREATE` or `ACTIVITY_EDIT`).

- `resultCode`: The result (or error) type returned by the call. A zero value indicates success, while non-zero values indicate failure or exceptions.

- `intent`: An Intent created by the returning Activity, used to return data through Extras.

Combining `startActivityForResult()` and `onActivityResult()` can be viewed as an asynchronous remote procedure call (RPC), forming a recommended way for one Activity to launch another and share services.

Complete definition of the `onActivityResult()` method:

```java

super.onActivityResult(requestCode, resultCode, intent);

Bundle extras = intent.getExtras();

switch(requestCode) {

case ACTIVITY_CREATE:

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mDbHelper.createNote(title, body);

fillData();

break;

case ACTIVITY_EDIT:

Long mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (mRowId != null) {

String editTitle = extras.getString(NotesDbAdapter.KEY_TITLE);

String editBody = extras.getString(NotesDbAdapter.KEY_BODY);

mDbHelper.updateNote(mRowId, editTitle, editBody);

}

fillData();

break;

}

```

In this method, we control both `ACTIVITY_CREATE` and `ACTIVITY_EDIT` Activity results. For creation, we retrieve the note's title and body from the extras (from the returned Intent) and use them to create a new note. For editing, we retrieve the title, body, and row index from the extras and use them to edit and update the note in the database. The final `fillData()` ensures everything is updated.

### Step 7: Open and Review the `note_edit.xml` File

This is the most complex interface layout we've handled so far. This file is provided to avoid issues with code formatting and alignment. XML is very strict about case and structure, and these errors often cause screen layout problems.

A new parameter we haven't seen before is used here: `android:layout_weight` (set to 1 in each case). `layout_weight` assigns importance to views in a linear layout. All views have a `layout_weight` value, defaulting to zero, meaning they take up as much space as needed. If a value greater than zero is assigned, the available space in the parent view is divided based on the ratio of each view's `layout_weight` value to the total `layout_weight` value in the current screen layout.

For example, consider a horizontal layout with a text label and two text editing elements. If the text label has no specified `layout_weight`, it will occupy the minimum required space. If both text editing elements have a `layout_weight` of 1, they will equally divide the remaining width of the parent layout (since their importance is declared equal). If the first text editing element has a `layout_weight` of 1 and the second has a `layout_weight` of 2, the first gets one-third of the remaining space, and the second gets two-thirds (since the second is declared more important).

This layout also demonstrates how to nest multiple layouts within others to achieve more complex and aesthetically pleasing layouts. In this example, a horizontal linear layout is nested within another vertical layout to align the title label and text field horizontally.

### Step 8: Create a `NoteEdit` Class Extending `android.app.Activity`

This is the first time we're creating an Activity without the help of the Android Eclipse plugin. When created this way, the `onCreate()` method is not automatically overridden. It's hard to imagine an Activity without an overridden `onCreate()` method, so this is the first thing you need to do.

Right-click on the `com.android.demo.notepad2` folder in the Eclipse folder tree, select **New > Class**, name the class `NoteEdit`, and set the superclass to `android.app.Activity`.

Once the class is generated, right-click in the editor window, select **Source > Override/Implement Methods...**, and check the box next to `onCreate(Bundle)`.

### Step 9: Implement the `onCreate()` Method for the `NoteEdit` Class

Set the title of the new Activity to "Edit Note" (a string defined in `strings.xml`) and set the content view using the `note_edit.xml` layout file. Retrieve handles for the note title, text body, and confirm button views.

Unpack the parameters passed via the Intent's extras bundle and pre-fill the title and note text body and view so the user can edit them. Also, retrieve and store the note row index (`mRowId`) to know which note the user is currently editing.

Implementation:

```java

setContentView(R.layout.note_edit);

mTitleText = (EditText) findViewById(R.id.title);

mBodyText = (EditText) findViewById(R.id.body);

Button confirmButton = (Button) findViewById(R.id.confirm);

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

}

```

### Step 10: Create an `onClickListener()` Method for the Button

Implement the `onClick()` method in an anonymous inner class to handle the confirm button click event. When the user clicks the button, extract the edited note's title and text fields, place these data into a bundle, and return them to the calling Intent.

```java

confirmButton.setOnClickListener(new View.OnClickListener() {

public void onClick(View view) {

Bundle bundle = new Bundle();

bundle.putString(NotesDbAdapter.KEY_TITLE, mTitleText.getText().toString());

bundle.putString(NotesDbAdapter.KEY_BODY, mBodyText.getText().toString());

if (mRowId != null) {

bundle.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

Intent mIntent = new Intent();

mIntent.putExtras(bundle);

setResult(RESULT_OK, mIntent);

finish();

}

});

```

### Step 11: Define the New Activity in the Manifest File

Before the new Activity can be recognized by the Android operating system, it needs its own Activity entry in the `AndroidManifest.xml` file. This allows the Android OS to know about the Activity and call it. You can also define which IntentFilters the Activity implements, but we will skip that for now and just let the Android OS know the Activity is defined.

Use the Manifest Editor included in the Eclipse plugin to conveniently edit the `AndroidManifest.xml` file. Alternatively, you can directly edit the file.

Double-click on `AndroidManifest.xml` in the folder tree to open it. Click the **Application** tab at the bottom of the Manifest Editor. In the application section, click **Add...**. Select the radio button at the top labeled "Create a new element at the application level". Before clicking "OK", ensure the "(A)Activity" option is selected in the dialog. In the Application Node section, click the new "Activity", enter `.NoteEdit` in the Name* field, and press Enter.

### Step 12: Run the Application

You should now be able to add real notes from the menu and delete existing notes. To delete a note, highlight it using the device's directional controls. Selecting a note from the list should bring up an editor allowing users to edit the note. After editing, pressing the confirm button will save the changes to the database.
