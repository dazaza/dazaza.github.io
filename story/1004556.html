Microsoft Edge可保存对PDF文件所做的编辑，而无需每次保存副本

2020-06-01 01:53:11

作为一个重大突破，Microsoft Edge现在支持原生文件系统API，这将把渐进式Web应用程序(PWA)及其使用提升到一个全新的水平。官方的路线图是通往新开发的切入点，这只意味着一件事：使用现代网络技术弥合本地应用程序的差距。

“Microsoft Edge现在支持本机文件系统API。Microsoft Edge现在支持通过本机文件系统API授予站点编辑文件或文件夹的权限。“。

Microsoft Edge最终支持Native File System API是一件大事。原因如下：

渐进式Web应用程序(PWA)显然是软件交付的未来。在过去的几年里，Web本身已经发展成为一个功能强大的平台，随着PWAS的引入和大量采用，跨设备的软件交付变得容易得多。这就是促使如此多的Web开发人员将他们的注意力转移到PWAS上的原因。

但是，无论PWA有多伟大，它们都有一些我们不可能忽视的局限性。这些限制使用户无法将本地应用替换为渐进式Web应用。简而言之，PWAS不能做原生应用程序能做的一切。这正是谷歌、微软和其他公司试图改变的。

正如Chrome OS开发者倡导者Sam Richard在2019年Chrome Dev峰会上指出的那样，捆绑在本地包装器中的Web应用程序迫使Web开发人员承担使浏览器和其本地包装器保持最新的安全和维护负担。

“对于那些确实想为Web构建但需要原生功能的人，他们被迫将Web应用捆绑在原生包装器中。”

这正是Microsoft Edge的Native File System API支持发挥作用的地方，Edge已经推出了使用此Native File System API的原生PDF编辑支持。

将来，Microsoft Edge用户可以轻松地将对PDF文档所做的编辑保存回文件，而不是每次都保存一份副本。

从Google Chrome83开始，针对所有桌面平台(包括Windows、Linux和MacOS)的Native File System API开始了新的原始试用。我们在文本编辑器演示中看到了它的实际应用。但首先，我们必须通过Chrome：//标志启用Native File System API实验。

现在，转到googlechromelab.github.io/text-Editor，在您的计算机上创建并保存一个.txt文件。有趣的是，您可以使用Web应用程序直接更改存储在上的保存的TXT文件。

Microsoft Edge正在扩展对编辑PDF文档的类似支持。原生文件系统API和原生PDF编辑都在向Microsoft Edge Beta Channel推出。
