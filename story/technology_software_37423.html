The Mystery of the Svchost.exe Process

by xloong on 2007-12-01 15:36:44

Repost: http://www.pccourse.cn/html/13/n-5613.html

The svchost.exe process is a very important component of the NT core system and is indispensable for Windows 2000 and XP. Many viruses and trojans also invoke it. Therefore, gaining a deep understanding of this program is one of the essential lessons for computer enthusiasts.

You are certainly familiar with the Windows operating system, but have you noticed the "svchost.exe" file in the system? Observant users will find that there are multiple "svchost" processes in Windows (you can see them by pressing "Ctrl+Alt+Del" to open the Task Manager and checking under the "Processes" tab). Why is this so? Let's uncover its mysteries.

### Discovery

In the Windows operating system family based on the NT kernel, different versions of Windows have varying numbers of "svchost" processes. Users can check the number of these processes using the "Task Manager." Generally, Windows 2000 has two svchost processes, while Windows XP has four or more (so if you see multiple such processes in your system, don't immediately conclude that your system is infected with a virus). In Windows 2003 Server, there are even more. These svchost processes provide many system services, such as RPCSS service (Remote Procedure Call), DMServer service (Logical Disk Manager), DHCP service (DHCP Client), etc.

If you want to know how many system services each svchost process provides, you can enter the "tlist -s" command in the command prompt window in Windows 2000 (this command is provided by the Windows 2000 Support Tools). In Windows XP, use the "tasklist /svc" command.

The svchost can contain multiple services.

### Deep Dive

Windows system processes are divided into independent and shared processes. The "svchost.exe" file is located in the "%systemroot%\system32" directory and belongs to the shared process category. As Windows system services continue to increase, to save system resources, Microsoft implemented many services in a shared manner, which are started by the svchost.exe process. However, the svchost process only acts as a service host and cannot implement any service functions itself. It merely provides the conditions for other services to be started within it and cannot offer any services directly to users. So how do these services function?

It turns out that these system services are implemented in dynamic link library (DLL) form. They point their executable programs to svchost, and svchost calls the corresponding DLLs to start the services. How does svchost know which DLL to call for a specific system service? This is achieved through parameters set in the system registry for each service. Below, we'll explain using the RPCSS (Remote Procedure Call) service as an example.

From the startup parameters, it’s clear that the service relies on svchost to start.

### Example

Take Windows XP as an example. Click "Start" -> "Run," input the "services.msc" command, and a services dialog box will pop up. Open the "Remote Procedure Call" properties dialog box, and you will see that the path of the rpcss service's executable file is "c:\windows\system32\svchost -k rpcss." This indicates that the rpcss service depends on svchost calling the "rpcss" parameter to be realized, and the content of the parameter is stored in the system registry.

Input "regedit.exe" in the Run dialog box and press Enter to open the Registry Editor. Find the [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\rpcss] item. Locate the key "magepath" of type "reg_expand_sz," whose value is "%systemroot%\system32\svchost -k rpcss" (this is the service startup command seen in the services window). Additionally, under the "parameters" sub-item, there is a key named "servicedll" whose value is "%systemroot%\system32\rpcss.dll," where "rpcss.dll" is the dynamic link library file used by the rpcss service. Thus, the svchost process reads the registry information of the "rpcss" service to start the service.

### Clarification

Since the svchost process starts various services, viruses and trojans try every means to exploit it, attempting to confuse users by leveraging its characteristics to achieve infection, intrusion, and destruction (such as the variant worm virus "W32.Welchia.Worm"). However, it is normal for the Windows system to have multiple svchost processes. Which one is the virus process in an infected machine? Here's an example to illustrate.

Assume that the Windows XP system is infected with "W32.Welchia.Worm." The normal svchost file exists in the "C:\Windows\System32" directory. If you find the file in another directory, be cautious. The "W32.Welchia.Worm" virus resides in the "C:\Windows\System32\wins" directory. Therefore, using a process manager to check the execution file path of the svchost process makes it easy to detect whether the system is infected with a virus. The built-in Windows Task Manager cannot view the process path, so you can use third-party process management software, such as the "Windows Optimization Master" process manager. With these tools, you can easily check the execution file paths of all svchost processes. If you find that the execution path is unusual, you should immediately conduct detection and handling.
