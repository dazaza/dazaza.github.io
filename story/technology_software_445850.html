**Linux Introduction - Disk Management**

### Disk Management

#### df

- **Usage**: Check disk status

#### mount

- **Usage**: Mount a disk

- To make it automatically mount on startup, add an entry to the `/etc/fstab` file.

#### Mounting ISO

- **Command**: `mount -o loop /xxx/xxx.iso /xxx`

- Example: `mount -o loop /soft/rhel-5.3-server-i386-dvd.iso /mnt`

#### Network Mount

- **Command**: `mount -t nfs 10.1.1.1:/share/rhel_source`

#### Mount CD-ROM

- **Command**: `mount -o ro -t iso9660 /dev/cdrom /xxx`

#### Mount All Devices in `/etc/fstab`

- **Command**: `mount -a`

#### Bind Two Mount Points

- **Command**: `mount --bind /home /mnt`

#### Move Mount Point

- **Command**: `mount --move /mnt /disk11`

#### umount

- **Usage**: Unmount a device or mount point

- **Command**: `umount device_name/mount_point`

#### fdisk

- **Usage**: Partition disk

- **Commands**:

- `n`: Create new partition

- `d`: Delete partition

- `p`: Display partitions

- `l`: List partition types

- `t`: Change partition type

- `m`: Show help menu

- `w`: Save changes

#### After partitioning, use `partprobe` to synchronize.

#### mkfs

- **Usage**: Format a partition

- **Syntax**: `mkfs -t filesystem_type partition`

#### Swap Partition:

- **Add Swap**:

- **Method 1**: Add a swap partition

1. Add a swap partition

2. Format with `mkswap` (e.g., `mkswap /dev/sda9`)

3. Enable with `swapon` (e.g., `swapon /dev/sda9`)

4. Add to `/etc/fstab`: `/dev/sda9 swap swap defaults 0 0`

- **Method 2**: Create a swap file

1. Create a swap file: `dd if=/dev/zero of=/opt/swapfile bs=2M count=1024`

2. Format with `mkswap`: `mkswap /opt/swapfile`

3. Enable with `swapon`: `swapon /opt/swapfile` (To enable on boot, add it to a startup script like `/etc/rc.local`)

#### `/etc/fstab`

- **Example Entry**: `LABEL=/ / ext3 defaults 1 1`

- **Columns**:

- Mounted device

- Mount point

- Filesystem

- Mount options

- Backup flag

- Partition check flag

### Processes

#### Process States:

- `r`: Running state

- `s`: Interruptible sleep state

- `d`: Uninterruptible sleep state

- `t`: Stopped state

- `z`: Zombie state

- `x`: Exit state (process is about to be destroyed)

#### ps

- **Usage**: View processes

- **Options**:

- `-e`: All processes

- `-f`: Full format

- `-l`: Long format

- `-w`: Wide format

- `-u`: Include user, PID, %CPU

#### top

- **Usage**: View processes dynamically

- **Shortcuts**:

- `M`: Sort by memory usage

- `P`: Sort by CPU usage

#### /proc

- Contains current kernel information

#### pstree

- **Usage**: View process tree

#### kill

- **Usage**: Terminate a process

- **Command**: `kill process_id`

#### killall

- **Usage**: Terminate a group of processes with the same name

- **Command**: `killall httpd`

#### pgrep

- **Usage**: Find processes by name

### Software Installation

#### 1. rpm

- **Installation Command**: `rpm -ivh package_name.rpm`

- **Options**:

- `-i`: Install

- `-v`: Verbose output

- `-h`: Show progress bar

- `-q`: Query installed package info

- `-qa`: Query all installed packages

- `-qpl`: Query package installation path and file list

- `-qf`: Query which package a file belongs to

- `-e`: Uninstall

- `-F`: Force upgrade

- `--nodeps`: Ignore dependencies

- `--replacefiles`: Overwrite existing files

- `--force`: Force installation

#### 2. src.rpm

- **Method 1**: Use `rpmbuild --rebuild` to generate an RPM package

- Note the output for the location of the generated RPM package.

- Then, go to the directory and install with `rpm -ivh`.

- **Method 2**: Use `rpm --rebuild`.

#### 3. Source Package Installation

- **Steps**:

1. Extract: `tar -xvf xxx.xxx`

2. Configure: `./configure` (Some software may not require this step)

3. Compile: `make`

4. Install: `make install`

#### 4. Binary Package Installation (.bin)

- These packages do not require installation; just extract and run.

- Ensure the extracted files have execution permissions.

#### 5. yum

- **Advantage**: Automatically resolves package dependencies

- **Steps**:

1. Specify software repository

2. Configure `/etc/yum.repos.d/rhel-debuginfo.repo`

- Example:

```

[Server]

name=Server

baseurl=file:///soft/rhel53/Server

enabled=1

gpgcheck=0

```

3. Install software with `yum install package_name`

4. Install groups with `yum groupinstall group_name`

5. View packages with `yum list` or `yum grouplist`

6. Uninstall with `yum erase package_name` or `yum remove package_name` (wildcards like `*` can be used)

### Packing, Compressing, and Decompressing

#### tar

- **Usage**: Pack/unpack files

- **Options**:

- `-cf`: Pack

- `-xf`: Unpack

- `-C`: Specify unpacking path

- `-z`: Use gzip for compression/decompression (commonly used with `.gz` files)

- `-j`: Use bzip2 for compression/decompression (commonly used with `.bz2` files)

- `-v`: Show progress

#### bzip2

- **Usage**: Compress/decompress `.bz2` files (original file will be deleted unless `-k` is used)

- **Options**:

- `-z`: Compress

- `-d`: Decompress

- `-k`: Keep original file

#### gzip

- **Usage**: Compress/decompress `.gz` files (original file will be deleted unless `-k` is used)

- **Options**:

- `-n`: Compress

- `-d`: Decompress

#### gunzip

- **Usage**: Decompress `.gz` files (original compressed file will be deleted)

#### zip

- **Usage**: Compress/decompress `.zip` files (original file will not be deleted)
