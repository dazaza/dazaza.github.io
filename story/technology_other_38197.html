OK, never mind the chatter, let's get down to business (cut the chit-chat). Batch processing, also known as batch scripts, translates to BATCH in English. The BAT file extension comes from the first three letters. Its structure has no fixed format, as long as the following rule is followed: each line can be considered a command, each command may contain multiple *** commands, execution starts from the first line until the last line ends, and it runs on the DOS platform. Batch processing has a distinct feature: convenient, flexible use, powerful functionality, and high degree of automation. Example one: First, here's an extremely easy batch script for everyone to familiarize themselves with. Save the following lines as name.bat and execute them (later in the text, only the code will be provided, saving and executing methods are similar):

ping sz.tencent.com > a.txt

ping sz1.tencent.com >> a.txt

ping sz2.tencent.com >> a.txt

ping sz3.tencent.com >> a.txt

ping sz4.tencent.com >> a.txt

ping sz5.tencent.com >> a.txt

ping sz6.tencent.com >> a.txt

ping sz7.tencent.com >> a.txt

exit

Can you all understand it? Is it very easy? But its function is quite practical. After executing this batch script, a file named a.txt will be created on your current disk. The information recorded inside can help you quickly find the fastest QQ server, thus avoiding the painful process of "relay through the server." Here > means placing the result of the preceding command into the location specified after it, and >> serves the same function as >, except it appends the result to the end of the previous result specifically on the next line while retaining the result of the previous command. This makes the a.txt file grow larger (are you thinking about how to cause damage??). By the way, this batch script can be combined with other commands to automatically judge server speed, directly displaying the IP address of the fastest server. Isn't that great? More details will be introduced later.

Example two: Another outdated example (a.bat):

@echo off

if exist C:\Progra~1\Tencent\AD\*.gif del C:\Progra~1\Tencent\AD\*.gif

Why is this an outdated example? It's simple, because almost no one uses QQ with ads anymore (KAO, my QQ still shows friends' body measurements!!), so it's almost useless. However, its role was significant at one time: deleting QQ ads to keep the chat window clean. The address used here is the default installation address for QQ, and the default batch script name is a.bat, though you can modify it according to your situation. In this script, the if command is used to achieve the effect of timely judgment and deletion of ad images. As long as you don't close the DOS window after the command is executed and don't press CTRL+C to forcibly terminate the command, it will continue monitoring for ad images (QQ also keeps checking whether its own ads have been deleted). Example three: Using a batch script to check if infected by Glacier. Script content follows:

@echo off

netstat -a -n > a.txt

type a.txt | find "7626" && echo "Congratulations! You have been infected by GLACIER!"

del a.txt

pause & exit

Here, the netstat command is utilized to check the status of all network ports. If you know the commonly used ports for trojans, it's easy to determine if you've been infected by Glacier. However, this isn't definitive, because the default port 7626 for Glacier can be modified. What's introduced here is merely the method and idea. With slight modifications, this can become a script to check for other types of trojans. Further modifications, adding parameters and port lists, can transform it into an automatic detection script for all trojans. Hehe, isn't that exciting? The script also utilizes combination commands && and pipe command |, which will be detailed later.

Example four: Use a batch script to automatically clear system junk. Script follows:

@echo off

if exist c:\windows\temp\*.* del c:\windows\temp\*.*

if exist c:\windows\Tempor~1\*.* del c:\windows\Tempor~1\*.*

if exist c:\windows\History\*.* del c:\windows\History\*.*

if exist c:\windows\recent\*.* del c:\windows\recent\*.*

Save the above script content into autoexec.bat, and the system trash will be automatically deleted every time you boot up. Two points to note here: one, DOS does not support long filenames, hence Tempor~1 appears; two, adjustments can be made according to your actual situation to meet your needs.

Example five, let's look at a practical instance. This is a very interesting script, a small and useful tool that fully embodies the "automation" characteristic of batch processing. Let's first introduce the origin of this script: everyone knows the process of running an assembly program (MASM), where you first assemble the source code, then link it, and finally execute it, requiring inputting many things along the way, which can be troublesome (only those who have experienced it truly understand). How can this process be simplified? During our assembly course design, I "had to" write this script, and using it feels great, hehe. Look at the script content:

@echo off

::close echo

cls

::clean screen

echo This programme is to make the MASM programme automate

::display info

echo Edit by CODERED

::display info

echo Mailto me : qqkiller***@sina.com

::display info

if "%1"=="" goto usage

::if input without parameter goto usage

if "%1"=="/?" goto usage

::if parameter is "/?" goto usage

if "%1"=="help" goto usage

::if parameter is "help" goto usage

pause

::pause to see usage

masm %1.asm

::assemble the .asm code

if errorlevel 1 pause & edit %1.asm

::if error pause to see error msg and edit the code

link %1.obj & %1

::else link the .obj file and execute the .exe file

:usage

::set usage

echo Usage: This BAT file name [asm file name]

echo Default BAT file name is START.BAT

::display usage

Don't be scared by this pile of stuff, stay calm and read carefully (recall the first paragraph of the first chapter!). Explanations for each line of commands are already given. Content after two colons is the E-translation of the previous line (don't worry if you're afraid of English, it's all easy, understandable at a glance, really don't know? Won't you look it up in a dictionary, so lazy?). A colon at the beginning of the fifth line from the bottom isn't a typo! Its specific function will be explained in detail later. The masm and link in this script are the assembly and linking programs, which must be in the current directory together with the edit program and the source code you want to edit (of course, including this script, duh!). Using this batch script can minimize manual input as much as possible. Throughout the process, you only need to press the enter key a few times to automatically convert the assembly source code into an executable exe file, with intelligent judgment functionality: if errors occur in the source code during assembly (assembly fails), it will automatically pause to display error messages and then automatically enter the source code editing interface after pressing any key; if the source code assembles successfully, it will proceed to link and automatically execute the generated exe file after linking. Additionally, due to the simplicity and flexibility of batch commands, this script has excellent modifiability, allowing simple modifications to suit different users' habits. Friends studying assembly should definitely try it out!

The following commands appear in this script: @, echo, ::, pause, :, goto, %, and if. These will be explained in this chapter.

1、@

Everyone is familiar with this symbol, essential for email, so how did it end up in batch processing? Haha, it's not its fault, batch processing simply can't do without it, otherwise it wouldn't be perfect. Its function is to prevent the command itself from being displayed in the execution window (such a convoluted sentence!). Simply put, if it's at the beginning of a line, the command on that line won't be shown. In Example Five, the @ in the first line @echo off prevents the script from displaying the subsequent echo off part during execution. Got it now? Still confused? No problem, after reading the brief introduction to the echo command, you'll naturally understand.

2、echo

Means "feedback" or "replay" in Chinese. It is essentially a toggle command, having only two states: on and off. Thus, there are the two commands echo on and echo off. Directly executing the echo command will display the current state of the echo command (off or on). Executing echo off will turn off the replay, so all subsequent commands won't show the command itself, only the results after execution, unless you execute the echo on command. In Example Five, the combination of the @ command and the echo off command in the first line achieves two purposes: not showing the echo off command itself, and not showing the subsequent commands themselves. It may seem a bit messy, but if you practice, you'll understand within three minutes, guaranteed!

Another use of the echo command: it can display information! For example, the second-to-last line in Example Five, Default BAT file name is START.BAT will be displayed in the window after the script executes, but the echo command itself won't be shown (why??).

Another use of the echo command: it can directly edit text files. Example Six:

echo nbtstat -A 192.168.0.1 > a.bat

echo nbtstat -A 192.168.0.2 >> a.bat

echo nbtstat -A 192.168.0.3 >> a.bat

The method to edit the above script content is to input each line directly into the command line and press enter after each line. Finally, an a.bat file will be generated in the current directory, and executing it will produce results.

3、::

The function of this command is simple; it is a comment command, equivalent to the rem command in batch scripts. The content after it is not displayed during execution and serves no function, as it is just a comment that enhances the readability of the script, similar to /*………*/ in C language. Everyone understands this, so no further explanation is needed.

4、pause

Means "pause" in Chinese (look at your workman). I always think it's the simplest command in batch processing, pure and practical. Its function is to pause the current program process and display a message: Press any key to continue... In Example Five, this command is used twice. The first time, it allows the user to clearly see the program information, and the second time, it displays the erroneous assembly code information (actually, it's not trying to display it, but rather the masm program is paused while displaying error information, allowing you to see where your source code went wrong).

5、: and goto

Why introduce these two commands together? Because they go hand in hand; either one missing or extra will cause an error. Goto is a jump command, and : is a label. When the program encounters goto, it automatically jumps to the part defined by : (aren't they inseparable?). In Example Five, a : appears at the beginning of the fifth line from the bottom, so when the program runs to goto, it automatically jumps to the part defined by the : label, resulting in the display of the script usage (usage is the label name). It's not hard to see that the goto command finds where to jump based on this colon and label name, and they correspond one-to-one. The goto command is often used in conjunction with the if command. For specific usage, refer to Example Five.

Another use of the goto command: prematurely ending the program. Using the goto command in the middle of the program to jump to a certain label, and the content of that label is defined as exit. For example:

……

goto end

……

:end

Here, :end is at the last line of the script! Actually, this example is quite silly, and you'll understand more after learning the if command and compound commands.
