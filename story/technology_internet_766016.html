Compromise or Progress? Take a Look at the Start Button in Windows 8.1

by anonymous on 2013-09-04 14:09:28

The biggest change in Microsoft Windows 8 compared to the past is the removal of the Start menu and the addition of the Start screen. Microsoft's move was aimed at enabling the new Windows to better leverage touch capabilities, aligning with the touchscreen trend brought by smartphones. However, the disappearance of the Start menu from Windows 95 through to Windows 7 undoubtedly challenged the usage habits of a wide range of users. In Windows 7, users could find recently opened programs and all applications via the Start menu, access system multimedia folders, and quickly perform shutdown or restart operations. In Windows 8, if users need to access all programs, they can either search directly on the Start screen using their keyboard or summon the application bar on the Start screen and then click "All Apps".

Such an application search method is indeed cumbersome for traditional users (those using mouse and keyboard). Many users, after using Windows 8, found this major change inconvenient and chose to downgrade, while some OEM manufacturers pre-installed their own developed Start menu applications on their computers to address user habit issues.

Clearly, Microsoft has been continuously listening to user feedback. In Windows 8.1, Microsoft reintroduced the Start button and provided rich Start screen navigation options: logging in directly to the desktop instead of the Start screen, displaying the desktop background on the Start screen, always showing the Start screen on multiple monitors, and automatically displaying the app view when switching to the Start screen.

Compared to Windows 8, the visual display of the app view in Windows 8.1 hasn't changed much, but users can choose how apps are categorized (such as by installation date or category). Although Windows 8 also had a similar app view, its cumbersome activation process and overly monotonous interface made it difficult for many users to adapt. Microsoft cleverly resolved the "split" issue between the Start screen and the traditional desktop in Windows 8 within Windows 8.1. Supporting the display of the desktop background on the Start screen undoubtedly strengthened the fusion between the traditional desktop and the Start screen; logging in directly to the traditional desktop saved traditional users an extra step. In Windows 8.1, the Start screen can finally be called a "full-screen version of the Start menu": users can enter the app view by pressing the Win key or clicking the Start button, and search for apps or use semantic zoom, functions that closely resemble those in the Start menus of Windows Vista and Windows 7.

The deeper the target users' habits are ingrained, the more product improvements should focus on adapting to these habits — this is what we've learned from Windows 8 and iOS 7. By choosing to bring back the Start button in Windows 8.1 and allowing the Start menu to return in full-screen mode, Microsoft demonstrated both compromise and progress. It might seem like Microsoft is retracing its steps, which often invites ridicule. However, recognizing mistakes and correcting them in a progressive manner is a quality and ambition that a great company should possess. Microsoft may appear tough, but it has a softer side. Fortunately, Microsoft continues to move forward on the path of improvement.

Currently, the RTM version of Windows 8.1 has been completed, with global official distribution set for October 17th. At that time, Windows 8 users will be able to update to Windows 8.1 for free without losing their data, while users of Windows 8.1 Preview will need to reinstall applications after the update.
