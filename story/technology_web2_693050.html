Development Exercise for Notepad Application (II)

by dream2012 on 2012-03-06 14:09:52

In this chapter's exercise, you will use lifecycle event callbacks to store and retrieve the state of your application. This exercise describes:

- Lifecycle functions and how your program invokes them

- Techniques for maintaining the state of your program

### Step 1: Import Notepadv3 into Eclipse

If you see an error related to `AndroidManifest.xml` or issues with the `Android.zip` file, right-click on the project and select **Android Tools > Fix Project Properties**. The starting point for this exercise is where Notepadv2 ended.

Currently, there are problems in the application:

- Clicking the back button while editing causes data loss.

- Other issues during editing can also lead to lost content.

To fix this, we need to move most of the functionality (functions) for creating and editing notes into the `NoteEdit` class and define a complete lifecycle for editing notes.

Delete the code that parses the title and body from the extras bundle in `NoteEdit`. Instead, we will use the `DBHelper` class to directly access the database for notes. We only need to pass `mRowId` into the `NoteEdit` Activity (but only when editing; if creating, we don't need to pass anything). Delete these lines:

```java

String title = extras.getString(NotesDbAdapter.KEY_TITLE);

String body = extras.getString(NotesDbAdapter.KEY_BODY);

```

We also don't need the properties passed into the `extras` Bundle, which were used to edit the body text in the UI. So delete:

```java

if (title != null) {

mTitleText.setText(title);

}

if (body != null) {

mBodyText.setText(body);

}

```

### Step 2: Create a Class Object for NotesDbAdapter

At the top of the `NoteEdit` class, create a class object for `NotesDbAdapter`:

```java

private NotesDbAdapter mDbHelper;

```

Also, in the `onCreate()` method, add an instance of `NotesDbAdapter` (right after the `super.onCreate()` call):

```java

mDbHelper = new NotesDbAdapter(this);

mDbHelper.open();

```

### Step 3: Check `savedInstanceState` for `mRowId`

In `NoteEdit`, we need to check `savedInstanceState` for `mRowId` to ensure that we overwrite any saved state in the bundle (this happens when the activity loses focus and is restarted).

Replace the current initialization of `mRowId`:

```java

mRowId = null;

Bundle extras = getIntent().getExtras();

if (extras != null) {

mRowId = extras.getLong(NotesDbAdapter.KEY_ROWID);

}

```

With the following:

```java

mRowId = savedInstanceState != null ? savedInstanceState.getLong(NotesDbAdapter.KEY_ROWID) : null;

if (mRowId == null) {

Bundle extras = getIntent().getExtras();

mRowId = extras != null ? extras.getLong(NotesDbAdapter.KEY_ROWID) : null;

}

```

Add a null-check for `savedInstanceState` and still load `mRowId` from the `extras` Bundle if `savedInstanceState` does not provide it. This is a quick and safe way to use ternary operators, which will return `null` if not provided.

### Step 4: Populate Fields

Next, if we have `mRowId`, we need to populate the fields related to it:

```java

populateFields();

```

This should be done before `confirmButton.setOnClickListener()`, which we will define shortly.

### Step 5: Simplify `onClick()`

Remove the creation of the bundle and setting bundle values in the `onClick()` handler. The `Activity` no longer needs to return any additional information to the caller. Since we don't need to return a value, we can use a simplified version of `setResult()`:

```java

public void onClick(View view) {

setResult(RESULT_OK);

finish();

}

```

### Step 6: Define `populateFields()`

Define the `populateFields()` method:

```java

private void populateFields() {

if (mRowId != null) {

Cursor note = mDbHelper.fetchNote(mRowId);

startManagingCursor(note);

mTitleText.setText(note.getString(

note.getColumnIndexOrThrow(NotesDbAdapter.KEY_TITLE)));

mBodyText.setText(note.getString(

note.getColumnIndexOrThrow(NotesDbAdapter.KEY_BODY)));

}

}

```

This method uses the `NotesDbAdapter.fetchNote()` method to find the note to edit and then calls the `startManagingCursor()` method from the `Activity` class, which is an Android-provided method to track the cursor lifecycle. Afterward, we simply retrieve the title and body from the cursor and populate their respective view elements.

### Step 7: Override Lifecycle Methods

Still in the `NoteEdit` class, override these methods: `onSaveInstanceState()`, `onPause()`, and `onResume()`. These are all part of the existing `onCreate()` function set.

When an `Activity` might be killed before being restarted, Android will call `onSaveInstanceState()`, allowing you to save useful state information so the `Activity` can be re-initialized upon restart. This corresponds to the `onCreate()` method, as the `savedInstanceState` bundle passed into `onCreate()` is the same one you create as `outState` in `onSaveInstanceState()`.

`onPause()` and `onResume()` are automatically called methods. `onPause()` is always called when an `Activity` ends, even if we explicitly call `finish()`. We'll use this to save the current note to the database. To reduce resource usage, it's good practice to release any resources that can be released during `onPause()`. Therefore, we close the `DbHelper` class and set its field to `null` so it can be garbage collected when appropriate.

On the other hand, `onResume()` can recreate the `mDbHelper` instance so we can use it. Then, read the note from the database again and populate the fields.

After defining `populateFields()`, add the following lifecycle methods:

#### `onSaveInstanceState()`:

```java

@Override

protected void onSaveInstanceState(Bundle outState) {

super.onSaveInstanceState(outState);

outState.putLong(NotesDbAdapter.KEY_ROWID, mRowId);

}

```

#### `onPause()`:

```java

@Override

protected void onPause() {

super.onPause();

saveState();

}

```

We'll define `saveState()` next.

#### `onResume()`:

```java

@Override

protected void onResume() {

super.onResume();

populateFields();

}

```

### Step 8: Define `saveState()`

Define the `saveState()` method to write data to the database:

```java

private void saveState() {

String title = mTitleText.getText().toString();

String body = mBodyText.getText().toString();

if (mRowId == null) {

long id = mDbHelper.createNote(title, body);

if (id > 0) {

mRowId = id;

}

} else {

mDbHelper.updateNote(mRowId, title, body);

}

}

```

Notice that we accept the return value of `createNote()`, and if it's valid, we save it to the `mRowId` field so we can update it later without needing to recreate it (which could happen if lifecycle events are triggered).

### Step 9: Update `onActivityResult()`

Now, remove the previously defined handling code in the `onActivityResult()` method of the `Notepadv3` class. All note retrieval and updates occur within the `NoteEdit` lifecycle, so the `onActivityResult()` method only needs to update the data view:

```java

@Override

protected void onActivityResult(int requestCode, int resultCode, Intent intent) {

super.onActivityResult(requestCode, resultCode, intent);

fillData();

}

```

Since another class is doing the work, this function only needs to update the data.

### Step 10: Remove Title and Body Setting Code

Remove the code that sets the title and body in the `onListItemClick()` method (they are unnecessary except for `mRowId`):

```java

Cursor c = mNotesCursor;

c.moveToPosition(position);

```

Also remove:

```java

i.putExtra(NotesDbAdapter.KEY_TITLE, c.getString(c.getColumnIndex(NotesDbAdapter.KEY_TITLE)));

i.putExtra(NotesDbAdapter.KEY_BODY, c.getString(c.getColumnIndex(NotesDbAdapter.KEY_BODY)));

```

The remaining code should be:

```java

super.onListItemClick(l, v, position, id);

Intent i = new Intent(this, NoteEdit.class);

i.putExtra(NotesDbAdapter.KEY_ROWID, id);

startActivityForResult(i, ACTIVITY_EDIT);

```

You can now also remove `mNotesCursor` from the class and reset it as a local variable in the `fillData()` method:

```java

Cursor notesCursor = mDbHelper.fetchAllNotes();

```

Note that the `m` in `mNotesCursor` stands for member variable, so when making `notesCursor` a local variable, remove the `m`. Remember to rename all instances of `mNotesCursor` in the `fillData()` method.
