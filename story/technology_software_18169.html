This software is a comprehensive industry management software specifically developed for small and medium-sized supermarkets. The software supports consumption by both regular customers and members. Member card types can be divided into stored-value cards, discount cards, count-based cards, points cards, and upgrade cards. The type of member card can be flexibly set according to promotional activities for members. Historical records of members can be viewed at any time, including deposit records, consumption records, points records, and reward records. This greatly reduces the hassle of manual operations. For retail items, settings can be made freely, and mnemonic codes can be automatically generated, allowing quick order placement during checkout. The inventory management of goods is clear and precise, with automatic deduction of inventory. It can automatically tally the ranking of product sales, employee commissions from card sales, and daily revenue. Products can also be promoted during specific sales periods, and certain products requiring clearance can be sold at special prices. There are operation logs for recording operator actions, so it's always known what operations have been performed on the software. Permissions are clearly defined in the software, with different operators responsible only for their own operational management. When operators leave the computer, the system can be locked to prevent non-operators from accessing and making mistakes. The software has features like automatic reminders for member birthdays and voice prompts for consumption amounts, making management more humanized, leaving a good impression on customers, and enhancing the overall level of the supermarket.

1. Different users can have different permission passwords. One software package can be used by multiple users, each having different permissions based on their responsibilities.

For example, the cashier's permission is related to cash register content; the manager mainly queries and tallies business revenue; finance mainly tallies some reports for accounting purposes; the general manager is a superuser with the authority to add, modify, or delete all contents in the system. Thus, one system can be divided into several parts managed by different personnel, equivalent to division of labor and cooperation. If something goes wrong in a particular step, it’s easy to find out which user's permission scope is involved, reducing the need for large-scale checks and effectively stopping some bad habits of employees, helping enterprises gradually move towards systematization and perfection, allowing each operator to manage all matters confidently.

2. Card types can be set as: Stored-value cards, Accumulation cards, Discount cards, Count-based cards, Stored-discount cards, Upgrade cards.

Stored-value card: Stored value + discount + points, every consumption directly deducts from the card, no need to pay cash, making it convenient for customers. Accumulation card: Equivalent to a points card, accumulating points based on consumption amount, the venue can exchange points for cash, consumption, or prizes to support some accumulation promotion activities. Discount card: Eligible consumption items can enjoy automatic discounts from the card, giving customers the feeling of VIP treatment. Count-based card: Deposit money in advance, total number of consumptions allowed, when consuming this item later, just swipe the card to automatically reduce the number of times, suitable for customers who repeatedly consume the same item, after the number of consumptions ends, the card can be recharged to purchase more times. Stored-discount card: Can deposit money and offer discounts but cannot accumulate points, combining the functions of stored-value and discount cards, applicable to different customers. Upgrade card: When the cumulative consumption amount reaches a certain amount, a lower discount can be enjoyed. For example, a new card might start at 95% off, but after cumulative consumption of 2000 yuan, it can enjoy 90% off, and so on.

3. Supports various membership card media types such as magnetic cards, barcoded cards, IC cards (contact and contactless).

Many single-machine version customers choose the first two options because these cards are relatively low-cost and since they are single-machine versions, there is no need to implement a one-card-for-all system. Therefore, making IC cards seems unnecessary. However, for large chain clients aiming for one-card-for-all consumption, IC cards must be used to achieve real-time deductions of consumption amounts on the card to ensure accurate account balances.

4. Supports various query methods

The system can query all operation records for each membership card. In addition to viewing detailed consumption, recharge, and points information, it can also query all operator logs, i.e., all operations performed by each operator in the system daily can be truly reflected. Whether due to operational errors or intentional mistakes, everything can be clearly seen. Through the function of checking operator logs, all bosses can completely rest assured about the cashier section.

5. Membership cards can be returned after use.

After the amount on the membership card is fully consumed, if the customer no longer uses the card or switches to another type of card, the card can be returned. The returned card can still be reissued for other customers to use, saving on card usage and production costs.

6. Lost membership cards can be reported lost, temporarily disabled, replaced, or transferred to another card.

If a card is lost, it can be reported lost and replaced. If a membership card is lost, it can be reported lost and temporarily disabled, ensuring that if someone else finds it, it cannot be used for consumption. Each card also has a password, equivalent to an additional security lock. If there is still an amount left on the card, the replacement card function can be used, transferring all data and amounts from the lost card to a new replacement card, avoiding inconvenience for customers who lose their cards due to loss of funds.

7. Forgotten passwords can be modified.

If a customer forgets their password, they can retrieve a new password with valid identification, eliminating all concerns for cardholders.

8. Automatic backup can be set when exiting the system.

Setting automatic backup when exiting the system means that each time the system is exited, it will prompt whether to back up the database, without requiring the operator to remember to click backup before exiting each time. The backup path is also pre-set in the system. As long as the database is backed up, data can be restored regardless of any issues encountered by the system, preventing data loss.

9. Male and female voice prompts during checkout.

Voice prompts in male or female voices can be set during checkout, letting customers know their consumption amount before printing receipts, avoiding many inconveniences caused by incorrect checkouts.

10. Hotkeys can be set to call out or minimize system windows.

After setting hotkeys for calling out or minimizing system windows, no matter which window the user is currently in, pressing the set hotkey can bring up the system operation window, and after completing the operation, hotkeys can also be used to minimize the window, making the operation both convenient and fast, facilitating simultaneous operation with other cashier software.

11. Member birthday reminders can be set.

Member birthdays can be reminded either before the birthday through queries or on the day of the birthday when the member comes to shop. This displays in the member information window. Whether early reminders or same-day reminders, it is all for tracking services to send birthday wishes, offering discounts or gifts on the birthday, making customers feel warm, bridging distances, and making customer service more humanized.

12. Manual reward points for member consumption can be set.

Member card consumption can automatically accumulate points, even without using a points card, enjoying consumption reward points, very suitable for special members receiving special point rewards.

13. A built-in calculator can be opened during checkout.

The system includes a built-in calculator that can be opened while checking out and operated simultaneously in the window, allowing all operations to be completed directly within the system.

14. Multiple consumption explanations can be set.

Consumption explanations can be set according to the specific circumstances of the using entity, choosing different consumption explanations based on the customer's consumption situation each time they settle accounts, making it clear for both customers and entities when querying.

15. The software can be locked when the operator leaves.

If the operator needs to temporarily leave the computer, to prevent others from operating the system while away, the software can be locked. Without the operator's password, the software cannot be entered for operation, making the system safer to use. Many current software systems cannot achieve this, and usually require the system to be closed upon departure, then reopened upon returning to work. In comparison, this system is more convenient and secure.

16. Member consumption amounts can be divided into discounted and non-discounted categories.

Member cards can enjoy discounts, but not all consumption items are eligible for discounts, depending on cost differences. Some products can be discounted, while others do not enjoy discounts. This feature makes all using entities' membership card management more perfect.

17. Receipt printers are supported.

Receipt printers include dot matrix POS printers and thermal POS printers, both of which can run in the system.

18. Member personal data, historical consumption, and points records can be permanently stored in the computer system.

Once member data enters the computer and operation records are created, they will remain stored in the system indefinitely, ready to be retrieved whenever needed. Data is safe and reliable, and operations are convenient.

19. Importing and exporting of data is supported.

Customer data can be obtained from the sales system, and club member data can be batch-imported into the club member management system. Data from the club can also be exported to other software, greatly reducing the workload of operators.

20. Cashier shift handovers and report printing are supported.

For venues operating across days, we provide shift handover functions for different shifts, and each handover prints out a revenue report for that shift. This facilitates settlement between finance and cashiers, and also statistically summarizes the work of each shift's cashiers, assigning responsibility to individuals.
