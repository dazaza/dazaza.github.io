How to Maintain the Health and Safety of a Virtualization Environment?

by xwkj0983 on 2012-02-22 14:47:16

After completing the virtualization transformation, our work is not over. Let's learn how to maintain a healthy virtualization environment in order to maximize your return on investment. If you have recently consolidated and virtualized physical servers, you need to consider the risks involved and how to improve data center efficiency. Perhaps in a perfect world, that would be all there is to it. However, after completing the virtualization process, there are still many things that need attention, such as backups and VM sprawl, which can keep your virtual environment healthy and secure. In order to keep your virtual machines in optimal condition after completing the virtualization project, you must immediately perform the following 8 tasks in your infrastructure:

1. Get rid of all old services that are no longer needed.

Since the purpose of using virtualization is to reduce energy consumption and cooling costs, why keep the root cause of these issues? Resist the temptation to use these old servers for other functions, as this will go against the goal of achieving virtualization. Unplug these old servers, remove them from the racks, and have someone haul them away. Although saying this, you may still want to retain some relatively newer servers and run free virtualization products on them, so as to complement your new virtual hosts.

2. Establish a regular work schedule to update operating system patches on your virtual hosts.

The importance of this task must be fully recognized. Numerous customers run on virtual hosts. If these customers are compromised due to an unpatched vulnerability, then regardless of how secure or timely the customers themselves are with patching, they will all suffer the same fate. Updating server patches is more troublesome than updating client patches because clients usually need to be updated while hosts need to be restarted. To simplify this process, you can use automatic patch update scripts or third-party applications. At the same time, ensure that you stay informed about patch releases. Not all suppliers release patches according to a fixed schedule. Subscribe to patch release emails to ensure you are promptly notified when patches are released, so you can regularly update your hosts.

3. Stay vigilant against VM sprawl and seek various methods to prevent it in your environment.

Because VM sprawl often occurs over time, preventing it becomes a challenging task. Virtual machines (VMs) are not actual physical entities, so administrators cannot easily create them. Many administrators view VMs as "free" servers that can be created on demand. Unfortunately, VMs are not free. Each VM consumes resources from the host, and the more VMs you have, the more resources your host will consume. If the issue of VM sprawl is not effectively controlled, it will lead to a lack of host resources and become a bottleneck that severely affects the performance of all your VMs. As a result, you will have to purchase additional host servers and shared storage devices to solve the problem, which will incur extra costs. Additionally, there are several options available to control VM sprawl, including virtual hosts. One option is to implement resource recovery systems like those from Vkernel or use Vizioncore’s vFoglight product for VMware to create reports on resource usage. Microsoft's Hyper-V has some applicable resource recovery standards but lacks tools for effective utilization. Another option is to restrict the administrators who create VMs. Establish a formal process for requesting new VMs, which is more effective than allowing every administrator in the data center to arbitrarily create VMs. You can require each new VM request to provide a valid justification and undergo review and approval, so users consider whether they truly need to create this new VM.

4. Use third-party backup products specifically designed to run in virtual environments.

VMware's Consolidated Backup is a product that indirectly backs up host VMs through a backup proxy server. Using traditional methods to back up a virtual environment is inefficient and time-consuming. Installing a backup agent on each VM and backing up the server through the Guest OS is an extremely resource-intensive method for the host server and provides no benefit to all VMs running on that host. Of course, there are alternative methods, such as copying their virtual disk files to other servers or disk storage devices.
